## 摘要
自进化框架通常优化任务解决方案，而将周围的框架视为固定。我们提出感知框架的自进化（HASE），一种智能体强化学习框架，其中单个模型可以在多轮行动空间中生成任务解决方案或编辑选定的框架组件。HASE使单个Qwen3-8B模型能够匹配使用Claude Code作为框架提议者的GPT-OSS-120B模型的文本分类性能。在阿尔法因子挖掘中，HASE优于所报告的GPT-OSS-120B基线。HASE还修复不完善的评估组件，并在圆填充算法发现中收敛到最先进性能。这些结果表明，HASE通过统一的智能体过程改进了框架和解决方案。

## Abstract
Self-evolving frameworks usually optimize task solutions while treating the surrounding harness as fixed. We introduce Harness-Aware Self-Evolving (HASE), an agentic reinforcement-learning framework in which a single model can generate task solutions or edit selected harness components in a multi-turn action space. HASE enables a single Qwen3-8B model to match the text-classification performance of a GPT-OSS-120B model that uses Claude Code as the harness proposer. In alpha factor mining, HASE outperforms the reported GPT-OSS-120B baseline. HASE also repairs imperfect evaluation components and converges to state-of-the-art performance in circle-packing algorithm discovery. These results show that HASE improves the harness and the solution through one unified agentic process.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：现有的自进化系统（如 FunSearch、AlphaEvolve、Meta-Harness 等）在迭代优化任务解决方案时，通常将外围的**框架组件**（如提示词、记忆、检索策略、评估器）视为固定不变的。这导致两个局限：
  - 人为编写的评估器可能不完善或被模型利用（奖励黑客），而非真正解决问题。
  - 指导性框架（prompt、记忆等）若提供不良搜索先验，会限制模型能力。
- **动机**：需要一种机制，让模型能够自主**改进其工作环境**（框架），而不仅是产出更好的解决方案。
- **整体含义**：提出**HASE（感知框架的自进化）**，一个统一的智能体强化学习框架，让**同一个模型**可以在多轮动作空间中**同时生成解决方案或编辑选定的框架组件**，实现模型权重、框架、任务解决方案的协同进化。

## 2. 论文提出的方法论

### 核心思想
- **统一动作空间**：模型可以在同一多轮交互中执行两类动作：提交解决方案（对象级 artifact）或使用沙盒工具（ls、cat、grep、replace、overwrite）修改可编辑的框架文件。没有独立的硬编码规划器，模型根据上下文决定行动。
- **两类框架组件区分**：
  - **指导组件（Guidance Components）**：包括提示词、记忆格式化、检索脚本、上下文选择等，改进搜索质量，可自由编辑。
  - **评估组件（Evaluation Components）**：即局部评估器，提供密集训练奖励。编辑此类组件需要**真实世界评估器（oracle）的锚定**，防止无约束的奖励重写。
- **训练流程**：基于 **GRPO（Group Relative Policy Optimization）** 算法。每个阶段（Phase）包含多个 GRPO 轮次，每个轮次中模型 rollout 多轮轨迹，提交解决方案后根据局部评估器获得任务奖励。指导组件的编辑通过**相位级审查（Phase-level Review）** 收集候选、筛选、排序并提交最佳编辑。评估组件的编辑需要在相位边界查询真实世界评估器，若局部代理分数与 oracle 分数不一致，则将错配案例加入**错配集 M**，修复后的评估器必须解决更多错配并保持非回归。

### 关键技术细节
- **奖励设计**：不奖励单纯的框架操作（如浏览文件或随机修改），只奖励最终提交的解决方案质量。指导编辑的信用通过**证明点（PoC）** 分配：只有编辑框架后获得更高任务分数的轨迹才计入候选池。评估编辑的奖励仅当代理分数与 oracle 不一致时给予，且与错配集一致性增加成正比。
- **Harness 选择**：相位边界处，指导候选通过验证性能评审，评估候选通过错配集评审。提交的稳定更新成为下一阶段的持久框架。
- **安全性**：可编辑组件是白名单的；评估编辑需要 oracle 差异触发；相位审查只提交接口有效的补丁。这些约束可能拒绝有用但侵入性的修改，但提高了安全性。

## 6. 主要结论与发现
- HASE 使单个 8B 模型在文本分类任务中匹配 **GPT-OSS-120B + Claude Code** 系统的性能（86.98% vs 86.8%）。
- 在阿尔法因子挖掘中，HASE Qwen3-8B 超越所有基线（包括 GPT-OSS-120B）：IC 0.0308、RankIC 0.0302、ICIR 0.3414、AER 14.35%、IR 1.70。
- 圆填充中，从**不完整的局部评估器**出发，HASE 自动修复评估器两次（增加边界和重叠检查、收紧容差 1e-8→1e-12），最终达到 SOTA 分数 2.635983。
- Heilbronn 三角形中，初始弱评估器被利用（代理高分但 oracle 0），HASE 修复后权重协同进化达到 0.036082（冻结策略仅 0.02188）。
- **核心结论**：联合进化框架组件和解决方案，且对评估器编辑进行 oracle 锚定，是一种有效的自我改进范式；较小的模型通过感知框架的自进化可超越大模型+外部提议者的系统。

## 7. 优点
- **统一动作空间**：无需外部强大编码提议者，同一模型既当解决方案生成器又当框架编辑者，降低了系统复杂度。
- **奖励设计合理**：不奖励无意义框架操作，只奖励最终方案质量，避免过度奖励表面探索。
- **相位级审查机制**：拒绝单一幸运轨迹直接改变持久框架，保证稳定性。
- **两类组件分开治理**：指导组件自由进化，评估组件受 oracle 锚定，防止评估器被随意弱化（奖励黑客）。
- **多领域验证**：从分类、量化金融到几何优化均取得出色结果，泛化性好。
- **开源与复现友好**：提供了详细算法伪代码、实验协议、代码演变示例（附录），以及其他鲁棒性检查（数据泄露检测）。

## 8. 不足与局限
- **不是穷尽规模研究**：虽然覆盖 4 个领域，但每个领域仅一个任务，未在大量不同任务上测试。
