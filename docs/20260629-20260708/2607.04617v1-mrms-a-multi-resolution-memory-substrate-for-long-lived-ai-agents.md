## 摘要
长期AI智能体需要在交互之间保持连续性，但这种连续性无法通过简单扩展提示窗口来实现。智能体必须保留有用的先前经验、有选择地检索、区分个人上下文与外部证据，并在底层情境变化时修正记忆。我们提出了一种架构化的记忆基板，沿两个正交轴组织：一个表示轴，涵盖结构化记录、向量表示和图关系；一个时间轴，涵盖短期痕迹、中期抽象和长期语义承诺。其关键设计约束是同步的结构化-向量-图记忆：结构化记录管理资格，向量表示支持回忆，图关系在门控上下文投影前裁决支持、矛盾和取代。其核心主张是，可靠的个性化是一个记忆设计问题：有用的记忆是结构化的、选择性暴露的、持续整合的、带认识论标签的，而不是存储为无差别的对话历史。在框架之外，我们将MRMS实例化为一个轻量级原型，实现结构化记录、向量检索、时间策略和基于图的修正。该原型在受控的长期交互场景中，通过明确的证据要求，演练了前代记忆选择、修正、边界执行和证据归因等核心基板机制。

## Abstract
Long-lived AI agents require continuity across interactions, but continuity cannot be obtained by simply extending the prompt window. An agent must preserve useful prior experience, retrieve it selectively, distinguish personal context from external evidence, and revise memory when the underlying situation changes. We propose an architectural memory substrate organized along two orthogonal axes: a representational axis spanning structured records, vector representations, and graph relations; and a temporal axis spanning short-term traces, medium-term abstractions, and long-term semantic commitments. Its key design constraint is synchronized structured-vector-graph memory: structured records govern eligibility, vector representations support recall, and graph relations adjudicate support, contradiction, and supersession before gated context projection. Its central claim is that reliable personalization is a memory design problem: useful memory is structured, selectively exposed, continuously consolidated, and epistemically labeled rather than stored as undifferentiated conversation history. Beyond the framework, we instantiate MRMS as a lightweight prototype implementing structured records, vector retrieval, temporal policies, and graph-based revision. The prototype exercises the core substrate mechanisms through pre-generation memory selection, revision, boundary enforcement, and evidence attribution under controlled long-lived interaction scenarios with explicit evidence requirements.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：长期AI智能体需要跨交互的连续性，但简单扩大语言模型的提示窗口（prompt window）无法实现有用记忆的保存、选择性检索和修订。现有方法把对话历史存储为无差别的转录，缺乏对记忆的**持久性、来源、范围、可信度**的区分，导致“虚假正面”错误（如使用过时偏好、被取代的事实）比“虚假负面”更具破坏性。
- **研究动机**：将智能体的连续性重新定义为**记忆设计问题**。作者认为可靠的个性化要求记忆系统在生成响应之前就能判断**某项记忆是否应该被允许影响行为**、其来源是用户偏好还是外部证据、是否存在矛盾或更新。这需要从“被动存储”转向“生成前控制”。
- **整体含义**：提出MRMS（Multi-Resolution Memory Substrate）框架，旨在为长期自主智能体提供一个可审计、可修正、可选择性暴露的记忆基板，将记忆组织视为架构问题而非简单的存储或检索。

- **核心思想**：沿两个正交轴组织记忆：
  - **表征轴**：结构化记录（structured records）、向量表示（vector representations）、图关系（graph relations）。
  - **时间轴**：短期痕迹（short-term traces）、中期抽象（medium-term abstractions）、长期语义承诺（long-term semantic commitments）。
- **关键技术细节**：

  - **三重视图同步**：
    - 结构化存储 \( S_t \)：权威记录，支持按元数据精确过滤（如用户、范围、状态、时间层）。
    - 向量索引 \( V_t \)：近似语义匹配，用于召回。
    - 记忆图 \( G_t \)：带类型的边（支持、矛盾、取代、派生自、会话内等），用于推理关系有效性。
  - **同步不变性**（表2）：
    1. 稳定跨视图标识符，使结构化、向量、图能关联。
    2. 图节点必须对应有效结构化记录。
    3. 高影响图边必须有证据记录支撑。
    4. 结构化状态（如退役、取代）主导检索，即使向量或图仍然可达，也不能用于生成。
  - **选择流程**：
    - 首先应用结构化门控得到授权集 \( A_t \)。
    - 在授权集内做向量搜索获得候选 \( N_k(x_t) \)。

    - 最终组装成**结构化上下文包**（context packet），包含本地状态、情景证据、语义声明、关系注释、外部证据、**负面上下文**（明确告知生成器哪些记忆被拒绝或已过时）。
  - **时间承诺与晋升**：短期记录可快速写入，长期语义承诺需要多次支持证据才晋升，提供稳定性-可塑性平衡。
  - **撰写、选择、合并、修订、边界投影**五阶段循环，其中合并和修订处理矛盾、衰减和修正。

### 3. 实验设计：使用的数据集/场景、基准、对比方法

- **使用的场景与基准**：论文设计了一个**诊断基准**（diagnostic benchmark），包含 **800 个确定性生成的合成任务**，分布在8个任务族：
  - 延迟回忆 (Delayed recall)
  - 近误边界 (Near-miss boundary)
  - 源边界 (Source boundary)
  - 修订 (Revision)
  - 陈旧抑制 (Stale suppression)
  - 证据归因 (Evidence attribution)
  - 矛盾处理 (Contradiction)
  - 时间承诺 (Temporal commitment)
- 每个任务族 100 个任务。场景平均含 17 条记忆记录，并包含元数据、图边、黄金标签（gold memory identifiers）。
- **对比方法（消融条件）**：
  - **Recent context only**：仅当前提示窗口。
  - **Vector only**：仅向量检索（按用户范围，无状态/范围/图）。
  - **Structured + vector**：在向量检索前加精确门控（用户、主题、任务、源）。
  - **Structured + vector + temporal**：额外加状态、层、时间策略（处理陈旧抑制、弃权）。
  - **Full MRMS**：加上图解析（支持、矛盾、取代、派生自边）。
- **评估指标**：总体准确率、修订成功率、陈旧抑制、边界控制、归因、弃权准确率、辅助指标（选中的记忆数、上下文令牌近似值）。

### 5. 实验数量与充分性

- **实验总量**：1 个主实验（800 个任务），5 个消融条件 + 全系统，共 8 个任务族 × 不同条件。
- **消融覆盖**：系统性地移除了“结构化门控”、“时间策略”、“图解析”等组件，对应每种组件的作用被清晰隔离。
- **充分性分析**：
  - **充分**：任务族覆盖了长期记忆中的关键失败模式（边界污染、陈旧召回、矛盾、归因缺失等）。每个族 100 个任务，统计上稳定（报告了 95% 自助法置信区间）。
  - **客观公平**：所有条件使用相同的合成数据生成器，黄金标签为记忆标识符，避免 LLM 判断偏差。但是**未与现有商业/开源系统（如 MemGPT、MemoryBank、Mem0、Zep）进行直接数值比较**，因为那些系统输出最终答案而非生成前记忆决策。作者认为 MRMS 专注于“什么记忆应该被使用”这一不同的测量层级，因此采用受控消融更合适。这一点既是特点也是局限。

### 6. 论文的主要结论与发现

- **全系统 MRMS 在 800 个任务上获得 98.8% 总体准确率**（95% CI: 98.0–99.4）。
- 每个消融组件的关键贡献：
  - 向量检索改善了回忆，但会召回过时或超出范围的对象。
  - 结构化门控完美地修复了边界控制（100%），但未能处理矛盾和归因。
  - 时间策略修复了陈旧抑制和弃权（100%）。
  - 图解析修复了矛盾处理（100%）和证据归因（95%），是归因任务的关键。
- **全 MRMS 保持上下文紧凑**：平均选择 1.11 个记忆对象（评分前候选者 1.8 个）。
- **可靠个性化是记忆设计问题**：记忆必须结构化、选择性暴露、持续合并、带认识标签，而不是无差异的历史转录。
- 结论强调了“生成前控制”的重要性：决定哪些记忆应该被使用，比最大化召回率更关键。

### 7. 优点

- **方法创新**：将同步结构化-向量-图记忆作为一个显式架构约束，而非实现选择。
- **设计清晰**：严格区分了三种表征在记忆生命周期中的不同权威（结构化控制生命周期、向量支持召回、图支持关系推理），并明确同步不变性。
- **评估系统化**：开发了专门针对预生成记忆决策的诊断基准，而非仅依赖下游任务完成率。消融设计精巧，每个组件的影响被透明地量化。
- **可审计**：输出结构化上下文包和决策轨迹（授权记录、排名候选、图扩展、拒绝记录），便于错误分析和后续集成。
- **与现有框架互补**：MRMS 可以叠加在 RAG、长期上下文模型、智能体反思之上，负责管理记忆的“准入”和“关联”。

### 8. 不足与局限

- **实验覆盖限制**：
  - 仅使用**合成确定性数据**，未在真实多轮对话或个人助理交互中测试。

- **对比偏差风险**：
  - **未与现有系统（MemGPT、MemoryBank、Mem0、Graphiti 等）进行直接数字比较**。作者声称那些系统的输出不是预生成记忆决策，但缺乏头对头对比可能让读者难以判断 MRMS 在实际部署中的相对优势。
