## 摘要
LLM代理在长周期任务中往往会退化：随着轨迹增长，它们会重新访问已探索的状态，重复失败的动作，并丢失之前有效的策略。测试时训练提供了一种使模型权重适应不断变化的任务状态的方法，但现有的LLM测试时训练方法大多是一次性适应固定输入。我们研究了多轮代理交互中的持续测试时训练，其中每次更新都会改变生成后续训练文本的策略。这创建了一个自训练循环，当新轨迹信息出现时有所帮助，但当代理陷入困境并重复训练相似文本时，可能会放大漂移。我们发现更新文本的重复性区分了这些情况，并提出了自主测试时训练，一种token级重新加权方法，降低先前更新中重复n-gram中出现token的损失权重，同时保持新token的完全权重。为了在实时交互中进行此类更新，我们使用vLLM的运行时LoRA API构建了一个并发服务系统，将开销限制为无测试时训练成本的1.9倍。自主测试时训练在ALFWorld上提升成功率最多5.0个百分点，在SWE-bench Lite上提升4.9个百分点。这些增益集中在模型已具备任务能力但在长轨迹中漂移的地方，表明自主测试时训练主要保持现有能力而非教授新技能。

## Abstract
LLM agents often degrade over long episodes: as trajectories grow, they revisit explored states, repeat failed actions, and lose strategies that previously worked. Test-time training (TTT) offers a way to adapt model weights to the evolving task state, but existing LLM TTT methods largely adapt once to a fixed input. We study continuous TTT in multi-turn agent episodes, where each update changes the policy that generates later training text. This creates a self-training loop that helps when new trajectory information appears, but can amplify drift when the agent gets stuck and repeatedly trains on similar text. We find that update-text repetition distinguishes these regimes and introduce Agentic Test-Time Training (aTTT), a token-level reweighting method that downweights the loss on tokens appearing in repeated $n$-grams from prior updates while leaving novel tokens fully weighted. To run such updates inside live episodes, we build a concurrent serving system using vLLM's runtime LoRA API, limiting overhead to 1.9$\times$ the no-TTT cost. aTTT improves success by up to 5.0 points on ALFWorld and 4.9 points on SWE-bench Lite. The gains concentrate where models already have task competence but drift over long trajectories, suggesting that aTTT mainly preserves existing competence rather than teaching new abilities.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：LLM 代理在长周期（multi-turn）任务中经常退化：会重复访问已探索状态、重复失败动作、丢失先前有效策略，导致任务失败。这种失败并非模型能力不足，而是随时间推移出现的**轨迹内漂移**。
- **现有方法局限**：Reflexion 等跨 trial 方法不改动单 trial 内的策略；长上下文窗口难以有效利用早期信息；解码时重复惩罚不改变策略。静态测试时训练（TTT）只适应一次固定输入，无法处理不断增长、自我生成的训练流。
- **核心挑战**：多轮代理中的持续 TTT 会产生内生的**自训练循环**——当代理陷入困境时，重复的行为变成重复的训练文本，进一步强化失败模式。成功轨迹与失败轨迹的关键区别在于**更新文本的新颖性**（novelty）。
- **本文意义**：提出一种能在 episode 内持续自适应、同时抑制有害自训练循环的框架，提升长轨迹任务的 agent 成功率。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：Agentic Test-Time Training (aTTT) —— 在 agent 与环境交互过程中，每隔 K 步进行一次权重更新，更新时使用 token 级损失重加权，降低重复 n-gram 的梯度贡献，保留新颖 token 的全权重，从而抑制反复训练相同模式导致的漂移。
- **关键技术细节**：
  - **更新信号来源**（三种）：
    - Self：agent 自身的推理和动作 token。
    - Env：环境给出的观察文本。
    - Summary：调用额外 LLM 生成的轨迹摘要。
  - **更新流程**（Algorithm 1）：
    1. 对当前更新文本 x_k 分词，得到 token 序列。
    2. 对每个 token 位置 j，计算其所在的所有 n-gram 在先前的更新历史 H_tok^k 中的最大出现次数 f_k(j)。
    3. 分配权重 w_k,j = max( w_min, 1/(1 + f_k(j)) )，频繁重复的 token 权重递减至 w_min。
    4. 用重加权后的交叉熵损失更新 LoRA 适配器。
    5. 累加当前 token 序列到历史。
  - **更新节奏**：默认每 5 个 agent 步骤更新一次（K=5）。
  - **适配器**：LoRA rank=8，α=16，学习率 5×10⁻⁴，每步 2 个梯度步；episode 内持续，episode 间重置。
- **重复性度量**：基于词级 Jaccard 相似度 ρ(x_k) = max_{x'∈H_k} J( W(x_k), W(x') )，用于序列级过滤 baseline。
- **系统实现**：基于 vLLM 运行时 LoRA API，16 个 episode 并发，单独训练 GPU 异步更新适配器并热切换，不阻塞其他 episode。
