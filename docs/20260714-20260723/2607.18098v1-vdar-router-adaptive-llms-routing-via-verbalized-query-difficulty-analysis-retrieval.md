## 摘要
大语言模型越来越多地应用于实际系统，使得高效的模型选择对于降低部署成本至关重要。LLM路由已成为一种在期望成本-性能权衡下为每个输入查询分配适当模型的实用解决方案。现有路由方法通常从输入查询的表面语义或嵌入相似性来估计模型适用性。然而，此类方法可能忽略查询的潜在难度，导致次优的路由决策。为解决这一挑战，我们提出了VDAR-Router，一种基于难度感知检索的路由框架。对于每个输入查询，VDAR-Router首先生成显式的难度分析。然后，它检索具有相似难度特征的历史示例。基于检索到的记录，它估计候选模型的适用性，并使用考虑性能和成本的奖励函数选择模型。在三个数据集上的实验表明，VDAR-Router比现有基线始终实现更好的成本-性能权衡。这些结果证明了难度感知检索对于无训练LLM路由的有效性。案例研究进一步表明，显式查询分析有助于检索更相关的示例并支持更可靠的路由决策。

## Abstract
Large language models are increasingly used in practical systems, making efficient model selection important for reducing deployment cost. LLM routing has emerged as a practical solution for allocating each input query to an appropriate model under a desired cost-performance trade-off. Existing routing methods often estimate model suitability from the surface semantics or embedding similarity of the input query. However, such methods may ignore the underlying difficulty of a query, leading to suboptimal routing decisions. To address the challenge, we propose VDAR-Router, a difficulty-aware retrieval-based routing framework. For each input query, VDAR-Router first generates an explicit difficulty analysis. It then retrieves historical examples with similar difficulty profiles. Based on the retrieved records, it estimates candidate model suitability and selects the model using a reward function that considers both performance and cost. Experiments on three datasets show that VDAR-Router consistently achieves better cost-performance trade-offs than existing baselines. These results demonstrate the effectiveness of difficulty-aware retrieval for training-free LLM routing. Case studies further show that explicit query analysis helps retrieve more relevant examples and supports more reliable routing decisions.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：大语言模型（LLM）在实际部署中面临成本与性能的权衡。LLM路由旨在为每个输入查询自动选择最合适的模型，以在给定预算下获得最佳响应质量。
- **现有方法的不足**：现有路由方法多依赖查询的表面语义或嵌入相似性（如KNN、检索增强），忽略了查询的内在难度差异。例如，两个文本外观相似的查询（如“建个人网站”与“建电商网站”）难度可能截然不同；而表面不相似的查询（如简单算术题）可能共享相同的难度特征。仅基于语义相似性检索会导致路由信号不可靠。
- **本文贡献**：提出**VDAR-Router**，通过**显式生成查询的难度分析**，并基于难度相似性检索历史示例，再利用检索到的性能-成本记录进行模型选择。该方法无需额外训练，可直接部署，且决策过程可解释。

## 2. 方法论
- **核心思想**：假设**难度相似的查询需要相似的模型能力**，因此各模型在难度相似的查询上表现一致。通过显式分析查询所需的各项能力维度（推理、理解、指令遵循、代理能力、知识检索、编码、多语言等），将查询映射到“难度特征空间”，用该空间进行检索，可更准确地估计模型适用性。
- **关键技术细节**：
  1. **难度分析生成（Difficulty Analyst）**：使用一个LLM（如Gemma4-31B或Qwen3.5-2B）作为分析器，对输入查询 `q` 生成自然语言描述的难度分析 `a`，涵盖所需能力的程度。
  2. **离线建库**：将训练集所有查询的难度分析通过嵌入模型 `E`（Qwen3-Embedding-0.6B）编码成向量，存入数据库。
  3. **检索**：测试时，对输入查询生成难度分析 `a`，嵌入后检索 `k` 个最相似的难度分析（含对应查询 `q_r` 和每个模型的性能分数）。
  4. **奖励排序**：基于检索到的 `k` 个查询，计算每个候选模型 `m` 的平均奖励：
     \[
     R(m|q) = \frac{1}{k}\sum_{i=1}^k \left( \alpha \cdot p_m(q_i^r) - \beta \cdot c_m(q_i^r) \right)
     \]
     其中 `p_m` 为性能分数，`c_m` 为归一化成本，`α + β = 1`。
  5. **选择**：选择奖励最高的模型 `\hat{m} = \arg\max_{m \in M} R(m|q)`。

- **无需训练**：整个框架不引入可训练参数，属于训练无关（training-free）方法。

## 6. 主要结论与发现
- VDAR-Router在**所有三个数据集上均取得最高奖励**（RouterBench和LLMRouterBench）以及**最佳成对精度和Spearman相关**（ArenaExpert5K），显著优于所有基线。
- **难度分析检索比查询嵌入检索更有效**：消融实验显示替换为查询嵌入后奖励下降；案例研究和Rasch难度差距分析证实难度检索能找到真正难度相似的示例。
- **奖励重排序（整合成本）是必要的**：移除后路由器倾向于选择最强模型，导致高成本、低奖励。
- **VDAR-Router对分析器模型大小不敏感**：使用2B模型仍能达到或超过大部分基线。
- **在小训练集（50/50分割）下优势更明显**，表明难度特征能更高效地从有限数据中提取有用信号。
- **阈值影响**：更大的 `α`（更重视性能）带来更高性能；更大的 `k` 在RouterBench和LLMRouterBench上稳定提升，在ArenaExpert5K上效果不那么单调（原因可能与偏好数据的主观性有关）。

## 7. 优点
- **免训练**：无需额外训练路由器，依赖预训练嵌入模型和难度分析LLM，可即插即用。
- **可解释性**：难度分析以自然语言呈现，每个路由决策都有明确依据。
- **成本意识**：通过奖励公式显式权衡性能与成本，支持动态调整偏好。
- **鲁棒性**：对小分析模型（2B）仍保持竞争力；在不同数据划分下均表现优异。
- **检索质量高**：Rasch难度对比证实检索到的示例在难度上更接近目标查询。

## 8. 不足与局限
- **测试时难度分析生成带来额外延迟**：虽可使用小模型缓解，但未深入评估延迟对实时性的影响。
