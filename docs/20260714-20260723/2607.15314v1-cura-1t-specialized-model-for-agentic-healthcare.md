## 摘要
医疗保健涉及高风险沟通、专业推理和工作流执行，然而同时涵盖这些用例的专业大语言模型仍然有限。一个医疗模型必须处理患者咨询、基于文本和图像的临床推理、交互式诊断以及电子健康记录（EHR）工具的使用。这些能力的失败方式各不相同，针对某一任务的狭义更新可能会损害另一任务。我们提出了Cura 1T，一个通过人类监督的自进化循环训练而成的医疗专业LLM。在每个进化轮次中，一个训练代理会规划目标能力、训练模型、评估基准轨迹，并根据观察到的失败改进数据混合。这种以数据为中心的循环通过有针对性的合成和精选示例来改进模型，而非单一的一般性医疗数据更新。在整个医疗评估套件中，Cura 1T在前沿基线中排名前列或接近前列，同时在域外推理和智能体基准测试中保持竞争力。

## Abstract
Healthcare spans high-stakes communication, expert reasoning, and workflow execution, yet specialized LLMs that cover these use cases together remain limited. A healthcare model must handle patient consultation, clinical reasoning over text and images, interactive diagnosis, and electronic health record (EHR) tool use. These capabilities fail in different ways, and a narrow update for one task can degrade another. We present Cura 1T, a healthcare-specialized LLM trained through a human-gated self-evolution loop. In each evolution round, a training agent plans a target capability, trains the model, evaluates benchmark trajectories, and refines the data mixture from observed failures. This data-centered loop improves the model through targeted synthetic and curated examples rather than a single generic medical-data update. Across the healthcare evaluation suite, Cura 1T ranks at or near the top among frontier baselines, while remaining competitive on out-of-domain reasoning and agentic benchmarks.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：社区在单个任务上有进展（如Med-PaLM、MedXpertQA、AgentClinic等），但缺乏一个统一模型。同时，手动迭代后训练（规划→训练→评估→修改数据配比）耗时且依赖专家。受自动化科研工作流启发，论文提出一个**人类门控的自我进化循环**，由LLM代理自动执行数据配比搜索与优化，以实现医疗专业化后的多任务平衡。

## 2. 方法论：核心思想、关键技术细节、公式与算法流程

- **核心思想**：采用“人类门控自我进化循环”进行后训练。循环对象不是超参数或网络结构，而是**后训练数据混合（data mixture）**。在每一轮进化中，一个训练代理（LLM Agent）按照“计划→训练→评估→精炼→人工审查”的顺序迭代，根据评估失败轨迹合成针对性数据，更新混合配比，从而逐步提升模型在所有目标能力上的表现，同时防止遗忘。

- **关键技术细节**：
  - **基座模型**：Kimi-K2.6（256K上下文，文本+视觉输入），参数量约1T（万亿）。
  - **训练栈流程**：每轮依次执行：
    1. **SFT（监督微调）**：作为低成本的配方和超参数验证步骤，确保数据混合和设置稳定。
    2. **RL（强化学习）**：基于奖励信号优化策略。
    3. **SDFT（自蒸馏微调）**（Shenfeld et al., 2026）：最终巩固步骤，使用**特权上下文**的教师分布引导学生分布，避免复制离策略轨迹破坏原生推理能力。其目标函数为：
       \[
       L(\theta) = D_{\text{KL}}\left(\pi_\theta(y|x) \parallel \pi(y|x, c)\right) = \mathbb{E}_{y \sim \pi_\theta(\cdot|x)}\left[\log \frac{\pi_\theta(y|x)}{\pi(y|x, c)}\right]
       \]
       其中 \(c\) 是额外信息（如干预后的轨迹、参考行为、验证知识等），训练后对学生模型只保留原提示 \(x\)。
  - **微调方式**：使用**LoRA适配器**（rank=32），保持基座权重不变，参数高效更新。
  - **数据合成技能**（精炼阶段）：
    - **Retention Anchor**：添加已解问题的少量示例，防止修复时遗忘。
    - **Reasoning Correction**：用前沿模型编辑失败推理轨迹，并生成相似问题。
    - **Knowledge Injection**：检索缺失知识，合成封闭书式QA。
    - **Behavior Calibration**：对比期望与观察行为，生成基于评分标准的示例。
    - **其他**：处理智能体工作流错误等。
    - **Data Mixture Curation**：合并、去重、加权、封顶。

- **算法流程（文字说明）**：
  1. **Plan**：训练代理定义目标行为、评估基准、指标、初始数据配比、候选超参数。需人类审批计划。
  2. **Train**：使用SFT验证混合；通过RL优化奖励；最后用SDFT巩固。
  3. **Evaluate**：运行基准测试，收集评分轨迹和失败摘要。
  4. **Refine**：根据失败原因分类（知识缺失、推理错误、行为偏离等），使用上述技能合成新数据，加入保留锚点，验证格式、安全、去重后输出新混合。
  5. **Human Review**：审查精炼后的数据混合和候选模型，决定保留（继续下一轮）、回滚（废弃本轮修改）、精细调整或部署。回滚的轮次记录但不作为后续基础。

## 7. 优点

- **方法创新**：提出**人类门控的自我进化循环**，将数据混合构造作为搜索对象而非超参数，适合数据稀疏且失败模式多样化的医疗领域。

- **开源发布**：模型、文档、代码开源，促进后续研究。
