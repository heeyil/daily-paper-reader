## 摘要
智能体在长周期LLM系统中的自我进化很大程度上是程序性的：有用的经验不仅仅是存储的信息，而是可重用的搜索、调试和验证程序。然而，当前的评估并未将这种形式的迁移分离出来。智能体基准测试评估单轮任务求解；记忆基准测试针对信息保留而非程序复用。我们引入了EvoAgentBench，这是一个通过能力引导迁移评估智能体自我进化的基准测试，涵盖四个智能体领域：网络研究、算法推理、软件工程和知识工作。EvoAgentBench从智能体执行中提取基于轨迹的能力，将其规范化到操作单元，并构建特定领域的能力图，连接共享程序重叠的任务。通过设计，每个测试任务都有经过验证的训练侧能力支持。在528/267的训练/测试分割、两种支架和三种基座模型上，精心策划的能力内容能够在不同模型系列之间可靠地迁移，但当前没有任何自动方法在所有设置中持续获得正向收益。EvoAgentBench将自我进化评估从总体准确性比较转向对经验编码、路由和采纳的细粒度诊断。该基准测试公开可用：https://huggingface.co/datasets/EverMind-AI/EvoAgentBench。

## Abstract
Agent self-evolution in long-horizon LLM systems is largely procedural: useful experience is not merely stored information, but reusable procedures for searching, debugging, and verification. Yet current evaluations do not isolate this form of transfer. Agent benchmarks test single-episode task solving; memory benchmarks target information retention rather than procedural reuse. We introduce EvoAgentBench, a benchmark for agent self-evolution via Ability-guided transfer across four agentic domains: web research, algorithmic reasoning, software engineering, and knowledge work. EvoAgentBench extracts trace-grounded Abilities from agent executions, canonicalizes them into operational units, and builds domain-specific Ability Graphs linking tasks that share procedural overlap. By design, every test task is backed by verified training-side Ability support. Across a 528/267 train/test split, two scaffolds, and three backbones, curated Ability content transfers reliably across model families, but no current automatic method sustains positive gain in all settings. EvoAgentBench shifts self-evolution evaluation from aggregate accuracy comparison to fine-grained diagnosis of experience encoding, routing, and uptake. The benchmark is publicly available at https://huggingface.co/datasets/EverMind-AI/EvoAgentBench.

## 2. 论文提出的方法论

### 2.1 核心思想
- 定义一个**能力（Ability）** 为从智能体执行轨迹中提取的可重用操作（如搜索策略、调试步骤、验证流程），具有触发条件、可重用步骤、适用边界和角色（方法/防护/工作流）。
- 构建**能力图（Ability Graph）**：节点是任务，边表示共享至少一个可通用能力。通过图分割保证测试任务与训练任务拥有共同的能力邻居，从而**确保迁移可能性**。

### 2.2 关键技术细节
- **多骨干轨迹收集**：对每个任务，使用三个构造骨干（Kimi-K2.5, GLM-5.1, DeepSeek-V3.2）和两个支架（OpenClaw, Nanobot）收集无进化的执行轨迹，形成多骨干证据池。
- **痕迹基础的能力提取**：提取器（Claude Sonnet 4.6）联合分析每个任务的多骨干轨迹，生成任务本地的原始能力卡片（包含触发条件、流程、证据、边界、角色）。
- **能力规范化**：通过嵌入相似性进行阻塞候选对 → 三个LLM独立裁决（操作等价性） → 全票通过自动接受，否则领域专家评审 → 组一致性检查后合并为规范能力单元。
- **能力图构建**：每个领域构建无向图，边代表任务至少共享一个可通用能力（边缘有效能力需至少支持两个不同任务且保持操作特异性）。
- **能力感知的拆分**：在社区（Louvain）约束下采样测试任务，确保每个测试任务与至少一个训练任务共享可通用能力，并被偏好选择基准奖励留下空间的（非全对/全错）任务。

## 3. 实验设计

### 3.1 数据集/场景
- **四个智能体领域**（来自已有基准的子集）：

### 3.2 Benchmark对比方法
- **Vanilla**：无进化状态。
- **自动自我进化方法**：
  - **Memento**：检索最相似训练案例（query, plan, outcome）
  - **ReasoningBank (RB)**：检索蒸馏的推理记忆池（自然语言策略）
  - **GEPA**：在训练集上进化单一提示并广播给所有测试任务（无检索）
- **诊断参考**：**Anchor Skill** – 使用策划师侧能力标签（来自提取流水线）检索能力根植的技能，内容仅来自训练侧证据，但路由标签来自测试任务的能力簇（非可部署方法）。

### 3.3 实验设置
- **两个支架**：OpenClaw（功能丰富）与 Nanobot（轻量级）
- **三个骨干模型**：Qwen3.5-27B, Qwen3.5-397B, Gemma-4-31B（模型系列互斥，与构造骨干不同）
- **共6个支架-骨干设置**，每个方法在各自域测试集上运行3次独立实验，报告标准误差。

## 6. 论文的主要结论与发现

1. **人工策划的能力可跨模型族迁移**：Anchor Skill在所有24个域-设置单元中均产生正向增益（平均+5.8%至+10.5%），证明能力内容原则上可迁移。
2. **当前自动方法仍脆弱**：每个自动方法（Memento, ReasoningBank, GEPA）至少在一个单元中出现负迁移（负误差带状）；平均增益偏低（-2.4%至+5.7%）。
3. **瓶颈在于方法侧的提取与路由**：Anchor使用策划师侧能力标签，自动方法需自行从训练轨迹提取并路由，效果远不及。
4. **成本-准确率解耦**：减少交互次数不一定对应提升准确率；许多方法在已解决任务上仍增加了turn数（产生额外开销）。
5. **多设置评估至关重要**：单一支架或骨干下的结果不能泛化；模型缩放不直接提升迁移效果。

## 7. 优点

- **创新性评估框架**：首次将自我进化评估从聚合准确率转向对“编码-路由-吸收”的细粒度诊断，通过能力图提供结构性保障。
- **严格的泄漏控制**：自动方法仅获得训练侧证据；Anchor技能内容完全来自训练侧原始卡片，测试侧信息仅借由路由标签进入（明确标记为非可部署方法）。
- **多领域、多支架、多骨干覆盖**：提供跨设置的稳定性检验，揭示方法-域-支架交互作用。
- **成本与性能联合报告**：暴露方法间行为异质性，避免仅以准确率误导结论。
- **开放性**：基准公开可用，促进复现与扩展。

## 8. 不足与局限

- **领域限制**：仅四个文本智能体领域（网络、算法、软件、知识），未涉及多模态或具身智能体。
- **提取器单一**：能力提取使用单一LLM（Claude Sonnet 4.6），可能引入该模型特有的偏见（虽然后续三法官评审+专家审查部分缓解）。

- **计算资源未披露**：阻碍可重复性和资源需求估计。
