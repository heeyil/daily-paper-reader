## 摘要
训练工具使用代理以从自身经验中改进仍然具有挑战性，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长程交互提供的监督较弱。我们提出DeepSearch-Evolve，一种基于DeepSearch-World构建的Web代理自我蒸馏框架，DeepSearch-World是一个确定且可验证的环境，配备可复现的搜索和页面阅读工具。DeepSearch-World包含从实体级随机游走构建的42万个多跳问答任务，并支持对自我进化有用的关键代理认知行为，包括进度验证、基于事实的反思和失败恢复。DeepSearch-Evolve迭代执行轨迹生成、过滤、数据混合和微调以训练更强的代理。在没有从更强大模型蒸馏的情况下，DeepSearch-World-9B在与开源代理相比取得了有竞争力的性能，在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，表明可验证环境能够实现长程Web代理的可扩展自我进化。我们将发布环境、42万训练池、验证集、模型和代码，以促进未来关于自我改进的深度搜索代理的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程
- **核心思想**：用确定性可验证环境替代不可控的实时网络，使工具调用行为的每一步都可以被自动验证（实体级答案匹配），从而生成高质量的行为轨迹；然后通过“支架教师→ReAct格式转换→SFT训练”的循环，将规划、记忆、反思、错误恢复等能力注入学生模型。
- **关键技术细节**：
  - **DeepSearch-World环境**：
    - 基于Wikipedia超链接图，通过实体级随机游走构建42万条多跳QA任务，每个任务有隐藏目标实体集。
    - 提供两个离线工具：`web_search_wiki`（BM25检索，返回标题+片段+URL）和`visit_wiki`（通过SQLite偏移索引返回完整文章），完全确定、可复现。
    - 环境支持**顺序无关的进度验证**：每次工具调用后自动匹配未解决的目标实体，生成基于事实的反射信号（`[REFLECTION]`）。
  - **支架教师代理**：分三阶段（Plan → Act → End）生成轨迹。Plan阶段初始化进度状态（completed_list, todo_list, experience, information）；Act阶段交替工具调用、状态更新和环境反射；End阶段基于已验证的工作记忆生成最终答案。
  - **自我进化训练循环**：
    1. 当前模型在DeepSearch-World中生成支架轨迹（教师）。
    2. 通过正确答案验证（拒绝采样）和LLM质量过滤（评估冗余性、目标对齐、逻辑一致性），保留高质量轨迹。
    3. **支架到ReAct转换**：移除阶段提示，将进度状态和反射重写为学生模型的`<think>`块，形成标准ReAct格式训练数据。
    4. SFT更新学生模型（等价于硬标签令牌级策略蒸馏），使用重要性采样混合来自不同轮次的数据（衰减因子γ=0.5）。
    5. 学生成为下一轮教师，迭代进行。
- **算法流程（文字说明）**：初始化模型→在确定性环境中生成支架轨迹→过滤（拒绝采样+质量过滤）→转换为ReAct格式→SFT训练→更新模型→重复。异步机制：生成和训练解耦，生成不阻塞训练。
