## 摘要
一个编码智能体在解决软件工程任务时，会经历数十步的推理、代码编辑和测试运行，但人们对底层语言模型内部如何表示正在处理的程序知之甚少。我们表明，编码智能体下的语言模型残差流线性地编码了演化程序的属性：一个基于隐藏状态的对数几率回归探针能够解码当前代码是否解析通过、是否通过测试套件、是否减少失败测试数量以及是否引入回归，在两个模型和两个基准测试中，正确性的AUC最高可达0.83。第二个发现更为令人惊讶：这些表示先于智能体自身的编辑。训练用于预测未来编辑结果（在它们被具体化并写入磁盘之前）的探针，其性能在约25步之前就超过了随机水平。我们称此为智能体的潜在编程视界。作为外部有效性的证明，我们展示了这些探针可以在无需重新训练的情况下跨基准测试迁移。我们的积极结果呼吁对编码智能体的机械可解释性进行更多研究。

## Abstract
A coding agent solving a software-engineering task spends dozens of steps reasoning, editing code, and running tests, yet little is known about what the underlying language model internally represents about the program it is working on. We show that the residual streams of language models under coding agents linearly encode properties of the evolving program: a logistic-regression probe on hidden states is able to decode whether the current code parses, passes its test suite, reduces the number of failing tests, and introduces regressions, reaching AUC up to 0.83 for correctness across two models and two benchmarks. Our second finding is more surprising: these representations run ahead of the agent's own edits. Probes trained to predict the outcome of future edits (before they are materialized and written on disk) achieve performance above chance up to roughly 25 steps in advance. We call this the agent's latent programming horizon. As a proof of external validity, we show that the probes transfer across benchmarks without retraining. Our positive results open calls for more research in mechanistic interpretability of coding agents.

## 1. 论文的核心问题与整体含义

编码智能体在解决复杂软件工程任务时，会经历多步推理、代码编辑和测试执行，但底层语言模型内部如何表示正在处理的程序几乎不为人知。本文旨在回答：**编码智能体在内部隐层状态中如何表示它正在编辑的程序？** 核心发现是：语言模型的残差流线性地编码了当前程序的属性（如编译通过、测试通过、回归等），并且这些表示具有前瞻性，能提前约25步预测未来编辑的结果。作者将这种前瞻能力称为“潜在编程视界”（Latent Programming Horizon）。这项工作填补了编码智能体可解释性研究的空白，证明模型在内部维护了一个不断更新的程序表示空间，即使这些表示并不直接体现在生成的输出中。

## 2. 方法论

### 核心思想
- 在编码智能体执行任务的过程中，记录每个步骤的语言模型隐藏状态（残差流向量 h^ℓ_t），然后训练线性探针（logistic regression）从这些隐藏状态中解码程序的二进制属性。
- 探针分为两类：**当前状态探针**（k=0）用于解码当前程序的属性；**前瞻探针**（k>0）用于预测未来 k 步后程序的属性，从而测量潜在编程视界的长度。

### 关键技术细节
- **隐藏状态提取**：在智能体生成的每个token位置（每5个token采样一次），提取不同层（ℓ ∈ {1, 11, 21, 31, 40}）的残差流向量，维度 d=2048。
- **程序属性定义**（共4个二进制属性）：
  - ▲ Well-formedness（编译/解析是否通过）
  - • Full correctness（是否通过测试套件，即任务成功）
  - ■ Partial correctness（测试失败数是否少于初始状态）
  - ♦ Regression（是否有原本通过的测试现在失败）
- **探针训练**：使用逻辑回归分类器，权重 w ∈ ℝ^d，通过随机搜索超参数（学习率、权重衰减、batch size、patience），以验证集AUC为目标选择最优配置。每个属性、每个层、每个模型、每个数据集独立训练。
- **标签分配**：标签基于每个编辑事件后的程序版本计算，并持久化到下一次编辑。将初始程序视为一个虚拟编辑事件，确保所有token都有标签。
- **前瞻实验**：对于每个属性 ϕ 和视界 k（0到50步），训练探针从步骤 t 的隐藏状态预测步骤 t+k 的程序属性。不同 k 对应不同的探针，比较AUC随 k 的变化。
- **控制实验**：对每个探针训练随机排列标签的版本作为基线，确保信号来自隐藏状态而非探针容量。

### 算法流程（文字说明）
1. 运行 mini-swe-agent 在每个任务上生成多条轨迹，记录每步编辑后的程序版本。
2. 对每个程序版本评估四个属性，得到标签序列。
3. 在轨迹中提取每个token位置的隐藏状态，与对应标签对齐。
4. 按任务ID划分训练/测试集（确保同一任务不跨集）。
5. 对每个（属性、层、模型、数据集、视界k）组合，训练logistic回归探针。
6. 计算测试AUC，并与随机标签基线比较。
7. 对跨基准迁移实验：将在一个数据集上训练的探针直接应用于另一数据集的隐藏状态，无需微调。

## 6. 主要结论与发现

1. **编码智能体编码当前程序属性**：线性探针能够从隐藏状态解码四个属性，其中Full Correctness AUC最高达0.83，Partial Correctness达0.84，显著高于随机基线0.5。这表明模型内部维护了丰富的程序语义表示。
2. **表示集中在中间层**：探针性能随层数先升后降（倒U型），在中间层（L21-L31）达到最佳，第一层最弱，最后一层略下降。
3. **模型差异**：Qwen3.6-35B-A3B的编码能力整体强于Laguna-XS.2（AUC高约0.1），但模式一致。
4. **跨基准迁移**：探针可以从SWE-Bench-Verified迁移到Pro（反之亦然），Full Correctness和Partial Correctness的AUC下降仅0.04-0.09，证明表示是通用的而非过拟合。
5. **长期潜在编程视界**：前瞻探针在k=25步时AUC仍高于0.55（Verified）和0.60（Pro），远优于随机基线；k=50步时仍保持微弱但显著的预测能力。这表明模型在编辑发生之前就已经“预知”了未来程序的属性。
6. **模式规律**：前瞻AUC随k增加平滑下降（冰球杆模式），在k接近0时最高，随后快速下降，进入平台期。不同层的排序与当前状态探针一致。

## 7. 优点

1. **新颖问题定义**：首次研究多步编码智能体中的隐层程序表示，并提出“潜在编程视界”这一概念，为智能体可解释性开辟新方向。
2. **大规模、多维度实验**：覆盖两个模型、两个基准、超过22K轨迹、222M隐藏状态，实验全面，结果具有统计意义和外部有效性。
3. **简洁有效的方法**：使用线性logistic回归探针，避免了复杂模型引入的过度拟合风险；控制实验和跨基准迁移证明了信号的稳健性。
4. **前瞻性结论的实证说服力强**：通过系统地扫描k从0到50步，展示了预测能力随时间的平滑衰减，并提供了k≤15的补充实验排除样本偏差。
5. **代码和数据公开**：论文承诺公开代码和数据，有利于可重复性。

## 8. 不足与局限

1. **解码不等于因果**（Decodability is not causality）：探针仅显示程序属性可解码，但不证明模型实际使用这些表示来指导行为。未来需要因果干预实验（如沿探针方向进行激活操控）来验证。
2. **标签不平衡**：Well-formedness属性在SWE-Bench-Verified中正例率极高（>0.92），导致AUC接近随机，但其实Brier分数较低（因为常预测器准确率高）。类似的，Full Correctness正例率仅0.08-0.30，但AUC却最高，表明预测较难属性时信号更明显。
3. **范围有限**：仅研究两个开源模型（且均非前沿模型）、一个智能体框架（mini-swe-agent）、两个基准（SWE-Bench系列）。基于其他模型（如GPT-4）、其他框架（如OpenHands）或其他任务的结果尚待验证。
4. **前瞻预测的非因果解释**：前瞻预测能力强可能是因为模型在当前状态已经隐含了未来步骤的推理上下文（如计划、讨论），而非纯粹的“对未生成代码的表示”。虽然论文认为这仍然是编程视界的证据，但需要进一步控制实验来区分。
5. **计算资源未具体说明**：无法评估实验的可重复成本，可能对资源受限的研究者构成障碍。
6. **前瞻实验的数据过滤**：k≥50时需排除短于50步的轨迹，可能引入选择偏差（更成功的轨迹往往更长）。作者用k≤15实验验证了模式一致，但高k值下的普适性仍需更多验证。
