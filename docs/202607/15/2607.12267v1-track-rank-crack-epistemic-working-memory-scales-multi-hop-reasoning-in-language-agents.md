## 摘要
语言代理在进行推理和工具调用时，随着推理链的延长，性能会急剧下降，即使每个单独步骤都很容易。我们将此归因于上下文稀释：代理的调查状态（它已确认了什么、怀疑什么、还需要什么）仅隐式地存在于不断增长的上下文窗口中，早期发现被后来的检索淹没。我们引入了SLEUTH，通过结构化的认知工作记忆使这种状态显式且可操作：代理维护基于来源的已确认事实、按证据排序的活跃假设以及直接驱动下一步行动的开放问题。在五个多跳基准和五个已有基线上，SLEUTH的优势随难度增加而增大，从HotpotQA上的+5分到4跳链上的+11分，超过了无多轮情节的Reflexion。分析剩余差距所在，我们识别出证据充分性问题：代理常常找到答案但未承诺，将预算浪费在不必要的验证上。一个轻量级的承诺触发机制解决了这个问题，但仅当代理已经维护了结构化状态时：将相同的触发机制应用于非结构化代理没有任何改进，这隔离出组织化的认知状态作为有效承诺的必要条件。最后，在较弱模型上强制执行协议遵循，在最难问题上恢复了高达+19分，表明代理如何组织其推理（而非原始模型能力）是扩展多跳推理的活跃因素。

## Abstract
Language agents that interleave reasoning and tool use degrade sharply as reasoning chains lengthen, even when each individual step is easy. We trace this to context dilution: an agent's investigative state (what it has confirmed, what it suspects, and what it still needs) lives only implicitly in a growing context window, where early discoveries are buried under later retrievals. We introduce SLEUTH, which makes this state explicit and actionable through a structured epistemic working memory: the agent maintains Confirmed Facts grounded to sources, Active Hypotheses ranked by evidence, and Open Questions that directly drive its next action. Across five multi-hop benchmarks and five established baselines, SLEUTH's advantage grows with difficulty, from +5 points on HotpotQA to +11 on 4-hop chains, surpassing Reflexion without multiple episodes. Analyzing where the remaining gap lies, we identify the evidence sufficiency problem: agents often find the answer but fail to commit, exhausting their budget on needless verification. A lightweight commitment trigger fixes this, but only when the agent already maintains structured state: the identical trigger applied to an unstructured agent yields no improvement, isolating organized epistemic state as the necessary condition for effective commitment. Finally, enforcing protocol adherence on a weaker model recovers up to +19 points on the hardest problems, showing that how an agent organizes its reasoning, not raw model capability, is the active ingredient for scaling multi-hop reasoning.

## 1. 核心问题与整体含义

- **研究动机**：语言代理在长链推理中性能急剧下降，即使每个单独步骤都容易。作者将其归因于**上下文稀释（context dilution）**——代理的调查状态（已确认、怀疑、仍需确定的信息）仅隐式存在于不断增长的上下文窗口中，早期发现被后续检索淹没。
- **整体含义**：提出通过**结构化认知工作记忆**显式维护推理状态，使得代理能像人类侦探一样分离已知、假设和未确定信息，从而提升多跳推理能力。

## 2. 方法论

- **核心思想**：SLEUTH（Structured Layered Evidence-Updated Tracking of Hypotheses），一种完全通过提示实现的工作记忆框架，维护三元组结构：
  - **Confirmed Facts（已证实事实）**：从工具输出中提取，并附来源引用，单调累积。
  - **Active Hypotheses（活跃假设）**：按证据支持度排序的候选答案，关联事实ID，置信度分级（高/中/低）。
  - **Open Questions（开放问题）**：尚未解决的疑问，每个带有建议行动和优先级，最高优先级问题直接决定下一工具调用。
- **更新协议**：每次工具调用后执行四步：提取事实 → 更新假设 → 更新开放问题 → 选择行动。开放问题为空且某假设高置信度时终止。
- **承诺机制（Nudge）**：解决**证据充分性问题**（找到答案但不承诺）。触发条件：回合数 ≥ α × 总预算（α=0.7）且已累积事实数 ≥ τ（τ=2）。触发时强制代理从已确认事实和假设中合成答案，停止探索。
- **实现**：完全通过系统提示，无微调或架构修改。对弱模型还用提示适配+运行时格式检查确保协议遵循（提升到~99%）。

## 6. 主要结论与发现

1. **SLEUTH在所有基准上取得最高EM**，且优势随难度增长：HotpotQA +4.7分 → 4跳MuSiQue +10.9分（vs 最强外部基线Reflexion）。
2. **结构化事实追踪是主要贡献**：Facts-Only已恢复大部分优势（4跳49.9% vs ReAct 42.0%）。完整三元组进一步带来+3.2分。
3. **证据充分性问题**：无承诺机制时，SLEUTH条件准确率高达64.6%但超时率24.7%；92%超时情节包含正确答案。轻量级承诺（Nudge@14）将超时率降至11.1%，EM提升4分。
4. **承诺机制仅在结构化状态下有效**：对ReAct应用相同触发（Nudge@14）无改善（+0.2），证实工作记忆是必要条件。
5. **协议遵循比模型能力更重要**：弱模型GLM-5在协议强制下4跳EM达48.9%，优于强模型Sonnet的ReAct（42.0%）；通过强制遵循恢复最高+19分。
6. **两个失败模式**：信息丢失（2-3跳主导）由事实追踪解决；信息麻痹（4跳主导）由承诺机制解决。

## 7. 优点

- **轻量性**：完全提示实现，无需训练、搜索或多轮情节，适用于任何指令遵循模型。
- **可解释性**：每步认知状态（事实、假设、问题）以自然语言输出，便于人类检查。
- **系统消融**：逐步分离组件贡献（事实追踪、假设排序、开放问题），揭示每个组件的必要性和协同效应。
- **深入问题分析**：分解超时与条件准确率，提出“证据充分性问题”，并通过控制实验隔离工作记忆的必要性。
- **跨模型泛化**：在Sonnet和GLM-5上一致表现，证明方法不依赖特定模型。
- **鲁棒性验证**：对预算、检索宽度、承诺时机等超参数进行了全面扫描，结果稳定。

## 8. 不足与局限

- **承诺阈值固定（70%预算）**：经在4跳上调参得到，可能不适用于任务结构差异大的场景（如极短或极长链条）。自适应阈值可能更好。
- **单调事实策略**：可能固化早期错误，例如事实正确但语境误导时无法纠正。开放域或有矛盾来源时风险更大。
- **封闭域假设**：仅使用给定文档集检索，未涵盖时间敏感、矛盾或动态的开放域知识。
- **额外token开销**：虽在困难问题上更高效（1.33×成本），但Easy任务上开销比例更大（1.50×），且需保证预算至少~2.5倍推理深度。
- **与更复杂体系比较缺失**：未与图结构或控制器驱动的方法（如Besta等）对比，但作者指出SLEUTH属于最轻设计点。
- **可重复性细节**：未提供随机种子、模型版本精确哈希，可能受托管模型更新影响。
- **推理错误仍是主要瓶颈**：4跳上32.1%错误源于正确证据上的错误推理，SLEUTH未解决该问题。
