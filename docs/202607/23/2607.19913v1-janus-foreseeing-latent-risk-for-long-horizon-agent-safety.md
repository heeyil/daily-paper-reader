## 摘要
智能体安全正从内容审查转向在工具使用智能体行动前预防操作失败。我们提出Janus，一种面向长期智能体安全的预见性框架，训练守卫者从部分轨迹中预判延迟风险。Janus通过多智能体模拟合成多样化智能体轨迹，并学习一个共享策略，该策略包含两个耦合任务：一个预测任务，用于预见与安全相关的未来；一个裁决任务，基于观察到的前缀和预见的未来来决定安全性。这两个任务通过CoAA-RL联合优化，该算法根据预测对下游安全判断的效用给予奖励。由此产生的守卫模型Vanguard能在执行前阻止不安全动作。在四个智能体安全基准测试中，Vanguard相比基线守卫平均提高了15.9个百分点的保护率，同时将良性任务完成率提高了5.1个百分点。

## Abstract
Agent safety is moving from content moderation toward preventing operational failures before tool-using agents act. We propose Janus, a foresight-oriented framework for long-horizon agent safety that trains guards to anticipate delayed risks from partial trajectories. Janus synthesizes diverse agent trajectories via multi-agent simulation and learns a shared policy with two coupled tasks: an anticipation task that forecasts safety-relevant futures and an adjudication task that decides safety from both the observed prefix and anticipated future. The two tasks are jointly optimized with CoAA-RL, which rewards forecasts by their utility for downstream safety judgment. The resulting guard model, Vanguard, blocks unsafe actions before execution. Across four agent-safety benchmarks, Vanguard improves average protection by 15.9 percentage points over baseline guards while increasing benign task completion by 5.1 percentage points.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：随着工具使用智能体（如 OpenClaw、Codex、Claude Code）的部署，其操作失败面远超传统语言模型。一次不安全动作可能删除文件、泄露数据、破坏环境或中断生产服务，而事后监督往往为时已晚。现有守卫方法大多是被动的，仅在风险已经发生或明确时检测，无法处理长期任务中早期信号逐步演变为延迟性危害的情形。
- **核心问题**：如何让守卫模型在长周期智能体执行过程中，从部分轨迹中预见潜在延迟风险，并在有害动作执行前进行干预？即从反应式守卫转向预测式守卫。
- **背景**：现有工作如 LlamaFirewall、ShieldAgent 等主要聚焦于内容审核或即时行为检测，缺少对“未来后果”的推理能力。语言世界模型（如 Qwen-AgentWorld）虽能预测未来状态，但未专门针对安全决策优化。Janus 填补了“预见性安全守卫”这一空白。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
Janus 是一个面向长期智能体安全的预见性框架，通过**多智能体模拟**合成多样化的长周期轨迹，训练一个共享策略同时完成两个耦合任务：
- **前瞻任务（Anticipation）**：从部分轨迹前缀预测与安全相关的未来摘要。
- **裁决任务（Adjudication）**：基于观测前缀和预见的未来摘要判断当前安全性。

两任务通过 Coupled Anticipation and Adjudication Reinforcement Learning（CoAA-RL）联合优化，其中前瞻奖励不仅考虑未来摘要与真实未来的相似度，还考虑其对下游安全裁决的效用。

### 关键技术细节

1. **数据构建**：
   - 风险分类：用户起源（有害意图伪装为良性任务）、环境起源（间接提示注入、工具输出污染等）、智能体起源（规划失败、过度遵守字面指令等）。
   - 多智能体模拟生成轨迹：使用 Manager、Instructor、ToolDesigner、Grader、EnvInjector、Decomposer、Executor、Simulator、Reviewer 等多个专用智能体，模拟 ReAct 风格执行，不执行真实 API。
   - 质量筛选：Reviewer 基于任务一致性、风险覆盖和轨迹完整性进行拒绝采样。
   - 步级标注：GPT-5.5 对每个决策点标注 Safe、Unsafe 或 Potential Unsafe，并生成未来摘要。共获得 75,180 条训练样本（34,100 Safe，18,665 Unsafe，22,415 Potential Unsafe）。

2. **耦合前瞻与裁决强化学习（CoAA-RL）**：
   - **输入**：用户指令 \(x\)、轨迹前缀 \(\tau_{\text{pre}}\)、真实未来摘要 \(s\)、安全标签 \(y\)。任务指示符 \(m \in \{A, D\}\) 区分模式。
   - **前瞻任务**：给定 \((x, \tau_{\text{pre}}, m=A)\)，采样 \(G_A\) 个未来摘要 \(S_k\)。使用双向 NLI 相似度 \(\text{Sim}(S_k, s)\) 衡量与真实未来的一致性。
   - **裁决任务**：对于每个 \(S_k\)，采样 \(G_D\) 个裁决输出 \(J_{k,j}\)，解析为预测标签 \(\hat{y}_{k,j}\) 和理由。裁决奖励 \(r^D_{k,j} = \lambda_D \ell_{k,j} + (1-\lambda_D)\phi_{k,j}\)，结合标签正确性 \(\ell_{k,j}\) 和格式有效性 \(\phi_{k,j}\)。
   - **耦合奖励**：每个摘要 \(S_k\) 的效用 \(u_k = \frac{1}{G_D}\sum_j r^D_{k,j}\)。前瞻奖励 \(r^A_k = (1-\lambda_A)\text{Sim}(S_k, s) + \lambda_A u_k\)，即平衡未来真实性与裁决有用性。
   - **组相对优化**：采用 GRPO 风格，对每个样本计算组内优势：
     - 前瞻优势：\(A^A_k = r^A_k - \bar{r}^A\)，其中 \(\bar{r}^A\) 是组内平均前瞻奖励。
     - 裁决优势：选择索引 \(\tilde{k}\) 使得该组内裁决奖励方差非零，计算 \(A^D_{\tilde{k},j} = r^D_{\tilde{k},j} - \bar{r}^D_{\tilde{k}}\)。
     - 联合目标：\(\theta^* = \arg\min_\theta \left[ \mathcal{L}^{\text{GRPO}}_A(\theta) + \eta \mathcal{L}^{\text{GRPO}}_D(\theta) \right]\)。
   - **推理阶段**：两步预测：先由前瞻任务生成未来摘要 \(\hat{s}\)，再在给定 \((x, \tau_{\text{pre}}, \hat{s})\) 下由裁决任务输出安全标签和理由。无真实未来可用。
