## 摘要
基于LLM的自主智能体需要外部记忆来克服其无状态性和有限的上下文窗口，以实现长期交互和动态知识推理。然而，现有的记忆检索方法往往缺乏适应性和样本效率，并且难以从异构存储中检索到正确的记忆混合。我们提出了探索性-同化性反思（EAR），一个旨在实现高初始检索性能和样本高效适应的框架。EAR结合了两种机制：探索性反思，其通过迭代搜索来启动检索并为每个查询收集有用经验；以及同化性反思，其从经验缓冲区中重放这些经验，以比仅依赖即时奖励的方法更高效地优化全局重排序器。实验表明，在两个长期对话基准上，EAR相比基线检索器将检索性能提升了高达17.9%。我们还展示了EAR具有高样本效率和对噪声反馈的鲁棒性。

## Abstract
LLM-based autonomous agents require external memory to overcome their statelessness and limited context window for long-term interaction and dynamic knowledge reasoning. However, existing memory retrieval methods often lack adaptability and sample efficiency, and struggle to retrieve the right mixture of memories from heterogeneous stores. We propose Exploratory-Assimilating Reflection (EAR), a framework for high initial retrieval performance and sample-efficient adaptation. EAR combines two mechanisms: Exploratory Reflection, which performs iterative search to bootstrap retrieval and collect useful experiences for each query, and Assimilating Reflection, which replays these experiences from an Experience Buffer to refine a global reranker more efficiently than methods relying only on immediate rewards. Experiments show that EAR improves retrieval by up to 17.9% over the baseline retriever on two long-term dialogue benchmarks. We also show that EAR is highly sample-efficient and robust to noisy feedback.

## 2. 方法论：核心思想、关键技术细节

### 2.1 整体框架

- EAR 由两个协同模块构成，模拟认知心理学中短时记忆与长时记忆的交互：
  - **探索性反思（Exploratory Reflection）**：针对每个查询进行迭代搜索，构建丰富的证据集，同时收集有用经验。
  - **同化性反思（Assimilating Reflection）**：通过经验缓冲（Experience Buffer）重放历史探索经验，高效更新全局重排序器。

### 2.2 关键技术细节

1. **残差检索适配器（Residual Retrieval Adapter）**
   - 学习可训练的线性变换 \( \phi = \{W_q, W_m\} \)，对查询和记忆的原始嵌入进行残差调整，并归一化。
   - 在候选集 \( C \) 上定义 softmax 分布用于排序。

2. **探索性反思算法**
   - 将每个候选记忆视为**本地赌臂**，用 UCB（Upper Confidence Bound）公式计算价值：  
     \[
     U_i^{(t)} = \hat{\mu}_i^{(t-1)} + \alpha \sqrt{\frac{\log(1+\sum N_\ell^{(t-1)})}{1+N_i^{(t-1)}}}
     \]
   - 选择时综合考虑三种信号：
     - 相关性（query–memory 相似度）
     - 多样性惩罚（已选记忆间的最大余弦相似度）
     - 探索奖励（UCB 值）
   - 进行 T 轮探索，每轮选择大小为 s 的记忆子集，从 LLM 获得反馈（+1 有用，-1 无用），更新经验平均和计数。

3. **当前查询反馈学习**
   - 使用**反馈加权排序损失**：
     \[
     \mathcal{L}_{\text{cur}}(\phi) = -\frac{1}{Ts} \sum_{t=1}^T \sum_{j=1}^s (y_{i_j^{(t)}} - b) \log p_\phi(i_j^{(t)} | x, C)
     \]
     其中 \( b \) 为基线（设为 0.5），放大正负反馈的影响。

4. **同化性反思与经验重放**
   - 维护一个经验缓冲 \( \mathcal{B} \)，存储每个查询的原始嵌入、候选集和聚合后的反馈。
   - 从当前查询的嵌入出发，选取缓冲中**嵌入最相似**的 B 个经验。
   - 对每个重放的经验，用当前适配器重新计算得分，并用 Gumbel 技巧随机采样子集，计算重放损失。
   - 最终适配器更新为：
     \[
     \mathcal{L}_{\text{EAR}}^{(n)} = \mathcal{L}_{\text{cur}}^{(n)} + \lambda_{\text{rep}} \mathcal{L}_{\text{rep}}^{(n)}
     \]

5. **训练与推理**
   - 训练时：每查询执行 T 轮探索，更新适配器。
   - 推理时有两种模式：
     - **仅适配器模式**：关闭探索，直接使用适应后的 logits 选择 Top-s。
     - **辅助探索模式**：启用探索，获得额外自适应，但需更多 LLM 调用。

## 8. 不足与局限

- **经验重放策略有待优化**：文中指出仅根据嵌入相似度选择过去经验可能并非最优，更复杂的采样或重放机制值得进一步研究。
