## 摘要
随着大语言模型(LLM)代理从独立提示扩展到长周期工作流，失败越来越多地出现在角色到实例的绑定边界，即特定任务的角色请求必须在当前服务、网络和查询条件下分配给具体的代理实例。现有的代理系统研究改进了角色专业化、工作流拓扑、记忆和工具使用，但通常假定一个固定的稳定执行环境。这种假设限制了部署的可靠性，因为相同的角色请求在不同服务区域和网络条件下运行的代理实例之间可能表现出不同的延迟、失败概率和输出质量。我们提出了对冲代理计算(HACO)，这是一种运行时控制方案，将每个角色请求视为对候选代理实例的一个可靠性约束选择问题，每个实例耦合了角色类型、LLM和具体的执行环境。与路由不同，HACO为每次调用自适应地选择一个对冲候选集合。其分配规则结合了乐观排序（优先考虑具有高估计质量、可靠性和信息不确定性）与保守可靠性累积（仅在对冲集合达到目标成功概率后才停止选择）。通过经验收集，HACO从所有执行的候选轨迹中更新候选和链接配置文件，包括质量、成功、延迟和网络统计。在各种基准测试以及运行时退化研究上的实验表明，HACO在变化的部署条件下提高了鲁棒性和输出质量，同时使用的token和延迟成本低于穷举并行执行。

## Abstract
As large language model (LLM) agents move from isolated prompting to longhorizon workflows, failures increasingly arise at the role-to-instance binding boundary, where task-specific role requests must be assigned to concrete agent instances under current service, network, and query conditions. Existing agent system research has improved role specialization, workflow topology, memory, and tool use, but often assumes a fixed stable execution environment. This assumption limits deployed reliability, because the same role request can exhibit different latency, failure probability, and output quality across agent instances operating under different service regions and network conditions. We propose Hedged Agent Computing (HACO), a runtime control scheme that treats each role request as a reliability-constrained selection problem over candidate agent instances, each coupling a role type, an LLM, and a concrete execution environment. Different from routing, HACO adaptively selects a hedge set of candidates for each invocation. Its allocation rule combines optimistic ranking, which prioritizes candidates with high estimated quality, reliability, and informative uncertainty, with conservative reliability accumulation, which stops selection only after the hedge set reaches a target success probability. Through experience harvesting, HACO updates candidate and link profiles from all executed candidate traces, including quality, success, latency, and network statistics. Experiments on various benchmarks, together with runtime degradation studies, show that HACO improves robustness and output quality under changing deployment conditions, while using lower token and latency cost than exhaustive parallel execution.

- **研究动机**：现实中，同一角色请求可能经由不同服务提供商和区域执行，其行为不仅取决于LLM能力，还受网络延迟、带宽波动、瞬态故障、资源争用等物理环境因素影响。近期基础设施事故（如红海海底光缆中断、Azure西欧热事件）进一步表明物理和区域条件会直接影响服务可用性。

- **整体含义**：将角色调用视为**角色类型、LLM选择和执行环境**三者耦合的运行时决策，而非固定调用。提出**对冲代理计算（HACO）**，通过自适应冗余执行来管理角色调用级的不确定性，同时利用冗余收集经验以优化未来分配。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
HACO将每个角色请求建模为一个**可靠性约束的子集选择问题**：在候选代理实例集合中，选择一个最小成本的对冲集合，使得集合整体达到目标可靠性τ。每个候选实例表征为三个维度：**认知能力**（输出质量，用Beta分布建模）、**物理执行可靠性**（内部失败率与网络丢失率）、**延迟感知资源成本**（对数化延迟折扣）。

### 2.2 关键技术细节
HACO包含四个阶段：

1. **角色抽象**：从MAS控制器接收角色调用事件 `(role_type, payload)`，由角色类型确定候选池。

2. **冗余分配**（核心）：
   - **乐观排序**：为每个候选计算乐观效用 `U⁺_i = [min(1, μ_i + λσ_i) · ρ_i] / D_i`，其中μ、σ是后验均值/标准差，ρ_i是执行成功概率，D_i是延迟折扣因子。按降序排列，鼓励探索低观测但潜在强的候选。
   - **保守可靠性累积**：按排序顺序依次加候选，利用悲观可靠性估计 `R⁻_i = min(0.98, max(ϵ_R, μ_i - γσ_i) · ρ_i)` 更新累积失败概率 `P_fail ← P_fail · (1 - R⁻_i)`，当 `1 - P_fail ≥ τ` 时停止分配。避免过早终止。

3. **对冲执行**：并行启动选中的候选集合，选择第一个成功的最高效用候选作为胜者，其余异步收集痕迹。

4. **经验收集**：从所有执行候选记录质量、成功、延迟、令牌使用、网络统计等，更新候选配置和链路配置（包括Beta后验更新、带宽抖动、丢包率等）。

### 2.3 关键公式
- 认知能力：Beta先验 `θ_i ~ Beta(α_i, β_i)`，后验更新 `α_i ← α_i + q_i, β_i ← β_i + (1 - q_i)`。
- 物理可靠性：`ρ_i = (1 - f_int⁽ⁱ⁾)(1 - ℓ_net⁽ⁱ⁾)`
- 延迟成本：`C_i = ln(1 + T_i / T_0)`，延迟折扣 `D_i = 1 + η C_i`
- 优化目标：`min_{S⊆A} Σ C_i s.t. P(qualified success|S) ≥ τ`

## 6. 论文的主要结论与发现

1. **HACO在变化条件下显著提升可靠性和输出质量**：在网络退化、候选退化、评估器退化、真实部署故障场景下，HACO的全失败率上升极小（0~4.1个百分点），而BestOne上升高达67~90个百分点。
2. **自适应冗余优于单路由和全并行**：HACO的雷达图在三个基准上均形成最大最平衡的轮廓，接近MoA的鲁棒性，但令牌和延迟开销大幅降低。
3. **经验收集驱动持续改进**：HACO能够自动从退化区域迁移到健康区域（例如全球选择比例下降，本地/区域选择上升）。
4. **τ提供可控效果-效率权衡**：增大τ从0.70到0.95，步骤失败率从1.87%降至0.17%，平均得分从53.77升至58.86，令牌成本增加但平滑可控。
5. **组件贡献明确**：去除环境建模、乐观探索或保守停止均导致性能下降，证实三组件联合设计的必要性。

## 7. 优点

- **问题建模新颖**：首次系统地将执行环境作为关键轴纳入角色调用决策，暴露了角色-实例绑定边界的可靠性瓶颈。
- **技术方案简洁有效**：乐观-保守双界协调机制避免了单一路由和全执行两种极端的缺陷；Beta分布和韦伯-费希纳定律使用合理。
- **高度实用**：无需训练，仅需在线经验收集，适用于现有MAS；可通过τ灵活调节可靠性-成本平衡。
- **实验设计严谨**：覆盖模拟和真实部署，多种退化场景，消融和对标完善，结论有说服力。
- **可扩展性**：HACO仅需MAS控制器发出角色调用事件，与控制器内部实现解耦，易于集成各种MAS框架。

## 8. 不足与局限

1. **任务范围局限**：实验仅涵盖数据科学和可视化领域的工作流，未验证于网络自动化、检索增强型代理、分布式智能系统等场景。作者承认需要在更广泛环境中验证。
2. **τ需手动调优**：目标可靠性τ是外部参数，实际部署中需根据应用要求和预算设定，缺乏自动调优机制。
3. **反馈信号有限**：经验收集目前只使用质量、成功、延迟、令牌等操作级信号，未引入错误类型、推理轨迹、代码级失败模式等语义反馈。作者计划加入时序适应性机制（如遗忘因子）。
4. **独立性假设**：候选间的条件独立性假设在实际中可能不完全成立，保守性证书在假设违反时仅作为近似估计，需要后续监控。
5. **对评估器质量的依赖**：虽然实验显示评估器退化影响较小，但质量评分仍依赖LLM-as-judge，在极端退化下可能偏差。
6. **未考虑安全性**：系统级鲁棒性不保证事实正确性、公平性、隐私或防恶意使用，需结合应用级防护。
