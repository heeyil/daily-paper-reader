## 摘要
长时域任务需要持续的感知、推理和探索，对大型语言模型（LLM）智能体而言是一个持续的挑战。这一差距体现在它们在诸如ARC-AGI-3等持续学习基准上的有限表现，尤其是当模型在开箱即用条件下评估时。为缩小这一差距，研究者提出了多种智能体框架，每种都采用不同的策略来处理长序列观测，即从环境中保存哪些信息以及如何将其加载到模型上下文中——我们认为这一选择至关重要。现有的上下文管理方法面临显著的权衡，因为保留更多信息会使检索相关细节变得更加困难。我们提出PRO-LONG，一种围绕程序化记忆构建的最小化上下文管理框架，适用于长时域、探索性场景下的LLM智能体。PRO-LONG通过维护完整、结构化的交互日志，并利用编码智能体在历史搜索方面的最新进展，解决了这一权衡问题。在完整的ARC-AGI-3公共游戏集上，PRO-LONG在多个前沿模型上相对于基础编码智能体平均提升了18.0个百分点，同时匹配或超过了最先进的专用框架（最高达76.1%的pass@1），且使用的令牌数减少了4.2至5.8倍。借助Fable 5，PRO-LONG在总成本为1,750美元的情况下达到了97.4%的best@2。相关代码和日志可在https://github.com/alexisfox7/PRO-LONG获取。

## Abstract
Long-horizon tasks require sustained perception, reasoning, and exploration, and are a persistent challenge for large language model (LLM) agents. This gap is reflected in their limited performance on continual learning benchmarks such as ARC-AGI-3, especially when models are evaluated out of the box. Various agent harnesses have been proposed to close this gap, and each commits to a strategy for handling long sequences of observations, i.e., what information to save from the environment and how to load it into model context, a choice we argue is particularly consequential. Existing methods for context management face a significant tradeoff, as preserving more information makes retrieving relevant details less tractable. We propose PRO-LONG, a minimal context management framework built around programmatic memory for LLM agents in long-horizon, exploratory settings. PRO-LONG addresses the tradeoff by keeping a complete, structured interaction log and capitalizing on recent progress in coding agents to search this history efficiently. On the full ARC-AGI-3 public game set, PRO-LONG improves over a base coding agent by an average of 18.0 percentage points across frontier models, and matches or exceeds state-of-the-art specialized harnesses (up to 76.1% pass@1) while using 4.2-5.8x fewer tokens. With Fable 5, PRO-LONG achieves 97.4% best@2 at a total cost of \$1,750. Relevant code and logs are available at https://github.com/alexisfox7/PRO-LONG.

## 1. 核心问题与整体含义（研究动机与背景）

大型语言模型（LLM）智能体在长期任务中面临持续感知、推理和探索的挑战。现有基准如 ARC-AGI-3 评估智能体通过探索推断环境动态并持续改进的能力，但开箱即用的前沿模型表现极低。为弥补差距，研究者提出了各种智能体框架（harness），其中核心问题是如何管理长序列观测的历史：保存多少信息以及如何检索到模型上下文中。现有方法面临**保真度-可追溯性权衡**：保存越多信息，后续检索越困难。许多方法在写入时进行有损压缩或摘要，但可能丢失对未来有用的细节。

论文提出 **PRO-LONG**（Programmatic Memory Enables Long-Horizon Reasoning），围绕**程序化记忆**设计最小化上下文管理框架，旨在解决这一权衡，使智能体在长时域探索任务中保持高效推理。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **写 (write)**：将每一观测、动作和结果**完整追加**到日志文件 `log.txt` 中，不做任何压缩或摘要（无损）。
- **读 (read)**：通过**程序化搜索**（正则表达式、grep、Python 代码）从日志中检索必要信息，利用编码智能体本身的能力高效处理长日志。
- 设计三个原则：
  - **简单性**：无需学习或启发式判断，无需专用检索系统。
  - **无损性**：日志是环境状态的忠实完整记录。
  - **兼容性**：充分利用编码智能体（coding agent）的程序化能力。

### 关键技术细节
- 构建于基础编码智能体之上（如 Codex 或 Claude Code），提供文件系统、shell 和 Python 执行环境。
- 每个动作轮次，智能体查看当前环境并表示（64×64 文本网格），输出计划并写入 `actions.json`，环境执行后返回新状态并追加到日志。
- 日志格式：包含动作编号、关卡、尝试、分数、智能体计划、动作及结果棋盘。
- 检索：智能体可使用 `grep`、`sed`、Python 脚本等程序化工具在日志中搜索模式，例如查找分数变化的关键动作。
- 无需子智能体，系统提示仅约30行。

## 3. 实验设计：数据集、基准与对比方法

### 数据集与基准
- **ARC-AGI-3**：25个公共游戏，每个包含6-10个关卡，规则未公开，需探索推断。棋盘64×64网格，16种颜色映射为ASCII符号。动作包括方向移动、点击、撤销、重置等。
- 评分标准：Relative Human Action Efficiency (RHAE)，比较智能体动作数与人类中位数，加权综合得分。

### 对比方法
- **No-Log**：基础编码智能体（无外部日志，仅依赖自我笔记和当前上下文）。
- **WorldModeler**（GPT-5.5）：Codex框架，约600行提示，使用视觉、子智能体和辅助脚本构建可执行世界模型。
- **Arcgentica**（Opus 4.6）：自定义SDK，主协调器+四个子智能体（探索者、理论家、测试者、求解者），共享记忆数据库。
- **Schema**（Opus 4.8 / Fable 5）：Claude Code框架，十二个自定义MCP工具，维护 `step(grid, action)` 函数并通过回溯和BFS搜索。

### 评估协议
- 默认配置：高模型推理努力，每游戏最大500动作，每轮最多20动作。
- 所有智能体使用相同动作预算和评分程序。对于先前方法，作者重新评分其公开运行以保持一致比较。
- 多次独立运行（如GPT-5.5重复5次），报告 pass@1 和 best@k（k=2,5）。
