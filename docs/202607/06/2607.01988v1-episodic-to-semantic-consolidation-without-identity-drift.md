## 摘要
长期运行的自主智能体在知识巩固与信息完整性之间面临结构性紧张。记忆巩固通常被视为一种改变智能体的操作：微调模型、重写提示、蒸馏策略、或在控制未来行为的上下文中追加反思。在受监管的自主部署中，这是一种负担，因为智能体在承诺和审计合同下运作，这些合同绑定于特定的、经加密认证的身份。我们提出将巩固视为并非对规划器或身份清单的突变，而是对情景记忆的确定性函数 f: M^ep -> M^sem，其输出是一个独立可寻址的语义知识层；身份哈希不读取 M^sem，因此巩固在不改变智能体认证身份的情况下更新知识。我们给出了智能体表示的形式化描述，通过关于清单哈希输入集的结构引理证明了身份不变性，指定了一种确定性聚合算法，其输出是可审计的数据库行，包含明确的置信度和支持事件来源，并通过综合实验验证了该结构，展示了逐字段正确性、跨巩固过程的字节相同身份，以及相对于校准的贝叶斯收缩基线，平均减少了79.82%的非生产性规划器尝试（10个种子上的95% BCa CI [78.02%, 81.49%]）。该结构是一种适用于自主智能体的知识更新规范，其中经验积累为可查询的事实，而智能体的认证身份在其运行生命周期内保持字节相同，并以一个具体的服务智能体作为运行案例研究。

## Abstract
Long-running adaptive intelligent agents face a structural tension between knowledge consolidation and information integrity. Memory consolidation is conventionally treated as an agent-changing operation: a model is fine-tuned, a prompt rewritten, a policy distilled, or a reflection appended to the context that governs future behaviour. In regulated autonomic deployment this is a liability because the agent operates under commitments and audit contracts that bind to a specific, cryptographically certified identity. We propose to treat consolidation not as a mutation of the planner or the identity manifest, but as a deterministic function f: M^ep -> M^sem over episodic memory whose output is a separately addressable semantic knowledge layer; the identity hash does not read M^sem, so consolidation updates knowledge without changing the agent's certified identity. We give a formal account of the agent representation, prove identity invariance through a structural lemma on the manifest's hash-input set, specify a deterministic aggregation algorithm whose outputs are auditable database rows with explicit confidence and supporting-event provenance, and validate the construction with synthetic experiments demonstrating per-field correctness, byte-equal identity across consolidation passes, and a mean 79.82% reduction in unproductive planner attempts (95% BCa CI [78.02%, 81.49%] across 10 seeds) against a calibrated Bayesian-shrunk baseline. The construction is a knowledge-update discipline for autonomic agents in which lessons accumulate as queryable facts while the agent's certified identity remains byte-equal across its operational lifetime, with an embodied service agent as the running case study.

- **核心问题**：长期运行的自主智能体（如机器人）在受监管环境中面临“知识巩固”与“信息完整性”之间的结构性张力。传统记忆巩固（模型微调、提示重写、策略蒸馏、上下文反思追加）会改变智能体本身，即改变其加密认证身份（identity certificate）的哈希值。在需要持续合规审计的部署场景（如医院、制造车间）中，每一次巩固都成为重新认证事件，不可持续。
- **整体含义**：本文提出**身份稳定的巩固（identity-stable consolidation）**，将记忆巩固视为从情景记忆到语义记忆的确定性函数，身份哈希不读取语义层，从而在不改变认证身份的前提下积累操作知识。这为自主智能体提供了可审计的知识更新规范。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心思想
- 将智能体状态形式化为四元组 (M, M^ep, M^sem, π)：
  - **M**：身份清单（manifest），包含固定字段（如规划器ID、能力注册哈希、策略版本等），其 SHA-256 哈希作为认证身份 `h`。
  - **M^ep**：情景事件日志（append-only 序列，按身份哈希分区）。
  - **M^sem**：派生语义记忆表，由确定性函数 `f: M^ep → M^sem` 生成。
  - **π**：规划器（planner），只能通过只读接口查询 M^sem。
- 关键创新：**f 只写入语义存储，不影响清单 M 的哈希输入集** → 身份哈希在任意次巩固后保持字节相同。

### 2.2 关键技术细节
- **结构身份不变性引理（Lemma 1）**：通过输入集审计证明——M 的字段集在类型设计时固定，不包含对 M^sem 的任何引用；任何仅写入 M^sem 的操作 `f ∈ F_sem` 均不改变 M 的规范序列化结果，因此 SHA-256 输出不变。
- **确定性 v1 巩固算法（§3.2）**：SQL 风格的聚合流程，包含五个步骤：
  1. **读取**：从情景存储中获取上次检查点后的新事件；
  2. **提取**：投影结构化字段（技能ID、目标类、环境、参数、成功/失败标志等）；
  3. **分组**：按稳定键（如 skill+target+environment）分桶；
  4. **聚合**：计算计数、成功率、参数分布（均值、IQR）、最常见的失败原因；
  5. **更新**：利用唯一键 (identity_hash, fact_kind, fact_key) 进行 upsert，写入语义事实表 `semantic_facts`，包含置信度、观测数、最后支持事件ID、巩固规则版本。
- 属性：**确定性、幂等性、顺序无关性** – 相同输入 + 相同规则版本 → 字节相同输出。审计链：每次巩固后追加 `consolidation_run` 事件，确保可追溯性。
- **只读规划器接口（§3.4）**：通过 HTTP GET 端点按技能、目标、环境查询最高置信度的事实，规划器可将这些事实作为 prompt 的 grounding 信息；规划器不拥有语义存储的写权限，保证信息而非权威。

## 3. 实验设计

### 3.1 数据集与场景
- **合成情景数据**：使用模拟生成的执行结果事件，对象为 `manipulation.grasp` 技能，涵盖玻璃杯（成功率高）和未知物体（成功率低）两种目标。
- 场景：1000 次规划决策（每种子），规划器在四种配置下运行。

### 3.2 对比的基线方法
| 控制组 | 说明 |
|--------|------|
| **no-memory** | 每次尝试所有抓取，不利用语义事实； |
| **uniform** | 使用语义事实的存在性（存在/不存在）驱动决策，但将所有已知技能的置信度设为 1.0，隔离结构信号； |
| **raw** | 使用巩固器直接输出的成功率作为置信度； |
| **calibrated** | 对每个技能的成功率应用贝叶斯收缩（先验均值 0.5、权重 10，基于 20% 历史切片）后与阈值比较。 |

### 3.3 实验分组
- **V1：确定性正确性**：N=1000 行（800成功，200失败），检验单次巩固后的逐字段正确性、幂等性、顺序无关性（6 个校验点全部通过）。
- **V2：身份不变性**：比较巩固前后身份哈希，并验证变异 M 字段后哈希变更。
- **V3：规划器实用性**：10 种子，每种子 1000 决策，比较四种控制组减少无效尝试的比例；同时进行超参数敏感性分析（阈值扫描[0.3,0.7]、先验均值/权重扫描、决策数 N 扫描[100,10000]）。

## 6. 主要结论与发现

- **身份不变性确证**：巩固操作前后身份哈希字节级相同（SHA-256 输出一致）；变异清单字段后哈希改变，证明身份由清单而非语义层界定。
- **大幅减少无效尝试**：校准控制组 vs 无记忆基线，平均无效规划尝试减少 **79.82%**，95% CI [78.02%, 81.49%]；均匀置信度控制组无改善（CI 跨零），证明改善来源于置信度信号而非仅有事实结构。
- **超参数鲁棒**：阈值在 [0.5, 0.7] 内、先验均值 ≤0.5 的网格、决策数 N ≥ 500 时结果稳定；仅当阈值过低或先验过于主导时出现退化。
- **运行时效率**：巩固延迟最多 309ms（N=10^5），语义查询 P99 <0.004ms，适合每次情景写入后在线调用。

## 7. 优点

- **结构新颖**：将身份认证与知识存储解耦，通过类型设计而非运行时断言保证身份不变性，具有形式化保证（引理与推论）。
- **审计友好**：每条语义事实都可通过 `last_supporting_event_id` 和 `consolidation_run` 事件追溯到原始情景，可解释性强。
- **实现简洁且高效**：确定性 SQL 聚合，无需复杂模型，资源消耗极低，适合受监管的自主部署环境。
- **实验设计严谨**：使用贝叶斯收缩基线消除“无记忆基线过于简单”的批评；多种子 BCa 置信区间提升统计可靠性；超参数扫描覆盖范围充分。

## 8. 不足与局限

- **仅支持结构化聚合**：v1 无法处理自由文本摘要、操作员注释等非结构化信息；v2（LLM 辅助）已提出但暂未实现，且非确定性会挑战审计性。
- **无遗忘机制**：不随时间衰减，一年前的事实与最近事实同等对待，可能导致过时信息误导规划。
- **无对抗防御**：假设情景存储可信，未考虑恶意注入污染事件的可能。
- **缺乏长期部署数据**：实验为纯合成数据，真实世界累积流尚未获得，无法验证长期行为漂移是否可控。
- **情景有效载荷结构假设**：依赖已有结构化字段（技能ID、参数等），否则无法聚合。
- **自适应性情（adaptive temperament）**：未涉及如何避免语义记忆成为改变智能体风险偏好或伦理限制的隐蔽途径。
