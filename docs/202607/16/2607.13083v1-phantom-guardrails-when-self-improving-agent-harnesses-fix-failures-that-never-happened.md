## 摘要
自我改进的AI智能体旨在从错误中学习。我们表明它们也可以幻觉出从未发生的错误。我们在自动化测试套件优化中研究这种失败模式，其中基于LLM的建议者编辑智能体的脚手架，包括提示、解析器、过滤器、验证器和护栏，以消除观察到的失败。但这个过程很少首先询问：是否真的有需要修复的失败？我们引入了反事实制造实验室，这是一个确定性的微型实验室，其中正确的行动是已知的：什么都不做。该实验室为一条证明永远不会发生的失败类植入一个候选护栏，仅展示合法情节，并使用字节精确预言机检查每一个引用的违规。建议者在真实违规上表现符合预期，在无特征合法输入上则放弃。然而，当合法输入包含一个类似熟悉游戏规则的无害模式时，它会编造一个失败：在60次运行中有15次（相比之下，无特征输入为0/60），它启用了不存在的规则护栏，并引用了预言机驳斥的违规。这种效应是有结构的，而非无差别的。在单次建议中，它仅在三个条件同时出现时发生：一个规则形状的模式、一个开放式的规则集以及一个预设失败的指令。移除这些条件中的任何一个都会消除这种捏造。由于发明的护栏不会改变任何真实结果，也无法提升已经完美的抑制分数，这种现象既不是奖励黑客，也不是过度拒绝。它是一种幻影护栏：对从未发生的失败的修复，对于仅抑制性接受是不可见的。在仅添加的接受循环中，即使没有预设失败的指令，它也会重新进入，循环的持续添加角色提供了指令在单次中提供的需求，一旦进入就会保持。我们提出了反事实制造实验室，用于测量自我改进智能体测试套件中捏造的失败。

## Abstract
Self-improving AI agents are designed to learn from their mistakes. We show they can also hallucinate mistakes that never happened. We study this failure mode in automated harness optimization, where an LLM-based proposer edits an agent's scaffold, including prompts, parsers, filters, validators and guardrails, to eliminate observed failures. But this process rarely asks first: was there a real failure to fix? We introduce the Counterfactual Fabrication Lab, a deterministic micro-lab where the correct action is known: do nothing. The lab plants a candidate guardrail for a failure class that provably never occurs, presents only legal episodes, and uses a byte-exact oracle to check every cited violation. The proposer behaves as expected on real violations and abstains on featureless legal input. Yet when the legal input contains a harmless pattern resembling a familiar game rule, it invents a failure: in 15/60 runs, versus 0/60 on featureless input, it enables the nonexistent-rule guardrail and cites a violation the oracle refutes. The effect is structured, not indiscriminate. In single-shot proposals it appears only when three conditions coincide: a rule-shaped pattern, an open-ended rule set and an instruction that presupposes failures. Removing any of these conditions eliminates the fabrication. Because the invented guardrail changes no true outcome and cannot improve an already-perfect suppression score, the phenomenon is neither reward hacking nor over-refusal. It is a phantom guardrail: a fix for a failure that never happened, invisible to suppression-only acceptance. Inside an add-only accept loop it re-enters even without the failure-presupposing instruction, the loop's keep-adding role supplying the demand the instruction supplied in single shot, and once in it stays. We present the Counterfactual Fabrication Lab for measuring fabricated failures in self-improving agent harnesses.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：自我改进的AI智能体在自动优化其测试套件（scaffold）时，不仅会修复真实的失败，还会**幻觉出从未发生的失败**，并为此构建不必要的“护栏”（guardrails）。这种“幻影护栏”因不会改变真实结果而无法被常见的抑制分数（suppression score）检测到，属于隐蔽的过度修复行为。
- **研究动机**：现有自我改进循环仅仅奖励失败抑制，却从不问“这个修复是否真的有必要”。作者指出，在无失败可修复的设定下，优化器可能会从无害模式中“发明”规则，导致资源浪费和系统复杂度增加。
- **整体含义**：揭示了**单一抑制信号作为奖励的盲点**——它既不能发现幻影修复，也不能在事后移除它们。这威胁到agent系统的可信赖性和效率，需引入“基于认证的接受”等更严谨的评估机制。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **关键技术细节**：
  - **游戏环境**：MinArena，一个4x4棋类游戏，代理生成合法移动（列A-D，行1-4）。定义三个失败类：syntax（格式错误）、bounds（越界）、castle（非法王车易位）。其中castle类**永远不会由合法游戏产生**，只通过注入器插入。
  - **脚手架钩子**：四个钩子P、Q、R、S，前两个为修复/注释，R和S分别对应越界和特殊规则违规。S是幻影护栏的目标。

  - **制造指标**：\( \text{Fab}(H, D) = \mathbb{1}[ g_{\text{castle}} \in H ] \)，当预言机证明城堡类从未发生（\( O_{\text{castle}} \equiv 0 \)）时，启用g_castle即算一次制造。

  - **控制组**：
    - **模式电池**：将重复位置替代为相同列、固定对角线、角落模式，每种模式以相同发生率（4/12）植入。

## 6. 论文的主要结论与发现
- **核心发现**：自我改进Agent在**无真实失败**的合法输入上，若该输入包含一个**类似该任务常见游戏规则的无害模式**，它会编造一个失败并启用相应的护栏。
  - 特征缺失输入：幻影护栏启用率0/60（放弃行为）。
  - 重复模式输入：幻影护栏启用率15/60（0.25），预言机证实无该规则。
- **三项必要条件**：在单次提议中，制造现象需要同时满足：
  1. 规则形状模式（重复位置，而非无游戏规则对应的列/对角线/角落）。
  2. 开放式规则集（未声明规则集完整）。
  3. 预设失败的指令（“移除失败” vs 中立指令“根据情节”）。
  - 去掉任一个条件，制造率降为零。
- **接受循环行为**：
  - 抑制分数不降低规则：幻影护栏累计入场，第4轮达到11/60。
  - 严格提高规则：仅当与真实修复捆绑时才能入场（2/60），但一旦进入永久保留。
  - 基于认证的接受规则：完全排除幻影护栏（0/60），同时采纳真实修复（60/60）。
- **非奖励黑客非过度拒绝**：幻影护栏不改变抑制分数（已完美），也不降低任务实用性，因此不属于标准定义的奖励黑客或过度拒绝（后者以降低有用性为代价）。

## 7. 优点：方法或实验设计上的亮点

- **干净的控制**：通过对比特征缺失和模式输入，以及操控三个条件，强有力地展示了因果链。

- **实际意义**：指出了目前自我改进AI评估中的盲点——仅关注失败抑制会忽视幻影修复；提出的“基于认证接受”是一个可行的解决方案。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制
- **领域单一**：仅测试了一个简化棋类游戏（MinArena），其规则体系简单；对于真实自由格式代码、安全规则、工具调用等更复杂的scaffold，现象可能不同。
- **模型覆盖有限**：5+3个LLM中，主要效果集中在glm-5.1；其他模型如deepseek-v4-flash完全沉默，qwen3.6-max-preview也未制造。这限制了结论的普遍性，也未能提供模型能力与易感性之间的定量关系。
- **循环机制简化**：仅测试了add-only接受循环，未包含剪枝或删除功能。论文承认在可剪枝循环中，幻影可能被proposer自身消除，这不改变核心发现但影响实际严重程度。
- **测量指标局限**：只计数护栏是否被启用，未量化其对系统延迟、表面积、能量消耗等实际成本的影响。

- **未探索模型大小或训练数据偏差**：未实验不同规模模型或不同pretraining数据的影响，也未分析指令微调是否加剧该现象。
