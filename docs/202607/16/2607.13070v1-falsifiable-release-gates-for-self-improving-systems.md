## 摘要
关于自我改进智能体运行时的安全性声明几乎总是自我评级的：一份策略文件、一条护栏或一份README承诺。我们描述了可证伪的发布门，以及构建和验证此类系统的方法论，使得每一项新能力在发布前必须通过预先指定的、机器可验证的验收套件，并且在每个门处保持一组固定的常驻不变量。我认为我们在Antahkarana（一个开放运行时）中应用了该方法，通过从基本可观测性到自我治理循环的七道门，该循环建议对其自身策略进行更改。没有动作会到达效应器，除非由控制环铸造的安全关键属性能力令牌被穷尽地机器检查，覆盖一百万条有界模型的已记录可达状态空间，并根据执行轨迹重新检查。一个故意损坏的模型给出最短的反例，因此检查器具有威慑力，这是显而易见的。自我增强循环受到正向约束：整个写入表面是策略规则，收紧更改。可能自动应用的放宽更改始终需要人工合并，而自动关闭的提案是那些错误预测自身效果的提案。我们发布了所有七道门的验收测量结果，精确定义了每个声明的范围（协调骨架的有界范围，而非学习组件），并发布了运行时（包括命令行工具和门套件），以便结果可复现，并且门可以针对其他智能体框架运行。审查者可以在几秒钟内重复执行单一命令的中央不可绕过检查。

## Abstract
Safety claims on self-improving agent runtimes are almost always self-graded: a policy file, a guardrail, or a README commitment. We describe falsifiable release gates, and a methodology to build and validate such systems, such that every new capability must pass a pre-specified, machine-verifiable acceptance suite before it ships, and a fixed set of standing invariants is preserved at each gate. I think we applied the method in the Antahkarana, an open runtime, via seven gates from basic observability into a self governing loop that suggests changes to its own policy. no action goes to an effector without a safety-critical property capability token minted by a control ring is exhaustively machine-checked over the one million recorded reachable state space of a bounded model and re-checked against execution traces. # 论文详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：自我改进的智能体运行时可以修改自身行为，这使它既强大又危险。当前的安全声明往往是自我评级的（self-graded），例如一个策略文件、一条护栏或README中的承诺，缺乏可证伪的保障。
- **核心问题**：如何确保自我改进系统在获得每项新能力后，不破坏已有的安全保证？即“每一个新能力是否都保留了上一个能力的安全性？”这一问题目前仅靠审查和良好意图，缺乏可验证、可重复的机制。
- **整体含义**：论文提出**可证伪发布门控（Falsifiable Release Gates）** 方法，将安全性构建为一个过程而非一次性的审计。通过预先声明、机器可检验的验收套件，以及跨所有门保持固定不变量，使得自我改进系统的安全性是可度量、可复制、可审计的。

## 2. 论文提出的方法论

### 2.1 核心思想
- **门控机制**：每个新能力（新功能或模型更新）对应一道“门”，门中包含一套预先写好的、机器可检验的验收测试套件。该能力只有通过该套件才能被视为“发布”。
- **常驻不变量**：一组固定的安全不变量跨所有门保持，确保后加入的能力不能破坏先前门所保证的安全性。
- **门先于代码**：验收套件在能力开发之前就写好，因此能力本身不能移动“门柱”。
- **单调收紧性**：系统可以自主将策略变得更严格（tighten），但任何放宽（loosen）的策略变更都必须有人工合并，以保证无人值守的系统只能趋向于更谨慎。

### 2.2 关键技术细节
- **控制环与能力令牌**：系统只有一个通向真实动作的路径：控制环（control ring）对动作进行裁决，仅在“允许”裁决时铸造一个能力令牌（capability token）。任何效应器（effector）不接受没有匹配且未过期的令牌的动作。此属性（INV-1）是安全关键。
- **机器可检验的有界模型**：使用TLA+等规范语言对协调骨架（ticks, verdicts, token mint, effector, supervisor control）进行建模，通过穷举状态枚举（可达状态空间291个状态）验证不变量。检查器执行时间在CI中以秒计。
- **“牙齿”规则**：每个不变量都配有一个故意损坏的模型（broken model），检查器必须能从中提取最短反例（最短反例如4步、8步、5步）。这确保检查器不是虚设的。
- **痕迹一致性**：记录的执行轨迹会回放到模型中，若模型拒绝轨迹则CI失败，从而防止模型与代码脱节。
- **自我改进循环的约束**：
  - 写表面仅限于策略规则（如阈值、分类）。
  - 系统不能修改对其自身进行评判的组件（评估管道、钳位器、验证器），这是结构性限制，不是可绕过的验证步骤。
  - 改进提案在发布前必须经过完整的接受套件（包括之前的门），并预测自身效果；错误预测的提案自动关闭。
  - 收紧的更改可自动应用，放宽的更改总是需要人工合并。

### 2.3 算法/流程（文字描述）
1. **定义常驻不变量**（如INV-1, INV-4, INV-5）。
2. **为每个新能力设计“门”**：门包含可证伪的接受套件（代码形式的测试）。
3. **建立有序的“阶梯”**：门按依赖顺序排列（例如，先实现可观测性（G7），再对抗性工具集成（G8），再无漂移学习（G9），等等，直到自我治理循环（G12））。
4. **开发能力并运行接受套件**：若失败则不能发布；若通过则能力生效，且必须通过后续所有门（增量测试）。
5. **机器检查安全属性**：使用有界模型验证不变量；通过故意破坏模型验证检查器有效性。
6. **运行痕迹一致性检查**：确保模型与实现一致。
7. **自我改进时**：提案写为策略差异，收紧可自动应用，放宽需人工合并；错误预测者自动关闭。
