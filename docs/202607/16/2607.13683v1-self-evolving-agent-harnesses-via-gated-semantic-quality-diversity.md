## 摘要
一个LLM智能体的实际任务性能既取决于其模型周围的框架（包括提示词、注入知识、运行时控制和配置），也取决于冻结的模型本身。在部署中，该框架通常是唯一可用的杠杆，因此自动改进它是提高性能的自然途径，而无需触及权重。难点不在于生成变化，而在于知道哪个变化真正有帮助。自我生成的反馈存在噪声，表面上的增益可能是测量伪影或仅仅是对其调优任务过拟合的编辑。我们提出了一个自我进化的智能体框架，将提出变化与归因变化分离开来：一个语言模型诊断故障并提出补丁，而所有采样、测量和显著性检验均由确定性代码控制，因此每个被归因的改进在构造上都是可信的。补丁填充一个门控分类质量-多样性存档（GSME），其键是编辑所解决的病理（WHERE x WHY），而不是它修复的任务，这是一种抗过拟合的归纳偏差；泛化能力仅在进化后在密封测试集上评分。在七个领域中使用冻结的开放权重模型，该框架经过训练选择并仅在密封测试集上评分一次；其归因增益为+9至+15.5个百分点，并保留了训练增益的86-147%，这表明它们泛化而非过拟合。获胜的补丁跟踪模型的主要病理，而不是其大小或系列：更改模型可能会改变病理和补丁，而相同的病理到补丁匹配在两个模型系列中重复出现。转移的是诊断和归因循环，而不是任何特定的框架。

## Abstract
An LLM agent's real-task performance is shaped as much by the harness around its model as by the frozen model itself: its prompts, injected knowledge, runtime control, and configuration. In deployment the harness is often the only lever available, so improving it automatically is the natural way to raise performance without touching the weights. The hard part is not generating changes but knowing which one truly helped. Self-generated feedback is noisy, and an apparent gain can be a measurement artifact or an edit that merely overfits the tasks it was tuned on. We present a self-evolving agent-harness framework that separates proposing changes from crediting them: a language model diagnoses failures and proposes patches, while all sampling, measurement, and significance testing are owned by deterministic code, so every credited improvement is trustworthy by construction. Patches populate a gated, categorical quality-diversity archive (GSME) keyed on the (WHERE x WHY) pathology an edit addresses rather than the tasks it fixes, an anti-overfitting inductive bias; generalization is measured on a sealed test scored only after evolution. Across seven domains with a frozen open-weight model, the harness is train-selected and scored once on a sealed test; its credited gains there are +9 to +15.5pp and retain 86-147% of the training gain, evidence they generalize rather than overfit. The winning patch tracks the model's dominant pathology, not its size or family: changing the model can change the pathology and the patch, while the same pathology-to-patch match recurs across two model families. What transfers is the diagnose-and-credit loop, not any specific harness.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 智能体在部署时，只能调整其外围框架（harness），包括提示词、注入知识、运行时控制与配置，而模型权重固定不变。但现有自动调优方法难以区分真实改进与测量噪声或过拟合，因为自生成反馈不可靠，单次评估波动大，训练集上的增益可能仅仅是过拟合。
- **整体含义**：如果能够自动、可信地改进 harness，就能在不修改权重的情况下持续提升部署性能。本文提出一种自我进化的框架，核心创新在于将提议（生成补丁）与信用分配（判定是否有效）分离，通过确定性代码确保每次增益都是可信的。

## 2. 论文提出的方法论

### 核心思想
- **分离提议与信用分配**：一个更强的模型（evolver，如 Claude Opus 4.8）负责诊断故障、提出补丁；所有采样、测量、显著性检验由确定性代码执行，模型从不自行计算定量判断。
- **门控语义质量-多样性存档（GSME）**：补丁被存入一个基于（where × why）病理分类的存档，每个细胞仅保留一个精英。where 指编辑触及的杠杆（提示、知识、运行时、配置），why 是 LLM 诊断的失败病理（如“思考跑飞”、“过早终结”）。这种按病理而非任务分类的方式是一种抗过拟合的归纳偏置。
- **三重门控**：
  - **有效性门**：环境健康检查 + 基础设施故障重试，确保测量环境可靠。
  - **激活门**：只有补丁实际上被触发的证据才被归因。
  - **显著性门**：基于配对 2σ 检验（z≥1.96），在逐任务配对差值上做统计检验。

### 技术细节
- **演化流程**：
  1. 冷启动：在训练集上以 K=3 运行基线，获得逐任务日志。
  2. 每轮：evolver 读取上一轮轨迹（仅训练集），设计 3-4 个候选补丁；先通过预筛选（零推理检查激活信标），然后用锚定子集（约 15-30 个高方差任务）以 K=1 筛选，再全量 K=3 评分，通过三重门后存入 GSME。
  3. 选择：采用**质量偏差选择**（每轮扩展全局最佳 harness）与**跨细胞重组**，利用存档多样性。
  4. 终止：连续 P=10 轮无候选超过基线，或达到轮次上限 R=20。
- **密封测试**：训练结束后，在从未被使用过的密封测试集上单次评分训练选择的最佳 harness，计算保留率（密封增益÷训练增益）。

## 6. 主要结论与发现

1. **自进化 harness 可以带来可信、泛化的增益**：在六个领域上密封测试增益为 +9 到 +15.5 pp，保留训练增益的 86-147%，所有增益通过配对 2σ 检验。
2. **病理-补丁匹配律**：获胜补丁跟踪模型的主导病理，而非模型大小或系列。在 AppWorld 上，27B 的主导病理是“空参与”，补丁为 verify-finalize；397B 和 Gemini 的病理是“粗心错误”，补丁为 submit-verify 清单。相同病理在两家模型系列间重复出现。
3. **可转移的是诊断-信用循环**，而非具体 harness：因为主导病理随模型变化，harness 不直接跨模型转移，但循环本身可重复应用。
4. **GSME 存档有效**：多数领域的最佳 harness 是跨细胞重组的结果，且接受的编辑覆盖所有四个杠杆，而非仅局限在提示词调优。
5. **置信度门控机制确有必要**：无显著性门时，中性机制可能被误认为是“≥3pp 的赢家”约 25% 的时间；激活门排除了从未触发的补丁。

## 7. 优点

- **可信度设计**：将语义提议与定量信用分配严格分离，确保每个归因增益可重现、可审计。
- **抗过拟合归纳偏置**：GSME 按病理而非任务分类，防止搜索坍缩到任务标识符。
- **严格的统计检验**：配对 2σ 检验消除了任务间方差，比简单均值门槛更可靠；密封测试一次评分避免多次比较膨胀。
- **跨模型泛化证据**：通过 AppWorld 上的跨模型实验证明病理-补丁匹配律，增强结果的外部有效性。
- **成本感知**：树感知分数借用从理论上证明并实证验证可以减少 67% 的评价试次，为可扩展性铺路。
- **清晰的问题框架**：区分内核（不可变）与可变面，编辑空间明确（where 分类）。

## 8. 不足与局限

- **验证器依赖**：归因可信度完全依赖于逐任务验证器的可靠性；如果验证器可被对抗性利用或本身有噪声，则保证不再成立。
- **短视优化目标**：当前仅优化基准成功率，未考虑鲁棒性、可维护性、长期效用等模糊或长期目标。
