## 摘要
当前代理式人工智能的发展依赖于Token最大化：用Token购买能力——更长的推理轨迹、更多的轮次、更广泛的工具载荷、更大的重放上下文——因此每个任务的Token增长速度快于任务价值。Token单价的下降掩盖了这一模式；总支出仍在上升。我们认为，对抗Token最大化的关键杠杆是控制层：编排层，它组装上下文、暴露工具、序列化轮次、委派工作，并承载企业可观测性和治理。我们通过受控交换将其隔离：22个锁定评估任务，六个基础模型（Claude Sonnet 4.6、Gemini 3.1、Gemini Flash 3.5、Qwen 3.6、GLM 5.1、Palmyra X6），仅更改编排层——一个冻结的传统生产循环与Writer Agent控制层。在保持模型不变的情况下，控制层使每任务混合成本降低41%（从0.21美元降至0.12美元），中位挂钟时间降低44%（从48秒降至27秒），每任务Token降低38%（从14.2k降至8.8k），任务完成质量持平（0.78对0.81，在该样本量下具有方向性）。效率与模型无关——每个模型都变得更便宜（33%-61%）——而质量提升取决于能力：模型提升与其基线强度几乎完全相关（r=0.99，n=6），我们将此现象称为控制杠杆。每美元质量提升82%；每百万Token的任务完成数从54.9上升至92.0。在此工作负载下，编排层对每任务成本的影响超过了模型菜单的完整差异。我们形式化了编排层的Token经济学（包括提示缓存下的有效输入价格），详细阐述了该效应背后的六个机制家族——从缓存形状纪律到失败支出治理——在同一维度上比较了六个广泛使用的代理系统，并认为控制层是唯一一个其效率会乘数放大组织所运行的每个模型（包括现在和未来）的组件。

## Abstract
Agentic AI development today runs on token maxing: buying capability with tokens -- longer reasoning traces, more turns, wider tool payloads, bigger replayed contexts -- so tokens per task grow faster than task value. Falling per-token prices mask the pattern; total spend rises anyway. We argue the decisive lever against token maxing is the harness: the orchestration layer that assembles context, exposes tools, sequences turns, delegates work, and carries enterprise observability and governance. We isolate it with a controlled swap: 22 locked evaluation tasks, six foundation models (Claude Sonnet 4.6, Gemini 3.1, Gemini Flash 3.5, Qwen 3.6, GLM 5.1, Palmyra X6), changing only the orchestration layer -- a frozen conventional production loop versus the Writer Agent Harness. Holding models constant, the harness cuts blended cost per task 41% ($0.21->$0.12), median wall-clock 44% (48s->27s), and tokens per task 38% (14.2k->8.8k), with task-completion quality at parity (0.78->0.81, directional at this sample size). Efficiency is model-invariant -- every model gets cheaper (33-61%) -- while quality gains are capability-dependent: a model's gain correlates almost perfectly with its baseline strength (r=0.99, n=6), a phenomenon we term harness leverage. Quality per dollar rises 82%; task-completions per million tokens rise from 54.9 to 92.0. On this workload the orchestration layer moved cost per task more than the full spread of the model menu did. We formalize token economics at the orchestration layer (including effective input price under prompt caching), detail the six mechanism families behind the effect -- cache-shape discipline to failure-spend governance -- compare six widely used agent systems on the same axes, and argue the harness is the one component whose efficiency multiplies across every model an organization runs -- present and future.

# 论文总结：《控制效应：编排设计如何设定企业代理式人工智能的Token经济学》

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前Agentic AI开发陷入“token maxing”陷阱——通过不断消耗更多token来换取能力提升（更长的推理轨迹、更多的代理轮次、更大的工具载荷、更大的回放上下文），导致每个任务的token消耗增长速度超过其价值增量。虽然token单价持续下降，但总支出仍在攀升，形成类似“杰文斯悖论”的动态。
- **整体含义**：作者提出对抗token maxing的关键杠杆在于**编排层（harness）**——即决定上下文组装、工具暴露、轮次序列、委托工作和可观测性/治理的软件层。通过受控实验证明：固定任务和模型，仅更换编排层可使成本降低41%、延迟降低44%、每任务token减少38%，同时任务质量持平。编排层对成本的影响甚至超过了模型选择本身，且效率提升跨模型泛化（33%-61%），为Agent系统降本增效提供了全新架构级杠杆。

## 2. 方法论：核心思想、关键技术细节、公式/算法流程

### 核心思想
- **形式化token经济学**：将单个agentic任务的总成本分解由编排层控制的各项（系统提示、历史、工具模式、检索、用户轮次、重试和死分支），强调输入侧主导（因每轮重放历史导致输入输出token比例约100:1）。
- **有效输入价格模型**：基于prompt缓存，若输入token中缓存命中率为h，则有效输入价格为 p_eff_in = p_in * (1 - h*(1-κ))，其中κ≈0.1（缓存读取折扣）。h完全由编排层决定——通过保持prompt的字节稳定性来最大化缓存命中率。
- **定义token maxing**：token强度τ持续增长，而每token边际质量下降。提出效率指标：每美元质量η=Q/C和每百万token完成数CPM=Q·10^6/τ。

### 关键技术细节（Writer Agent Harness的六个机制家族）
1. **缓存形状纪律**：设计“两区prompt”——字节稳定的前缀（工具模式目录、稳定系统提示、只追加的副本）由缓存读取（99.9%为缓存命中），每轮重建的易变尾部（时钟、文件列表、计划状态等）排除在缓存外。利用提供商的缓存断点策略，将大部分输入token以约0.1×列表价计费。
2. **结构化、增量、缓存感知的压缩**：当长度达模型输入预算80%时，折叠历史为检查点（耐久记忆、执行摘要、保留的用户需求等），保留最近4-12条消息原文。压缩由廉价辅助模型离线执行，不破坏缓存前缀稳定性，将累积输入复杂度从O(k²)变为O(k)。
3. **上下文卸载**：多种机制使信息可用但不在上下文中——子代理担任上下文防火墙（返回摘要及侧边栏引用，父代理不支付探索token）；技能使用渐进式披露；大工具输出将完整内容写入工作区文件、上下文中仅放头尾预览；计划/画布状态事件源化、每轮紧凑渲染；后台任务结果一次性传递；已加载媒体有界。
4. **零token等待**：等待（如人工审批、后台作业）以持续状态挂起，不消耗token，通过入站事件恢复，不会通过轮询消耗token。持久化日志确保故障恢复时无需重放已完成的轮次。
5. **失败支出治理**：失败分类（速率限制、超时等），仅白名单类型允许回退到下一个提供商；流中断成为“废弃尝试”；断路器和原因感知的转向；循环上限50次、工具并行度4。限制重试带来的token乘数效应。
6. **模型无关地板**：模型选择、提供者、回退顺序作为数据提供，循环不分支模型名；原生工具调用仅一条路径，对弱模型进行模式卫生处理（内联$refs、恢复双重编码JSON参数等）。固定地板，模型决定天花板。

## 3. 实验设计：数据集/场景、benchmark、对比方法

- **任务集**：22个锁定基线prompt，覆盖9个能力领域：模型与系统感知（MSA）、接地与检索（GDR）、内容生成（CNG）、Playbooks（预定义多步工作流）、MCP工具使用、演讲生成（早期）、声音（品牌声音和自定义指令）、图像分析与生成，以及子代理（仅控制层有新能力）。
- **Benchmark**：质量指标为任务完成率（LLM作为评判者，锁定pass/fail标准）；效率指标为每任务成本、中位壁钟时间、每任务token量；衍生指标：每美元质量（η）和每百万token完成数（CPM）。还包含人工评分UX定性评估。
- **对比方法**：主要对比是**传统生产代理循环（baseline）** 与**Writer Agent Harness**。两者使用相同的22个任务、6个模型（Claude Sonnet 4.6、Gemini 3.1、Gemini Flash 3.5、Qwen 3.6、GLM 5.1、Palmyra X6）和相同的评判者与定价表。此外，第4.4节从架构层面对比了六个广泛使用的agent系统（Claude Code、Claude Cowork、LangGraph、CrewAI、AutoGen/AG2、Hermes Agent）在token经济机制上的差异（缓存策略、压缩合约、防火墙委托、零token等待、每任务记账等），但未进行实际测量。

- **充分性与公平性**：

## 6. 主要结论与发现

1. **效率大幅提升且跨模型一致**：混成成本降低41%（$0.21→$0.12），中位时间降低44%（48s→27s），每任务token减少38%（14.2k→8.8k）。每个模型成本均降低33%-61%，效率增益为模型无关。
2. **质量持平且有条件提升**：整体任务完成质量0.78→0.81（方向性持平）。30个能力-模型单元改善、7个退步（全出现在三个小模型上，集中在编排密集型能力如MCP工具使用、Playbooks）。质量增益与模型基线能力几乎完美相关（r=0.99），称为“控制杠杆”（harness leverage）。
3. **每美元质量提升82%，每百万token完成任务数从54.9升至92.0**，证明效率和能力并非此消彼长。
4. **子代理（净新增能力）仅在两个最强模型上达到可用阈值**（0.85-0.86），体现了编排功能的“能力地板”。
5. **编排层对成本的影响超过模型选择**：同组任务下，基线中最贵模型与最便宜模型成本差36%，而采用harness可节省33-61%。编排层是更大的成本杠杆。

## 7. 优点

- **实验设计清晰干净**：采用受控交换实验，通过固定任务、模型、评判者来隔离编排层效应，对比内部因果性强。
- **形式化框架实用**：系统定义了token经济学（token消耗分解、有效输入价格、token maxing定义、效率指标CPM和η），为该方向提供可量化的分析工具。
- **机制拆解详尽**：给出六个机制家族（缓存纪律、结构化压缩、上下文卸载、零token等待、失败治理、模型无关地板），每一机制对应公式中的具体项，可操作性强。

- **结果实用性**：提供了实际运营场景（百万级任务/月）的成本估算（每年节省超100万美元），并讨论了KPI导向（建议用CPM替代仅看质量）与路由优化（按功能需求路由而非仅难度），对工业界有直接指导意义。
- **诚实披露局限性**：明确声明样本量小、质量结论方向性、利益冲突、跨系统比较基于文档而非测量等，提升了整体可信度。

## 8. 不足与局限

- **样本量有限（n=22）**：质量差异（0.78→0.81）无法达到统计显著，属于方向性结果。效率提升虽大但仅覆盖一个基线-控制对比，未测量基线运行间自然方差。
- **单厂商控制层**：比较仅针对Writer的Harness与其自身基线，结果来自于单一实现，无法直接推广到其他编排设计。跨系统比较（第4.4节）仅基于公开文档和设计研究，无实际实验数据，属架构分析而非实证对比。
- **模型数量较少（n=6）**：harness leverage相关系数r=0.99仅基于6个数据点，虽趋势明显但需更大模型面板验证。
- **任务集偏企业助手**：包括接地、多步工作流、内容生成等，但未覆盖长时域编程基准（如SWE-bench），结果可能不适用编码类Agent场景。单次运行未取平均，缺乏随机性控制。

- **利益冲突**：所有作者来自开发该Harness的Writer公司，尽管实验设计旨在透明化，仍存在潜在偏差（如未公布其他编排设计的负面结果）。
- **未讨论训练或部署算力资源**：论文未提及训练或推理硬件成本，限制了对其整体能效比的评估。
