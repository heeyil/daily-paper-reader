## 摘要
基于可验证奖励的强化学习（如GRPO）是当前推理模型的引擎，但它仅对最终答案进行评分。在困难问题上，这导致模型更倾向于写更多内容而非提升思考质量，因为推理过程本身从未被评分，且没有用于评估良好思考的标签。我们提出Agon，它使两个竞争模型互为评分者。两个模型尝试解决同一问题；在交替角色中，一个模型起草解决方案，另一个在解题的同时阅读该方案，每个模型因比对手更优解题而获得奖励。为获胜，一个模型必须战胜已看到其工作的对手，因此推理在训练过程中被隐式评判，无需过程标签或奖励模型。由于两个模型均被优化，每个模型都面临逐渐增强的对手，这是单模型强化学习无法提供的。两者仅需实力相当且行为不同。在推理时，这对模型按训练方式部署，形成一个两阶段级联：一个模型起草，另一个阅读草案后作答。在DeepMath的困难子集上使用Qwen3时，该方法使GRPO的pass@1翻倍，相较于未经训练的混合智能体（Mixture-of-Agents）在同一基线上的提升约高出八倍。这一排序在竞争性编程代码及不同模型族（Qwen3.5、Gemma 4）上得到复现。目前模型以文本形式交互；下一步将让它们在隐空间中共同推理。

## Abstract
Reinforcement learning from verifiable rewards (e.g. GRPO) is the engine behind today's reasoning models, yet it grades only the final answer. On hard problems this trains models to write more rather than to think better, since the trace itself is never graded and no label for good thinking exists. We introduce Agon, which makes two competing models each other's graders. Both attempt the same problem; in alternating roles, one drafts a solution and the other reads it while solving, and each is rewarded for out-solving the other. To win, a model must out-reason a rival that has seen its work, so reasoning is judged implicitly during training, with no process labels and no reward model. Because both models are optimized, each faces a progressively stronger rival, which single-model RL cannot provide. The two need only be comparably strong and behaviorally different. At inference the pair deploys as it trains, a two-stage cascade in which one model drafts and the other answers after reading the draft. On the hard split of DeepMath with Qwen3, this doubles GRPO's pass@1, roughly eight times the gain of an untrained Mixture-of-Agents pass over the same base. The ordering replicates on competitive-programming code and across model families (Qwen3.5, Gemma 4). For now the models talk in text; the next step is to let them reason together in latent space.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前基于可验证奖励的强化学习（如 GRPO）仅对最终答案评分，不评价推理过程（trace）。在困难问题上，这会诱导模型通过延长输出长度来提高答对概率（“overthinking”），而非提升每 token 的推理质量。同时，直接标注“好思考”的步骤级标签不可行，学习过程奖励模型成本高且本身不可验证。
- **整体含义**：本文试图通过引入**第二个模型作为隐式评分者**，让训练过程中推理过程被隐式评判，从而打破“长度膨胀、质量停滞”的困境。这为提升 LLM 推理能力提供了一种无需过程标签或额外奖励模型的新范式。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：让两个竞争模型（A 和 B）在 RL 训练中互为评分者。每个问题下，角色交替：一个起草（drafter），另一个阅读起草者的解并挑战（challenger）。奖励取决于谁更好地解决了问题（out-solving the rival）。推理质量通过竞争结果被隐式评判。
- **关键技术细节**：
  - **角色旋转**：每轮训练步交换起草者和挑战者角色，使两个模型都能学习两种能力。
  - **奖励设计**：
    - 合作模式（coop）：`R(b_i) = 2 * c(b_i) + λ*ϕ(b_i)`。
    - 竞争模式（adv，即 Agon）：增加“转换奖励” `c(b_i)*(1-c(a_i))`，使正确解决对手失败的题目获得更高权重。这形成了难度加权，而不仅仅是简单差值。
  - **训练流**：每个问题生成 N 个起草者 rollout（独立），以及每个对应起草者下生成 1 个挑战者 rollout（N 个）。起草者组和挑战者组各自计算组内相对优势（group-relative advantage），分别更新。
  - **高效实现**：两个策略通过两个 LoRA 适配器（rank=16）共享同一个冻结基础模型，内存开销约 2%，而非双倍模型。两个适配器初始化不同、更新流不同以保持多样性。
  - **推理部署**：两阶段级联：模型 A 起草，模型 B 阅读摘要后给出最终答案；方向可互换。

## 6. 论文的主要结论与发现

- 信息交换（cooperative）比 vanilla GRPO 和自精炼提高显著（pa s s@1 46 vs 30 vs 32），支持“交换有益”论断。
- 加入竞争（Agon）进一步显著提升（61 vs 46），支持“竞争超越合作”论断。
- Agon 在 DeepMath-hard 上将 GRPO 的 pass@1 翻倍（30→61），增益约为未经训练的 MoA 的 8 倍（34→61，MoA 仅 +4）。
- 在相同推理预算下，训练后的级联远优于未训练的级联。
- Agon 的最终阶段轨迹平均更短（3.5k vs 8.1k GRPO 单通道），是竞争的自然结果而非显式惩罚。
- 增益随模型规模增大而减小，但跨模型族一致；小模型受益最大。
- 密度杠杆（长度 tiebreak）可在几乎不损失准确率的情况下进一步缩短轨迹。

- **效率**：通过双 LoRA 适配器实现两个策略，几乎不增加参数量和显存，训练开销主要来自 rollout 生成而非模型复制。
