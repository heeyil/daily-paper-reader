## 摘要
仓库操作由标准操作程序（SOP）管控，这些程序编码了复杂的多系统决策逻辑，必须在严格的时间限制下可靠执行。然而，LLM代理缺乏强制执行程序合规性的机制，并且在完整SOP规范引入的上下文过载下性能下降。我们提出Eluna，一个生产部署的代理系统，用于可靠的SOP执行。Eluna是一个图引导的多代理框架，将SOP编码为具有渐进式公开的有向无环图，并将独立任务委托给并行子代理，每个子代理具有持久的代码执行和实时数据访问。为了满足生产延迟和准确性需求，我们采用非对称情景蒸馏，其中强大的教师通过情景错误记忆得到改进，然后较小的学生在去除记忆的修正轨迹上进行微调，在不增加推理时间开销的情况下内化修正。在13个任务的基准测试和两个生产应用中，我们的微调模型匹配或超过了它们的教师，击败了所有更大的现成基线，并在工单处理应用中达到了94%的专家一致性。

## Abstract
Warehouse operations are governed by Standard Operating Procedures (SOPs) that encode complex, multi-system decision logic, which must be executed reliably under strict time constraints, yet LLM agents lack mechanisms to enforce procedural compliance and degrade under the context overload full SOP specifications introduce. We present Eluna, a production-deployed agentic system for reliable SOP execution. Eluna is a graph-guided, multi-agent framework that encodes SOPs as directed acyclic graphs with progressive disclosure and delegates independent tasks to parallel sub-agents, each with persistent code execution and live data access. To meet production latency and accuracy needs, we use asymmetric episodic distillation where a strong teacher is improved through episodic error memories, then a smaller student is fine-tuned on the corrected trajectories with memory stripped, internalizing corrections without inference-time overhead. On a 13-task benchmark and two production applications, our fine-tuned models match or exceed their teacher, beat all larger off-the-shelf baselines, and reach 94% expert agreement on the ticket processing application.

- **核心问题**：现代仓库操作由标准操作程序（SOP）管控，这些SOP包含复杂的多系统决策逻辑，必须严格、可重复地执行。然而，现有LLM代理（如ReAct、Reflexion）依赖松散结构的提示词，缺乏强制程序合规的机制；同时，将完整SOP规范一次性输入会导致上下文过载，使模型性能急剧下降，无法可靠地执行复杂工作流。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将SOP编码为有向无环图（DAG），通过渐进式披露只暴露当前可到达的子图和节点细节，结合层级多代理架构，将独立节点评估委派给并行子代理，实现上下文隔离与并行执行，从而避免上下文过载；再通过非对称情节蒸馏训练，让小型学生模型内化教师轨迹中的修正，无需推理时依赖外部记忆。
- **关键技术细节**：
  1. **图结构化SOP表示**：每个SOP表示为有向无环图 \( G=(V,E) \)，节点类型包括观察、根因、计算、约束、动作；边编码依赖关系。渐进式披露：先返回可达子图，评估每个节点时才加载完整规范。
  2. **层级多代理架构**：主代理（orchestrator）持有子图并决定下一个要评估的节点，分发任务给并行子代理（sub-agent），每个子代理获得独立上下文（仅含该节点规范、角色指令和工具）。子代理产生结构化结论后返回给主代理，主代理摘要性记录，避免上下文膨胀。动作节点由主代理自身执行。
  3. **技能打包（Skill Packaging）**：每个工作流作为自包含的“技能”加载，包含决策图、MCP工具、节点规范、CodeAct函数、执行指令。框架保持用例无关，新增工作流只需编写技能。
  4. **非对称情节蒸馏（Asymmetric Episodic Distillation）**：教师模型在完整框架内生成轨迹，通过情节学习（episodic learning）迭代修正错误（不更新权重），生成高质量轨迹；然后对轨迹进行拒绝采样（LLM作为裁判），保留正确轨迹；最后对学生模型进行监督微调（LoRA），**且训练时完全剥离情节记忆**，迫使学生将修正内化到权重中，从而消除推理时的记忆依赖。
  5. **工具生态**：包括持久代码解释器（CodeAct）、任务跟踪列表（TodoList）、MCP工具（实时数据访问）、代理RAG、持久化记忆（跨调用积累知识）。

## 6. 论文的主要结论与发现

1. **图引导框架显著提升程序合规性**：渐进式披露和子代理隔离有效缓解上下文过载，使模型能够可靠跟随SOP。
2. **蒸馏比规模化更有效**：微调后的32B模型（Eluna-Q）匹配甚至超越355B教师（GLM-4.7），击败所有更大现成模型，表明领域特定训练远胜于单纯增加模型规模。
3. **非对称蒸馏的关键性**：消融实验表明，仅教师使用情节记忆+学生训练时剥离记忆（即Eluna方式）达到与推理时也使用记忆相当的性能，而训练时使用记忆、推理时剥离则性能灾难性下降（-40%至-52.7%），证实了“内化修正到权重”的必要性。
4. **生产部署效果**：机器人输送应用在仓库中处理约8,000次触发，SOP遵守近乎完美，错误源于规范缺陷或数据质量而非推理；工单处理应用自动化94.4%的工单，精度96.7%，召回99.2%，处理时间从约6分钟降至平均210秒，大幅提升效率。

## 7. 优点

- **方法创新**：将SOP编码为DAG并结合渐进式披露和并行子代理委派，直接解决上下文过载问题；非对称情节蒸馏巧妙地将教师修正知识转移到学生权重中，既提升准确率又不增加推理延迟。
- **实用导向**：设计充分考虑生产延迟约束（通过小型学生模型满足SLA）和可扩展性（技能打包，新增工作流只需编写技能）。
- **实验充分**：涵盖多任务基准和两个实际生产应用，消融实验验证核心设计，对比模型覆盖不同规模与家族，结果统计严谨（4次运行平均）。
- **可复现与透明度**：开源框架使用Strands Agents SDK，训练配置详细给出，便于后续研究。
