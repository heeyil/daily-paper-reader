## 摘要
大型语言模型的高效推理需要确定在哪些地方可以减少计算量，同时保持模型质量。我们通过多层感知机（MLP）激活稀疏化和令牌级条件路由来研究这一问题。我们首先提出基于感知灵敏度的稀疏化阈值设置方法（SATS），该方法利用局部MLP输出灵敏度代理来选择每层的门控阈值，而不是直接根据激活百分位数校准阈值。SATS保留了通过门控激活阈值来稀疏化MLP激活的现有机制，但将基于百分位数的校准替换为灵敏度感知的选择规则。然后，我们引入一个轻量级的令牌路由框架，它在每个令牌的基础上动态选择基础路径或修改路径，而不是将修改后的计算统一应用于所有令牌。我们在多个最近的开源权重大型语言模型上评估了这两种方法。我们的结果表明，在匹配的实际稀疏度下，SATS优于基于阈值的稀疏化基线，并且令牌路由比静态激活修改基线实现了更有利的质量-吞吐量权衡。总体而言，我们的结果表明，改进的阈值校准和令牌路由可以改善大型语言模型中的质量-吞吐量权衡。

## Abstract
Efficient inference in Large Language Models (LLMs) requires deciding where computation can be reduced while preserving model quality. We study this problem through multilayer perceptron (MLP) activation sparsification and token-level conditional routing. We first propose Sensitivity-Aware Thresholding for Sparsity (SATS), a threshold calibration method to choose layerwise gate thresholds using a local MLP output sensitivity proxy rather than calibrating thresholds directly from activation percentiles. While SATS retains the existing mechanism of sparsifying MLP activations by thresholding gate activations, it replaces percentile-based calibration with a sensitivity-aware selection rule. We then introduce a lightweight token routing framework that dynamically selects between a base path and a modified path on a per-token basis, rather than applying the modified computation uniformly to all tokens. We evaluate both methods on multiple recent open-weight LLMs. Our results show that SATS improves over the threshold-based sparsification baseline at matched actual sparsity and that token routing yields a more favorable quality-throughput trade-off than static activation modification baselines. Overall, our results suggest that improved threshold calibration and token routing can improve the quality-throughput trade-off in LLMs.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）推理成本高昂，其中MLP层计算占比大。现有工作通过阈值门控激活实现稀疏化（如CATS），但阈值校准仅依赖激活百分位数，未考虑不同阈值对MLP输出造成的实际损伤；此外，稀疏化计算被统一应用于所有令牌，忽略了不同令牌对修改计算的敏感性差异。
- **研究动机**：在保持模型质量的前提下，进一步减少计算量，需要更“损伤感知”的阈值校准和更灵活的令牌级计算路径选择。
- **整体含义**：提出两种互补方法——灵敏度感知阈值校准（SATS）和令牌路由（Token Routing），旨在改善LLM推理的质量-吞吐量权衡。

## 2. 论文提出的方法论

### 2.1 SATS：灵敏度感知阈值校准

- **核心思想**：不使用百分位数选择阈值，而是为每个候选阈值计算一个局部MLP输出失真（Err_ℓ(t)），并设定层误差预算b，选择满足失真≤b的最大阈值。
- **关键公式**：
  - 局部失真：Err_ℓ(t) = E_x[ ||y_ℓ(x) - ŷ_{ℓ,t}(x)||₂ / (||y_ℓ(x)||₂ + ε) ]
  - 阈值选择：t*_ℓ(b) = max{ t ∈ T_ℓ : Err_ℓ(t) ≤ b }
  - 全局稀疏度目标：通过扫描b找到最接近目标稀疏度s_target的分配。
- **流程**：①在校准数据集上构建候选阈值集T_ℓ（基于激活幅度）；②估计每个t的局部失真；③对给定预算b，逐层选择阈值；④外循环搜索b以实现目标稀疏度。
- **保留CATS运行时机制**：推理时仍使用门控激活掩码（masking），仅修改阈值校准方式。

### 2.2 令牌路由

- **核心思想**：每个令牌独立选择走密集MLP路径（base path）还是稀疏MLP路径（sparse path，即应用SATS阈值化的路径）。路由仅基于令牌身份（token ID），实现为预计算查找表，推理时开销忽略不计。
- **构建路由表**：在校准语料上，对每个令牌出现位置计算稀疏路径相对于密集路径的损失差：d_i = max(0, ℓ_sparse - ℓ_base)；对同一令牌身份x，平均其d_i得到s(x) ≈ E[d_i | x_i = x]。
- **路由决策**：若s(x) < τ，则走稀疏路径；否则走密集路径。τ通过目标路由比例（如25%、50%、75%）在留出集上校准。

## 6. 论文的主要结论与发现

- **SATS优于CATS**：在相同实际稀疏度（30%、50%）下，SATS在多数指标（平均准确率、困惑度）上超过CATS，尤其在高稀疏度（50%）时优势更明显。
- **令牌路由改善质量-吞吐权衡**：路由SATS在保持大部分吞吐增益的同时，准确率接近密集模型，且优于静态稀疏模型。例如，在LLaMA 3.1 8B上，50%路由比例平均准确率76.45 vs 静态稀疏76.20 vs 密集77.11，而吞吐接近静态稀疏。
- **微调后路由效果**：微调后稀疏路径损失降低，但路由仍能带来质量提升（尤其Qwen 3，路由75%时准确率甚至超过密集模型）。但微调后路由的吞吐增益因LoRA额外计算而略有缩小，通过定制稀疏核可缓解。
- **局部失真代理有效**：局部MLP输出失真与端到端模型性能一致，可作为可靠的校准信号。

## 7. 优点

- **方法创新**：SATS在保留CATS高效运行时掩码的同时，引入损伤感知校准，使得稀疏化更精确；令牌路由轻量级（查找表），无需额外网络或计算。
- **实验验证充分**：在两个主流开源模型上进行了训练-free和微调两种设置，对比了多组稀疏度和路由比例，并报告了精度/困惑度/吞吐三个维度。
- **实用导向**：关注实际推理部署，考虑了精度格式（FP32 vs BF16）、微调兼容性（LoRA），并设计了定制Triton核减少开销。

## 8. 不足与局限

- **模型规模单一**：仅测试8B模型，未验证在更大（如70B）或更小（如1B）模型上的泛化性。
