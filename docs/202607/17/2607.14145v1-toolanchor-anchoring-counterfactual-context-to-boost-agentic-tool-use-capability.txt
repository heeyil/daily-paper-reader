## Abstract 

Tool-augmented large language model agents excel at long-horizon tasks, yet they are typically post-trained on fixed toolsets. When tasks demand new tools, these agents struggle to incorporate them effectively, and retraining from scratch is often impractical. We identify the core obstacle in such toolset expansion problem as behavioral inertia : the tendency of agents to fall back on familiar tools and established reasoning patterns despite having access to new ones. We demonstrate that injecting counterfactual anchor contexts at critical decision points can break this inertia, recovering failed trajectories by eliciting suppressed agent capabilities. To scale this insight, we propose ToolAnchor, a framework that uses teacher models to hypothesize these counterfactual contexts, verifies them via student rollouts, and 

internalizes the successful interventions through agentic post-training. Extensive evaluations across general AI assistant (GAIA), textual search (BrowseComp), and visual search (VDR-Bench) tasks demonstrate that ToolAnchor consistently exhibits competitive performance under expanded toolsets. Our work bridges the gap between static post-training and dynamic adaptation, charting a new path for scalable agentic reinforcement learning. 

## 1 Introduction 

Recent advances in tool-augmented agents have significantly broadened the real-world utility of Large Language Models (LLMs) [ 1, 2, 3 , 4 , 5 , 6 ]. By integrating external tools into the reasoning loop, LLM-based agents have achieved strong performance across a wide spectrum of downstream tasks, spanning text-based question answering [ 7 , 8, 9 , 10 ], visual question answering [ 11 , 12 , 13 ], long-form text report generation [ 14 , 15 ], and domain-grounded tool use [ 16 , 17 ]. Central to this progress are the ReAct paradigm [ 18 ], which interleaves reasoning with tool invocation, and agentic reinforcement learning (agentic RL) methods [ 19 , 20 , 21 ], which further refine tool-use capabilities via post-training. However, these successes largely assume a fixed, predefined toolset available throughout training. In practice, task requirements evolve continuously, often necessitating new tools such as web search for enhanced retrieval [ 22 , 23 ], python interpreters for scientific computation [ 24 , 25 ], file parsers for document understanding [26], and visual tools for multimodal perception [12, 4]. A natural question then arises: how can an existing agent efficiently adapt to an expanded toolset? 

The most straightforward approach is to directly apply agentic RL, allowing the agent to discover effective reasoning strategies and action policies via exploration. However, this strategy faces a fundamental difficulty. Since the agent’s policy has already converged during prior agentic RL training, it tends to fall back on familiar behavioral patterns, underutilizing newly introduced tools and repeating suboptimal reasoning strategies. This behavioral inertia leads to sparse rewards and inefficient exploration, as the agent rarely ventures beyond its established routines. An alternative

Yet, as our preliminary analysis in Section 3.3 reveals, generalization alone is insufficient. To better understand this bottleneck, we conduct a motivation study on failed student trajectories. A teacher model first identifies a decision round where an alternative local thought–action pair could plausibly redirect the subsequent rollout; we call this decision point a counterfactual anchor round (Section 3.2). The teacher then proposes this alternative using only the history before that round, yielding a teacher-revised context from which the student resumes generation. Compared with resuming from the original context, these teacher-revised contexts improve task success in both textual and visual search settings. This suggests that failures arise not only from limited tool-use capability, but also from missing contextual anchors that redirect reasoning and tool selection at key decision points. In this work, we propose ToolAnchor , a framework that bootstraps new tool-use capabilities through verified counterfactual context anchoring. ToolAnchor operates in three stages. In the Counterfactual Anchor hypothesis stage, diverse teacher models audit the student agent’s failed rollout trajectories and hypothesize anchor rounds where the agent’s decision deviated from a promising path. While the primary focus is on rounds where the agent failed to invoke a newly introduced tool, the teacher also considers suboptimal uses of existing tools and flawed reasoning steps, as these often compound the difficulty of adopting new tools effectively. For each identified anchor round, the teacher hypothesizes alternative reasoning and actions that would plausibly redirect the trajectory toward task completion. In the Counterfactual Anchor Verification stage, the student agent resumes generation from these teacher-anchored contexts under agentic rollout. Crucially, we retain for training only those anchor contexts that satisfy both verification criteria, namely task-level success and anchor-level usefulness, rather than accepting teacher hypotheses uncritically. In the Counterfactual Anchor Internalization 

stage, the verified contexts are used for agentic supervised fine-tuning (agentic SFT), allowing the agent to absorb the anchored behaviors into its own policy. This is followed by agentic RL, which realigns the internalized anchor behaviors with the agent’s previously learned agentic policy and supports robust, autonomous use of the expanded toolset. Our contributions are summarized as follows: ( 1) We formalize the practical challenge of expanding a post-trained agent’s toolset. Through a motivation study, we identify behavioral inertia as the core bottleneck and reveal that counterfactual contextual anchoring at critical decision points can effectively redirect failed trajectories. ( 2) We propose ToolAnchor, a scalable, three-stage framework for capability expansion. It automates the discovery of counterfactual anchor rounds via diverse teacher models, strictly verifies their effectiveness through student rollouts, and internalizes these validated behaviors via agentic SFT and RL. ( 3) Extensive evaluations across general AI assistance (GAIA), textual search (BrowseComp), and visual search (VDR-Bench) demonstrate that ToolAnchor seamlessly integrates new visual tools. Furthermore, it simultaneously boosts the base agent’s reasoning and existing tool mastery, offering a robust paradigm for continuous agentic evolution. 22 Related work 

Tool-augmented deep research agents. Deep research agents tackle open-ended question answer-ing through long-horizon tool use: iteratively planning, invoking external tools, verifying evidence, performing intermediate reasoning, and synthesizing final answers [ 3 ]. This paradigm naturally encompasses general AI assistance, textual search, and visual search. Textual search remains the most extensively explored direction, where agents retrieve external evidence to answer complex queries. For instance, Search-R1 [ 22 ], Search-o1 [ 23 ], and WebThinker [ 29 ] directly integrate web search into reasoning, while subsequent work advances this via agentic SFT and RL [ 30 ], systematic uncertainty reduction [ 31 ], scalable data synthesis [ 32 ], dual-environment RL frameworks [ 31 ], and continual pre-training [ 33 ]. Building on these techniques, the Tongyi DeepResearch Agent [ 26 ] achieves competitive performance, and OpenSeeker [ 34 ] provides high-quality training datasets. Visual search extends this paradigm to queries requiring visual evidence, demanding agents to not only retrieve web pages but also understand and ground visual content in external knowledge. Benchmarks like VDR-Bench [ 35 ] and VisBrowse [ 13 ] evaluate this capability, alongside VLM-driven agents such as T3-Agent [36] and Vision-DeepResearch [12] which explore multimodal tool integration. However, existing visual search approaches predominantly build agents from scratch with jointly designed toolsets, rather than extending well-trained text-oriented agents. This leaves a practical gap: given the immense cost of training capable deep research agents, incrementally augmenting them with visual tools is far more practical than starting over. We address this gap as a toolset expansion problem. 

Agentic reinforcement learning. Reinforcement learning has emerged as a powerful post-training paradigm for LLM-based agents [ 21 , 37 , 38 ]. GRPO [ 19 ] introduced group relative policy optimiza-tion for mathematical reasoning, and DAPO [ 20 ] further enhanced its performance through a series of techniques. Although these methods can be applied to agentic RL settings, multi-turn tool-integrated reasoning introduces distinct challenges, such as distributional shift caused by tool feedback that deviates from the LLM’s training data. To mitigate these issues, several methods have been proposed: SimpleTIR [ 25 ] filters trajectories containing void turns, GiGPO computes both episode-level and step-level relative advantages, and ARPO [39] and AEPO [40] use entropy-based adaptive beaming strategies to execute partial rollouts at high-entropy tokens. Despite these advances, existing agentic RL methods share a common assumption: the toolset is fixed throughout training. When new tools are introduced to a post-trained agent, its policy has already converged around the original toolsets, and standard agentic RL exploration mechanisms may be insufficient to overcome this behavioral inertia . Our work targets this setting through a teacher-guided anchoring mechanism that is then validated by student rollout. This design contrasts with standard knowledge distillation [ 41 ] and on-policy imitation methods such as DAgger [ 42 ], which treat teacher-generated reasoning and actions as ground-truth supervision. Our framework treats teacher guidance as a hypothesis to be verified through student rollouts rather than as a direct supervised target. 

## 3 Preliminaries 

3.1 Tool-use agent setup 

Following Tongyi DeepResearch Agent [ 26 ], we consider a post-trained tool-use agent equipped with an original toolset Uold . In our experiments, Uold consists of web search, web browsing, a Python interpreter, scholar search, and a file parser. To study toolset expansion, we introduce a set of new visual tools Unew , including an image descriptor, a bounding-box extractor, and crop-and-image search. The resulting expanded toolset is U = Uold ∪ Unew . Detailed tool descriptions are provided in Appendix C.1. We follow the ReAct framework, which consists of thought, action, and observation in each round. At each round, the agent first generates a thought and an action, and then receives an observation from the environment. This process forms a trajectory, denoted as HT , which is a sequence of thought–action–observation triplets: HT = ( τ0, a 0, o 0, . . . , τ i, a i, o i, . . . , τ T , a T ),where aT represents the final answer to the given question. For each non-terminal round t < T , the agent samples (τt, a t) ∼ πθ (· | H t−1), after which the environment executes at and returns ot. The terminal action aT corresponds to the final answer, and no observation oT is returned. 33.2 Counterfactual anchor contexts

4.1 Problem formulation 

Building on the setup in Section 3.1, we study toolset expansion for a post-trained tool-use agent πθ .The agent has been trained with the original toolset Uold and is then exposed to the expanded toolset 

U = Uold ∪ Unew . The goal is to learn an updated policy πθ′ that improves task success under U

while preserving the agent’s existing capability on tasks solvable with Uold .Our preliminary analysis in Section 3.3 reveals two key findings. First, the post-trained agent exhibits 

behavioral inertia , disproportionately relying on Uold even when Unew is available (Table 1). Second, failed trajectories can often be recovered when the agent resumes generation from a teacher-proposed counterfactual anchor context at a critical decision round (Table 2). These findings suggest that the bottleneck is not only missing tool-use capability, but also the lack of effective contextual anchors that redirect the agent away from its established behavioral patterns. This motivates ToolAnchor, which hypothesizes, verifies, and internalizes such contexts, as illustrated in Figure 2.

We first roll out the base policy πθ on the training datasets under the expanded toolset U , collecting both successful and failed trajectories. We define the failed trajectory set as Dfail = {H T | Success( HT ) = 0 }, which serves as the input for the teacher-guided counterfactual anchor hypothesis. 

Counterfactual round hypothesis. We define an anchor round hypothesis as a candidate round at which an alternative local decision may induce a different subsequent rollout and potentially change the task outcome. Forming this hypothesis requires assessing how earlier decisions, observations, and subsequent reasoning steps interact across the trajectory, rather than relying on a fixed heuristic such as choosing the first or last suspicious step. We instantiate this step with a prompt-guided teacher audit: given the full failed trajectory, a teacher model sequentially reviews each round’s thought 

τt, action at, and observation ot, and selects a candidate anchor round t∗ where a revised context is likely to redirect the subsequent rollout. The prompt used in our implementation is provided in Appendix B.2. In our implementation, the teacher is instructed to use the following criteria: 

Criterion 1: Downstream impact. Prefer a round whose alternative local decision is likely to influence multiple subsequent steps in the trajectory. 

Criterion 2: Path redirection. Prefer a round that acts as a branching point, where a different local decision may redirect the subsequent rollout toward a different reasoning or tool-use path. 

Criterion 3: Strategic leverage. Prefer a round involving high-level agentic decisions, such as rea-soning direction, tool selection, query formulation, evidence interpretation, information integration, or planning. 

Criterion 4: Counterfactual potential. Prefer a round where replacing the original local decision under the same history appears most likely to improve the final task outcome. These criteria are intended to avoid trivial anchor-selection heuristics, such as always choosing the earliest suspicious step or the final decision step, both of which may fail to identify the round with the greatest potential to redirect the subsequent rollout.

A teacher may select an ineffective anchor round, propose a plausible but unhelpful alternative decision, or 6generate an observation that does not support the student’s subsequent reasoning. Policy realignment and exploration (post-SFT agentic RL). Following agentic SFT, we apply agentic RL (GRPO) on the expanded toolset U , using a binary task-level reward (1 for correct answers, 0 otherwise). This stage serves to integrate the internalized anchor behaviors from SFT with the agent’s prior RL policy, consolidating these local updates into a policy that is both aligned with prior knowledge and capable of effective inference under the expanded toolset. Simultaneously, it enables the agent to explore additional tool-use strategies beyond those explicitly provided by the teachers. Collectively, agentic SFT and agentic RL ensure that local counterfactual behaviors are internalized, properly aligned with the prior policy, and generalized, allowing the agent to leverage both newly introduced tools and previously learned capabilities. 

## 5 Experiments 

5.1 Experimental setup Student and teacher models. We use the Tongyi DeepResearch Agent [ 26 ] (Qwen3-30B-A3B) as our base model πθ , which is trained via agentic RL on a fixed toolset Uold consisting of web search, 7web browse, a Python interpreter, scholar search, and a file parser. We extend it with three visual tools Unew : an image descriptor, a bbox extractor, and crop-and-image search. We choose the Tongyi DeepResearch Agent not only because its system prompts, toolsets, and model weights are open-source (allowing for training and customization), but also because it demonstrates strong performance on general AI assistant and textual search tasks. For the counterfactual anchor hypothesis, we employ Gemini-2.5-Pro and GPT-5 as diverse teacher models to identify anchor rounds and generate alternative reasoning and actions. 

Training datasets and benchmarks. We use two training datasets that span both textual and visual search tasks: (1) OpenSeeker-1.5k, which contains 1,500 instances sampled from OpenSeeker-v1-Data [ 34 ]; and (2) VDR-1.5k [ 12 ], a visual search dataset that also contains 1,500 samples. We evaluate our method on three benchmarks corresponding to distinct tasks: GAIA [ 10 ] (General AI assistant, 165 samples), BrowseComp [ 8] (Textual search, 200 samples subset following [ 34 ]), and VDR-Bench [35] (Visual search, test-mini split with 500 samples, following [12]). 

Evaluation metrics and baselines. Following prior work [ 12 , 34 ], we adopt an LLM-as-a-judge approach for evaluation, using Qwen3-235B-A22B as the judge model. Specifically, we use the offi-cial prompts provided by each benchmark to assess the correctness of the agent’s final answer against the ground truth. Our primary evaluation metric is task success rate under Pass@1, where each query is evaluated with a single agent rollout. All benchmark results are reported with Pass@1. Pass@3 is used only during counterfactual anchor verification, where a candidate context passes task-level verification if at least one of three student rollouts solves the task. Please refer to Appendix B.4 for the corresponding evaluation prompts. We compare ToolAnchor against both proprietary and open-source deep research agents, including GPT-5 [ 45 ], Gemini-2.5-Pro [ 46 ], Claude-4-Sonnet, OpenAI Deep-Research, Qwen3-VL-30B-A3B-Thinking [ 47 ], Qwen3-VL-30B-A3B-Instruct, OpenSeeker [ 34 ], Vision DeepResearch [ 12 ], and Tongyi DeepResearch [ 26 ]. Implementation details, including training hyperparameters, inference hyperparameters, and compute resources, are provided in Appendix C.2.

Post-training method. Figure 3 ablates the post-training pipeline after visual-tool expansion. Agentic SFT on unverified counterfactual contexts performs poorly, especially on BrowseComp, indicating that teacher-hypothesized contexts can introduce noisy or ineffective supervision if applied without student-side verification. Using verified contexts improves over unverified agentic SFT on BrowseComp and VDR-Bench, supporting the utility of counterfactual anchor context verification. Agentic RL trained from scratch achieves moderate performance (70.3%, 41.5%, 21.6%), indicating that without access to counterfactual anchor contexts via SFT, RL alone exhibits limited exploration and fails to fully leverage the available toolset. However, agentic SFT with verified contexts alone still underperforms the ToolAnchor, suggesting that anchor-context internalization through SFT is insufficient to fully recover and improve the agent’s tool-use policy. By combining verified SFT with a subsequent agentic RL, ToolAnchor achieves the best performance across all three benchmarks. This indicates that verified counterfactual contexts provide effective local supervision, while the post-SFT RL stage aligns the internalized anchor behaviors with the agent’s previously learned policy.

How many counterfactual contexts survive verification? For each teacher-generated counterfac-tual context, we sample three student rollouts and use Pass@3 task-level verification, where a context passes if any rollout solves the task. We then apply anchor-level verification to successful rollouts, retaining only contexts whose success is judged materially attributable to the anchor intervention. Overall, 33.67% of candidates pass task-level verification and 31.98% remain after anchor-level verification, showing that verification filters noisy teacher hypotheses while retaining contexts that both recover the task and validly intervene on the failure point. 

5.5 Case study 

In Figure 1, we present a case study to illustrate how ToolAnchor redirects a failed visual-search trajectory. In this example, the base Tongyi DeepResearch Agent first uses visual tools to locate the green-hooded character and invoke the crop-and-image search tool, but then dismisses the returned visual-search leads as generic and switches to broad text search, eventually producing an incorrect answer. The teacher hypothesizes that this decision is the counterfactual anchor round, since it is the key point where the agent abandons the visual-evidence path. The generated counterfactual context instead keeps the agent on the visual-search path by analyzing the crop-and-image search results and issuing a targeted follow-up query. This qualitative example illustrates how a local counterfactual context can redirect the subsequent rollout. Details are provided in Appendix A. Consistent with this qualitative example, Appendix C.3 shows that hypothesized counterfactual anchor rounds are broadly distributed across different trajectory positions, with most occurring within the first 10-20 rounds, and that each can potentially produce distinct effects on the agent’s subsequent behavior. 

## 6 Conclusion and limitation 

In this paper, we propose ToolAnchor, a framework for extending post-trained tool-use agents with new tools through verified counterfactual context anchoring. Our main insight is that toolset expansion is not merely about adapting tool-selection and reasoning patterns to newly introduced tools. Post-trained agents may also need effective contextual anchors to break out of established reasoning and tool-use patterns. ToolAnchor turns teacher-hypothesized counterfactual contexts into training signals only after student-side verification, allowing the agent to internalize local high-leverage redirections rather than imitate full teacher trajectories. Experiments across general assistance, textual search, and visual search show that this mechanism improves new visual-tool use while also benefiting existing tool use and agentic reasoning, suggesting a practical path for capability expansion without retraining agents from scratch. In this work, the experiments rely on a limited set of teacher models and verified anchor contexts. Future research could investigate incorporating a more diverse set of teacher models and expanding the training dataset to enhance coverage, robustness, and generalization of internalized behaviors across a broader range of tasks and toolsets. 10 References

Python interpreter A Python interpreter tool that executes Python code within a secure sandbox. 

Scholar search A scholar search tool that executes Google Scholar searches and retrieves the top 10 results. It accepts natural language queries and returns structured data, including page titles, URLs and snippets. 

File parser A file parser tool that analyzes various files and extracts their content as plain text. Bbox extractor A bounding box (bbox) extractor that uses the multimodal model Qwen2.5-72B-Instruct to locate specific entities within an image. It accepts both local image files and URLs, as well as entity queries, and returns the bounding box coordinates of the requested entity. 

Crop-and-image search An image cropping and search tool that executes Google Lens and retrieves the top 10 results. It accepts an image as input and returns structured data, including titles, URLs, and snippets. The snippets are generated by the multimodal model Qwen2.5-72B-Instruct, which compares the similarities and differences between the query image and the webpage images, a design motivated by VC-STaR [48]. The VDR toolset consists of a web search tool, a web browsing tool, a Python interpreter, and a visual search tool.
