## 摘要
本文提出PrismAD，一种基于语义混合规划器的解耦端到端自动驾驶框架。现有规划器通常将异质场景标记聚合到一个耦合表示空间中，迫使单个规划分支联合建模智能体交互、道路几何和驾驶意图。这种耦合可能削弱特定因素的推理，并模糊不同规划线索的贡献。为解决这一限制，PrismAD将场景标记划分为交互、几何和意图三组，并将它们分配给具有相同架构但参数独立的规划专家。每个专家学习一个专门的运动规划表示，而语义感知路由器利用运动预测和自车规划的独立路由权重自适应地聚合专家预测。进一步引入带噪声门控的稀疏top-K激活，以提高路由鲁棒性并减少不必要的专家计算。在nuScenes开环数据集和NeuroNCAP闭环基准上的大量实验表明，PrismAD表现出竞争力的性能。我们的代码将很快开源。

## Abstract
This letter presents PrismAD, a decoupled end-to-end autonomous driving framework based on a Semantic Mixture-of-Planners. Existing planners usually aggregate heterogeneous scene tokens into a coupled representation space, forcing a single planning branch to jointly model agent interaction, road geometry, and driving intention. Such coupling may weaken factor-specific reasoning and obscure the contribution of different planning cues. To address this limitation, PrismAD partitions scene tokens into interaction, geometry, and intent groups, and assigns them to independent planning experts with the same architecture but separate parameters. Each expert learns a specialized motion-planning representation, while a semantics-aware router adaptively aggregates expert predictions with separate routing weights for motion prediction and ego planning. Sparse top-$K$ activation with noisy gating is further introduced to improve routing robustness and reduce unnecessary expert computation. Extensive experiments on the nuScenes open-loop dataset and NeuroNCAP closed-loop benchmark demonstrate that PrismAD exhibits competitive performance. Our code will be released soon.

## 2. 方法论

### 核心思想
- 将场景令牌划分为三个语义组：**交互（Interaction）**、**几何（Geometry）**、**意图（Intent）**。
- 每组令牌分配给一个独立的规划专家（Expert），所有专家共享相同的网络架构但拥有独立的参数。
- 每个专家学习一个专门的运动规划表示。
- 引入**语义感知路由器（Semantics-Aware Router）**，为运动预测和自车规划分别计算独立的路由权重，自适应地聚合各个专家的预测结果。
- 采用**稀疏top-K激活**与**噪声门控（Noisy Gating）**，提升路由鲁棒性并减少不必要的专家计算开销。

### 关键技术细节
- **令牌分组**：根据语义属性将原始场景令牌分为三组：
  - 交互组：包含其他交通参与者的动态信息。
  - 几何组：包含道路结构、车道线、可行驶区域等静态几何信息。
  - 意图组：包含自车的目标意图（如变道、直行、转弯）和高层指令。
- **专家网络**：每个专家是一个轻量级规划模块，输入对应的语义令牌，输出两种规划结果：运动预测（其他智能体未来轨迹）和自车规划（自车未来轨迹）。
- **路由器设计**：
  - 路由器接收所有专家的输出，为运动预测和自车规划分别学习一套路由权重。
  - 采用**稀疏门控机制**：仅激活 top-K 个专家（K < 专家总数），其余专家输出被抑制。
  - 门控中引入**高斯噪声**以增加探索性，避免路由坍塌（routing collapse）。
- **聚合方式**：最终规划结果 = 各专家输出与对应路由权重的加权和。

### 公式/算法流程（文字说明）
1. 输入：异构场景令牌集合。
2. 按语义分组，得到三组令牌：`T_interact`, `T_geom`, `T_intent`。
3. 将各组分发给对应的专家 `E_interact`, `E_geom`, `E_intent`。
4. 每个专家独立计算运动预测和自车规划。
5. 语义感知路由器接收所有专家输出，通过带噪声的稀疏门控（top-K softmax）生成两组路由权重：`w_pred`（用于运动预测聚合）和 `w_plan`（用于自车规划聚合）。
6. 加权聚合：`MotionPrediction = Σ w_pred_i * ExpertOutput_i`，`EgoPlanning = Σ w_plan_i * ExpertOutput_i`。
7. 输出最终规划轨迹。
