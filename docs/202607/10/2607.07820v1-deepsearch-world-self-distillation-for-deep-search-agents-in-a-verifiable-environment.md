## 摘要
训练工具使用型智能体从自身经验中改进仍具挑战，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长程交互提供的监督较弱。我们提出DeepSearch-Evolve，一种基于DeepSearch-World构建的Web智能体自蒸馏框架。DeepSearch-World是一个确定且可验证的环境，配有可复现的搜索和页面阅读工具，包含从实体级随机游走构建的42万个多跳问答任务，并支持对自我进化有用的关键智能体认知行为，包括进度验证、基于经验的反思和故障恢复。DeepSearch-Evolve通过迭代执行轨迹生成、筛选、数据混合和微调来训练更强的智能体。无需从更强模型中蒸馏，DeepSearch-World-9B在与开源智能体的竞争中达到了具有竞争力的性能：在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，表明可验证环境能使长程Web智能体实现可扩展的自我进化。我们将发布环境、42万训练池、验证集、模型和代码，以促进未来对自我改进深度搜索智能体的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

- **核心挑战**：现有工具使用型智能体（如 Web 搜索代理）的自我改进方法存在瓶颈：
  - **监督微调（SFT）** 依赖固定的教师蒸馏轨迹，性能受限于教师能力，且容易饱和。
  - **稀疏奖励强化学习（RL）** 对长程交互的监督不足，难以定位失败原因（如查询构造、工具选择、证据提取等环节）。
- **研究动机**：当前方法缺乏可验证、可复现的环境来提供**过程级（process-level）监督**。受限于奖励稀疏性、静态模仿或不可靠的密集监督，代理难以从自身交互中持续进化。
- **整体含义**：提出一种**可验证环境 + 自蒸馏框架**，使代理能基于自身经验（而非更强模型）实现可扩展的自我改进，为长程代理的自主进化探索新路径。

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
- **DeepSearch-World**：一个**确定性、可验证的离线 Wikipedia 环境**，包含 42 万个多跳 QA 任务（基于实体级随机游走构造），支持进度验证、基于经验的反思和故障恢复。
- **DeepSearch-Evolve**：一种**迭代式自蒸馏框架**，代理通过以下循环自我改进：
  1. **轨迹生成与验证**：当前模型作为**教师**，在 DeepSearch-World 中使用**脚手架（scaffold）** 策略生成轨迹（含规划、行动、反思、状态更新），并通过答案正确性和轨迹质量过滤。
  2. **Scaffold-to-ReAct 转换**：将脚手架轨迹中的规划、记忆、反思等信号重写为标准 ReAct 格式的 <think> 块，作为学生训练的目标。
  3. **监督式自蒸馏更新**：对过滤后的 ReAct 轨迹进行 SFT 训练（等效于硬标签 token 级 KL 散度优化），同时采用**重要性采样**混合多轮轨迹（衰减因子 γ=0.5），优先使用最近验证数据，防止灾难性遗忘。

#### 关键技术细节
- **环境构造**：
  - 基于 Wikipedia 超链接图，随机游走生成 2-4 跳实体链，构造多跳问题，并通过**特征模糊化**防止记忆。
  - 工具接口：`web_search_wiki`（BM25 检索，返回标题/摘要/URL）和 `visit_wiki`（通过 SQLite 偏移索引读取全文），模拟真实搜索–浏览流程。
- **脚手架教师**：分为三个阶段：**PLAN**（初始化进度状态，含 `completed_list`, `todo_list`, `experience`, `information`）、**ACT**（最多 30 步，工具调用→观察→环境反射→状态更新）、**END**（基于已验证证据生成最终答案）。
- **轨迹过滤**：两步筛选：① 答案正确性（拒绝采样）；② 轨迹质量（由 Qwen3.5-9B 评估冗余性、目标一致性和逻辑合理性）。

### 6. 论文的主要结论与发现

1. **可验证环境是自我进化的关键**：DeepSearch-World 提供确定性、过程级验证，使代理能从自身轨迹中稳定学习。
2. **自蒸馏框架有效替代更强教师**：DeepSearch-World-9B 无需从 GPT-4 等模型蒸馏，即达到与依赖强教师或多代理合成的开源代理相近的性能（如 BrowseComp 31.2%、GAIA 61.5%、HotpotQA 93.4%）。
3. **数据池规模影响进化天花板**：420K 训练池比 100K 池使代理在后期轮次中仍保持提升，且工具使用行为更稳定（更低格式错误、更高实体命中率）。
4. **脚手架与转换策略至关重要**：状态内化（将进度状态注入 <think>）和反思重写（将环境反射改写为自修正）均产生显著增益，缺失时性能大幅下降。

### 7. 优点

- **环境设计的创新性**：提供确定性、可复现的离线环境，支持细粒度的实体级进度验证，避免真实 Web 的不可靠性和高成本。
- **方法简洁有效**：纯 SFT 更新，无需复杂的 RL 目标或在线策略匹配，训练稳定且计算高效（异步生成 + 训练）。
- **自我进化的可扩展性**：迭代自蒸馏机制使代理能力随轮次持续增长，且通过重要性采样保留历史经验。
- **开源友好**：计划发布完整环境、数据集、验证集、模型和代码，便于复现和后续研究。
- **公平性**：对比了大量开源代理（7B-9B 规模），且明确区分了监督来源（是否依赖更强模型），突出自身方法的独特性。

### 8. 不足与局限

- **领域局限**：环境仅基于 Wikipedia，知识覆盖有限，泛化到其他领域（如实时新闻、多模态、代码任务）尚未验证。
- **训练范式单一**：仅采用 SFT，未探索 RL 式（如 GRPO、PPO）或在线策略蒸馏（OPSD）能否进一步释放能力，作者也承认这是未来方向。
- **未处理真实 Web 动态性**：虽然工具接口对齐，但最终评估仍主要在离线环境进行，真实 Web 的延迟、噪声、页面结构变化等未充分考虑。
- **可能继承数据偏差**：Wikipedia 本身可能存在历史或地域偏差，模型训练后可能放大这些偏差。
- **实验覆盖有限**：仅 9B 模型实验，未验证更大模型（如 32B/70B）下该方法的有效性；且中文 benchmarks 提升较小（受限于仅英文训练轨迹）。
- **安全与伦理**：论文虽声明使用公开数据，但未讨论恶意查询或对抗性输入下的安全对齐问题。
