## 摘要
在长周期任务中，决策相关的状态通常分散在日益增长的轨迹中，而行动智能体必须从中提取信息并采取行动。随着轨迹增长，任务要求、环境事实、先前尝试、诊断以及未完成的子目标可能被埋藏在上下文窗口中，甚至超出其范围，从而在需要时无法影响决策。我们将这种失败模式称为“行为状态衰减”。我们将记忆视为一种主动干预机制，而非被动检索。一个独立的记忆智能体与未经修改的行动智能体并行运行，从最近的轨迹中更新结构化的记忆库，并决定是注入一个基于记忆的提醒还是保持沉默。该模块可与前沿行动智能体及现有智能体框架即插即用。在Terminal-Bench 2.0和τ²-Bench上，它分别提升了弱行动智能体和强行动智能体的pass@1指标，在Terminal-Bench上提高了+8.3个百分点，在τ²-Bench上提高了+6.8个百分点。消融实验表明，选择性干预优于被动记忆库暴露、持续注入、仅顾问指导以及通用检索。作为向开放权重记忆策略迈出的早期一步，我们使用SETA上的SFT和GRPO训练了Qwen3.5-27B，提升了验证奖励，并实现了向Terminal-Bench的部分迁移。

## Abstract
In long-horizon tasks, decision-relevant state is often scattered across an expanding trajectory, while the action agent must surface it and act. As trajectories grow, task requirements, environment facts, prior attempts, diagnoses, and open subgoals can be buried in the context window or pushed beyond it, failing to influence decisions when needed. We call this failure mode "behavioral state decay". We study memory as an active intervention mechanism rather than passive retrieval. A separate memory agent runs alongside an unmodified action agent, updating a structured memory bank from the recent trajectory and deciding whether to inject a memory-grounded reminder or remain silent. The module is plug-and-play with frontier action agents and existing agent harnesses. Across Terminal-Bench 2.0 and $τ^2$-Bench, it improves pass@1 for both weaker and stronger action agents, with gains of +8.3 pp on Terminal-Bench and +6.8 pp on $τ^2$-Bench. Ablations show that selective intervention outperforms passive bank exposure, always-on injection, advisor-only guidance, and general retrieval. As an early step toward open-weight memory policies, we train Qwen3.5-27B on SETA using SFT and GRPO, improving validation reward and achieving partial transfer to Terminal-Bench.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建一个**独立的记忆智能体（Memory Agent）**，与未经修改的行动智能体并行运行。记忆智能体在固定间隔被调用，执行两个阶段：
  1. **阶段一（记忆管理）**：基于最近轨迹窗口和当前记忆库，通过工具调用更新结构化的记忆库（包含状态字段、知识记忆、程序性记忆）。
  2. **阶段二（主动干预）**：根据更新后的记忆库和轨迹，决定是否注入一个基于记忆的提醒（memory-grounded reminder）到行动智能体的下一次调用中，或保持沉默（null intervention）。
- **关键技术细节**：
  - **记忆库结构**：三元组 `(status, knowledge, procedural)`。
    - `status`：私有状态字段，记录进展、未解决问题等，**从不暴露给行动智能体**。
    - `knowledge`：稳定的事实（任务要求、环境属性、文件路径等）。
    - `procedural`：尝试和结果（失败命令、成功修复、诊断信号等）。
  - **工具调用接口**：阶段一通过 `memory_update_status`、`memory_save_knowledge`、`memory_save_procedural`、`memory_delete` 等工具显式管理记忆库。
  - **干预决策**：阶段二输出要么是 `<context_for_action>` 提醒，要么是 `<no_intervention/>`。提醒仅限**记忆驱动的简洁提醒**，避免泛泛的策略建议。
  - **触发机制**：在第一步和之后每隔固定步数调用一次记忆智能体（本文使用每步都调用，以隔离干预策略本身的效果）。
- **训练开源记忆策略**（初步探索）：使用**监督微调（SFT）** 蒸馏提示式记忆智能体的轨迹，再用**GRPO（强化学习）** 进一步优化干预时机决策。仅在SETA环境中训练，保持行动智能体冻结。

- **对比/消融方法**：

## 6. 论文的主要结论与发现

- **主结论**：主动记忆干预能有效缓解长周期任务中的行为状态衰减，显著提升pass@1。在Terminal-Bench上，Sonnet 4.5提升+8.3 pp，Opus 4.6提升+2.4 pp；在τ²-Bench上，Sonnet 4.5提升+6.8 pp，Opus 4.6提升+2.5 pp。
- **消融结论**：选择性干预（两阶段完整系统）优于被动暴露全记忆库、始终注入、仅顾问指导、通用记忆检索（Mem0）。特别是，在宏观平均上，完整系统表现最平衡。
- **训练结论**：记忆干预策略可以部分地通过SFT+GRPO学习并迁移，Qwen3.5-27B在SETA上训练后，在Terminal-Bench上带来+3.5 pp的提升。
- **定性发现**：记忆在重新激活特定执行状态（如任务要求、环境事实、失败尝试、诊断信息）时最为有效；失败主要由校准误差（如不必要提醒、过度自信推断）导致。

## 7. 优点：方法或实验设计上的亮点

- **问题定义清晰**：首次明确界定“行为状态衰减”这一失败模式，并区分于一般记忆缺失问题。
- **即插即用设计**：记忆智能体可作为独立模块与任何行动智能体及现有框架集成，无需修改行动智能体。
- **干预策略的核心创新**：将记忆使用从存储/检索问题提升为**控制问题**，强调“何时干预”比“记住什么”更重要，且通过沉默动作抑制不必要的干预。
- **两阶段分解**：将记忆管理（写/更新）与干预选择（何时注入）解耦，便于单独优化和消融。
- **结构化记忆库**：区分私有状态（status）、知识记忆（knowledge）和程序性记忆（procedural），避免污染行动上下文，同时支持精细更新。
- **实验设计系统**：多基准、多模型、多消融变体、定性分析、训练迁移验证，层次分明。
- **开源尝试**：训练开源模型（Qwen3.5-27B）作为记忆智能体，展示可迁移性，有利于社区复现和扩展。
