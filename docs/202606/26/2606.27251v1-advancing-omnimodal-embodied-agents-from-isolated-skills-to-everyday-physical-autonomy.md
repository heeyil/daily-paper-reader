## 摘要
在非结构化环境中构建持久化具身智能体，需要统一编排跨越信息（API、物联网）与物理（操作、导航）领域的异构工具，并具备对长时间运行中必然出现的物理故障的自主恢复能力。现有系统将这些视为分离问题：基于视觉语言模型的规划器缺乏统一的信息物理动作空间，智能体框架积累无界上下文从而降低时间连贯性，且视觉语言动作策略以开环方式执行而不检测自身故障。我们认为，持久化自主性需要的不是单一模型，而是一种层次化异步架构，显式分离规划、记忆与验证。为此，我们提出OmniAct框架，它集成多模态语义规划器以实现跨统一动作空间的技能路由，采用具有事件边界驱动压缩的自适应层次记忆以实现次线性上下文增长，并配备异步视觉抢占引擎以在物理执行期间闭合语义环路。在两个机器人平台协调四个物联网设备的40项真实世界长期任务中，OmniAct在所有复杂度层级上实现端到端成功率的一致提升，在超过10万累积交互令牌中保持近乎平坦的令牌消耗，并将中等规模开放权重模型提升至专有级别性能。

## Abstract
Building persistent embodied agents in unstructured environments demands unified orchestration of heterogeneous tools spanning both cyber (APIs, IoT) and physical (manipulation, navigation) domains, coupled with autonomous recovery from physical failures that inevitably arise over extended operation. Existing systems treat these as separate problems: VLM-based planners lack a unified cyber-physical action space, agent frameworks accumulate unbounded context that degrades temporal coherence, and VLA policies execute open-loop without detecting their own failures. We argue that persistent autonomy requires not a monolithic model but a hierarchical asynchronous architecture with explicit separation of planning, memory, and verification. To this end, we present OmniAct, a framework integrating a multimodal semantic planner for skill routing across unified action spaces, an adaptive hierarchical memory with event-boundary-driven compression for sub-linear context growth, and an asynchronous visual preemption engine that closes the semantic loop during physical execution. Across 40 real-world long-horizon tasks on two robotic platforms coordinating four IoT devices, OmniAct achieves consistent improvements in end-to-end success across all complexity levels, maintains near-flat token consumption over under 100k+ accumulated interaction tokens, and elevates mid-scale open-weight models to proprietary-level performance.

- **整体含义**：实现“日常物理自主”需要层次化异步架构，显式分离规划、记忆与验证，而非依赖单一端到端模型。本文旨在通过结构化的模块协同，统一跨域工具编排、压缩上下文并实现闭环故障恢复。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
OmniAct 框架通过三个协同模块实现持久化自主：多模态语义规划器（统一动作空间）、自适应分层记忆（事件边界驱动压缩）、异步视觉抢占引擎（语义级闭环验证）。

### 关键技术细节
- **多模态语义规划器**：接收语音、图像、状态等多模态输入，执行意图理解、任务分解与技能路由。输出结构化JSON，支持单步并行调度多个技能（如同时调用物联网设备与执行操作）。
- **异步视觉抢占引擎**：
  - 物理执行期间，异步视觉监视器周期采样图像，提交给VLM进行语义级进度与安全检查。
  - 若检测到异常（如物体未被抓起），立即中断执行器，将故障上下文返回规划器进行重新规划。
  - 该机制基于高层语义而非几何约束，可捕获低层控制器无法感知的故障模式。
- **自适应分层记忆（三级）**：
  - **感官记忆**：FIFO缓冲近期视觉帧，供控制策略与视觉监视器使用，不进入规划上下文。
  - **情景记忆**：通过事件边界驱动压缩，在子任务完成或状态变化时，将原始交互序列压缩为紧凑关键帧摘要（保留语义、状态转移、动作结果），丢弃程序化冗余。上下文增长仅与语义事件数成正比，实现次线性扩展。
  - **反思记忆**：存储静态先验（工具模式、技能清单）与动态累积经验（执行失败时的根因分析、纠正策略），在后续规划中检索注入上下文，实现累积适应。

### 公式与算法流程（文字说明）
系统在每一步接收多模态观测（视觉帧+状态信息），结合当前用户意图与系统状态形成多模态上下文。规划模型根据上下文和从层级记忆检索的历史经验，从统一的信息-物理动作空间中采样一个动作（目标技能+技能参数）。物理执行期间，视觉监视器异步采样图像并调用VLM校验，若检测到异常则中断执行并触发重规划及反思记忆写入。

## 3. 实验设计

### 数据集/场景与benchmark

## 6. 主要结论与发现

1. **架构有效性**：OmniAct在所有复杂度层级上端到端成功率一致优于基线（L3任务：操作场景50.0% vs 最佳基线43.8%；导航场景54.2% vs 37.5%）。子任务成功率提升更显著（L3达80%以上，基线约60%）。
2. **记忆与上下文管理**：事件边界驱动的层次记忆在10万+累积令牌下保持每调用仅4k-8k令牌的平坦消耗，而原始历史线性增长至160k+令牌。消融实验表明，完整系统（情景记忆+反思记忆）在长期偏好约束满足率上比最佳消融变体高13-20个百分点。
3. **跨模型泛化**：OmniAct提升了所有具备基础能力的骨干模型，中等规模开源模型（如Qwen3-VL-30B-A3B）从50.0%提升至80.0%，达到与Gemini-3.1-Pro原始性能相当的水平，表明结构化编排可弥合开源与专有模型差距。
4. **异步视觉预emption**：成功将因VLA执行失败（如滑落）导致的级联错误转化为可恢复的局部延迟，大幅缩小了子任务与端到端成功率之间的差距。

## 7. 优点

1. **方法创新性强**：首次提出统一信息-物理动作空间与事件边界驱动的分层记忆压缩，异步视觉预emption机制新颖且实用。
2. **实验设计扎实**：在两种真实机器人平台、40个真实任务上验证，覆盖多种难度与模态组合，消融实验系统且公平。
