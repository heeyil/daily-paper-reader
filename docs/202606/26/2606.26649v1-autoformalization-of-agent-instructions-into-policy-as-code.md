## 摘要
在高风险领域中，智能体的安全性需要正式的策略执行，但现有方法要么依赖概率性护栏（微调分类器、基于提示的引导）而无法提供正式保证，要么依赖手写符号执行但无法扩展到实际策略规范的广度。我们提出了一种自动形式化管线，利用基于LLM的生成器-批评者循环，将智能体提示、MCP工具描述和自然语言策略文档转化为经过形式验证的策略。生成的策略使用Cedar策略语言编写。在MedAgentBench基准测试上，我们的自动形式化策略覆盖的自然语言规范比先前工作中手写的符号执行要广泛得多。

## Abstract
Agent safety in high-stakes domains requires formal policy enforcement, but most existing approaches either rely on probabilistic guardrails (fine-tuned classifiers, prompt-based steering) that offer no formal guarantees, or on hand-coded symbolic enforcement that does not scale to the breadth of real policy specifications. We present an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural language policy documents into formally verified policies using an LLM-based generator-critic loop. The resulting policies are written in the Cedar Policy Language. On the MedAgentBench benchmark, our autoformalized policies cover substantially more of the source natural-language specification than the hand-coded symbolic enforcement in prior work.

## 2. 方法论

- **核心思想**：采用“验证三明治”（Verification Sandwich）架构，通过 LLM 驱动的生成器-批评者循环（Generator-Critic Loop），将非结构化输入（系统提示、MCP 工具定义、自然语言策略文档）自动形式化为 Cedar 策略语言。

- **关键技术细节**：
  - **三层架构**：
    - **接地层**（Grounding Layer）：从输入中提取实体、工具模式，定义主体-资源-动作本体，并程序化生成 Cedar Schema（基于 MCP 工具定义）。
    - **模型层**（Model Layer）：使用 LLM（Gemini 3 Pro）作为策略生成器，基于输入生成候选 Cedar 策略。
    - **安全层**（Safety Layer）：包含两类批评者：
      - **硬批评者**（Hard Critic）：Cedar 引用工具进行语法检查、模式合规、静态分析（空策略、矛盾规则）。确定性验证，可重试最多 3 次并将错误反馈给生成器。
      - **软批评者**（Soft Critic）：LLM-as-Judge（Gemini 2.5 Flash，两阶段：Judge → Verifier），评估语义对齐和定性评分。
  - **生成器-批评者循环**：迭代生成→验证→反馈，直到硬批评者通过且软批评者达到质量阈值。最终输出经验证的 Cedar 策略集。
  - **运行时执行**：通过外部策略引擎（Cedar）在智能体调用工具前决策是否允许，实现确定性强制执行。

- **形式化表示**：Cedar 策略语言（Amazon 开源），基于属性访问控制（ABAC），支持类型安全、静态分析和人类可读性。

## 7. 优点

- **完全自动化**：无需人工编写符号规则，直接输入自然语言策略即可生成形式化策略，大幅降低门槛。
- **形式化保证**：Cedar 提供类型安全、矛盾检测、语法验证等确定性保证，硬批评者重复验证，避免 LLM 幻想实体。
- **人类可审计**：Cedar 策略语言（类似自然语言）可读性高，安全工程师可直接审查。
- **可扩展性**：生成器-批评者循环可迭代优化，适用于任意自然语言策略文档。
- **实验设计清晰**：区分原始/对抗性数据集、按写操作分解结果，揭示了真实性能。

## 8. 不足与局限

- **Cedar 缺少状态**：Cedar 是设计为无状态的，不支持多轮工作流中的时序依赖（如“必须先 A 后 B”）。论文提及未来工作需引入时序逻辑和记忆感知。
- **软批评者基于 LLM**：语义对齐评估仍依赖 LLM，存在幻觉和偏见风险（尽管温度较低）。
