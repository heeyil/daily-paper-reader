## 摘要
在生产环境中高效部署大语言模型（LLM）需要在准确性和成本之间进行权衡。运营商通常默认使用单一模型，该模型要么对简单查询过于昂贵，要么对困难查询能力不足。为应对这一挑战，我们提出了一种两阶段级联解决方案。第一阶段对传入查询进行聚类，并将每个聚类分配给其最具成本效益的模型。该路由过程的成本预算由一个可解释的超参数设定，并在离线状态下调整。第二阶段增加了质量估计（QE）级联；当第一阶段的输出被判断为低质量时，该查询将被升级到更强的模型。这确保了只有困难或低置信度的案例才会到达昂贵的模型。在测试数据集上，该级联系统保留了最强模型97-99%的准确率，同时减少了每输出令牌时间（TPOT）。它仅需要任务正确性标签，并且无需手动重新配置即可适应模型池的变化。

## Abstract
Efficient deployment of large language models (LLMs) in production forces a trade-off between accuracy and cost. Operators often default to a single model that is either expensive for easy queries or insufficient for hard ones. To address this challenge, we propose a two-stage cascaded solution. Stage 1 clusters incoming queries and assigns each cluster to its most cost-effective model. The cost budget for this routing process is set by an interpretable hyperparameter, tuned offline. Stage 2 adds a quality estimation (QE) cascade; when an output from Stage 1 is judged low-quality, the query is escalated to a stronger model. This ensures only hard or low-confidence cases reach the expensive models. On the test datasets, the cascaded system retains 97-99% of the strongest model's accuracy while reducing Time Per Output Token (TPOT). It requires only task-correctness labels and adapts to changes in the model pool without manual reconfiguration.

- **问题**：LLM 生产部署面临准确率与成本之间的根本矛盾。运营者通常选择单一最强模型（过度投资于简单查询）或单一轻量模型（在困难查询上表现不足），造成效率浪费。
- **背景**：开源模型覆盖广泛的大小和成本-准确率权衡，但缺乏自动化机制将不同难度的查询路由到最合适的模型。
- **目标**：提出一个只需任务正确性标签即可训练的框架，在显性推理成本预算下联合优化路由，并通过后生成质量估计恢复准确率。

## 2. 方法论

### 核心思想
两阶段级联框架：
- **Stage 1 – 基于聚类的路由**：查询经 k-means 聚类（基于 all-MiniLM-L6-v2 嵌入），每个聚类根据成本调整后的错误率分配给最经济的模型。
- **Stage 2 – 质量估计（QE）级联**：对 Stage 1 中分配到高效模型的输出，由轻量 QE 分类器判断质量，低质量输出升级到更强模型。

### 关键技术细节
- **路由评分公式**：
  ```
  Score(m, c) = Error(m, c) + λ · Cost_norm(m)
  ```
  其中 `Cost_norm(m) = (TPOT(m) - TPOT_min) / (TPOT_max - TPOT_min)`，λ 为可解释超参数（控制准确率-延迟权衡）。
- **λ 自动选择**：在训练数据上求解 `λ* = argmax_{λ} [Acc(λ) | TPOT(λ) ≤ B]`，B 为用户指定的 TPOT 预算。
- **Pareto 分析**：剔除被支配模型（如 TeleQnA 中四模型池剪枝为两个有效模型）。
- **QE 分类器**：基于 ModernBERT-base 的二元分类器（标签：accept/escalate），输入为查询+模型输出+生成长度，训练数据来自 Stage 1 高效模型的任务正确性标签。
- **流程**：离线计算聚类中心和各聚类路由表；推理时仅需一次嵌入查询和一次 argmin 查找。

## 6. 主要结论与发现

- **准确率-延迟平衡**：完整系统（Stage 1+2）在 AIME 上以 18% 更低的 TPOT（9.7 ms vs. 11.8 ms）达到最强模型准确率的 99.3%（88.4% vs. 89.1%）；在 TeleQnA 上以 2.3% 准确率损失节省 2.9% TPOT（23.8 ms vs. 24.5 ms）。
- **无需额外标注**：仅需任务正确性标签（来自标准模型评估），无需人工偏好或置信度数据。
- **自动适应模型池变化**：新增模型时 Pareto 分析自动更新路由表（附录 B 证明 Qwen3.5-35B 加入后获得 Pareto 改善）。
- **η 效率指标**：提出 `η = ΔAcc / ΔTPOT`，量化每毫秒节省的准确率代价，较低的 η 更好。

## 7. 优点

- **可解释性**：λ 直接表示“对使用最强模型的成本惩罚”，路由区域可离线计算并可视化（如图 2）。
- **自动化预算满足**：λ 自动选择确保 TPOT ≤ B 约束，运营商只需指定 B 即可。
- **轻量附加成本**：QE 分类器为现代 BERT-base，推理时增加约 0.52 ms/token，远小于模型生成时间。
- **模块化与可扩展**：聚类、路由、QE 各组件可独立替换；模型池增减无需重新标注，仅需重跑 Pareto 分析。
- **适用场景明确**：特别适合电信领域等要求按需推理、数据保密的部署（需本地运行模型）。

## 8. 不足与局限

- **静态聚类与路由表**：离线计算，无法适应在线查询分布漂移；更新需收集新标注数据。
- **TPOT 度量不完整**：不考虑排队、预填充（TTFT）、批处理影响、网络延迟等，在真实负载下可能高估效率。
- **QE 分类器误警率高**：TeleQnA 上约 51% 的升级为不必要，导致额外成本；后续可优化（使用更大模型或更多数据）。
- **GPU 内存限制**：所有候选模型需同时驻留显存，在 VRAM 有限场景下缩小可选模型池。
- **硬件依赖**：TPOT 值依赖 A100-specific FP8 方式（W8A16 vs. H100 的 W8A8），换硬件后 Pareto 曲线可能变化。
- **效率指标 η 未考虑批处理**：实际吞吐量受批处理影响，简单的 TPOT 均值可能误导。
- **用户预算 B 可能未严格达成**：如 TeleQnA 中 B=20 ms，但 Stage 1+2 实际 TPOT=23.8 ms，超过预算，作者未详细讨论。
- **AIME 测试集太小**：仅30条，结论的统计显著性不足。
