## 摘要
基于结果的强化学习为语言智能体提供了稳定的优化基础，但其稀疏的轨迹级奖励对哪些中间决策应被强化或抑制提供的指导很少。基于策略的自我蒸馏提供了密集的令牌级监督，但现有的技能条件变体通常依赖于外部技能记忆或检索到的特权上下文，这些维护成本高，且可能与多轮交互中当前策略引发的状态分布不匹配。我们提出**OPID**（**O**n-**P**olicy Sk**i**ll **D**istillation），一个直接从完成的基于策略轨迹中提取技能监督的框架。OPID将轨迹事后信息表示为分层技能：回合级技能捕获全局工作流或避免失败规则，而步骤级技能捕获关键时间步的局部决策知识。一个关键优先路由机制在识别到关键决策时使用步骤级技能，否则回退到回合级技能作为默认指导。选定的技能被注入交互历史中，允许旧策略在原始和技能增强的上下文中重新评分相同的采样响应。由此产生的对数概率偏移提供了一个令牌级自我蒸馏优势，与结果优势相结合用于策略优化。OPID因此保留了强化学习作为主要训练目标，同时引入了密集、分布匹配的事后监督。在ALFWorld、WebShop和基于搜索的QA上的实验表明，OPID在智能体性能、样本效率和鲁棒性方面普遍优于纯结果强化学习和现有技能蒸馏基线。我们的代码可在https://github.com/jinyangwu/OPID/tree/main获取。

## Abstract
Outcome-based reinforcement learning provides a stable optimization backbone for language agents, but its sparse trajectory-level rewards provide little guidance on which intermediate decisions should be reinforced or suppressed. On-policy self-distillation offers dense token-level supervision, yet existing skill-conditioned variants often rely on external skill memories or retrieved privileged context, which are costly to maintain and can be mismatched with the state distribution induced by the current policy in multi-turn interaction. We propose \textbf{OPID} (\textbf{O}n-\textbf{P}olicy Sk\textbf{i}ll \textbf{D}istillation), a framework that extracts skill supervision directly from completed on-policy trajectories. OPID represents trajectory hindsight as hierarchical skills: episode-level skills capture global workflows or failure-avoidance rules, while step-level skills capture local decision knowledge at critical timesteps. A critical-first routing mechanism uses step-level skills when critical decisions are identified and falls back to episode-level skills as default guidance otherwise. The selected skill is injected into the interaction history, allowing the old policy to re-score the same sampled response under both original and skill-augmented contexts. The resulting log-probability shift yields a token-level self-distillation advantage, which is combined with the outcome advantage for policy optimization. OPID thus preserves RL as the primary training objective while introducing dense, distribution-matched hindsight supervision. Experiments on ALFWorld, WebShop and Search-based QA demonstrate that OPID generally improves agent performance, sample efficiency, and robustness over outcome-only RL and existing skill-distillation baselines. Our code is available at https://github.com/jinyangwu/OPID/tree/main.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：基于结果的强化学习（如 GRPO）为语言智能体提供了稳定的优化主干，但其稀疏的轨迹级奖励难以对中间决策提供细粒度指导。一个终局奖励只能指示轨迹成功与否，却无法区分哪些中间步骤导致了最终结果，尤其在长程交互中，早期错误可能影响全局。
- **现有方法的局限**：基于策略的自蒸馏（on-policy self-distillation）能提供密集的 token 级监督，但现有技能条件变体（如 Skill-SD、SDAR）依赖外部技能库、检索技能文件或持久技能记忆，维护成本高，且检索到的技能可能与当前策略诱导的状态分布不匹配。这种不匹配在多轮交互中尤为严重，微小偏离会导致状态漂移，使技能失效。
- **研究目标**：提出 OPID，直接从当前策略已完成的轨迹中提取分层事后技能（episode 级和 step 级），并通过关键优先路由机制选择合适技能注入交互历史，生成 token 级自蒸馏优势，与结果优势结合进行策略优化，无需外部技能库或推理时的特权上下文。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
OPID 将已完成轨迹视为分布匹配的事后监督源，从中提取分层技能（episode 级和 step 级），通过关键优先路由选择最合适的技能增强上下文，让旧策略在原始和技能增强上下文中对同一采样响应重新评分，利用 log-probability 差异形成 token 级技能优势，与回合优势结合进行 PPO 风格的策略优化。

### 关键技术细节
- **问题建模**：将智能体任务建模为部分可观察马尔可夫决策过程（POMDP），策略 πθ 基于交互历史 ht 生成响应 yt。
- **三阶段流程**：
  1. **在线技能提取**：对每个完成的轨迹 τ，使用 LLM 分析器 A 提取：
     - 回合级技能 s^ep_τ：总结全局工作流或失败避免规则。
     - 步骤级技能 s^step_{τ,t}：捕获关键时间步的局部决策知识，仅对稀疏的关键步骤 t ∈ C_τ 提取。
  2. **关键优先路由与自蒸馏**：
     - 路由规则：若 t ∈ C_τ 则用步骤级技能，否则用回合级技能。
     - 将选定技能注入历史得到增强上下文 ĥ_{τ,t}；旧策略 π_{θ_old} 对同一响应 token 分别在原始和增强上下文下计算 log-probability：ℓ^old 和 ℓ^skill。
     - 技能优势 A^skill_{τ,t,ℓ} = (ℓ^skill - ℓ^old) * m_{τ,t,ℓ}（m 为有效 token 掩码）。
  3. **策略优化**：
     - 回合优势 A^ep_τ 采用 GRPO 风格：在任务提示 q 的组内标准化奖励（A^ep_τ = (R(τ) - μ_q)/σ_q）。
     - 最终优势 A^OPID = A^ep + λ_skill * A^skill。
     - 优化标准裁剪策略目标 L_policy(θ) = -E[min(ρ A^OPID, clip(ρ, 1-ε, 1+ε) A^OPID)] + β L_KL。
- **推理时无需分析器、技能检索或特权上下文**，仅有原始交互历史。

## 3. 实验设计

### 数据集 / 场景
- **ALFWorld**：物品家庭基准，6 种任务类型（Pick, Look, Clean, Heat, Cool, Pick2），训练 2400 样本，测试 140（seen split）和 134（unseen split）。
- **WebShop**：电子商务网站交互，训练 2400 样本，测试 128 任务，报告得分和成功率。
- **Search-based QA**：与搜索环境交互问答，包括 NQ、TriviaQA、PopQA、HotpotQA、2WikiMultiHopQA、MuSiQue、Bamboogle 七个子集，训练 19200 样本，测试 51713 样本。

### 基准方法
- **无训练方法**：Vanilla（原始提示）、Skill-Prompt*（验证时使用技能）。
- **基于结果RL**：GRPO（纯结果）、Skill-GRPO（训练时用技能但测试时移除）、Skill-GRPO*（训练和测试均用技能）。
- **自蒸馏 / 技能蒸馏方法**：OPSD、GRPO+OPSD、Skill-SD、RLSD、SDAR 等。

- WebShop：归一化任务分数和成功率。
- Search-based QA：各子集准确率及平均准确率。

## 6. 主要结论与发现

- OPID 在大多数模型-领域组合上一致优于纯结果 RL（GRPO）和现有的技能蒸馏基线。
