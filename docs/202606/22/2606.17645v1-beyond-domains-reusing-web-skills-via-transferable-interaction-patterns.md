## 摘要
大型语言模型（LLM）的Web代理通常作为工具调用者部署：每轮，模型读取新的页面观察并发出一个结构化的工具动作。当每个动作都是低级原语时，交互轮次迅速增长，面向策略的LLM补全也随之增加，在Mind2Web和WebArena等基准测试中主导了延迟和成本。因此，近期的系统将重复的交互片段包装为Web技能：从成功轨迹或归纳程序构建的可调用工具，使一次调用能替代多个原语。然而，先前的技能库仍主要通过指令相似性或粗略的站点元数据来触发，导致在未见站点上技能复用率低，并留下大量潜在的步骤和令牌缩减空间。我们提出了SkillMigrator，一个通过学习可复用Web技能并通过匹配布局结构而非特定元素引用来跨站点迁移的代理。每个归纳的技能存储为可迁移交互模式（TIP）：技能与归纳时刻快照的结构化草图配对。在测试时，SkillMigrator通过布局相似性检索TIP，并将其引用锚定在实时页面上。其余部分为标准流程：使用稳定引用的可访问性快照观察，以及在原语和技能调用上的固定工具调用。与最先进的方法相比，SkillMigrator在WebArena和Mind2Web上以匹配的成功率将成功轨迹的平均LLM动作计数减少了8-10%。

## Abstract
Large language model (LLM) web agents are usually deployed as tool callers: each turn, the model reads a fresh page observation and emits one structured tool action. When every action is a low-level primitive, horizons grow quickly and so do policy-facing LLM completions, dominating latency and cost on benchmarks such as Mind2Web and WebArena. Recent systems therefore wrap repeated interaction fragments as web skills: callable tools built from successful trajectories or induced programs, so one call can replace several primitives. However, prior skill libraries are still triggered mainly by instruction similarity or coarse site metadata, which yields low skill reuse on held-out sites and leaves much of the potential step and token reduction on the table.   We present SkillMigrator, an agent that learns reusable web skills and transfers them across sites by matching layout structure rather than specific element references. Each induced skill is stored as a transferable interaction pattern (TIP): the skill paired with a structural sketch of the snapshot at induction time. At test time, SkillMigrator retrieves TIPs by layout similarity and grounds their references on the live page. The rest of the stack is standard: accessibility-snapshot observations with stable references, and fixed tool calling over primitives plus skill invocations. Compared with the state-of-the-art approaches, SkillMigrator reduces the average LLM-action count on successful trajectories by 8-10% across both WebArena and Mind2Web at matched success rate.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：当前 LLM 驱动的 Web 代理（如 ReAct）以每步调用一个低级原语动作（fill、click 等）的方式运行，导致在长任务中产生大量 LLM 调用，延迟和成本高昂。
- **现有方案不足**：已有工作通过构建可复用的 Web 技能（如 ASI、SkillWeaver、PolySkill）来减少 LLM 调用，但这些技能库的检索主要依赖指令语义相似性或粗粒度的网站元数据，导致：
  - 在未见过的同领域站点上复用率已较低；
  - 在**跨领域**（如从电商到开发者工具）的网站上几乎无法复用。
- **核心意义**：现有方法忽视了不同领域网站在交互模式上可能高度相似（如“填写表单并提交”是一个通用模式），因而无法实现真正的跨域技能迁移。本文首次系统研究了**超越领域和网站的 Web 技能复用**，旨在通过匹配布局结构而非文本标签来突破这一瓶颈。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 将每次归纳的技能存储为**可迁移交互模式（Transferable Interaction Pattern, TIP）**，每个 TIP 包含：
  - 技能的自然语言意图 \( \iota_k \)
  - 操作模板 \( \sigma_k \)（如 `fill-and-submit`）
  - 槽位模式 \( \Phi_k \)（每个槽位包含键、描述符、同义词池）
  - 来源页面的可访问性树骨架 \( \tau_k \)（布局结构草图）
- 测试时，通过**布局相似性**（树编辑距离）结合文本信号检索最匹配的 TIP，然后通过两个阶段的绑定过程将其接地到实时页面的具体元素上。

### 关键技术细节
- **布局信号**：使用 APTED 计算两个可访问性树的小树编辑距离，用于衡量页面结构的相似性。
- **检索分数**：
  \[
  \text{score}(k, s_i, \tilde{o}_{i-1}) = \alpha \cdot \text{text\_signal} + (1-\alpha) \cdot \text{layout\_signal}
  \]
  其中文本信号编码技能描述和当前子任务/页面摘要的句子嵌入相似度，布局信号由树编辑距离归一化得到。
- **门控机制**：仅当最高分 TIP 的分数超过阈值 \( \beta \) 时才执行技能模式，否则回退到原语 ReAct 模式。
- **槽位绑定**：
  - 阶段 A：从指令（或实例化字典）提取槽位值，通过匈牙利算法进行槽位-值匹配。
  - 阶段 B：将每个槽位绑定到实时页面对应角色和名称的控件上，同样使用匈牙利算法。
- **操作模板执行**：每个模板有固定计划（如 `fill-and-submit` 模板依次填充绑定槽位后点击提交按钮），未绑定的必要槽位则回退到 LLM 原语生成。

### 算法流程（文字说明）
1. 对当前子任务 \( s_i \) 和观察 \( \tilde{o}_{i-1} \)，生成页面摘要和节点列表。
2. 计算所有技能 k 的分数，选择最优 \( k^* \)。
3. 若分数 < β，回退到 ReAct（执行 LLM 原语动作）。
4. 否则，执行阶段 A：从指令中解析槽位值。
5. 执行阶段 B：将槽位绑定到实时页面的控件引用。
6. 按操作模板顺序发出填充/点击等原语动作，未绑定的必要槽位交给 LLM。

## 6. 主要结论与发现

1. **效率提升**：在保证任务成功率（SR）基本持平的前提下，SkillMigrator 在 WebArena 上将成功轨迹的平均 LLM 动作计数从 PolySkill 的 5.9 降至 5.4（降低 8.5%），在 Mind2Web 跨域集上从 PolySkill (+Update) 的 6.9 降至 6.2（降低 10.1%）。
2. **技能复用率更高**：在 Mind2Web 跨域拆分中，SkillMigrator (+Update) 的技能复用率达到 35.4%，高于 PolySkill 的 31% 和 ASI 的 23.6%。
3. **布局信号是关键**：消融实验表明，去掉布局信号后 SR 急剧下降（WebArena 45.7% → 39.2%），接近无技能基线；而同义词池和门控机制贡献了额外的泛化能力。
4. **门控可做利用率调节**：关闭门控（β=0）进一步降低 \( \bar{N} \)（5.4 → 4.7）但 SR 仅轻微下降，表明 β 可作为部署时权衡效率与安全的旋钮。
5. **与其他技能库正交**：将 SkillMigrator 的回退路由到 PolySkill 可达到最低 \( \bar{N} \)（5.3），说明它可以作为现有系统的检索层堆叠使用。

## 7. 优点

- **方法创新性**：首次提出利用**布局结构**（可访问性树骨架）作为跨域技能检索的核心信号，突破了纯语义检索的局限，实现了超越领域和网站的技能迁移。
- **设计简洁而有效**：TIP 记录仅包含骨架和槽位模式，不存储具体页面引用，确保了可迁移性；两阶段槽位绑定（值解析+控件对接）机制灵活，能处理变长度表单。
- **工程实用性**：门控机制避免了弱匹配技能的执行，保证了整体成功率不下降；操作模板的设计使得技能可落地执行，无需额外 LLM 推理。
- **实验全面且有说服力**：在两个主流基准（WebArena、Mind2Web）上进行了系统性比较，包含消融、敏感性、正交性研究，验证了各组件贡献和参数鲁棒性。
- **可扩展性**：SkillMigrator 可以堆叠在其他技能库之上，兼容性良好。

## 8. 不足与局限

- **布局结构假设的鲁棒性**：方法假设共享相似可访问性树结构的页面也具有相似的交互语义。对于视觉同构但控制流不同的页面（如看似相同的表单但实际按钮功能不同），布局匹配可能失效，导致误用技能。
- **操作模板覆盖范围有限**：仅支持预定义的几种操作模板（如 fill-and-submit、click-by-text），不透明模态（拖拽、模态对话框）只能回退到原语，需要扩展模板库。
- **依赖强 LLM 骨干**：实验仅使用了 GPT-4.1 和 Claude-4 两种前沿闭源模型，未在更小或开源模型上验证效果，可能限制其实际部署成本。
- **未涉及多模态信息**：仅使用文本快照，忽略了视觉布局、截图等多模态特征，可能错失某些视觉引导的交互模式。
- **仅评估了两个基准**：缺少对真实网络开放站点（如维基百科、购物网站实时交互）的测试，生态位验证有限。
- **算力和资源消耗不透明**：未报告训练/推理过程中的 GPU 小时数等指标，不利于成本对比。
- **隐私与合规风险**：技能库通过复用交互模式可能绕过站点认证边界，虽然论文在局限中提到应遵守政策，但未讨论实际部署约束。
