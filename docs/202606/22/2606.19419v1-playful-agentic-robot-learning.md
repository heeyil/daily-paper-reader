## 摘要
当前的具身智能体机器人系统能够编写可执行的代码即策略程序、观察反馈，并在多次尝试中修正行为，但它们主要仍是任务驱动的：只有在明确指令后才获取可复用的技能。我们研究游乐式具身智能体机器人学习，其中具身编码智能体将自我导向的游玩作为下游任务到来前的持续技能学习阶段。我们引入了RATs（机器人智能体团队），专门设计用于游玩时的技能获取。在游玩过程中，RATs提出新颖但可学习的探索性任务，规划并执行机器人代码策略，验证中间进度，诊断失败，在密集的步骤级反馈下重试，并将成功执行提炼为持久化的代码技能库。在测试时，智能体从该冻结库中复用相关技能以帮助解决新任务。在LIBERO-PRO和MolmoSpaces上的实验表明，与无游玩和随机游玩基线相比，游玩学习到的技能在保留的下游任务上表现更优，在LIBERO-PRO和MolmoSpaces上分别比CaP-Agent0提高了20.6和17.0个百分点。此外，只需将学习到的技能检索到上下文中，即可将其插入其他推理时的代码即策略智能体，在无需微调基础模型的情况下，分别在RoboSuite和真实世界迁移上提高了8.9和8.8个百分点。

## Abstract
Current agentic robot systems can write executable Code-as-Policy programs, observe feedback, and revise behavior across multiple attempts, but they remain largely task-driven: reusable skills are acquired only after explicit instructions. We study Playful Agentic Robot Learning, where an embodied coding agent uses self-directed play as a continual skill-learning stage before downstream tasks arrive. We introduce RATs, Robotics Agent Teams designed for play-time skill acquisition. During play, RATs proposes novel yet learnable exploratory tasks, plans and executes robot-code policies, verifies intermediate progress, diagnoses failures, retries with dense, step-level feedback, and distills successful executions into a persistent code skill library. At test time, the agent reuses relevant skills from this frozen library to help solve new tasks. Experiments in LIBERO-PRO and MolmoSpaces show that play-learned skills improve held-out downstream tasks over no-play and random-play baselines, with 20.6 and 17.0 percentage-point gains over CaP-Agent0 on LIBERO-PRO and MolmoSpaces, respectively. Moreover, the learned skills can be plugged into other inference-time Code-as-Policy agents by simply retrieving them into the context, improving RoboSuite and real-world transfer by 8.9 and 8.8 points, respectively, without finetuning the underlying model.

# 论文总结：Playful Agentic Robot Learning

## 1. 核心问题与整体含义

**研究动机与背景**：当前具身智能体系统（如Code-as-Policy方法）能够编写代码策略、观察反馈、多次尝试修正行为，但**它们本质上是任务驱动的**——只有在接收到明确的外部指令后，才会获取可复用的技能。这种“被动学习”模式限制了机器人的自主探索和持续能力积累。与此相对，自然智能（如儿童通过玩耍学习技能）展示了另一种模式：在明确目标给出之前，通过自我导向的探索来发现可控效果并积累可复用技能。  
**整体含义**：本文提出**游乐式具身智能体机器人学习（Playful Agentic Robot Learning）**，即让具身编码智能体在下游任务到达之前，通过自我驱动的“游玩”阶段，主动地持续学习可复用的技能。这种方法旨在突破传统任务驱动框架，使机器人具备主动、持续的学习能力，从而提升对未见任务的泛化能力。

## 2. 方法论

### 核心思想
通过一个多智能体系统（RATs）在游玩阶段自主提出探索任务、执行代码策略、基于密集反馈诊断并重试，将成功行为蒸馏为可复用的代码技能库；测试时，从该库中检索相关技能以辅助解决新任务。

### 关键技术细节
- **整体流程**：包含两个阶段。**游玩阶段**：每次迭代，系统提出一个自驱任务，执行任务并利用多智能体团队（规划器、策略编写器、验证器、诊断器）进行Write-Execute-Verify-Diagnose循环，成功后将策略提炼为技能加入库，失败则存储经验教训到失败记忆。**测试阶段**：冻结技能库，仅利用库中技能结合原始基元解决外部任务。
- **任务提议团队**：基于**好奇心驱动**和**金发姑娘原则**（Goldilocks Principle）选择任务。候选任务生成使用LLM基于场景、技能库和失败记忆。选择时最大化`新奇性 × 能力前沿`的乘积。新奇性鼓励未探索的组合，能力前沿项`4 * r_bar * (1 - r_bar)`在平均成功率约0.5时最大，避免过易或过难的任务。
- **执行团队**：包含规划器（生成步骤级计划，标明所需技能）、策略编写器（将计划转为代码）、质量检查器、步骤验证器、目标验证器、失败诊断器，以及可选的子智能体（针对持续失败的子动作进行隔离练习）。
- **内存管理团队**：成功时提取可复用的参数化函数（技能），进入**实验性**状态；失败时记录为教训并提炼。技能经历实验性→验证性→弃用三个可靠性层级，定期进行记忆整理和技能提议（针对反复缺失能力时从基元合成候选技能）。

### 公式或算法流程（文字说明）
- **算法1**：RATs在游玩阶段的伪代码。
  1. 初始化技能库为基元库，失败记忆为空。
  2. 对于N次迭代：
     - 任务提议者根据当前场景、技能库、失败记忆生成候选任务，通过金发姑娘评分选择任务τ。
     - 执行团队运行任务，尝试最多重试预算（如5次），成功则提取技能，失败则存储教训。
  3. 游玩结束后，冻结技能库。
  4. 在测试环境中，使用完整RATs执行系统或仅插件式CaP-Agent0评估任务，技能库通过检索嵌入上下文。
- **关键公式**：任务评分`score(τ) = N(τ) · F(τ)`，其中`N(τ)=1/(Σ√(N(o,s)+1))`，`F(τ)=4·r̄(τ)(1−r̄(τ))`，`r̄(τ)`为Wilson界下技能平均成功率。

## 3. 实验设计

### 数据集 / 场景
- **LIBERO-PRO**：包含物体、目标、空间三种泛化任务，每种有初始位置交换（Pos）和任务扰动（Task）两种子集，共60个保留任务。
- **MolmoSpaces**：包含打开、关闭、拾取、拾放四种任务类别，每类10个任务，共40个任务。
- **RoboSuite**（仅用于跨环境迁移）：包含立方体拾取、堆叠、螺母组装、擦拭、双臂协作等7个任务。
- **真实世界**：在物理机器人上执行“拾取红立方体”和“放置立方体到碗”两个任务，以及额外补充的“交换立方体”和“关闭抽屉”任务。

### 对比方法
- **VLA基线**：OpenVLA、π0、π0.5（均基于Transformer架构，以成功率衡量）。
- **Code-as-Policy基线**：CaP-Agent0（仅使用基元库，无游玩）。
- **本文方法（RATs）**：在游玩中使用RATs系统学习技能库，测试时使用完整RATs执行或插件到CaP-Agent0。

### Benchmark结果
- **LIBERO-PRO**：RATs平均成功率43.8%，CaP-Agent0为23.2%（+20.6pp）；所有VLA基线最高仅12.8%（π0.5）。
- **MolmoSpaces**：RATs平均38.0%，CaP-Agent0为21.0%（+17.0pp）。
- **跨环境迁移（RoboSuite）**：CaP-Agent0 + RATs技能（来自LIBERO-PRO游玩）从40.3%提升至49.1%（+8.9pp）。
- **真实世界**：LIBERO-PRO技能 + CaP-Agent0提升8.8pp；MolmoSpaces技能 + CaP-Agent0提升21.7pp（额外任务）。

### 消融实验
- **游玩策略**：无游玩、随机游玩（50次随机任务）、好奇游玩（RATs提议）。在LIBERO-PRO上，好奇游玩在CaP-Agent0测试下从23.2%提升至32.3%，随机游玩仅24.7%。
- **测试时系统**：CaP-Agent0 vs RATs Exec。无游玩下RATs Exec（36.3%）已优于CaP-Agent0（23.2%），好奇游玩+ RATs Exec达到44.3%。
- **计算匹配**：将30M游玩token均摊到测试任务，使CaP-Agent0拥有15次重试而非10次，结果仅从23.2%升至26.0%，而好奇游玩+10次重试达到32.3%，表明游玩计算优于额外测试时重试。

## 8. 不足与局限

- **主要局限**：
  1. **仿真为主，真实世界验证有限**：仅进行了少量真实世界任务（2-3个简单操作），大规模物理部署和复杂任务下的鲁棒性尚未验证。

  3. **不恰当技能复用可能损害性能**：当检索到的技能与下游任务不匹配时，可能造成性能下降，未来需更好的检索和上下文感知选择机制。
- **实验覆盖方面**：
  - 未评估在更复杂的灵巧操作（如双手协作、关节装配）上的表现（RoboSuite中螺母组装任务成功率仍为0%）。

  - 未研究技能库规模对性能的饱和点，以及不同游玩迭代次数的影响。
- **其他局限**：
  - **成本高**：RATs严重依赖VLM（如gemini、gpt）进行规划、验证、诊断，推理开销大。
  - **依赖高质量验证器**：步骤级验证的质量直接影响诊断准确性和技能提取。
  - **受限于底层控制API**：当前基元库（如简单抓取、移动）限制了可到达的操作复杂度。
  - **论文中LLM版本不一致**（gemini-3.1pro-preview vs gpt-5.5），可能影响实验可复现性。
