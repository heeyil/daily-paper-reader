## 摘要
对于多步交互环境中的LLM智能体，一个关键挑战是如何有效利用积累的交互经验。现有工作通常将这种经验的两种用途分开：将其作为自然语言规则保留在模型外部以供后续提示，或使用轨迹和反馈更新模型参数。前者易于解释，但可能逐渐与不断演化的策略脱节；后者能更广泛地改进策略，但在稀疏奖励设置中对局部错误的纠正有限。我们提出了面向LLM智能体的经验规则与策略联合学习（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时，JERP检索任务相关规则，并将其与交互历史一起作为智能体的条件。每轮结束后，JERP使用收集的轨迹来优化策略，并通过将当前展开与参考成功轨迹进行比较来修订规则池。这种耦合使得规则池与不断演化的策略保持一致，同时将稳定且高效的行为逐渐吸收到模型自身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务的决策性能上取得了持续提升。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

- **核心问题**：LLM智能体在多步交互环境（如网页导航、具身交互、工具使用）中需要有效利用累积的交互经验。现有的两种经验利用方式——将经验规则化（External rules）和通过参数更新（Policy optimization）——被分离开来，导致各自的局限：
  - **规则化方法**（如Reflexion、ExpeL）：规则可解释且易于修改，但容易随着策略演化而变得过时或误导，因为相同的模型可能无法正确应用静态规则，且规则与策略之间的对齐会逐渐丢失。
  - **参数更新方法**（如GRPO、Reinforcement Learning fine-tuning）：能全局改善策略行为，但在稀疏奖励环境下对局部错误（如中间步骤的失误）缺乏及时的针对性纠正，往往等不到有效信号。
- **整体含义**：本文提出将这两种经验利用方式**联合**起来，使得**同一批交互轨迹同时用于更新规则池和优化策略参数**，从而让规则池与策略动态对齐，逐步将稳定高效的行为内化到模型参数中，同时保留规则的可解释性和可更新性。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **JERP (Joint Learning of Experiential Rules and Policies)**：在一个训练循环中，维持每个任务实例的长期经验规则池（Long-term experiential-rule pool），决策时从中检索出工作规则集（Working rule set）并与交互历史一同作为提示（prompt）。每轮结束后，采样N条完整轨迹，**同时**用于：
  - **参数更新**：基于组相对优势（Group-relative advantage）的GRPO风格目标函数优化策略参数。
  - **规则池更新**：通过对比反射（Contrastive reflection）生成规则编辑操作（ADD/EDIT/UPVOTE/DOWNVOTE/MERGE），并更新规则效用分数，维持规则池在容量限制下。

### 关键技术细节
1. **问题建模**：部分可观测马尔可夫决策过程（POMDP），策略决策上下文包括任务描述d、交互历史h_t、当前工作规则集 \(\tilde{K}(d)\)。
2. **轨迹采样**：每个任务实例d，采样N条完整轨迹 \(\{\tau_i\}_{i=1}^N\)。
3. **参数更新（基于GRPO）**：
   - 计算每条轨迹的终局奖励 \(R(\tau_i)\)。
   - 组内标准化得到优势 \(A_i = (R(\tau_i) - \mu_d) / (\sigma_d + \delta)\)，轨迹内所有时间步共享同一优势。
   - 优化目标 \(L_{GRPO}(\theta)\) 包含裁剪（clipped）的代理目标项和KL正则项。
4. **规则池更新（Contrastive Reflection）**：
   - 输入：当前规则池 \(K(d)\)、当前轨迹组 \(T_d\)、参考成功轨迹集 \(T^+(d)\)（来自训练中累积的成功轨迹）。
   - 调用一个冻结参数的LLM生成结构化编辑操作（ADD, EDIT, UPVOTE, DOWNVOTE, MERGE），分别对应补充、修订、强化、弱化、合并。
   - 将操作应用到规则池，并按规则更新效用分数：初始分\(s_0\)，UPVOTE加\(\delta_+\)，DOWNVOTE减\(\delta_-\)，低于阈值则移除。MERGE合并后的分数取原规则中最大值。
   - 规则池保持容量上限，超出时移除低分规则。
5. **算法流程（Algorithm 1）**：
   - 对每个任务初始化空规则池和参考成功轨迹集。
   - 每轮：采样任务d → 选择top-k工作规则 → 采样N条轨迹 → 计算优势并更新策略参数 → 生成编辑操作并更新规则池 → 将当轮成功轨迹加入参考集。

## 6. 主要结论与发现

- **性能提升显著**：JERP在AlfWorld总体成功率达到61.5%，超过GRPO（57.8%）和RLOO（48.7%）；在WebShop上成功率达64.1%，平均分数79.0，均优于所有基线。
- **收益集中在长序列、多约束任务**：如Clean、Heat、Cool、Pick2等需要更严格中间步骤的任务上，JERP相比于纯参数更新的GRPO提升更明显；而在较简单的Pick和Look上GRPO略优，说明规则池对于**纠正局部反复出现的错误**更有效。
- **动态规则池更新是关键**：消融实验（固定规则池）显示，性能提升在训练中期之后停滞，而JERP持续改善，证实了“规则池与策略同步演化”的重要性。
- **归纳性结论**：将经验规则和策略参数在同一循环中联合学习，可以互补两者的优势，使智能体既能通过规则获得有针对性的局部指导，又能通过参数更新实现全局行为改进。
