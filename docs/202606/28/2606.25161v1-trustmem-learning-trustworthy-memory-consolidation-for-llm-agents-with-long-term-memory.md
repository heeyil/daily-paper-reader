## 摘要
大语言模型（LLM）智能体依赖长期记忆来支持超出有限上下文窗口的扩展交互和个性化辅助。现有的记忆智能体通过生成的写入、修订和删除操作主动更新外部记忆，但这些更新可能会遗漏重要信息、破坏现有记忆或引入不可支持的幻觉内容。一旦存储，这些错误将成为持久的系统状态故障，可能影响未来的推理和生成。本文提出TrustMem框架，旨在提高记忆巩固的可信度。TrustMem依赖记忆转换验证器，从覆盖度、保留度和忠实度三个方面评估记忆更新的转换过程。它进一步在同一记忆状态下构建候选更新之间的偏好对，使偏好引导的强化学习能够直接优化记忆更新行为。大量实验表明，TrustMem在记忆效用和可靠性上均有提升：在MemoryAgentBench、HaluMem和Mem-alpha验证集上取得了最先进的结果，将HaluMem记忆提取的F1分数提高了12.14分，并在每种错误类型的最强基线上，分别将转换层面的遗漏、破坏和幻觉降低了40.1%、79.1%和50.0%。

## Abstract
Large language model (LLM) agents rely on long-term memory to support extended interactions and personalized assistance beyond finite context windows. Existing memory agents actively update external memory through generated write, revise, and delete operations, but these updates may omit important information, corrupt existing memory, or introduce unsupported hallucinated content. Once stored, such errors become persistent system-state failures that can affect future reasoning and generation. In this paper, we propose TrustMem, a framework designed to improve the trustworthiness of memory consolidation. TrustMem relies on a Memory Transition Verifier to evaluate the transition process of memory updates in terms of coverage, preservation, and faithfulness. It further constructs preference pairs among candidate updates under the same memory state, enabling preference-guided reinforcement learning to directly optimize memory updating behaviors. Extensive experiments demonstrate that TrustMem improves both memory utility and reliability: it achieves state-of-the-art results across MemoryAgentBench, HaluMem, and the Mem-alpha validation set, improves HaluMem memory extraction by 12.14 F1 points, and reduces transition-level omission, corruption, and hallucination by 40.1\%, 79.1\%, and 50.0\%, respectively, compared with the strongest baseline for each error type.

## 1. 核心问题与整体含义（研究动机和背景）

**长期记忆的可靠性危机**：  
LLM 智能体依赖外部长期记忆来超越上下文窗口限制，实现持续交互与个性化服务。现有记忆智能体（如 Mem-1、Mem-T）通过自主生成的写入、修订、删除操作来更新记忆。然而，这些操作可能：

- **遗漏**（Omission）：关键信息未写入记忆；
- **破坏**（Corruption）：已有记忆被错误修改或扭曲；
- **幻觉**（Hallucination）：引入输入或历史记忆均不支持的无根据内容。

一旦错误被存储，便成为**持久性系统状态故障**，会反复影响后续推理和生成。终端奖励（如最终任务成功率）无法定位具体哪一步更新出了错，存在严重的**信用分配鸿沟**。因此，核心挑战从“如何保留信息”转变为**如何构建可信的记忆巩固过程**。

## 2. 方法论：核心思想、关键技术细节

### 整体框架概述
TrustMem 将记忆巩固建模为**持续记忆更新问题**：输入流按块（chunk）顺序处理，每步智能体根据当前块 `c_t` 和之前累积的记忆 `M_{t-1}` 生成结构化的记忆编辑动作 `a_t`（WRITE / REVISE / PRUNE），由确定性记忆执行器 `E` 更新记忆为 `M_t`。每个局部转换 `z_t = (c_t, M_{t-1}, a_t, M_t)` 被 **Memory Transition Verifier** 评估。

### 核心创新一：Memory Transition Verifier（记忆转换验证器）
- 用一个冻结的 LLM 作为验证器，对每个局部转换从三个维度打分（0~1）：
  - **Coverage（覆盖度）**：当前块中的重要信息是否被保留；
  - **Preservation（保留度）**：已有有效记忆是否被无理由删除或扭曲；
  - **Faithfulness（忠实度）**：新增或修改的内容是否受输入或先前记忆支持。
- 输出标量信任分数 `v_t = V(z_t) ∈ [0,1]`，作为过渡级奖励。

### 核心创新二：多粒度奖励设计
- **样本级奖励**：
  - 任务奖励 `R_task`：最终记忆用于下游问答的指标（F1/LLM评分）；
  - 效率奖励 `R_eff`：惩罚记忆膨胀，鼓励紧凑记忆。
- **过渡级奖励**：
  - 动作执行奖励 `R_act`：动作是否可解析、可执行；
  - 内容类型奖励 `R_type`：修改内容是否具体、非空；
  - **验证器奖励 `R_ver`**：即上述 `v_t`。
- 总奖励：  
  `R_i,t = R_task + λ_eff R_eff + λ_act R_act + λ_type R_type + λ_ver R_ver`（λ 为权重系数）。

### 核心创新三：Transition-Ranked GRPO（过渡级排序 GRPO）
- 基于 GRPO（Group Relative Policy Optimization）进行全局轨迹优化：采样 G 个 rollout，计算组归一化优势。
- **偏好对构建**：对于同一状态下的多个候选过渡，用验证器分数选出最高和最低分者（分数差超过阈值 δ），构成偏好对 `(z+, z-)`。
- **DPO 风格排序损失**：  
  `L_rank = -log σ(β_rank [Δ_θ - Δ_ref])`，其中 `Δ` 为长度归一化对数概率差。
- **联合目标**：  
  `L = -J_GRPO + β_KL L_KL + λ_rank L_rank`，同时优化标量奖励、KL 正则化和过渡级排序。

- **记忆效用**：在 Mem-α 验证集平均分 66.3（超 Mem-α 4.4 分），MemoryAgentBench 平均分 65.7（超 Mem-α 6.5 分），尤其 TTL 任务提升显著。

## 7. 优点
- **精细信用分配**：通过 Memory Transition Verifier 直接监督每步记忆更新，解决了终端奖励无法定位的问题。
- **多粒度奖励组合**：兼顾最终任务性能、记忆紧凑性、动作可执行性和内容可信度。
- **偏好排序优化**：利用验证器分数构建同类状态下的偏好对，提供比标量奖励更鲁棒的信号。
- **实验设计严谨**：在三个互补基准（综合验证、OOD 泛化、操作级可靠性）上评测，并进行了充分的消融、案例分析和错误类型分析。

## 8. 不足与局限
- **模态限制**：当前仅处理文本输入，现实应用中用户输入常为多模态（图像、表格等），尚未验证对多模态记忆的适用性。
- **模型泛化性**：仅基于 Qwen3-4B 训练和评测，未探索更大模型（如 7B/14B/72B）或其他基座（如 Llama 系列），结果可能存在模型依赖。

- **验证器成本**：Memory Transition Verifier 使用冻结 LLM，每步更新都需运行一次，推理开销较高，未讨论实时部署的延迟问题。
