## 摘要
解决带有模糊和不完整描述的问题，尤其是涉及复杂漏洞的问题，需要一个复杂且长周期的工作流程。智能体必须导航代码库以定位根本原因、复现故障、实施修复并验证生成的补丁。低效的上下文管理可能导致上下文快速退化和上下文污染，阻碍成功解决。我们提出了icat-agent，一种去中心化的多智能体脚手架，它用同步的、基于事件的消息传递取代了共享上下文。利用基于评估标准的问题质量检查，icat-agent策略性地调整其工作流程：对于明确定义的问题，它启动并行打补丁和验证；而对于低质量的问题，则进行初步探索。在SWE-bench Verified和SWE-bench Pro上的全面评估表明，icat-agent在使用相同底层模型的情况下，在所有难度级别上持续优于包括SWE-agent、mini-SWE-agent和Claude Code在内的突出基线，在SWE-bench Verified上改进3.6-8.4%，在SWE-bench Pro上改进6.3-18.5%。icat-agent在计算上也是高效的，与多智能体Claude Code基线相比，每个实例平均降低1.18美元成本。我们的发现表明，像icat-agent这样的稳健脚手架能够解锁固定模型中的大量潜在能力，相同的骨干模型在icat-agent下解决的问题数量显著多于现有脚手架。icat-agent + GPT-5.4-xhigh解决了SWE-bench Pro中67.4%的问题，比当前SWE-bench Pro上最佳结果（59.10%，mini-SWE-agent+GPT-5.4-xhigh）高出8.3个百分点。

## Abstract
Resolving issues with ambiguous and incomplete descriptions, particularly concerning complex bugs, requires a sophisticated, long-horizon workflow. Agents must navigate codebases to locate the root cause, reproduce the failure, implement a fix, and validate the resulting patch. Inefficient context management, thereby, can lead to rapid context degradation and context poisoning, preventing successful resolution. We propose icat-agent, a decentralized, multi-agent scaffolding that replaces shared context with synchronous, event-based message passing. Utilizing a rubric-based issue quality check, icat-agent strategically pivots its workflow: it initiates parallel patching and validation for well-defined issues, while deploying preliminary exploration for low-quality ones. A comprehensive evaluation of icat-agent on SWE-bench Verified and SWE-bench Pro demonstrates that it consistently outperforms prominent baselines across all difficulty levels, including SWE-agent, mini-SWE-agent, and Claude Code, while using the same underlying models, improving by 3.6-8.4% on SWE-bench Verified and 6.3-18.5% on SWE-bench Pro. icat-agent is also computationally efficient, reducing the average cost by $1.18 per instance compared with the multi-agent Claude Code baseline. Our findings reveal that a robust scaffold such as icat-agent unlocks substantial latent capability within a fixed model, with the same backbone resolving markedly more issues under icat-agent than under existing scaffolds. icat-agent +GPT-5.4-xhigh resolves 67.4% of SWE-bench Pro problems, outperforming the current best result on SWE-bench Pro (59.10%, mini-SWE-agent+GPT-5.4-xhigh) by 8.3 percentage points.

## 2. 提出的方法论

### 2.1 核心思想
提出 **icat-agent**（isolated Collaborative Agentic Team），一种去中心化、自适应多agent脚手架，核心思想包括：
- **上下文隔离**：三个专用agent（Explorer, Patch Editor, Validator）**不共享全局上下文**，而是通过**同步、基于事件的消息传递**交换结构化信号（如补丁、测试结果），避免上下文退化和中毒。
- **自适应工作流**：通过**rubric-based issue质量检查**，动态决定工作流路径：
  - **高质量issue**（明确定义bug文件、函数、修复策略、复现信息）：**绕过探索**，直接启动Validator和Patch Editor并行执行。
  - **低质量issue**（缺少上述信息）：先调用**Explorer**进行探索，收集必要上下文后，再并行执行Validator和Patch Editor。
- **测试与补丁独立性**：Validator不共享测试代码，只共享pass/fail结果；Patch Editor不能访问测试实现，只能基于结构化反馈修改补丁，从而避免测试过拟合和补丁过拟合。

### 2.2 关键技术细节

- **Issue Quality Checker**（§2.2）：用LLM按照四个二元准则（buggy文件、函数、修复策略、复现信息）评估issue质量，只有全部满足才判为高质量，否则触发Explorer。
- **Explorer Agent**（§2.3）：使用基于AST的工具（tree-sitter）进行仓库探索，输出结构化上下文摘要（相关文件、函数、调用链、可疑行），但不包含任何生成的测试或补丁，保持独立性。
- **Validator Agent**（§2.4）：负责生成**复现测试**（必须在bug版本上失败）、运行**回归测试**（基于覆盖率选择已有测试），并与Patch Editor通过消息总线通信。遇到持续失败时会自我反思（Reflect），必要时修正测试本身。
- **Patch Editor Agent**（§2.5）：根据issue描述和探索结果生成初始补丁，然后根据Validator的结构化反馈（如“补丁是否修复了bug”、“是否引入了新失败”）迭代修改补丁，但看不到测试代码。
- **Event-Driven Synchronous Communication**（§2.6）：通过消息总线在Validator和Patch Editor之间传递三种事件类型：CandidatePatch、TestResult、PatchAccepted。每条消息只包含预定义的有效载荷，避免传播完整推理历史。

### 2.3 算法流程（文字说明）

1. 初始化消息总线。
2. Issue Quality Checker评估issue描述质量。
3. 若为低质量，则调用Explorer探索仓库，等待其返回结构化上下文。
4. 并行启动Validator和Patch Editor，并将探索结果（如有）发送给它们。
5. Validator生成复现测试并在bug仓库上运行，确认失败后发布BugEvidence。
6. Patch Editor生成初始补丁并发布CandidatePatch。
7. 循环：Validator评估每个候选补丁（运行复现测试+回归测试），发布TestResult；Patch Editor根据结果诊断并修订补丁；直到补丁被接受或预算耗尽。
