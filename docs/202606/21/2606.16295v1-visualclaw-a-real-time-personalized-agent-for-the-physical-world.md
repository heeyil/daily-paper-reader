## 摘要
视觉语言模型正作为通用接口服务于复杂多模态任务。然而，部署仍面临三个差距：VLM在处理密集视频帧和长提示时通常带来高延迟和高成本；智能体框架在部署后保持静态；标准的视频问答基准测试并不检验智能体能否在使用工具的工坊内利用视觉证据。我们提出VisualClaw，一个围绕两个原则构建的自我进化多模态智能体。首先，混合编码通过级联门控过滤信息量较少的流式帧，并通过热/冷top-k注入压缩文本技能库，从而降低部署成本。其次，技能进化让智能体从失败中学习：检索到的记忆作为直接拼接的上下文或引导证据条件化一个进化器，产生技能库更新以帮助未来问题。在包含2个VLM的4个视频问答基准上，VisualClaw相比全帧上传平均降低每次查询API成本-98%，相比离线均匀8帧基线降低-25.9%，同时在大多数设置中提升准确率，例如在EgoSchema上使用Gemini 3 Flash平均提升+3.85%，峰值+15.80%。为解决差距，我们整理了VisualClawArena，一个包含200个场景的多模态智能体基准，通过严格的五阶段流程构建；模型必须在工坊内使用视频证据、文档、动态更新和可执行检查。在VisualClawArena上，相同的框架结合计算机使用智能体后端，相比无进化基线，Codex (GPT-5.5)的宏观准确率提升+2.9%，Claude Code (Sonnet 4.6)提升+3.2%，相比均匀采样基线成本降低-9.5%。这些特性使VisualClaw自然适用于边缘应用，其中级联将1小时的流式会话从大约3,600次API上传减少到仅5-20次调用，而自我进化使其成为完美的个性化助手。

## Abstract
Vision language models are serving as general-purpose interfaces for complex multimodal tasks. However, deployment still faces three gaps: VLMs typically incur high latency and cost when processing dense video frames and long prompts, the agent scaffold remains static after deployment, and standard video-QA benchmarks do not test whether agents can use visual evidence inside tool-using workspaces. We present VisualClaw, a self-evolving multimodal agent built around two principles. First, hybrid encoding reduces deployment cost by filtering less informative streaming frames with a cascaded gate and compressing the text skill bank through hot/cold top-k injection. Second, skill evolution lets the agent learn from failures: retrieved memories condition an evolver as direct concatenated context or as guided evidence, producing skill-bank updates that help future questions. Across 4 video-QA benchmarks with 2 VLMs, VisualClaw cuts per-question API cost by an average -98% versus full-frame upload and by -25.9% over the offline uniform 8 frame baseline, while boosting accuracy in most settings, e.g., an average +3.85% and a peak +15.80% on EgoSchema with Gemini 3 Flash. To address the gap, we curate VisualClawArena, a 200-scenario multimodal agentic benchmark built through a strict five-stage pipeline; models must use video evidence, documents, dynamic updates, and executable checks inside a workspace. On VisualClawArena, the same framework with computer-use agent backends improves macro accuracy by +2.9% for Codex (GPT-5.5) and +3.2% for Claude Code (Sonnet 4.6) over no-evolution baselines, with a -9.5% cost reduction compared to the uniform-sampled baseline. These properties make VisualClaw a natural fit for edge applications, where the cascade reduces a 1-hour streaming session from ~3,600 API uploads down to only 5-20 calls and the self-evolution makes it a perfect personalized assistant.

- **研究意义**：实现低成本、实时、可个性化进化的多模态智能体是推进边缘AI应用（如可穿戴设备）的关键。VisualClaw同时解决了上述三个问题，并引入新的多模态智能体基准VisualClawArena以弥补评估缺失。

## 2. 方法论

### 核心思想
VisualClaw采用**混合编码**（Hybrid Encoding）与**自我进化**（Self-Evolution）两大支柱。  
- 混合编码：在边缘端用轻量级级联门控过滤冗余帧，并通过热/冷top-k注入控制技能库的提示代价。  
- 自我进化：利用存储的成功案例（Memory Bank）和失败轨迹唤醒离线进化器（Evolver）更新技能库（Skill Bank），全程不更新VLM权重。

### 关键技术细节

1. **级联编码门控 (Cascaded Gate G)**  
   - 每帧到达时依次应用：感知哈希（dHash）去重 → 128维CPU场景编码器（HSV直方图、亮度、边缘密度、纹理） → 自适应变化门（与滚动参考帧比较，输出MAJOR/MINOR/SKIP）。  
   - MAJOR帧上传云端，MINOR仅更新参考，SKIP丢弃。决策仅依赖过去帧，支持流式在线处理。

2. **混合热/冷技能注入 (Hot/Cold Skill Injection)**  
   - 技能库S由种子技能（12条跨领域视觉推理模式）和进化新增技能构成。  
   - 对每个问题q，句子嵌入检索出top-k技能作为热技能（Hot），完整正文注入系统提示；其余技能作为冷目录（Cold），以“名称+一行描述”提供，可按需获取。  
   - 提示代价由k决定，与技能库大小解耦。

3. **记忆增强的元进化 (Memory-Augmented Meta-Evolution)**  
   - 记忆库Mv存储高置信度正确案例的嵌入。失败累计达Nevo（=15）次时，检索相关记忆并调用进化器E生成新技能ΔS。  
   - 实验两种记忆→进化器通道：  
     - Cat.（拼接）：将检索到的记忆直接附加到进化提示末尾；  
     - Guide（引导）：添加前缀指令，引导进化器抽象可复用技能而非具体场景细节。  
   - 两个卫生过滤器：F1（Jaccard去重）拒绝高重复条目；F2（按效用剪枝）定期移除命中率低的技能。

### 算法流程（文字说明）
- 对于每帧：dHash去重 → 编码 → 变化门决策，仅MAJOR帧加入关键帧集合K。  
- 对于每个问题q：检索技能（热技能+冷目录）与记忆 → 拼接提示 → 调用VLM生成答案 → 评分 → 失败存入队列。  
- 当失败队列达到Nevo：进化器读取相关记忆，提出新技能 → 去重后加入技能库 → 清空失败队列。  
- 每100个问题执行F2剪枝。

## 8. 不足与局限

- **实验覆盖有限**：  
  - 视频QA仅测试4个基准，未涵盖更多样化的自然视频（如体育、纪录片）。  
  - VisualClawArena仅200场景，主要来自三个源，场景多样性仍有提升空间，可能存在特定领域偏差。  
- **跨VLM转移风险**：附录A.7指出，针对一种VLM演化的技能库可能对另一种VLM的准确率造成损害（如格式执行技能），提示部署时需要匹配目标VLM的弱点。  
- **技能膨胀与奖励作弊**：自动评分可能存在系统偏差，技能库可能放大这些偏差（如位置偏好）；作者虽设置了卫生过滤器，但长期稳定性未充分验证。  
- **对系统提示敏感**：记忆引导的进化器指令（Pguide）设计依赖经验，可能效果不稳定；不同任务可能需要调整记忆长度、top-k等超参数。  
- **仅适用于非交互式视频流**：级联门控假设帧到达顺序固定，对于随机访问或跳帧场景不适用（如回溯关键帧）。
