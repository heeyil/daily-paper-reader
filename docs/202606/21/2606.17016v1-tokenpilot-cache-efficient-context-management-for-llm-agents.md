## 摘要
随着LLM代理被部署到长期会话中，上下文累积推高了推理成本。现有方法利用文本剪枝或动态内存驱逐来最小化令牌占用；然而，它们未受约束的序列突变改变了布局，引入了前缀不匹配和缓存失效。这揭示了文本稀疏性与提示缓存连续性之间的关键权衡。为解决这一问题，我们提出了TokenPilot，一个双粒度上下文管理框架。在全局层面，摄入感知压缩作为框架约束，稳定提示前缀并在摄入入口消除开放世界环境噪声。在局部层面，生命周期感知驱逐监控上下文段的持续剩余效用，强制执行保守的批量轮次调度，仅在任务相关性过期时卸载内容段。在PinchBench和Claw-Eval上的实验（包括隔离模式和连续模式）表明，TokenPilot在隔离模式下分别降低了61%和56%的成本，在连续模式下降低了61%和87%，同时与先前系统相比保持了有竞争力的性能。TokenPilot已集成到LightMem2中，代码位于https://github.com/zjunlp/LightMem2。

## Abstract
As LLM agents are deployed in long-horizon sessions, context accumulation drives up inference costs. Existing approaches utilize text pruning or dynamic memory eviction to minimize token footprints; however, their unconstrained sequence mutations alter layouts, introducing prefix mismatches and cache invalidation. This reveals a critical trade-off between text sparsity and prompt cache continuity. To address this, we present TokenPilot, a dual-granularity context management framework. Globally, Ingestion-Aware Compaction acts as a framework harness to stabilize prompt prefixes and eliminate open-world environmental noise at the ingestion gate. Locally, Lifecycle-Aware Eviction monitors the ongoing residual utility of context segments, enforcing a conservative batch-turn schedule to offload content segments only when task relevance expires. Experiments on PinchBench and Claw-Eval under both isolated and continuous modes demonstrate that TokenPilot reduces costs by 61% and 56% in isolated mode, and 61% and 87% in continuous mode, while maintaining competitive performance compared to prior systems. TokenPilot has been integrated into LightMem2 at https://github.com/zjunlp/LightMem2.

- **整体含义**：提出了TokenPilot双粒度框架，在全局（摄入时）和局部（运行时）两个层面协调这一权衡，旨在实现低成本、高性能的长期代理部署。

## 2. 论文提出的方法论

### 核心思想
TokenPilot将上下文管理分为两个互补的层次：
- **全局层面**：摄入感知压缩（Ingestion-Aware Compaction）——在消息进入上下文前，稳定布局、过滤噪声、压缩低效用内容。
- **局部层面**：生命周期感知驱逐（Lifecycle-Aware Eviction）——在线评估上下文段的残余效用，仅在任务相关性彻底过期时才保守地批量卸载。

### 关键技术细节

#### 2.1 全局摄入感知压缩
- **前缀稳定化**：将系统提示中的运行时易变字段（工作目录、时间戳、会话ID等）替换为静态占位符；并将工具定义下移至提示末尾，确保首个指令前缀跨任务字节一致，从而将昂贵的预填充转化为缓存命中。
- **观察缩减**：对开放世界环境消息（如HTML、日志）进行确定性缩减：去重、截断长输出、HTML瘦身、格式清理等。同时引入**恢复工具**，当压缩后的消息缺少关键信号时，代理可调用恢复工具动态获取完整负载，避免信息丢失。
- **效用密度公式**：内部消息（提示、思考、工具调用等）效用密度为1；环境消息通过基密度γ(m)和摄入门G(m)控制，G(m)基于内容哈希的访问频率自适应升级。

#### 2.2 局部生命周期感知驱逐
- **三状态追踪**：每个上下文段维护状态 `{active, completed, evictable}`。completed状态作为中间缓冲层，即使子任务已完成，只要其残余效用非空（通过工具依赖模式判断），仍保留在上下文中，避免立即驱逐。
- **批量调度**：状态估计模型E（使用Qwen3.5-35B-A3B作为轻量零样本验证器）每B轮触发一次（实验确定B=3），处理压缩历史视图，输出语义增量更新任务注册表。仅当段状态变为evictable时，才执行一次性结构清除。
- **成本模型**：总成本 = 缓存命中令牌数×折扣单价 + 缓存未命中令牌数×全价 + 输出令牌数×输出单价。TokenPilot追求在保持效用最大化下最小化成本。

## 3. 实验设计
