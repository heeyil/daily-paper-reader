## 摘要
长期机器人部署需要一种紧凑且可扩展的记忆系统，该系统能够保留细粒度的视觉语义，将观测结果在空间和时间上联系起来，并实现高效存储与检索。本文提出RAVEN，一种用于长时域机器人问答与导航的代理记忆系统。RAVEN将视觉嵌入及其位姿和时间信息存储在向量数据库中，并在空间地图上基于检索来回答问题并导航至目标。通过直接操作视觉嵌入，RAVEN避免了有损的图像到文本描述过程，实现了大规模精确的语义、空间和时间检索。在多个模拟与真实世界的视频问答基准测试中，RAVEN持续超越基于文本描述的记忆系统，并在长时域任务中匹配前沿视觉语言模型，同时检索成本降低10倍。最后，我们将RAVEN部署在Unitree Go1机器人上，用于自然语言目标到达的长时域导航任务，并在多个大型室内环境中展示了成功部署。

## Abstract
Long-term robot deployment requires a compact and scalable memory that preserves fine-grained visual semantics, grounds observations in space and time, and enables efficient storage and retrieval. In this paper, we propose RAVEN, an agentic memory system for long-horizon robotic question answering and navigation. RAVEN stores visual embeddings with pose and time in a vector database, and grounds retrieval in a spatial map to answer queries and navigate to goals. By operating directly on visual embeddings, RAVEN avoids lossy image-to-text captioning and enables accurate semantic, spatial, and temporal retrieval at scale. Across several simulated and real-world video question-answering benchmarks, RAVEN consistently surpasses caption-based memory systems and matches frontier VLMs on long-horizon tasks at 10$\times$ lower retrieval cost. Finally, we instantiate RAVEN on a Unitree Go1 robot for the task of long-horizon navigation for natural language goal-reaching, and show successful deployment over several large indoor environments.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：长期机器人部署需要记忆系统，但现有方法存在**有损图像→文本描述瓶颈**（captioning bottleneck），丢失细粒度视觉细节（形状、纹理、空间关系等）。闭集语义地图（如3D点云）或原始视频流要么词汇有限，要么过于冗余。
- **背景**：机器人问答与导航中，用户可能提出如“带粉色蝴蝶结的蓝帽子在哪”等细粒度、需要时空联合检索的问题。纯文本描述无法可靠保留此类信息。
- **整体含义**：RAVEN旨在构建一种**紧凑、可扩展、信息高保真**的记忆系统，直接基于视觉嵌入，避免中间文本压缩，同时实现高效的大规模检索。

## 2. 方法论：核心思想、关键技术细节
### 2.1 核心思想
- **直接使用视觉嵌入**：将机器人探索过程中采集的RGB帧通过多模态编码器（如CLIP、SigLIP、Seed1.6、QQMM-v2）编码为紧凑向量，**跳过图像→文本描述步骤**。
- **三维元组记忆**：每条记忆包含 `(视觉嵌入, 机器人位姿(x,y,z), 时间戳)`，称为**视觉-空间-时间记忆**（visuo-spatio-temporal memory）。
- **基于向量数据库的检索**：使用FAISS或Milvus等框架，支持语义、空间、时间维度的最近邻搜索。
- **代理推理循环**：VLM（如Gemini、GPT）作为智能体，迭代调用四种检索工具（文本、时间、位置、图像），积累证据后生成答案或导航目标。

### 2.2 关键技术细节
- **记忆构建阶段**：
  - 机器人进行前沿探索（frontier exploration），同时运行RGB-D SLAM构建3D占用地图。
  - 每隔一定时间采集RGB帧，用预训练多模态编码器获得视觉嵌入，与对应位姿和时间一起存入向量数据库。
- **查询阶段**：
  - 用户问题输入后，VLM先进行推理决定使用哪种工具（默认文本检索）。
  - 工具返回top-K个记忆条目（包含图像、位姿、时间），VLM评估是否足够；若不足则继续检索，最多迭代数次。
  - 最终输出结构化结果：文本答案、目标位置(x,y,z)或时间参考。
- **导航执行**：预测的目标坐标传给A*路径规划器，驱动机器人到达目标点，无需额外探索。

## 6. 主要结论与发现
1. **RAVEN持续优于caption-based方法**：在RAVEN-QA上，RAVEN（QQMM-v2+Gemini-2.5-Flash）在困难查询上准确率92.4%，而ReMEmbR仅74.8%。差距在细粒度（次要对象）查询上更明显（约30%）。
2. **匹配甚至超越VLM Only**：在NaVQA长视界上，RAVEN优于VLM Only（整体准确率75.2% vs 75.2%持平，但位置/时间误差更小）；在FindingDory长视界上RAVEN（32.2%）优于VLM Only（28.2%），且RAVEN记忆效率是VLM Only的10倍（检索帧数仅7.43% vs 100%）。
3. **高度压缩能力**：即使帧率降至0.12fps（250×压缩），RAVEN仍保持90%以上性能；相比原始RGB视频实现22,315×压缩。
4. **开放型小模型效果差**：27B~32B的开放VLM（如Gemma3-27B）性能不如直接使用嵌入器（Embedder Only），提示小模型不适合此类推理任务。
5. **真实部署成功**：在Unitree Go1机器人上执行自然语言导航，成功率97.1%（SigLIP+GPT-5.2）。

## 7. 优点
- **避免caption瓶颈**：直接使用视觉嵌入，保留细粒度语义，对次要对象、文本信息召回等任务特别有效。
- **统一时空检索**：通过向量数据库同时支持语义、空间、时间查询，且检索效率高（亚线性）。
- **训练自由**：所有模块均为预训练模型，无需额外微调，易于部署。
- **模块化设计**：编码器、VLM、向量数据库均可独立替换，适应不同硬件约束。
- **充分真实验证**：不仅在模拟环境，还部署到真实四足机器人，验证系统实用性。

## 8. 不足与局限
- **依赖嵌入质量**：检索能力受限于预训练多模态编码器的对齐质量。若嵌入模型对某些视觉特征（如非常规物体、极端光照）表现不佳，记忆会出错。
- **静态记忆假设**：当前记忆为一次构建后静态使用，无法更新或遗忘；在动态环境（物体移动、状态变化）中会失效。作者将其列为未来工作。
- **未覆盖极端动态场景**：实验环境主要为室内稳定环境，对室外、人类活动密集等动态场景未充分评估。
- **计算成本未全面对比**：RAVEN虽然检索效率高，但VLM代理可能需要多步工具调用（增加推理次数），而VLM Only只需一次前向。总延迟和成本对比需更详细分析。
- **实验偏差风险**：自建RAVEN-QA数据集虽然分类细致，但问题数量有限（273个），可能不完全代表现实分布。作者也承认这一点。
- **可扩展性验证有限**：虽提到可处理数千帧，但未测试数十万帧级超大规模记忆的检索性能。
