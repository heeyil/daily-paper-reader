## 摘要
混合专家模型架构通过将多个LLM代理组织成层次化推理管线来改进推理时缩放。然而，现有的MoA变体在深度增加时无法维持增益，表现出退化、早期平台期或饱和。我们提出ReM-MoA，一种记忆增强的MoA框架，通过两种机制维持缩放：(1) 排序推理记忆，它使用比较评审代理持久存储和排序所有层的推理轨迹；(2) 策划多样化记忆路由方案，它将成功和失败的轨迹以不同组合暴露给不同代理，保留探索多样性的同时传播高质量推理。我们进一步引入可选的跨领域评审代理蒸馏管线，通过前沿模型监督提高排序质量。在涵盖数学、形式逻辑、代码、知识和常识的五个推理基准上，ReM-MoA在深度和宽度缩放方面始终优于先前的MoA变体，并且其优势随深度增加而扩大，将结构化的跨层推理记忆确立为可缩放多代理推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

- **核心问题**：现有 Mixture-of-Agents (MoA) 架构在多 LLM 协同推理时，随着层数和宽度增加，性能出现退化（degradation）、早期平台期（plateauing）或饱和（saturation），无法实现持续的扩展收益。论文指出三个驱动机制：原始跨层拼接导致错误累积；仅有局部通信无法保留早期高质量推理；智能体输入趋同导致探索多样性崩溃。
- **整体意义**：该问题揭示了现有 MoA 架构缺乏有效的跨层推理记忆传递机制，而论文提出的**结构化跨层推理记忆**是实现可缩放多智能体推理的关键缺失组件。将 MoA 扩展问题从“聚合或交互”重新定义为“质量—多样性联合维持”问题，对提升多智能体系统的实用深度和广度具有重要价值。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 提出 **ReM-MoA** 框架，通过**排名推理记忆（Ranked Reasoning Memory, M）** 持久存储和排序所有层的推理轨迹，配合**策展多样化记忆路由（Curated Diversified Memory Routing, Π）** 为不同智能体分配不同的成功/失败轨迹组合，同时维持推理质量和探索多样性。

### 关键技术细节
- **排名推理记忆**：
  - 每层调用一个 **评审代理（Reviewer Agent, R）**，对所有 N 个推理轨迹进行**比较评分**（1-10 整数分）并生成针对性的推理理由（rationale）。
  - 每个轨迹与分数、理由打包为三元组 \(\langle r_{l,j}, s_{l,j}, \rho_{l,j} \rangle\)，追加到记忆库 \(M_{\le l}\) 中。
  - 记忆对所有后续层开放，实现非相邻层之间的跨层访问，避免仅靠相邻层传播的错误累积。

- **策展多样化记忆路由**：
  - 在每个后续层，每个提议代理收到从记忆库中**策展**的参考轨迹集 \(S_{l,j}\)，该集包含高分段（Top）、低分段（Bot）或对比组合（Con）。
  - 不同代理收到**不同**的参考集（如对于 N=3，一个代理收到 Top 1 ∪ Bot 1，一个仅 Top 2，一个仅 Bot 2），从而保持层内探索多样性。
  - 对于更大的 N 值，论文在附录中提供了预定义的路由分配表（Table 5），确保三种路由家族均有覆盖且互不重复。

- **可选增强：评审蒸馏（Reviewer Distillation）**：
  - 用前沿模型（GPT-5.5）作为教师，在 ReM-MoA 自身运行产生的轨迹数据上，对小型评审代理（Qwen2.5-7B-Instruct）进行 LoRA 微调。
  - 多领域训练（500 个问题，跨 5 个领域）使蒸馏后的评审代理泛化到未见域，进一步提高排序质量和框架最终性能。

### 公式/算法流程（文字说明）
1. 输入问题 x。
2. 对第 1 层：每个提议代理仅根据 x 生成推理轨迹；调用评审代理对所有 N 个轨迹进行比较评分，得到分数和理由，写入记忆。
3. 对于 \(l \ge 2\)：从记忆 \(M_{\le l-1}\) 中根据路由方案为每个代理选取不同的策展览参考集 \(S_{l,j}\)；每个代理结合问题和参考集生成新的轨迹；再次调用评审代理评分并更新记忆。
4. 最后一层 L 后，聚合器从记忆中选取三个参考集（同样来自三种路由家族），合成最终答案。
5. 流程可表示为：\(\mathbf{r}_l = A_l(\Pi(M_{\le l-1}, \text{problem}))\), \(M_l = \text{Reviewer}(\mathbf{r}_l)\)，聚合器最终输出。

## 6. 主要结论与发现
1. **现有 MoA 变体在深度扩展时存在明确的三种失败模式**：标准 MoA 退化、RMoA 平台期、AttentionMoA 饱和。
2. **ReM-MoA 随深度增加持续改进**，在五个基准上均超越所有基线，且与基线差距随深度单调增大（深度从 1 到 9，差距扩大 5.9 倍至 6.8 倍）。
3. **跨层访问和路由多样化是核心驱动**：消融实验显示移除跨层访问或路由多样化均会导致性能从“持续提升”回退至“平台期”；移除去理由影响较小，仅调节增益幅值。
4. **评审蒸馏可进一步提升性能并泛化到未见域**：Multi-FT 使 HellaSwag 的缩放从平台期转为持续提升；LOO 变体仅用四个域训练即可恢复全领域训练 44% 的平均增益，表明蒸馏捕捉了可迁移的评价技能。
5. **结构化跨层推理记忆是解决 MoA 缩放崩溃的关键缺失机制**。
