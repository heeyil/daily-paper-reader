## 摘要
设备端语言模型代理通过积累检索记忆中的经验而非更新权重来改进。这种记忆受到硬边界限制且暴露在外：它消耗RAM和能量，通过窄上行链路到达同伴，并且由于代理读取的内容可写入而成为攻击面。现有系统各自覆盖了问题的一部分：代理记忆无预算地增长，设备端方法仅凭成功保留条目，而投毒主要作为攻击而非记忆治理问题被研究。我们提出\sys{}，一种每个字节净价值评分，用于管理代理的经验记忆生命周期。核心思想是让预算充当策展人：每个条目按每个字节的价值减去危害来评分，因此一个标尺决定保留、共享和信任什么。\sys{}做出三个决策：(1) \textbf{保留}在RAM和能量预算下驱逐低价值字节；(2) \textbf{共享}仅在见解价值超过上行链路成本时发送；(3) \textbf{信任}通过来源对同伴条目进行门控。在语言模型代理任务漂移基准测试和包含两个机械臂节点与一个集线器的真实异构Jetson测试平台上，\sys{}将内存减少2.7倍，上行链路减少2.4倍，将注入成功率从0.75降至零，并在被毒化或陈旧记忆破坏的案例上提高准确性。通过净价值策展同时减少了占用空间、能量、上行链路和注入成功率，而不降低准确性。在这种设置下，通过净值遗忘增强了代理而非削弱它。

## Abstract
On-device language-model agents improve by accumulating experience in retrieved memory rather than by updating weights. This memory is hard-bounded and exposed: it consumes RAM and energy, reaches peers through a thin uplink, and becomes an attack surface because it is writable by what the agent reads. Existing systems each cover one part of this problem: agentic memories grow without a budget, on-device methods keep entries by success alone, and poisoning is studied mainly as an attack rather than as a memory-governance problem. We propose \sys{}, a single net-value-per-byte score that governs an agent's experience-memory lifecycle. The main idea is to let the budget act as the curator: each entry is scored as value minus harm, per byte, so one ruler decides what to keep, share, and trust. \sys{} makes three decisions: (1) \textbf{KEEP} evicts low-value bytes under the RAM and energy budget; (2) \textbf{SHARE} sends an insight only when its value exceeds its uplink cost; and (3) \textbf{TRUST} gates a peer entry by provenance. On language-model-agent task-drift benchmarks and a real heterogeneous Jetson testbed with two robot-arm nodes and a hub, \sys{} reduces memory by $2.7\times$ and uplink by $2.4\times$, drives injection success from 0.75 to zero, and raises accuracy on cases corrupted by poison or stale memory. Curating by net value reduces footprint, energy, uplink, and injection success together without reducing accuracy. In this setting, forgetting by net value improves the agent rather than weakening it.

#### 1. 核心问题与整体含义（研究动机和背景）

设备端语言模型（LLM）代理通过积累经验记忆（如成功/失败的任务摘要）来持续改进，而不是更新权重。这种记忆受到三个硬约束：
- **资源预算**：RAM和能量有限，不能无限增长。
- **暴露性**：记忆可通过上行链路共享给其他代理，且可由代理读取的任何内容写入，成为攻击面（注入恶意条目）。
- **三个决策困境**：保留什么（KEEP）、共享什么（SHARE）、信任什么（TRUST）本质上是同一个问题：一条经验是否值得其占用的字节成本。

现有工作分别处理部分问题：记忆架构（A-MEM）只增长不驱逐；设备端方法（naive-LRU）仅按成功保留；记忆投毒主要作为攻击而非治理问题。缺乏一个统一机制同时应对三个约束和目标。本文提出 **CURATOR**，通过单一的**净字节价值评分**，以预算为策展人，统一管理记忆生命周期，实现遗忘即改进。

#### 2. 方法论：核心思想、关键技术细节、公式/算法流程

**核心思想**：对每条记忆条目计算 \(\rho(m) = (\hat{V}(m) - \lambda \hat{H}(m)) / b(m)\)，其中 \(\hat{V}\) 是估计价值，\(\hat{H}\) 是估计危害，\(b\) 是字节数，\(\lambda\) 是权衡系数。以此决定KEEP、SHARE、TRUST。

**关键技术细节**：
- **价值 \(\hat{V}\)**：由三因子乘积组成：
  - 检索倾向 \(p(m)\)：基于设备滑动查询草图与条目嵌入的 softmax 概率。
  - 条件帮助 \(q(m)\)：加入条目后对任务结果的期望边际效用，通过标签自由的自我一致性估计。
  - 抽象增益 \(a(m)\)：原始轨迹与蒸馏后字节之比，压缩越高效，价值密度越高。
- **危害 \(\hat{H}\)**：包含两项：
  - 负迁移风险 \(\eta(m)\)：结合条目特异性和与当前任务中心的马氏距离，惩罚过拟合或过时的条目。
  - 来源风险 \(\pi(m)\)：logistic 门控函数，基于条目来源（外部/同伴/自衍生）、自加固次数、指令似然度。固定权重，不通过训练更新，防止攻击者通过微调绕过。
- **三个决策**（算法伪代码）：
  1. **KEEP**：按 \(\rho\) 降序排列，贪心选取直到 RAM 预算 \(B\) 耗尽；能量通过虚拟队列 \(Q\) 控制准入。
  2. **SHARE**：对 KEEP 集合，仅当 \(\rho_s(m) > \tau\) 且非冗余时发送；\(\rho_s\) 使用同伴的草图重新计算倾向，但价值/危害部分继承。
  3. **TRUST**：对收到的同伴条目，仅当 \(\rho(m') > \theta\) 时接纳，来源风险项自动阻断恶意条目。

**算法流程**（文本描述）：
1. 遍历所有记忆条目，计算每条目的 \(\rho\)。
2. KEEP：按 \(\rho\) 排序，依次加入直至达到内存预算，若加入前发现条目可蒸馏则先蒸馏（可能提升 \(\rho\) 使其保留）。
3. SHARE：从 KEEP 集合中，按 \(\rho_s\) 降序，逐一检查是否超过阈值且不重复，在带宽预算内发送。
4. TRUST：对每条同伴条目，若 \(\rho\) 超过信任阈值则加入本地记忆。
