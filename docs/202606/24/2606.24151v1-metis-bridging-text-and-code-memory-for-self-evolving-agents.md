## 摘要
自进化代理通过从过去执行中提炼经验并在未来任务中复用来随时间改进。现有系统将这类经验表示为注入代理上下文的自然语言文本，或表示为作为可调用工具暴露的代码。然而，在这两种表示之间的选择通常在设计时做出，而非源自经验本身的特性，导致它们之间的权衡关系尚不明确。我们提出了首个控制实验，在完全相同的经验集上分离文本记忆和代码记忆。结果表明，两种形式在构建成本、执行效率和可迁移性方面表现出互补的权衡，以至于单一表示均不足以胜任。基于这些发现，我们提出了Metis，一种构建于分层双表示记忆之上的自进化代理系统。Metis将文本经验组织为执行计划、环境事实和常见陷阱，并选择性地将重复出现的计划结晶为经过验证的可调用工具。这种设计结合了文本记忆的广泛适用性和代码记忆的执行效率，同时仅在重复复用合理时才产生工具生成成本。我们在AppWorld（一个具有挑战性的交互式代理基准）上评估了Metis。结果表明，与ReAct相比，Metis将任务准确率提升了最高20.6%，同时降低了最高22.8%的执行成本。与代表性的自进化代理系统相比，Metis在准确率、执行效率和记忆构建成本之间始终取得了更好的平衡。

## Abstract
Self-evolving agents improve over time by distilling experience from past executions and reusing it in future tasks. Existing systems represent such experience either as natural-language text injected into the agent context or as code exposed as callable tools. However, the choice between these representations is typically made at design time rather than derived from the characteristics of the experience itself, leaving the trade-offs between them poorly understood. We present the first controlled study that isolates text memory and code memory over an identical set of experiences. Our results show that the two forms exhibit complementary trade-offs in construction cost, execution efficiency, and transferability, such that neither representation alone is sufficient. Guided by these findings, we propose Metis, a self-evolving agent system built on a hierarchical dual-representation memory. Metis organizes textual experience into execution plans, environment facts, and common pitfalls, and selectively crystallizes recurring plans into validated callable tools. This design combines the broad applicability of text memory with the execution efficiency of code memory while incurring tool-generation cost only when justified by repeated reuse. We evaluate Metis on AppWorld, a challenging benchmark for interactive agents. The results show that Metis improves task accuracy by up to 20.6% over ReAct while reducing execution cost by up to 22.8%. Compared with representative self-evolving agent systems, Metis consistently achieves a better balance between accuracy, execution efficiency, and memory-construction cost.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前自进化（Self-evolving）智能体系统，通常将过往执行经验存储为**自然语言文本**（如经验提示）或**可执行代码**（如可调用的工具函数）。然而，对于何种类型的经验应该采用何种表示形式，现有系统主要依赖于设计阶段的“预先固定”选择，而非根据经验本身的特性动态决定。这导致了对文本与代码两种表示形式在构建成本、执行效率和泛化能力上的根本性权衡缺乏系统性理解，从而阻碍了智能体在部署环境中的持续有效改进。

- **研究动机与背景**：大型语言模型（LLM）驱动的智能体在解决用户任务时，其“无状态”特性导致经验无法跨对话复用，重复犯错和资源浪费问题突出。自进化智能体通过提炼经验并存储于“记忆”中来克服此局限。然而，现有工作在`经验表示层面`存在盲点：是存储为文本，还是代码？系统无法区分，也无法在两者间动态协同。本研究正是为了填补这一空白，旨在通过受控实验揭示两种表示的内在差异，并设计一种能动态选择最优表示的智能体系统。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：本文提出 **Metis**，一个**层级双表示记忆（Hierarchical Dual-Representation Memory）** 系统。其核心思想是：**以文本记忆为基础，以代码记忆为高效补充**。具体而言，首先通过低成本、高泛化性的文本记忆广泛捕获经验；然后，仅对其中表现出高频复用和稳定模式的“执行计划”，才判定为值得投入较高成本，并将其“结晶”（Crystallize）为可调用的、高效率的代码工具。

- **关键技术细节**：
    1.  **差分文本记忆（Differentiated Text Memory）**：文本记忆不再是扁平的经验提示，而是被主动组织为三类：
        - **执行计划 (Execution Plans)**：可复用的子程序模板，是唯一可升级为代码的记忆类型。
        - **环境事实 (Environment Facts)**：任务无关的环境与API约束。
        - **常见陷阱 (Common Pitfalls)**：格式化的失败模式，如（触发条件、错误操作、后果）。
    2.  **模式感知的代码生成 (Pattern-Aware Code Generation)**：
        - **基于复用的提升机制（Recurrence-Based Promotion）**：只有当一个执行计划`p`被选取（`S_plan_i`）的次数达到预设阈值`θ`后，才触发代码生成。任务是否成功并非必要条件，因为这有助于覆盖更广的程序族谱。代码生成器（Codify）仅接收计划体`body(p)`和该计划被选取的所有历史任务查询`QB(p)`与`QR(p)`，**明确排除原始执行轨迹**，避免过拟合。
    3.  **工具准入验证 (Tool Admission Guard)**：生成的工具必须通过沙盒环境中的编译和依赖完整性检查。该系统保证了新工具的鲁棒性。
    4.  **渐进式信息披露 (Progressive Disclosure)**：反思模块（Reflector）在分析轨迹时，使用压缩视图（只保留头和尾），必要时才请求展开完整细节，从而提升反思效率。

- **算法流程概述（文字说明）**：
    1.  **执行**：给定任务`qi`，记忆管理器（Memory Manager）分别从文本和代码库中检索相关条目，并连同依赖信息一并注入到执行器（Executor）的上下文中，执行器产生执行轨迹τ_i。
    2.  **反思**：执行完毕后，系统进行双路径反思。
        a. 先进行**模式感知的代码生成**：检查被选中的计划是否已累积足够的复用证据（`|B(p)| ≥ θ`），若是，则启动代码生成流程，通过验证后添加至代码库。
        b. 再进行**差分文本反思**：分析轨迹，根据成功/失败情况，生成或合并/更新三类文本记忆（计划、事实、陷阱）。
    3.  **更新**：将产生的文本记忆和代码工具写回记忆库，供后续任务使用。

### 6. 论文的主要结论与发现

- **核心发现**：文本记忆与代码记忆在**构建成本**（文本更低）、**执行效率**（代码更高）和**迁移可靠性**（文本更好）上存在根本性、互补性的权衡。没有任何一种表示是完美的。
- **系统结论**：Metis通过**层级双表示策略**有效地利用了这种互补性。它使用结构化的文本记忆作为低成本、高泛化的基础，并通过基于复用的门控机制，将成熟的、有复用价值的计划选择性升级为代码工具。结果证明：
    - **准确性最高**：在AppWorld的两种分法上，Metis的准确率（TGC）均显著高于所有基线（Official: 60.1% vs. No Memory 51.8%；Resampled: 66.1% vs. No Memory 54.8%）。
    - **执行效率最高**：Metis的平均执行Token数和ReAct轮数在所有方法中最低（Official: 97.4K tokens; Resampled: 78.5K tokens）。
    - **构建成本可控**：虽然Metis同时生成文本和代码，但其总构建成本（7.9M tokens Official）低于或与最强的基线（SkillX: 9.4M tokens）相当，其代码生成是源于复用的“物有所值”。

- **实验泛化性有限**：实验仅在一个模拟环境（AppWorld）上完成。Metis的设计和策略是否能在更多样、更复杂的真实世界环境（如网页、机器人控制等）中保持优势，仍是一个开放问题。
