## 摘要
随着自主智能体处理日益复杂的多步骤、多智能体任务，它们的执行轨迹已扩展至超出最大上下文窗口的限制。当前有效诊断智能体故障的方法是将完整轨迹加载到LLM的上下文窗口中，这会导致注意力稀释，并在智能体痕迹不可避免地超出上下文限制时失效。为解决这一问题，我们提出了SAFARI（通过主动调查扩展长程智能体故障归因），该框架用工具增强的诊断循环取代了线性上下文加载。通过为LLM配备专门工具箱以读取和搜索轨迹片段，并配备持久短期记忆（STM）以进行跨轮推理，SAFARI有效地将诊断准确性与架构上下文限制解耦。实验表明，在Who&When数据集上，SAFARI在1M token预算下优于现有最佳结果20%，在TRAIL GAIA子集上，在25K token预算下提升19%。最重要的是，即使目标故障位于模型原生上下文窗口5倍之外（传统评估器在此情况下完全失败），SAFARI仍能保持0.58的精确度。

## Abstract
As autonomous agents tackle increasingly complex multi-step, multi-agent tasks, their execution trajectories have scaled beyond the constraints of even the largest context windows. Current methods for effectively diagnosing agent failures load the full trajectory into an LLM's context window, which suffers from attention dilution and fails when agentic traces inevitably exceed context limits. To address this, we introduce SAFARI (Scaling long-horizon Agentic Fault AttRibution via active Investigation), a framework that replaces linear context loading with a tool-augmented diagnostic loop. By equipping LLMs with a specialized toolbox to read and search trajectory segments alongside a persistent Short-Term Memory (STM) for cross-turn reasoning, SAFARI effectively decouples diagnostic accuracy from architectural context limits. Our experiments demonstrate that SAFARI outperforms state-of-the-art results by 20% on the Who&When dataset within a 1M token budget, and by 19% on TRAIL GAIA subset on a 25K token budget. Most significantly, SAFARI maintains a 0.58 precision even when the target fault resides 5x beyond the model's native context window, a scenario where traditional evaluators fail entirely.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：自主智能体（LLM驱动）执行长程、多步、多智能体任务时，其执行轨迹长度动辄数百万 token，远超当前最大 LLM 上下文窗口（如 1M tokens）。现有故障归因方法将完整轨迹被动加载到上下文窗口中，导致**注意力稀释**，并在轨迹超出窗口时完全失效。
- **研究动机**：设计一种能够**低成本、可扩展**的故障诊断方法，使其**诊断精度不依赖于上下文窗口大小**，即使故障位于模型原生上下文之外也能有效定位。
- **整体含义**：SAFARI 通过**主动调查**（active investigation）替代线性上下文加载，为长程自主系统的可靠性监控提供了新范式。

## 2. 方法论：核心思想与关键技术细节
- **核心思想**：构建一个 **Investigator Agent**，它不一次性读入全部轨迹，而是通过**迭代工具调用**（工具增强的诊断循环）有选择地查询轨迹片段，逐步收集证据，形成假设，并通过原子评估器验证后得出结论。
- **关键技术组件**：
  - **工具箱**：
    - `read(offset, limit)`：按偏移量和长度顺序读取轨迹片段。
    - `search(pattern)`：大小写不敏感的正则表达式搜索，快速定位关键模式（如 span_id、错误值）。
  - **短期记忆（STM）**：持久化跨轮关键信息的 JSON 块，包含四个字段：
    - 任务目标（`task_goal`）
    - 故障假设与证据（`faults_identified_so_far`：step_id、理由、引用证据）
    - 尚未完成的研究缺口（`still_need`）
    - 已执行过的工具调用记录（`past_tool_calls`）
    - 每次 LLM 调用时 STM 追加在对话末尾，防止早期证据因上下文淘汰而丢失。
  - **原子推理评估器（Reasoning Evaluator）**：将假设分解为最多 3 个**原子声明**（如“步骤3出现了幻觉值”），由独立评估 LLM 仅根据引用证据给出置信度（0-100）。平均置信度 ≥ θ（默认 70）时允许提交结论；否则低分声明指明逻辑缺陷，Agent 需继续收集证据。
  - **强制终止机制**：迭代预算 K=30 轮，若未达成结论则强制综合已有证据提交最佳答案。
- **算法流程**（文字说明）：
  1. 初始化对话历史 H₀（系统提示 + 用户提示）。
  2. 每轮 k，Investigator Agent 读取当前历史 Hₖ 和工具模式，输出工具调用 a（read 或 search）。
  3. 系统返回观察 o，更新历史 Hₖ₊₁ = Hₖ ∪ {(a, o)}。
  4. 当 Agent 认为证据足够时，将假设分解为 E≤3 个原子声明，并行发送给评估器。
  5. 评估器返回平均置信度 C̅ 和逐声明反馈 v。若 C̅ ≥ θ，Agent 输出最终故障步 t' 和理由 R'；否则根据低分声明缺陷返回步骤 2 继续调查。
  6. STM 在每个 LLM 调用末尾注入，仅保留关键信息，旧 token 被推离时 STM 仍可用。

## 3. 实验设计：数据集、基准、对比方法

- **基准方法**：

  - **Step-by-step**（Zhang et al., 2025）：逐步骤扫描直到定位故障。
  - **RAFFLES**（Zhu et al., 2026）：迭代多轮推理基线，但受限于原生上下文窗口。
- **评估指标**：
  - **Step-level Accuracy**（精确匹配故障步）。

## 7. 优点
- **创新性**：首次将“主动调查”范式系统应用于智能体故障归因，用工具和迭代推断代替被动上下文消化，从根本上突破上下文限制。
- **工程实用性**：提供的 read/search 工具、STM 机制、原子评估器设计易于集成到现有 LLM API 中。附录提供了完整的 Prompts，具有良好的可复现性。
- **实验设计严谨**：
  - 提出 **缩放因子 ς** 量化超出上下文程度，并设计固定预算和固定缩放因子两种评估方式，为长程推理评测提供了新方法论。
  - 消融实验和延迟分析增强了结论的可信度。
- **消融实验明确 STM 贡献**：证明 STM 不仅是简单总结，而是逻辑锚点，有助于维持跨轮推理连贯性。

## 8. 不足与局限
- **数据偏差风险**：
  - 论文自身承认 Who&When 和 TRAIL 很可能被包含在 LLM（Claude-Opus）的预训练语料中，模型可能记住了正确答案，导致故障定位精度被高估。该偏差对所有方法均存在，但难以量化。
- **性能退化场景**：
