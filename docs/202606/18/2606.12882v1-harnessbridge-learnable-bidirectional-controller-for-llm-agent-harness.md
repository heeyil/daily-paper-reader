## 摘要
大型语言模型越来越多地被部署为用于长周期任务的智能体，然而其性能不仅受模型能力和环境设计的影响，还受中介智能体-环境交互的框架影响。现有的框架大多是手动设计的，随着轨迹变长和交互变得更加复杂，它们难以扩展。在这项工作中，我们探究框架是否可以通过一个可学习的即插即用模块生成，该模块可以以端到端的方式进行训练。我们引入了HarnessBridge，一个轻量级的可学习框架控制器，它将智能体-环境接口参数化为双向投影。HarnessBridge学习两种双向投影：观测投影，将原始轨迹蒸馏为紧凑的、决策相关的状态；以及动作投影，将提议的动作转换为可执行的转换或基于轨迹的拒绝。我们通过统一指令微调在框架监督数据集上训练HarnessBridge。在Terminal-Bench~2.0和SWE-bench Verified上，HarnessBridge匹配或超越了强大的专用框架，同时大幅减少了令牌使用量和轨迹长度，并从较小的生成器泛化到较大的商业模型。

## Abstract
Large language models are increasingly deployed as agents for long-horizon tasks, yet their performance is shaped not only by model capability and environment design, but also by the harness that mediates agent--environment interaction. Existing harnesses are largely manually engineered, making them difficult to scale as trajectories grow longer and interactions become more complex. In this work, we ask whether harness can be generated by a learnable plug-in module that can be trained in an end-to-end fashion. We introduce HarnessBridge, a lightweight learnable harness controller that parameterizes the agent--environment interface as a bidirectional projection. HarnessBridge learns two bidirectional projections: observation projection, which distills raw trajectories into compact, decision-relevant states, and action projection, which converts proposed actions into executable transitions or trajectory-grounded rejections. We train HarnessBridge on a harness supervision dataset via unified instruction tuning. On Terminal-Bench~2.0 and SWE-bench Verified, HarnessBridge matches or surpasses strong specialized harnesses while substantially reducing token usage and trajectory length, and generalizes from smaller generators to larger commercial models.

## 1. 论文的核心问题与整体含义（研究动机和背景）

大型语言模型（LLM）越来越多地被部署为执行长周期任务的智能体（agent），例如自主软件工程、网页导航、搜索等。现有工作主要关注两个方向：更强的生成器（智能体）和更丰富的环境。然而，随着任务周期变长，第三个方向变得至关重要：智能体与环境之间的**接口** – 即决定哪些信息传递给生成器、哪些动作提交给环境的“框架（harness）”。

当前，框架（如轨迹格式化、上下文管理、工具调用解析、输出验证等）几乎全部依赖**手动工程**。随着交互轨迹增长和复杂性增加，手动设计的框架难以扩展，导致两个方向的问题：
- **环境→智能体方向**：轨迹累积冗余上下文、过期错误、低价值细节，增加token成本并模糊决策关键状态。
- **智能体→环境方向**：模型可能重复无效动作、陷入空循环、发出格式错误的命令，浪费环境步骤。

近期有工作开始将框架视为可优化的目标（如Meta-Harness），但它们通常优化外部脚手架，而非学习运行时交互策略。本文提出一个根本问题：**能否将框架公式化为一个端到端可学习的生成问题？**

## 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：将智能体-环境接口参数化为一个**可学习的双向投影策略**，称为HarnessBridge。它包含两个方向的投影：
- **观测投影 (Observation Projection)**：将原始交互历史映射为生成器可见的状态，保留决策关键信息，同时压缩或抑制过时、冗余、被取代的内容。
- **动作投影 (Action Projection)**：将生成器提议的动作映射为可执行的环境转换或基于轨迹的拒绝（附带反馈），防止低价值或无根据的动作。

**关键技术细节**：

1. **形式化定义**：
   - 给定系统提示`s`、任务指令`q`、当前历史`H_t`和生成器提议动作`at`，HarnessBridge参数化一个双向交互策略`πh`，输出投影后的历史`Ĥ_t`和投影后的动作`a'_t`。
   - 生成器策略`πg`保持冻结，仅优化框架策略`πh`。

2. **观测投影 (P_obs)**：
   - 投影后的历史包括一个**活跃状态索引 (active-state index)** 和每个历史单元的投影表示。活跃状态索引记录未解决错误、打开约束、已确立事实、待办目标等，放在投影历史前。
   - 每个历史单元被赋予一个类型标签`z_i ∈ {PASS, COMPRESS, DROP}`，决定其投影形式（保留原文、压缩、丢弃）。
   - 压缩要求保持来源意识：压缩语句必须可追溯到原始轨迹的特定片段。

3. **动作投影 (P_act)**：
   - 给定历史`H_t`和提议动作`at`，预测决策`dt ∈ {PASS, REJECT}`。
   - 当`dt=REJECT`时，不执行环境步骤，返回基于轨迹的反馈`ρ_t = (concern, evidence, suggestion)`；若无法提供轨迹证据，则默认PASS。

4. **统一指令微调训练**：
   - 将两个投影统一为条件生成问题，共享同一个轻量级LLM（基于Qwen3.5-0.8B），通过指令区分任务。
   - 数据构建：从SWE-bench Verified的成功轨迹中采样，使用提示方法和LLM评判筛选高质量监督数据。观测投影示例需满足模式一致性、忠实压缩、保留决策关键证据；动作投影示例需通过/拒绝决策及反馈基于轨迹且可操作。最终获得5,405个高质量示例（2,682个动作投影 + 2,723个观测投影）。

5. **原始轨迹保留**：原始历史`H_t`始终作为权威记录保留，压缩仅在每轮触发，而非不可逆更新。
