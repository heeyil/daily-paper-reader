## 摘要
记忆已成为自进化智能体的标准基础，但保留经验并不等同于学习如何通过经验进化。现有的记忆智能体可以存储轨迹、检索反思或积累技能，但往往缺乏全面能力来选择有用经验、据此行动、编写可复用知识并维护不断增长的存储库。我们提出OPD-Evolver，一种通过在线策略自蒸馏实现慢-快协同进化的框架，用于培育此类智能体进化器。在快速循环中，OPD-Evolver与四层记忆层次结构交互，以读取、使用、写入和维护经验，实现快速测试时进化。在慢速循环中，经过结果校准的记忆归因和特权事后分析将这些能力蒸馏到可部署的策略中。跨多个领域基准测试，OPD-Evolver在性能上超过ReasoningBank等记忆系统高达11.5%，并比Skill0等基于训练的方法高出约5.8%。进一步分析表明，OPD-Evolver内化了高价值经验与记忆管理，使OPD-Evolver-9B能够挑战如Qwen3.5-397B-A17B和Step-3.5-Flash等大型对手，超越了仅依赖记忆增强的智能体，迈向真正合格的智能体进化器。

## Abstract
Memory has become a standard substrate for self-evolving agents, yet retaining experience is not the same as learning how to evolve through it. Existing memory agents can store trajectories, retrieve reflections, or accumulate skills, but often lack the holistic competence to select useful experience, act on it, write reusable knowledge, and maintain a growing repository. We introduce OPD-Evolver, a slow-fast co-evolution framework that cultivates such an agent evolver through on-policy self-distillation. In the fast loop, OPD-Evolver interacts with a four-level memory hierarchy to read, use, write, and maintain experience for rapid test-time evolution. In the slow loop, outcome-calibrated memory attribution and privileged hindsight distill these four abilities into the deployable policy. Across multi-domain benchmarks, OPD-Evolver surpasses memory systems such as ReasoningBank by up to 11.5%, and training-based methods such as Skill0 by ~5.8%. Further analysis shows that OPD-Evolver internalizes high-value experience and memory management, enabling OPD-Evolver-9B to challenge giant counterparts such as Qwen3.5-397B-A17B and Step-3.5-Flash, pointing beyond memory-augmented agents toward genuinely qualified agent evolvers.

## 1. 核心问题与整体含义（研究动机和背景）

现有记忆型智能体虽然可以存储轨迹、检索反思、积累技能，但往往只优化了演化过程的某一环节（如仅检索或仅写作），缺乏 **主动选择有用经验、据此行动、编写可复用知识、并维护不断增长的存储库** 这四种耦合能力。作者将具备这四种能力的智能体定义为“合格智能体进化器”（holistic agent evolver）。研究动机是：如何训练一个智能体获得完整生命周期级的能力，使其能够通过经验持续自我改进，而不仅仅是增强记忆。

## 2. 方法论

### 核心思想
提出 **OPD-Evolver**，一个**快慢双循环协同进化框架**：
- **快循环**：在线与环境交互，操作四级记忆层次（轨迹、提示、技能、工具），进行测试时快速演化（选择→执行→写作→维护）。
- **慢循环**：将快循环中收集的结果校准记忆归属（outcome-calibrated attribution）和特权事后信息（教师能看到学生看不到的校准信号），通过**在线策略自蒸馏**（on-policy self-distillation）将四种能力内化到可部署策略中。

### 关键技术细节
- **四级记忆层次**：轨迹（完整但冗长）、提示（局部警告/启发）、技能（可复用过程）、工具（可执行命令/代码模板）。
- **结果校准记忆归属**：对每个记忆，比较其被选中和不被选中时的任务回报差异，转化为记忆分数 \( V(m) \)（式11,12）。
- **统一事后自蒸馏**：对选择、执行、写作、维护四种决策，学生只能看到公开输入，教师额外看到特权后见（如每个候选记忆的价值、成功轨迹、写作出的记忆的未来价值、维护时的诊断信息）。教师对学生已生成前缀给出密集KL散度监督（式15,16），联合训练四种能力。

### 算法流程（文字描述）

2. 对每个任务：形成查询，从四级记忆层检索候选，由选择器压缩为上下文；策略基于记忆上下文交互获得回报；写出新记忆并更新存储库；每Q=30任务执行一次维护（查找、合并、删除）。
