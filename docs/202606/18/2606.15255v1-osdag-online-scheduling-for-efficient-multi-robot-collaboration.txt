Abstract: Coordinating heterogeneous multi-robot systems (MRS) for complex, long-horizon tasks requires both flex-ible high-level reasoning and efficient low-level scheduling. Existing LLM-based approaches address the reasoning side but introduce two critical bottlenecks: (1) repeated LLM inference during execution, which inflates latency with agent count, and (2) offline, pre-committed scheduling, which forces robots to idle while waiting for sequentially or-dered predecessors even when independent work is available. This paper presents OSDAG, a novel framework that integrates LLM-based task reasoning with Directed Acyclic Graph (DAG) representation and constraint-aware online scheduling. The LLM is invoked once to decompose a natural-language instruction into a dependency-annotated task graph, and a lightweight online scheduler then allocates ready tasks to idle agents in real time. The DAG representation encodes both precedence and resource constraints, ensuring correctness while exposing all available parallelism. Ex-periments across five benchmark scenarios demonstrate that OSDAG achieves 5–15 × faster reasoning time compared to dialogue-based methods, reduces makespan by up to 38% over sequential baselines, and maintains competitive suc-cess rates. Both simulation and real-world experiments on dual-arm manipulation tasks validate the effectiveness and practicality of the proposed approach for efficient multi-robot coordination. 1. INTRODUCTION 

Multi-Robot Systems (MRS) enable heterogeneous agents with distinct capabilities to collaboratively ac-complish complex tasks. Compared to single-robot sys-tems, MRS offer superior task completion time, broader workspace coverage, and flexible utilization of individ-ual expertise, making them essential for search and res-cue, warehouse automation, and industrial manufactur-ing [1]. Traditional MRS coordination relies on manually designed control logic and planning algorithms requiring explicit behavior specification and deep domain knowl-edge [2, 3]. While ensuring accuracy and predictability, these methods lack flexibility—any task or environment change necessitates costly reprogramming. Furthermore, classical approaches cannot interpret natural language, creating a gap between human intent and robot execution. Recent advances in Large Language Models (LLMs) such as Gemini 2.5 Pro [4] and GPT-4 [5] have opened new directions for robot coordination. Systems like SayCan [6] and ChatGPT-Prompts [7] demonstrate that natural language commands can be translated into ex-ecutable robot actions with strong zero-shot general-ization, enabling robots to adapt to novel tasks with-out reprogramming. Several studies have integrated LLMs into MRS using both centralized and decentralized paradigms. Decentralized methods like RoCo [8] assign each robot an LLM agent for dialogue-based coordina-tion, but the conversation history grows linearly with the number of agents, leading to higher latency than single-query approaches. Centralized methods reduce this over-head but typically generate flat action sequences with-out modeling inter-task dependencies. More critically, existing frameworks—including recent works that com-bine LLMs with no-task-dependence execution [9], plan-ning [10], or Linear Programming [11]—employ offline scheduling pipelines in which the entire plan is computed before any robot moves. This forces agents to execute predetermined sequences; even when Robot A finishes early, it remains idle waiting for Robot B due to rigid pre-allocated orderings, wasting valuable execution time. Recent work further combines LLMs with struc-tured planners to improve long-horizon reasoning and robustness. LaMMA-P [12] integrates LLMs with PDDL planning, REBEL [13] augments task alloca-tion with experience-based rules, ELHPlan [14] re-duces token usage through intention-aware collabora-tion, and CoMuRoS [15] introduces event-driven re-planning for heterogeneous teams. Other methods use no-task-dependence execution [9], planning [10], or optimization-based scheduling [11]. TwoStep [16] de-composes goals into PDDL sub-problems solved by sym-bolic planners, while LiP-LLM [11] constructs depen-dency graphs and formulates Linear Programming opti-mization for makespan minimization. Nevertheless, most existing frameworks still follow an offline pipeline in which the complete plan or schedule is computed before execution. As a result, robots may remain idle while wait-ing for pre-allocated task chains, even when independent

2. METHODOLOGY 

OSDAG converts a natural-language instruction into a dependency-aware task graph and executes the graph with a lightweight online scheduler. Given an instruc-tion I, the framework first grounds the task in the current multi-robot workspace, asks the LLM once to generate agent-assigned subtasks with dependencies, validates the generated structure, and then dispatches ready tasks to idle agents while preserving precedence and feasibility constraints. 2.3 Task Graph Formalization 

After validation, the structured task list is converted into a Directed Acyclic Graph G = ( V, E ). Each task 

Ti corresponds to a vertex vi ∈ V , and each prerequisite relation defines a directed edge: 

V = {vi | Ti ∈ T } , E = {(vj , v i) | j ∈ D i}. (13) Thus, an edge (vj , v i) indicates that task Tj must be com-pleted before task Ti can start. The graph G can also be represented by an adjacency matrix A ∈ { 0, 1}m×m:

Aji = I[( vj, vi) ∈ E]. (14) A topological sort is applied to verify that G is acyclic. If a cycle is detected, the task graph is rejected be-cause the dependencies are logically inconsistent. This DAG representation allows OSDAG to explicitly encode both sequential dependencies, such as pick →place ,and inter-agent dependencies, such as object handoff or shared-resource constraints. Unlike flat action sequences, the DAG exposes independent task branches that can be executed in parallel. This structure is essential for re-ducing idle time in multi-agent execution because a task can be dispatched as soon as all its predecessors are com-pleted, without waiting for unrelated tasks.

3. EXPERIMENTAL RESULTS 

3.1 Experimental Setup 

We evaluate our system in both simulation and real-world environments. Simulation experiments are con-ducted using the PyBullet physics engine with custom tabletop manipulation environments. We employ Gem-ini Flash as the LLM backbone to validate planning effi-ciency and reasoning performance. To comprehensively assess performance, we design five evaluation scenarios covering different task complexities and spatial configu-rations. Task 1 - Place cubes into matching bowls with 2 agents; Task 2 - Place cubes into matching bowls with 3 agents to evaluate parallel execution capability. Task 3 

- Clean table to test implicit dependency reasoning ( e.g. ,opening a drawer before placing items); Task 4 - Sort objects in order: apple → banana → cup to assess se-quential dependency handling. Task 5 - Place cubes into matching bowls to evaluate inter-agent collaboration with object handoffs. 

Evaluation Metrics . We evaluate performance using four complementary metrics: Planning Success Rate (SR, 

%), Reasoning Time (RT, seconds), Makespan (MS, sec-onds), and Success-weighted Step Efficiency (SSE) [11]. 

Baselines . We compare against three LLM-based multi-agent planning baselines: RoCo [8], a dialogue-based decentralized framework; TwoStep [16], a hierar-chical LLM-PDDL planner; and ChatGPT-Prompts [7], a direct flat-sequence generation method. All methods use the same task instructions and environment states. 

3.2 Simulation Results 

Table 1 reports performance across all five tasks, aver-aged over 10 trials. OSDAG achieves substantially lower reasoning time than dialogue-based methods and reduces makespan in dependency-heavy tasks, while maintaining competitive success rates. 

Reasoning Time. OSDAG requires 9.1–14.0 s per task, achieving a 5–15 × speedup over RoCo across all scenarios. RoCo’s dialogue-based coordination is the pri-mary bottleneck: each agent appends to a shared con-versation history, so latency scales with the number of robots— Task 2 with three agents incurs 206.3 s for RoCo versus 14.0 s for OSDAG. ChatGPT-Prompts achieves the shortest RT (8–10 s) by generating a flat sequence in a single call, but this comes at the cost of dependency modeling: its SSE drops to 0.36–0.51 and its makespan is 33–48 s, substantially higher than OSDAG’s 19–26 s across the same tasks. OSDAG matches or approaches ChatGPT-Prompts in RT while providing full dependency awareness, since the LLM is also invoked exactly once but is asked to produce a richer, structured output. 

Success Rate. Our system achieves 80% SR on Task 3, a complex scenario involving implicit dependen-cies requiring LLM reasoning (e.g., opening a drawer be-fore placing items). By embedding current object states 

σ into the prompt, OSDAG surfaces this hidden precon-dition in most trials, outperforming both RoCo (0%) and TwoStep (0%), which lack the object-state context to rea-son about it at all. 

Makespan. Our system reduces makespan by 33% in Task 4 (15.2 vs 22.7) and 38% in Task 5 (26.0s vs 41.6s) compared to baselines. 3.3 Ablation Study 

To isolate each component’s contribution, we con-duct ablation studies with the following variants: w/o Em-bedding - Removes object reachability, action descrip-tions, and status embeddings from the LLM prompt; w/o Graph : Generates sub-tasks without dependency anno-tations, resulting in a flat sequence; w/o Online Sched. -Replaces online scheduling with sequential per-agent ex-ecution. Table 2 presents the ablation results across all tasks. 

Effect of Embedding. Removing object and action embeddings increases reasoning time (16.5s vs 9.1s in Task 1) and reduces success rate (70% vs 90% in Task 2). Without grounded context, the LLM occasionally hallu-cinates invalid actions or incorrect agent assignments. 

Effect of Dependency Graph. Without dependency annotations, all tasks fail (SR = 0 ) due to resource con-flicts and lack of coordination. Agents attempt concurrent manipulation of the same object or violate precedence constraints, causing execution failures. 

Effect of Online Scheduling. Disabling online scheduling increases makespan significantly despite cor-rect plans—Task 4 rises from 15.2s to 22.7s (+49%), and Task 5 from 26.0s to 41.6s (+60%). This confirms that dynamic task allocation is essential for efficient parallel execution. 

3.4 Real-World Experiments 

To validate practical applicability beyond simulation, we deploy our framework in a human-robot collaboration (HRC) scenario in which the human follows text-based instructions generated by our scheduler. The robotic agent consists of a UF850 manipulator equipped with an Intel RealSense D435 RGB-D camera and a parallel grip-per. The human agent receives task instructions via a text display interface, which presents the scheduled actions in real time ( e.g. , “Human: Task 5 – pick(green cube)”). We employ YOLOv11-seg for object detection and in-stance segmentation, followed by depth alignment to ob-tain 3D object poses. The perception module outputs object categories, positions, and dimensions, which are embedded into the LLM prompt for task decomposition. We evaluate a collaborative cube-sorting task where the robot and human work in parallel to place colored cubes into matching bowls. Fig. 5(c) shows the generated task graph, and Fig. 5(d) illustrates execution snapshots at key timestamps. These results demonstrate that LLM-DAG effectively coordinates heterogeneous agents with different execution interfaces, validating its applicability to real-world HRC scenarios. 

4. CONCLUSION 

We presented OSDAG, a novel framework for multi-robot coordination that integrates the high-level reason-ing capabilities of Large Language Models with DAG-based task representation and real-time online schedul-ing. By embedding agent reachability, manipulation capabilities, and object states into structured prompts, the LLM generates dependency-aware subtasks that are dynamically assigned to available robots. This design strictly preserves task dependencies while enabling paral-lel execution of independent subtasks, achieving 5–15 ×

faster reasoning time than dialog-based methods and up to a 38% reduction in makespan compared to offline scheduling approaches. However, our framework relies on the LLM to generate dependency edges, and reason-ing errors may increase as environmental complexity and agent count increase. The current task selection policy may not yield globally optimal schedules.
