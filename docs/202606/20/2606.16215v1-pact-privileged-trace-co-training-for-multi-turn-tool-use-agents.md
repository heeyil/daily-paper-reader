## 摘要
多轮工具使用智能体必须在多个交互轮次中进行推理、调用工具并适应观察结果。这类智能体的后训练具有挑战性，因为强化学习尽管与仅提示推理设置相匹配，但常受稀疏奖励和弱信用分配问题的困扰；而基于专家轨迹的监督微调虽提供了密集的过程监督，却可能将模型过度约束在固定轨迹上。为解决这一问题，我们提出PACT——一种用于多轮工具使用智能体的特权痕迹协同训练框架。核心思想是仅将专家轨迹作为训练时的优化信号，而非 rollout 时的提示。PACT 保持 rollout 生成仅基于提示，然后通过两种互补信号利用专家轨迹指导优化：一种是基于痕迹条件的 RL 替代目标，在专家轨迹上下文中评估仅提示的 rollout；另一种是成分感知的 SFT 损失，以退火强度监督推理前缀和工具调用。为减少对仅训练时的痕迹上下文的过度依赖，PACT 进一步引入了仅提示锚定。我们还提供了一个潜在痕迹视角，将两个基于痕迹的目标联系起来，并解释专家轨迹如何在不用于 rollout 生成的情况下指导优化。在 FTRL、BFCL 和 ToolHop 上的实验表明，PACT 在强 SFT 和 RL 基线基础上持续改进，突显了特权痕迹协同训练在多轮工具使用学习中的价值。

## Abstract
Multi-turn tool-use agents must reason, call tools, and adapt to observations across several interaction turns. Post-training such agents is challenging, as reinforcement learning often suffers from sparse rewards and weak credit assignment despite matching the prompt-only inference setting, while supervised fine-tuning on expert traces provides dense process supervision but can over-constrain the model to fixed trajectories. To tackle this, we propose PACT, a Privileged trAce Co-Training framework for multi-turn tool-use agents. The key idea is to use expert traces only as training-time optimization signals rather than rollout-time hints. PACT keeps rollout generation prompt-only, then uses expert traces to guide optimization through two complementary signals: a trace-conditioned RL surrogate that evaluates prompt-only rollouts under expert-trace context, and a component-aware SFT loss that supervises reasoning prefixes and tool-calls with annealed strength. To reduce over-reliance on the training-only trace context, PACT further introduces a prompt-only anchoring. We also provide a latent-trace view that connects the two trace-based objectives and explains how expert traces can guide optimization without being used during rollout generation. Experiments on FTRL, BFCL, and ToolHop show that PACT consistently improves over strong SFT- and RL-based baselines, highlighting the value of privileged trace co-training for multi-turn tool-use learning.

# 论文总结：PACT — Privileged Trace Co-Training for Multi-Turn Tool-Use Agents

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：多轮工具使用（multi-turn tool-use）智能体需要交替进行推理、调用工具、接收环境观察并做出决策。后训练（post-training）此类智能体面临两难：
  - **强化学习（RL）** 虽匹配推理时的仅提示（prompt-only）条件，但奖励稀疏且信用分配弱，难以有效学习长程决策序列。
  - **监督微调（SFT）** 在专家轨迹上提供密集过程监督，但容易将模型过度约束至固定轨迹，忽略其他可行解法，且不直接反映中间决策对最终成功的影响。
- **核心目标**：在不改变推理时仅提示生成的情况下，充分利用专家轨迹指导优化，解决上述矛盾。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **PACT（Privileged trAce Co-Training）**：将专家轨迹仅用作**训练时的优化信号**，不用于 rollout 生成。Rollout 保持仅基于用户提示（prompt-only），专家轨迹仅在优化阶段作为“特权上下文”参与计算。

### 关键技术细节
1. **Prompt-only Rollout 生成**  
   - 从旧策略 π_old 采样 K 条完整多轮轨迹：τ_{i,k} ~ π_old(·|x_i)，完全不使用专家轨迹 h*_i。
2. **Trace-conditioned RL 替代目标**  
   - 在计算策略比率时，将上下文替换为特权轨迹上下文：c_h = [x_i; h*_i]。  
   - 对采样 rollout 中的每个 token 计算 trace-conditioned 策略比率：r^h_{i,k,n}(θ) = π_θ(τ_{i,k,n} | c_h, τ_{i,k,<n}) / π_old(...)。  
   - 结合来自 prompt-only rollout 的优势估计 A_{i,k,n}（采用 MatchTIR 的硬信用分配），形成 clipped RL 损失 L^h_RL(i)。  
   - 保持奖励和优势来自 prompt-only rollout，只改变概率计算上下文。
3. **Component-aware SFT 损失**  
   - 不对完整专家轨迹进行模仿，仅监督**模型可控制组件**：推理前缀（reasoning prefixes）和完整工具调用（tool-calls）。  
   - **推理前缀退火**：基于语义段（task_restate, tool_scan 等）构建多个前缀级别 L0~L4，训练时从 L4 逐步退火到 L0，避免截断语义单元。  
   - **工具调用退火**：工具调用保持完整，但模仿权重 λ_a(s) 从 λ_start^a 线性退火到 λ_end^a，早期强调格式正确性，后期允许探索。  
   - 最终 SFT 损失 L_SFT(i; s) 是带掩码的负对数似然，掩码 m_{i,n}(s) 综合推理前缀选择和工具调用权重。
4. **Prompt-only Anchoring**  
   - 以概率 q 将部分提示路由到标准 prompt-only RL 损失 L^x_RL(i)（不使用任何 trace），以减轻对特权上下文的过度依赖，保持推理时泛化能力。  
   - 最终每提示目标：L_i = (1-z_i)L^trace_i + z_i L^x_RL(i)，z_i ~ Bernoulli(q)。

### 潜在轨迹视角（Latent-trace View）
- 将 prompt-only rollout 似然分解为：log π_θ(τ|x) ≥ log p_θ(h*|x) + log π_θ(τ|x,h*)。  
  - 第一项对应 trace 似然，由 component-aware SFT 近似（学习轨迹结构）。  
  - 第二项对应特权 rollout 似然，由 trace-conditioned RL 优化（在专家上下文中评分 rollout）。  
- 该视角解释了为何两个目标互补。

## 6. 主要结论与发现

- PACT 在两种模型上均**一致优于所有基线**。以 Qwen3-4B 为例，平均分从最佳基线 MatchTIR 的 33.07 提升至 36.90。
- **Trace-conditioned RL 单独即可提升**：相比 prompt-only RL，Solve-P 从 30.60 升至 31.42（表 2）。
- **Component-aware SFT 与 trace-conditioned RL 互补**：两者结合达到最佳性能（Solve-R 40.45 vs 单独 L^h_RL 35.29）。
- **组件消融显示**：工具响应（观测）贡献最大，推理前缀和工具调用均不可缺少。
- **语义段退火优于词级退火**，且工具调用退火比固定权重好。
- **prompt-only 锚定（q=0.4）提供最佳权衡**：无锚定（q=0）时 ToolHop 下降，锚定太大则 FTRL 下降。

## 7. 优点

- **核心创新**：巧妙地将专家轨迹作为“特权上下文”仅用于优化，不改变 rollout 分布，从而避免推理-训练不匹配。
- **组件感知 SFT 设计**：区分推理前缀和工具调用，分别采用语义段退火和损失尺度退火，既提供早期过程指导又避免过度模仿。
- **理论支撑**：提供潜变量轨迹分解，为两个损失项提供了统一的概率解释。
- **实验坚实**：在多个基准上持续超越强基线，消融实验系统全面，设计选择被逐一验证。
