## 摘要
大型语言模型（LLM）智能体通过将语言模型与外部工具和环境集成，越来越多地自动化复杂任务。然而，它们的自主性带来显著的安全风险：智能体可能执行破坏性命令、泄露敏感数据或违反领域约束。现有安全方法面临根本性权衡：手工规则可解释但脆弱，过于保守的规则阻碍安全操作（高误报率），而宽松的规则则遗漏不安全行为（高漏报率）。神经分类器缺乏安全关键部署所需的可解释性。我们提出AutoSpec，一个通过反例引导的归纳综合（CEGIS）自动从用户安全/不安全标注中演化部署的专家设计安全规则的框架，该框架由归纳逻辑编程（ILP）引导。从专家规则和带标注的轨迹流开始，AutoSpec迭代评估规则，挖掘误报和漏报反例，使用ILP学习哪些谓词区分它们，生成候选规则编辑，并验证候选规则以选择最佳修订。关键见解是ILP高效识别出在漏报中频繁出现但误报中很少出现（或反之）的谓词，极大地剪枝了规则编辑的指数级搜索空间。该过程持续直至收敛，产生平衡精确率和召回率的可解释规则。我们在涵盖代码执行和具身智能体领域的291条执行轨迹上评估AutoSpec。AutoSpec在两个领域将规则F1分别提升至0.98和0.93，在保持高召回率的同时实现了高达94%的误报率降低，并在4-5次迭代内收敛。与启发式CEGIS相比，ILP引导的方法实现了高达4.8倍的F1提升。学习到的规则人类可读、可审计，并能泛化到未见场景。

## Abstract
Large language model (LLM) agents increasingly automate complex tasks by integrating language models with external tools and environments. However, their autonomy poses significant safety risks: agents may execute destructive commands, leak sensitive data, or violate domain constraints. Existing safety approaches face a fundamental tradeoff: hand-crafted rules are interpretable but brittle, with overly conservative rules blocking safe operations (high false positives) while permissive rules miss unsafe behaviors (high false negatives). Neural classifiers lack the interpretability required for safety-critical deployments.   We present AutoSpec, a framework that automatically evolves deployed expert-designed safety rules from user safe/unsafe annotations through counterexample-guided inductive synthesis (CEGIS) guided by inductive logic programming (ILP). Starting from the expert rules and a stream of annotated traces, AutoSpec iteratively evaluates rules, mines false-positive and false-negative counterexamples, uses ILP to learn which predicates discriminate them, generates candidate rule edits, and verifies candidates to select the best revision. The key insight is that ILP efficiently identifies predicates that appear frequently in false negatives but rarely in false positives (or vice versa), dramatically pruning the exponential search space of rule edits. This continues until convergence, producing interpretable rules that balance precision and recall.   We evaluate AutoSpec on 291 execution traces spanning code execution and embodied agent domains. AutoSpec raises rule F1 to 0.98 and 0.93 across the two domains, achieving up to 94% false positive reduction while maintaining high recall, and converges within 4-5 iterations. The ILP-guided approach achieves up to 4.8x higher F1 than heuristic CEGIS. The learned rules are human-readable, auditable, and generalize to unseen scenarios.

## 1. 核心问题与研究动机

LLM智能体通过调用外部工具、执行代码、操作物理环境等自动化复杂任务，但其自主性带来了严重安全风险（如执行破坏性命令、泄露敏感数据、违反领域约束）。现有安全方法面临两难：

- **手工规则**：可解释但脆弱，过于保守导致高误报（阻碍正常操作），过于宽松导致高漏报（错过不安全行为），且静态规则无法适应智能体模型、工具、提示的持续演变。
- **神经分类器**：可自适应但黑箱不可解释，无法用于安全关键部署，且不能用积累的用户标注来自动改进规则。

核心问题：如何将专家设计的初始安全规则，从用户对执行轨迹的安全/不安全标注中自动演化，保持可解释性同时平衡精确率与召回率？

## 2. 方法论：核心思想与技术细节

**核心思想**：将安全规则维护问题形式化为反例引导的归纳综合（CEGIS），由归纳逻辑编程（ILP）指导搜索。用户标注作为反例（漏报/误报），ILP高效识别区分反例的关键谓词，大幅剪枝指数级规则编辑空间。

**技术流程**（4阶段迭代循环，算法1）：
1. **反例挖掘**：在标注轨迹集上评估当前规则，收集误报（FP）和漏报（FN）作为反例。
2. **ILP谓词学习**：将FN作为正例、FP作为负例，调用ILASP求解器，学习最简谓词组合（析取合取式）以区分反例。输出排名靠前的候选谓词及编辑方向（收紧/放宽）。
3. **候选生成与选择**：基于ILP建议，使用4种编辑算子（AddConjunct、AddException、AddDisjunct、Relax）生成候选规则集。采用单步编辑、混合修复链、失败驱动分支三种策略。按字典序评分（F1>精确率>未解决反例数>规则数）选择最佳候选。
4. **验证与收敛**：若候选得分提升则接受，否则终止。循环最多K次，或达到目标阈值。引入软新颖性压力避免重复建议。

**谓词库**：每个领域提供16-24个布尔谓词（如`destructive_os_inst`、`near_danger_zone`）。规则以事件级逻辑公式定义，轨迹级安全分类为是否存在事件触发任何规则。

**关键创新**：ILP引导CEGIS，相比启发式搜索，避免了均匀探索指数空间，聚焦于高区分性谓词，实现4-5次迭代收敛。

## 8. 不足与局限

- **领域覆盖有限**：仅测试2个领域（代码执行、具身）。Web代理、数据库代理、多智能体协作等场景未验证。
- **谓词库依赖**：效果依赖于预定义的谓词库（16-24个谓词）。若领域谓词不充分（如缺失重要安全属性），ILP无法学习有效规则。论文指出未解决的反例可提示需要新谓词，但实际中需专家扩展库。
- **标签获取**：依赖用户手动标注轨迹为安全/不安全，实际部署中标注可能稀疏、有噪声或不一致。
- **时态与状态属性**：当前规则仅基于单事件谓词，未原生支持跨事件时序逻辑（如“必须先登录才能访问文件”）。论文提及可在谓词内部编码历史，但更丰富的LTL/MTL扩展留作未来工作。
- **全局最优性无保障**：由于问题NP-难，算法仅保证局部最优（单步编辑无法提升），不保证全局最优。但在实践中取得显著改进。
- **实验规模较小**：总轨迹数291条，领域内最多191条。更大规模、更多样化的评估可能揭示更多问题。
- **LLM分类器基线**：使用的LLM分类器（B2）是少样本提示，而非专门微调的分类器，可能未充分代表神经方法潜力。但论文强调对比点是“用相同标注信号”而非最优神经基线。
- **用户研究样本**：仅10名从业者，未公开人口统计细节，可能存在抽样偏差。
