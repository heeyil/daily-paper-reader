## 摘要
大型语言模型（LLM）智能体的长期记忆系统通常试图"记住一切"，统一提取记忆以保留尽可能多的事实。然而在生产环境中，推理成本和有限的上下文预算使得这一做法难以为继：除了将原始对话整合为记忆外，智能体还必须施加"写入控制"，高效地只保留每个用户真正关心的信息。否则，长期个性化交互会出现"记忆膨胀"，无关的琐事挤占有用信息，并逐步侵蚀问答准确性。我们认为，什么值得记忆取决于角色，并提出AdaMem（自适应记忆），一种从反馈中学习为每个用户记住什么的方法。AdaMem维护一个结构化的、角色特定的记忆策略，并通过轻量级的、带有失败回滚的补丁式自我反思步骤，从每周的问答反馈中优化该策略。为了研究这一设置，我们构建了AdaMem-Bench基准，模拟数周的交互并逐周进行问答。在两种提取模型和两种反馈模式下，与统一的Mem0基线相比，AdaMem将问答准确性提高了最多+9.0%，同时将记忆体积缩小了9%。

## Abstract
Long-term memory systems for Large Language Model (LLM) agents typically try to \emph{remember everything}, extracting memories uniformly to retain as many facts as possible. In production, however, inference cost and finite context budgets make this untenable: beyond consolidating raw dialogue into memory, an agent must exert \emph{write control}, efficiently keeping only the information each user actually cares about. Otherwise, long-horizon personalized interactions suffer \emph{memory bloat}, where irrelevant trivia crowds out useful information and steadily erodes question-answering (QA) accuracy. We argue that what is worth remembering is role-dependent, and propose \textbf{AdaMem} (Adaptive Memory), a method that \emph{learns what to remember} for each user from feedback. AdaMem maintains a structured, role-specific Memory Policy and refines it from weekly QA feedback through a lightweight, patch-style self-reflection step with failure rollback. To study this setting, we build \textbf{AdaMem-Bench}, a benchmark that simulates weeks of interaction with week-by-week QA. Across two extraction models and two feedback modes, AdaMem improves QA accuracy by up to \textbf{+9.0\%} over the uniform Mem0 baseline while shrinking memory volume by \textbf{9\%}.

## 1. 论文的核心问题与整体含义（研究动机和背景）

大型语言模型（LLM）智能体在长期个性化交互场景中需要具备长期记忆能力。现有记忆系统（如Mem0、MemoryBank）通常采用“记住一切”的均匀提取策略，试图将所有事实性信息存入记忆库。然而在实际生产部署中，存在两个硬约束：推理成本和有限的上下文窗口。随着交互时间延长，这种无差别记忆会导致“记忆膨胀”——不相关的琐碎信息（如日常问候、微小日程调整）挤占了有用信息的存储空间，并在检索时引入噪声，逐步降低问答准确性。作者认为，什么信息值得记忆是角色依赖的（role-dependent），例如与经理对话应关注业务决策，而与健身伙伴聊天应关注锻炼计划。因此论文提出：智能体不应记住一切，而应**学习为每个用户记住什么**，即通过偏好引导的写入控制，实现个性化、紧凑的记忆管理。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 核心思想
AdaMem（Adaptive Memory）构建一个结构化的、角色特定的**记忆策略（Memory Policy）**，并通过每周问答反馈的闭环循环来不断优化该策略。记忆策略是一组自然语言规则，指导提取器优先保留符合用户偏好的信息，过滤无关信息。

### 关键技术细节
- **记忆策略结构**：一个JSON对象，将每个角色（character）映射到自然语言提取偏好（例如：“仅提取最终商业决策和截止日期，忽略日常闲聊”）。
- **补丁式自我反思（Patch-Style Self-Reflection）**：每周结束后，智能体对本周错误回答的QA对进行诊断，识别两种失败类型：
  - 召回失败（Recall Failure）：对话包含信息但提取器遗漏（策略过于严格）。
  - 精度失败（Precision Failure）：检索到无关记忆误导回答（策略过于宽松）。
- **策略更新**：基于诊断生成一个“补丁”（patch）来调整策略。并采用**失败回滚（Failure Rollback）**机制：如果更新后的策略在历史QA验证上性能下降，则拒绝更新并回滚。
- **时间锚定（Time Anchoring）**：在提取时注入会话的绝对日期，要求LLM将所有相对时间表达（如“明天”）解析为绝对日期后再保存，避免时间歧义。

### 算法流程（文字描述）
1. **初始化**：创建初始记忆策略P1，记忆数据库D为空。
2. **每周循环**（共T周）：
   - 每日提取：解析该日对话中的相对时间，根据当前策略Pt提取偏好记忆，存入D。
   - 每周评估：回答本周的QA问题，获取反馈（正确/错误）。
   - 如果存在错误回答：进行反思诊断，生成补丁，更新策略得到P‘_{t+1}，并在历史QA上验证，若退化则回滚到Pt，否则采用新策略。
   - 写回：用更新后的策略重新生成答案并写回。

## 6. 论文的主要结论与发现

- **整体性能**：在所有设置下，AdaMem一致优于Mem0。在DeepSeek-V4-Flash+显式反馈下，准确率绝对提升+9.0%（85.2% vs 76.2%），记忆体积减少9.1%（371 vs 430）。
- **信息类别增益**：增益集中在“软偏好”类别（情感态度+15.9%、日程时间+9.9%、决定结论+9.0%），而事实/数字类别提升微小（+1.1%），说明AdaMem针对性保留主观、上下文敏感信息。
- **检索质量**：随着周数增加，Mem0召回率下降，AdaMem保持更高且稳定的Recall@5和MRR，验证了缓解记忆膨胀的效果。
- **策略学习瓶颈**：模型遵循策略的能力很强（策略遵循率>80%），但**策略推断**是主要瓶颈：在隐式反馈下策略对齐度停滞在53%-63%，而显式反馈可达88.3%。说明当前LLM难以从正确性信号中反向工程出用户偏好。

## 7. 优点

- **问题定位精准**：将长期记忆问题从“容量扩展”重新定义为“写入控制”，贴合工业部署中的成本约束。
- **方法轻量且实用**：补丁式自我反思和失败回滚机制无需训练，直接插入现有RAG记忆栈。
- **构建专用基准**：AdaMem-Bench提供黄金记忆标注和偏好对齐QA，填补了现有长期记忆基准中缺乏角色依赖反馈闭环的空白。
- **多维度评估**：不仅报告准确率，还引入提取F1、内存效率比（MER）、类别粒度分析、时间演化分析，充分揭示方法优势。
- **清晰揭示瓶颈**：通过解耦策略遵循与策略推断，指出弱反馈下学习偏好的困难，为后续研究指明方向。
