## 摘要
多轮工具使用智能体必须在跟踪对话状态和策略约束的同时协调长程工具序列。现有方法通常将推理时的编排与参数级学习分离，导致工具选择结构薄弱，且偏好更新容易受到训练与部署提示不匹配的影响。为了实现基准内的自我改进，ToolGraph结合了从模式派生的拓扑结构、从成功轨迹中估计的转移权重，以及针对写入前提和重复搜索循环的历史感知控制。随后，我们通过基于状态的匹配和基于前缀的对齐来定位分歧点，再经行动正确性注释过滤，构建了161个偏好对，并在推理时使用的相同ToolGraph上下文中训练DPO。在375个tau2-bench任务中，ToolGraph将加权平均奖励从0.304提升至0.338（相对提升11.2%），而ToolGraph+DPO达到0.355（相对于基线提升16.8%），其中DPO的增益主要集中在航空和零售领域。细粒度的诊断进一步表明，大约一半的电信轨迹在行动执行前耗尽了步骤预算，并且在我们评估的16种DPO配置中，选定奖励的正向性是最有用的检查点信号。

## Abstract
Multi-turn tool-using agents must coordinate long-horizon tool sequences while tracking dialogue state and policy constraints. Existing approaches often separate inference-time orchestration from parameter-level learning, leaving tool selection weakly structured and preference updates vulnerable to train--deployment prompt mismatch. For within-benchmark self-improvement, ToolGraph combines schema-derived topology, transition weights estimated from successful rollouts, and history-aware controls for write prerequisites and repeated-search loops. We then construct 161 preference pairs by locating divergence points via state-based matching and prefix-based alignment, filtered through action-correctness annotations, and train DPO under the same ToolGraph context used at inference. Across 375 tau2-bench tasks, ToolGraph raises the weighted average reward from 0.304 to 0.338 (+11.2% relative), while ToolGraph+DPO reaches 0.355 (+16.8% over the baseline), with the DPO gain concentrated in airline and retail. Fine-grained diagnostics further show that roughly half of telecom trajectories exhaust the step budget before action execution and that chosen reward positivity is the most useful checkpoint signal across our 16 evaluated DPO configurations.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **ToolGraph（工具图）**：将每个领域的工具表示为带权有向图。图拓扑由工具模式和共享参数构建，边权重从智能体自身的成功轨迹中估计。在推理时，通过Beam Scoring + Best-of-N选择工具调用序列。
- **分歧点偏好学习（Divergence-Point Preference Learning）**：将成功与失败轨迹配对，定位第一个行动分歧的“分歧点”，构造偏好对（选择成功行动，拒绝失败行动），在**与推理时相同的ToolGraph上下文中训练DPO**，避免提示不匹配。

### 关键技术细节
1. **ToolGraph构造**：
   - **节点**：每个工具的名称、描述、参数、必需参数集。
   - **有向边**：基于共享参数名构建（若两个工具共享同一参数名，则双向加边）。按执行阶段分类（identify/gather/act），并添加act→gather的后向边（当共享实体参数时），允许行动后回退收集信息。
   - **边权重**：从532条成功轨迹（reward=1.0）中计算两个统计量：工具成功率（轨迹中包含该工具且最终成功的比例）和转移概率$P_{\text{TG}}(\text{tgt}\mid\text{src})$（该工具立即跟随前一工具的频率）。最终边得分 = $P_{\text{TG}} \times \max(\text{success\_rate}(\text{tgt}), 0.01)$。每个任务贡献相等权重。
   - **规划提示（Planning Prompt）**：注入系统提示，包含执行阶段分组、并行集、Critical Rules（人工编码的领域控制规则，如“取消/修改前必须先调用get_reservation_details”）。
2. **Beam Scoring与Best-of-N选择**：
   - 每次决策生成最多3个候选工具调用序列（通过温度退火采样），去重。
   - 每个候选初始得分1.0（纯文本回答0.5），应用5种调整：重复惩罚、边奖励、前提门控、回溯奖励/惩罚、电信循环检测。选择最高分候选，平局时选工具调用数少的。
3. **DPO自进化**：
   - **失败归因**：将低奖励轨迹分类为5种症状（冗余读取、进度停滞、工具执行错误、步数耗尽、策略错误），每个任务最多保留2个失败案例。
   - **分歧点定位**：
     - Stage 1（状态匹配）：提取失败轨迹在决策点的状态签名（实体ID、已知字段、上一个工具、行动类型、待完成目标类型），在同任务的成功轨迹中搜索兼容状态（≥40% Jaccard重叠且行动不同）。
     - Stage 2（前缀对齐）：若状态匹配失败，则使用LCP₅₀（最长公共前缀，比较消息角色和前50个字符）对齐轨迹，找到首个助手消息不同的位置。
   - **过滤**：利用tau2-bench的action-correctness注释，排除“选择路径在分歧前已有错误”或“拒绝的行动被标记为正确”的配对。
   - **训练**：4-bit NF4 QLoRA（rank=16, α=32, dropout=0.05），目标Qwen注意力层和MLP层。损失函数为标准DPO损失：  
     $\mathcal{L}_{\text{DPO}} = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \left( \Delta_\theta(x, y_w) - \Delta_\theta(x, y_l) \right) \right) \right]$
   - **16种配置**：β∈{0.05,0.10,0.15,0.20}，学习率∈{2,5,8}×10⁻⁶，训练步数∈{60,100,150,200}（非全网格）。最终部署R2#4：β=0.20，lr=5e-6，100步。
4. **上下文一致性**：DPO训练数据中注入与推理完全相同的ToolGraph规划提示（包括工具描述和图提示），避免版本1中的提示不匹配。

## 6. 主要结论与发现

1. **性能提升**：
   - ToolGraph将加权平均奖励从0.304提升至0.338（+11.2%相对）。
   - ToolGraph+DPO进一步达到0.355（+16.8%相对于基线）。
   - 增益集中在航空（0.600→0.667→0.713）和零售（0.219→0.289→0.319）。
   - 电信变化小（0.465-0.480），银行仍然困难（0.058-0.072）。
2. **瓶颈诊断**：
   - 电信的主要瓶颈是**行动前步数耗尽**（MAX_STEPS）：约50%的轨迹在行动执行前终止。一旦进入行动阶段，action all-OK达95%+。DB pass率（29%）有误导性。
   - 银行领域9B模型能力显著不足（action all-OK仅13%，DB pass 15%）。
3. **DPO训练关键信号**：
   - **chosen reward的正向性**是最有用的检查点选择指标：配置中chosen reward保持正的（如R2#4）能提升在线奖励；chosen reward为负的配置（如R2#6）即使准确率高、margin大，也会出现灾难性遗忘。
