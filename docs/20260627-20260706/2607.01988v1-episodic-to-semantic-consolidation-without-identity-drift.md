## 摘要
长期运行的适应性智能体在知识巩固与信息完整性之间面临结构性张力。记忆巩固通常被视为一种改变智能体的操作：模型被微调、提示被重写、策略被蒸馏、或者反思被附加到控制未来行为的上下文中。在受监管的自主部署中，这是一种责任，因为智能体在绑定特定加密认证身份的承诺和审计合约下运作。我们提议将巩固视为不是对规划器或身份清单的变异，而是作为情景记忆上的确定性函数 f: M^ep -> M^sem，其输出是一个单独可寻址的语义知识层；身份哈希不会读取 M^sem，因此巩固在更新知识的同时不会改变智能体认证的身份。我们给出了智能体表示的形式化描述，通过对清单哈希输入集的结构引理证明了身份不变性，指定了一种确定性聚合算法，其输出是可审计的数据库行，具有明确的置信度和支持事件来源，并通过合成实验验证了该构造，展示了逐字段正确性、跨巩固传递的字节相等身份，以及相对于校准的贝叶斯收缩基线，平均减少了79.82%的无成效规划器尝试（10个种子下95% BCa置信区间[78.02%, 81.49%]）。该构造是一种自主智能体的知识更新规范，其中教训积累为可查询的事实，而智能体的认证身份在其整个运行生命周期内保持字节相等，并以一个具体的服务智能体作为运行案例研究。

## Abstract
Long-running adaptive intelligent agents face a structural tension between knowledge consolidation and information integrity. Memory consolidation is conventionally treated as an agent-changing operation: a model is fine-tuned, a prompt rewritten, a policy distilled, or a reflection appended to the context that governs future behaviour. In regulated autonomic deployment this is a liability because the agent operates under commitments and audit contracts that bind to a specific, cryptographically certified identity. We propose to treat consolidation not as a mutation of the planner or the identity manifest, but as a deterministic function f: M^ep -> M^sem over episodic memory whose output is a separately addressable semantic knowledge layer; the identity hash does not read M^sem, so consolidation updates knowledge without changing the agent's certified identity. We give a formal account of the agent representation, prove identity invariance through a structural lemma on the manifest's hash-input set, specify a deterministic aggregation algorithm whose outputs are auditable database rows with explicit confidence and supporting-event provenance, and validate the construction with synthetic experiments demonstrating per-field correctness, byte-equal identity across consolidation passes, and a mean 79.82% reduction in unproductive planner attempts (95% BCa CI [78.02%, 81.49%] across 10 seeds) against a calibrated Bayesian-shrunk baseline. The construction is a knowledge-update discipline for autonomic agents in which lessons accumulate as queryable facts while the agent's certified identity remains byte-equal across its operational lifetime, with an embodied service agent as the running case study.

### 1. 论文的核心问题与整体含义（研究动机和背景）

长期运行的自主智能体（如医院、工厂、仓库中的具身服务机器人）面临一个根本性矛盾：  
- **适应性需求**：智能体必须从经验中学习，更新置信度、风险参数等，以提高任务表现。  
- **身份完整性需求**：在受监管环境下，智能体绑定一个加密认证的身份证书（基于清单哈希），任何知识更新若改变该证书，都会触发重新认证事件，这在合规和审计上不可持续。  

传统记忆巩固方法（微调、反思、提示重写、策略蒸馏）都直接修改智能体本身的参数或上下文，导致身份漂移。本文旨在解决“如何在保持身份证书字节不变的前提下，让智能体从情景经验中积累可操作的知识”。

### 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：  
将记忆巩固视为一个**确定性函数** f: M^ep → M^sem，从情景事件日志输出到单独可寻址的语义知识层。身份哈希**不读取**语义层，因此更新语义知识不会改变身份证书。

**关键技术细节**：

1. **形式化智能体状态**：  
   元组 (M, M^ep, M^sem, π) 分别表示身份清单、情景日志、语义记忆、规划器。身份证书为 SHA256(M)。

2. **结构身份不变性引理（Lemma 1）**：  
   通过输入集检查证明：语义层 M^sem 不在清单 M 的哈希输入集中，因此任何只写语义层的 f 都不会改变身份哈希。该性质是编译时类型保证，而非运行时断言。

3. **确定性 v1 巩固算法**：  
   - 读取未处理的情景行，提取结构化字段（skill_id, target, env, outcome 等）。  
   - 按稳定键（如 skill_id + target + env）分组，聚合计数、成功率、参数分布等。  
   - 合并更新（upsert）到语义表 semantic_facts，每行包含置信度、观测数、最后支持事件 ID、规则版本。  
   - 算法是幂等、顺序无关的：相同输入和规则版本产生字节相等的输出。  
   - 每次巩固后，追加一个 `consolidation_run` 事件回情景日志，形成完整审计链。

4. **只读规划器接口**：  
   通过 HTTP GET 端点暴露语义事实，规划器读取但不写入；事实仅作信息参考，不构成权威决策，所有行动仍经过运行时安全门控。

**未来 v2 方向（LLM 辅助）**：  
用于抽象非结构化文本，但当前暂不部署，因为非确定性会破坏审计，需先建立来源签名、人工审查门控等。

### 3. 实验设计：数据集 / 场景、基准、对比方法

**合成数据与场景**：

- 每个种子注入 200 条情景行（glass_cup）+ 50 条（unknown_object），然后运行 1000 次决策。

**对比的控制方法（Planner grounding ablation）**：
1. **No-memory**：规划器无视语义事实，每次都尝试抓取。
2. **Raw**：直接使用巩固器给出的成功率作为置信度，阈值≥0.5。
3. **Uniform**：将所有已知技能的置信度强制设为1.0，仅区分事实是否存在。
4. **Calibrated**：对每个技能的成功率应用贝叶斯收缩（先验均值0.5，先验权重10），再与阈值0.5比较。

**基准**：以无记忆控制为基线，计算其他控制相对于基线的无成效尝试减少百分比。校准控制作为主要比较对象，用于反驳粗糙基线可能夸大效果的批评。

- **确定性 v1 有效**：能从情景日志提取结构化事实，审计链完整。

- **主要启示**：自主智能体可以在不改变“身份”（证书）的情况下积累操作性知识，实现“操作上变化，身份上不变”，为受监管长期部署提供可持续审计的知识更新范式。

### 7. 优点

1. **结构正确性保证**：身份不变性不是运行时检查，而是编译时类型系统推导，可靠性高。  
2. **可审计性设计**：每行语义事实包含置信度、观测数、支持事件 ID、规则版本，审计链完整，从决策可回溯至原始情景。  
3. **确定性幂等**：相同输入+规则版本产生字节相等输出，便于回放验证和多方审阅。  
4. **低资源消耗**：CPU 即可运行，适合资源受限的机器人部署。  
5. **清晰分离信息与权威**：语义层只提供信息，不绕过运行时安全门控，保留监管控制。  
6. **诚实报告局限**：明确指出 v1 仅处理结构化字段、无遗忘、不抗恶意注入，并规划 v2 需先解决审计问题。

### 8. 不足与局限

1. **仅限于结构化聚合**：无法处理叙事性、自由文本形式的经验，如操作员口头反馈。  
2. **无遗忘机制**：事实置信度不随时间衰减，一年前的老数据与新数据权重相同。  
3. **无对抗防御**：假设情景日志可信；恶意写入可污染语义事实。  
4. **依赖情景负载结构**：若能力输出仅为自由文本摘要，则无法聚合。  
5. **长期行为漂移**：虽身份不变，但累积的事实可能间接改变规划器行为；需要额外监控。  
6. **实验全部为合成数据**：缺乏真实长期部署数据，无法验证现实世界复杂性与规模下的表现。  
7. **v2 LLM 辅助未实现**：处理非结构化知识仍需未来工作，且需解决审计难题。  
8. **跨身份知识共享未考虑**：语义存储严格按身份哈希分区，无法在监管下进行受控的群体学习。
