## 摘要
长时域LLM智能体的记忆是一种契约，规定了每个未来决策允许查看的内容。最简单的契约将过去的观察、工具调用和思考过程附加到每个提示中，这使得先前的上下文易于访问，但也将其变成了一个混杂的混合物，其中任何单个记忆组件的影响都难以分离。我们引入并实现了一种替代的有界契约：每个决策都基于通过类型化检索组装的新用户消息，而不附加原始的跨决策记录。因此，提示的长度在任何运行长度下都保持有界，并且任何单层都可以独立进行消融。我们在《Slay the Spire 2》中实例化了该契约，这是一个封闭规则的随机卡牌构筑游戏，其运行需要数百个战术和战略决策。在同一游戏上的前沿LLM公开在线基准测试报告显示，在五种配置中，最低难度下零胜率，而开发者报告的人类在相同难度下的胜率为16%；该任务困难但尚未饱和。在我们的测试框架中，当启用触发的战略技能时，固定A0消融显示出了最大的观察差异：无存储基线赢得3/10局游戏，而添加技能层后赢得6/10局。在此样本量下，比较是方向性的而非统计上决定性的（费舍尔精确检验p≈0.37）；跨骨干网络探测和公共累积上下文基线作为操作比较报告，而非对契约变量本身的控制测试。我们发布了一个可复现的测试平台：298个带条件标签的已完成轨迹、冻结的记忆/技能快照、提示记录和分析脚本——这是一个智能体设计以及一个经过验证的、可复用的方法论，用于研究显式记忆层如何塑造长时域LLM智能体的决策。

## Abstract
Memory for a long-horizon LLM agent is a contract about what each future decision is allowed to see. The simplest contract appends past observations, tool calls, and reflections to every prompt, which makes prior context easy to access but also turns it into a jumbled mixture in which the effect of any single memory component is hard to isolate. We introduce and instrument an alternative bounded contract: every decision is made from a fresh user message assembled by typed retrieval, with no raw cross-decision transcript appended. The prompt thus stays bounded across runs of any length, and any single layer can be ablated in isolation. We instantiate the contract in Slay the Spire 2, a closed-rule stochastic deck-building game whose runs require hundreds of tactical and strategic decisions. A public online benchmark of frontier LLMs on the same game reports zero wins at the lowest difficulty across five configurations, and the developer-reported human win rate at the same difficulty is 16%; the task is hard but not saturated. Within our harness, a fixed-A0 ablation shows the largest observed difference when triggered strategic skills are enabled: the no-store baseline wins 3/10 games and adding the skill layer 6/10. At this sample size the comparison is directional rather than statistically decisive (Fisher exact p\approx0.37); a cross-backbone probe and public accumulating-context baselines are reported as operational comparisons rather than controlled tests of the contract variable itself. We release a reproducible testbed: 298 completed trajectories with condition tags, frozen memory/skill snapshots, prompt records, and analysis scripts -- an agent design and a validated, reusable methodology for studying how explicit memory layers shape long-horizon LLM-agent decisions.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：长时域（long-horizon）LLM智能体的记忆管理通常采用“全历史追加”的方式，将过往的观察、工具调用和反思全部附加到当前提示中，导致上下文混杂、长度随决策数线性增长，无法隔离单个记忆组件（如策略规则、经验摘要）对决策的影响。
- **研究背景**：现有工作如ReAct、Reflexion等通过追加历史记录实现记忆；而结构化记忆（MemGPT、Mem0）、技能库（Voyager）等虽提供了外部存储，但缺乏统一的、可消融的接口来评估不同类型记忆在决策中的作用。同时，长时域任务（如复杂游戏、软件工程）中，上下文增长导致成本高昂且难以归因。
- **整体含义**：论文主张将记忆视为一种“契约”——每个决策允许查看的内容应当是有界、类型化的，而非无限制增长的上下文。作者通过设计一种有界记忆契约（bounded-memory contract），使得记忆层可独立消融、审计和复用，从而在困难但尚未饱和的游戏《Slay the Spire 2》上建立了一个可复用的测试平台。

## 2. 论文提出的方法论

### 2.1 核心思想：有界记忆契约
- **契约定义**：每个决策不再依赖原始跨决策消息记录，而是由五个类型化知识层（L1–L5）通过检索动态组装成一个全新的用户消息。每个决策的提示大小被预算上限固定，不随运行长度增加。
- **公式化表示**：对于决策 \( d \) 和状态 \( s_d \)，引擎从五层中检索并组合出用户消息：
  \[
  u_d = \pi\big(L1,\ L2(s_d),\ L3(s_d),\ L4(s_d),\ L5(s_d)\big)
  \]
  发送给LLM作为 `<sys, u_d>`。其中L1、L2是固定协议和模式；L3是可过滤的游戏规则；L4是事后摘要（情景记忆）；L5是触发式战略技能库。

### 2.2 关键技术细节
- **五层知识层**：
  - **L1（协议）**：固定角色和状态类型模板。
  - **L2（状态模式）**：不可变的决策格式，包括战斗、卡组构筑、地图、事件等合法动作格式。
  - **L3（游戏知识）**：可枚举规则数据（卡牌、遗物、敌人等），按补丁刷新。
  - **L4（情景记忆）**：按角色-难度-幕-敌人类型存储的事后摘要，基于示例回忆。
  - **L5（技能库）**：由日志蒸馏得到的、由触发器索引的战略指南，每个技能包含显式触发条件和散文式策略，以及四级写入门控（余弦、Jaccard、LLM评判、可选收割）。

- **路由与战斗截断**：决策分派到四个模型层级（快速、战略、分析、进化），战斗仅维持局内对话对象，每轮最多三条消息（战斗开始、确认、最新状态），避免回合级记录追加。

- **技能发现**：L5的填充通过两种方式：错误驱动自我进化（mistake-driven discovery）和模板填充（Mode B）。写入门控确保技能质量，大多数候选被拒绝或合并。

- **上下文有界性**：通过顶k检索和项目大小上限，提示复杂度为 \( O(|sys| + s_{\text{thread}} + \sum_i k_i \cdot s_i) \)，不随决策数增长。

## 3. 实验设计

### 3.1 场景与benchmark
- **游戏**：Slay the Spire 2，一个封闭规则的随机卡牌构筑Roguelike游戏，具有数百个战术和战略决策。难度系统为Ascension等级（A0最低，A10最高）。
- **公开基准**：AGI-Eval在A0难度上报告五个前沿LLM配置0胜率；开发者报告人类A0胜率16%。任务困难但未饱和。

### 3.2 实验设置
- **固定A0消融矩阵**：五个条件单元（各10个已完成游戏）：
  1. `baseline-strict`：无记忆、无技能、无战略线索，严格知识过滤。
  2. `prompt-only`：保留完整提示和对话助手，禁用L4和L5。
  3. `mode-a`：添加人类编写L5技能种子。
  4. `mode-b-frozen`：将L5种子替换为模板填充体。
  5. `full-frozen`：在Mode A基础上添加冻结的L4情景记忆。
- **跨骨干网络探针**：相同冻结L4+L5栈迁移到Qwen 3.6 27B、DeepSeek V4 Pro、Gemini 3.1 Pro（各5个游戏）。
- **自动模式梯子**：胜利后提升难度，失败后重试当前难度，测量端点难度（A6–A8）。
- **开源累积上下文智能体对比**：与STS2MCP和CharTyr（使用累积聊天转录）在相同骨干（Gemini-3.1-pro）上各5局A0游戏比较效果、速度、成本。

### 3.3 对比方法
- 内部：五种条件之间比较（消融L5、L4等）。
- 外部：与AGI-Eval报告、Mega Crit社区人类胜率、以及两个开源智能体（STS2MCP、CharTyr）的同等条件比较。
- 注意：外部对比是操作比较，非控制消融。

## 6. 论文的主要结论与发现

1. **方向性发现**：在固定A0矩阵中，最大观察差异出现在启用触发的L5战略技能时（3/10→6/10），但Fisher精确检验p≈0.37，非统计显著。L4情景记忆在A0上饱和（mode-a与full-frozen胜率相同）。
2. **有界契约可行**：通过类型化检索，提示大小保持有界（约5k token/决策），不随运行增长，示意图展示了累积上下文智能体提示可达500k token/决策。
3. **跨骨干迁移敏感**：Gemini训练的L4+L5栈提升Qwen的分数（+84.5%）但降低DeepSeek的分数（-18.1%），胜负均为0/5，说明技能栈与骨干密切相关。
4. **自动梯子可达A6–A8**：启用事后写入记忆的流可以攀登较高难度。
5. **操作比较**：两个开源累积上下文智能体在同一骨干和难度下表现远低于我们的底线有界基线（0/5 vs 3/10），且每得分点消耗66–90倍新鲜令牌（新鲜令牌每点≥422k vs 6.4k）。

## 7. 优点

- **方法创新**：将长时域LLM智能体的记忆形式化为有界、类型化、可消融的契约，与常见的“追加历史”做法形成鲜明对比，为研究记忆组件提供了结构化工具箱。
- **实验设计严谨**：
  - 固定A0消融采用平衡子集，明确区分方向性与统计显著性。
  - 公开所有轨迹、条件标签、冻结快照、提示记录和脚本，支持第三方复现和扩展。
  - 承认外部对比的非控制性质，并详细披露差异。
- **基准选择**：Slay the Spire 2是一个困难但未饱和的任务，具有封闭规则、符号状态、长视界和随机性，非常适合评测长时域记忆。
- **应用潜力**：有界契约的设计可移植到其他封闭规则、回合制智能体任务（如软件工程中的工具调用），提供一种可审计的记忆管理范式。

## 8. 不足与局限

- **样本量不足**：固定A0每条件仅10局，导致L5差异方向性而非统计决定性；跨骨干探针每条件5局更不足以得出可靠结论。
- **缺乏同代码库累积上下文对比**：论文不能直接回答“有界契约是否比同基线的累积上下文更好”，因为开源智能体有多个其他差异（路由、提示、补丁）。作者将其列为未来工作。
- **单一角色与游戏版本**：主要实验仅针对Silent角色，且游戏版本可能变化（不同批次使用了v0.103.1和v0.103.3），虽释放了版本标签但可能引入混淆。
- **技能种子与模板**：L5的初始种子由人类创作，Mode B模板填充也依赖人工参数，并非完全自主学习；技能发现算法有特定门控，泛化性待验证。
- **适用范围**：有界契约针对封闭规则、回合制、符号状态的任务设计，不适用于连续控制流、视觉输入、多智能体或开放式生产环境。
- **训练无关**：所有智能体均为推理时无训练，未测试微调或强化学习对记忆层的影响。
- **资源消耗报告模糊**：未提供训练或推理所用的具体GPU资源和能耗，影响可重复性中的硬件需求评估。
