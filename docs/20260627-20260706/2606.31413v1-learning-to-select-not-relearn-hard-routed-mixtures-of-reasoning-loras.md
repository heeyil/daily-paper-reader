## 摘要
将独立训练的LoRA适配器组合成单个大型语言模型对于多领域自适应非常有用，尤其是在无法共享原始训练数据时。一种常见的方法是对LoRA专家采用MoE风格的路由，但对于冻结的预训练适配器，软加权组合可能会改变每个LoRA模块最初训练时的单位尺度加性更新。我们提出了\textbf{硬路由MoR-LoRA}，一个两阶段框架，通过单位尺度硬选择来组合冻结的推理LoRA专家。首先，使用来自可验证反馈的强化学习独立训练领域特定的LoRA适配器以获得推理专家。然后，所有专家被冻结，从中蒸馏出推理轨迹，仅训练一个轻量级共享路由器和一个小的注意力LoRA用于集成。该路由器使用硬top-1路由为每个token精确选择一个专家，同时直通估计器支持基于梯度的训练。在五个基准测试、多个模型规模和额外模型系列上的实验表明，硬路由MoR-LoRA保留了专家行为，同时需要的可训练参数远少于软路由混合基线。我们的分析进一步表明，归一化的软混合通常将大部分路由质量集中在一个专家上，这表明硬单位尺度路由为冻结的LoRA专家组合提供了一种简单高效的抽象。

## Abstract
Composing independently trained LoRA adapters into a single large language model is useful for multi-domain adaptation, especially when the original training data cannot be shared. A common approach is to use MoE-style routing over LoRA experts, but for frozen pretrained adapters, soft weighted combinations can change the unit-scale additive update under which each LoRA module was originally trained. We propose \textbf{Hard-Routed MoR-LoRA}, a two-stage framework for composing frozen reasoning LoRA experts through unit-scale hard selection. First, domain-specific LoRA adapters are trained independently using reinforcement learning from verifiable feedback to obtain reasoning experts. Then, all experts are frozen, reasoning traces are distilled from them, and only a lightweight shared router together with a small attention LoRA is trained for integration. The router selects exactly one expert per token using hard top-1 routing, while a straight-through estimator enables gradient-based training. Experiments across five benchmarks, multiple model scales, and additional model families show that Hard-Routed MoR-LoRA preserves expert behavior while requiring substantially fewer trainable parameters than soft-routing mixture baselines. Our analysis further shows that normalized soft mixtures often concentrate most routing mass on a single expert, suggesting that hard unit-scale routing provides a simple and efficient abstraction for frozen LoRA expert composition.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在多领域自适应场景中，独立训练的LoRA适配器因数据隐私限制无法共享原始数据，只能共享训练好的LoRA模块。如何将这些冻结的、独立训练的LoRA专家有效组合成一个统一模型，使其支持异构任务，同时保持每个专家的原始推理能力。
- **现有方法不足**：标准软路由（soft MoE）对多个专家的输出进行加权组合，改变了专家训练时假设的单位尺度加性更新（unit-scale additive update），导致缩放不匹配（scale mismatch）。先前工作如LoRA-Mixer虽通过额外正则化损失微调专家来缓解，但增加了可训练参数数量，可能改变专家原有知识。
- **本文核心思想**：提出“学习选择而非重新学习”（Learning to Select, Not Relearn），将推理获取与专家选择解耦。先独立训练推理专家（使用强化学习从可验证反馈RLVF），然后冻结专家，仅训练一个轻量级硬路由器和少量注意力LoRA，通过硬top-1选择（单位尺度）组合专家，保持专家行为不变。

## 2. 方法论：核心思想、关键技术细节

- **两阶段框架（Hard-Routed MoR-LoRA）**：
  - **Stage I（推理专家获取）**：针对每个领域数据集，独立训练一个LoRA适配器（应用于FFN层），使用RLVF（采用GRPO算法）进行优化。奖励基于最终答案自动验证（正确+1.0，错误-1.1），无需人类偏好。该阶段产生一组领域特定的推理专家 {E1, E2, ..., EK}。
  - **Stage II（混合集成）**：
    1. **蒸馏数据生成**：用冻结的专家在各自领域提示上生成推理轨迹，构成小规模SFT数据集（最多每领域1000样本）。
    2. **训练对象**：一个共享的线性路由器（token级）+ 一个小型注意力LoRA（应用于q/k/v/o投影）。训练目标为标准下一个token预测（next-token likelihood）。
    3. **硬路由机制**：对每个token，路由器生成softmax概率P，选择arg max得到离散的one-hot决策（硬top-1）。在前向传播中只应用被选中的专家（单位尺度加性更新）。反向传播使用直通估计器（STE）：forward用硬决策，backward用soft概率的梯度，使路由器可训练。
    4. **关键设计**：归一化的top-1软路由虽能保单位尺度但梯度为零（常数函数），无法训练路由器。硬路由+STE同时实现了单位尺度应用和梯度传播。

## 6. 主要结论与发现

1. **RLVF比SFT更有效地训练推理专家**：在中等和大模型（3B、8B）上，RLVF显著提升推理密集型任务（如GSM8K、ARC-C）的准确率（例如3B从68.60提升至72.69，8B从72.72提升至79.72），而1B模型受益有限（容量不足）。SFT在大型指令微调模型上反而可能削弱已有CoT能力。
2. **硬路由保留专家行为且参数高效**：在冻结专家设置下，Hard-Routed MoR-LoRA与需要继续训练专家的软路由方法（LoRAMixer）性能相当或更好，但可训练参数少得多（例如3B模型：73M vs 606M；8B模型：109M vs 1.133B）。
