## 摘要
部署在受监管行业的大型语言模型需要在合规执行和成本效率两个约束下运行。用户查询中的个人身份信息（PII）可能在系统确定该数据是否应离开其管辖边界之前就已到达模型端点。通过单个大型模型服务所有查询会消耗全部GPU容量，无论查询复杂度如何，且无法提供地理路由机制。专家混合架构并未解决此问题——路由发生在数据已到达端点后的模型内部专家层之间，且无论查询复杂度如何，所有专家均加载在内存中。我们提出一种分类器门控路由架构，通过设计强制执行合规性。训练好的编码器分类器置于任何解码器推理之前，评估每个查询的复杂度和数据敏感性，然后将其路由到适当地理位置适当大小的小型密集模型。包含PII的查询在开始任何LLM计算前即路由至本地端点，使数据驻留违规在结构上不可能发生。简单查询以极低成本到达小型快速模型。我们在600个查询上的评估表明，中位延迟降低39%，根据查询分布不同节约33-52%的成本，生成吞吐量为122-200 tokens/秒，而基线为50-64 tokens/秒。编码器分类器在7ms推理开销下达到99.2%的准确率，PII召回率近乎完美，将推理前分类确立为实现设计合规型LLM部署的实用路径。

## Abstract
Large language models deployed in regulated industries operate under two constraints: compliance enforcement and cost efficiency. Personally identifiable information (PII) in user queries can reach model endpoints before the system determines whether that data should leave its jurisdictional boundary. Serving all queries through a single large model consumes full GPU capacity regardless of query complexity while offering no mechanism for geographic routing. Mixture-of-Experts architectures do not address this routing occurs between expert layers within the model after data has already arrived at the endpoint, with all experts loaded in memory regardless of query complexity. We propose a classifier-gated routing architecture that enforces compliance by design. A trained encoder classifier sits before any decoder inference, evaluating each query for complexity and data sensitivity, then routing it to an appropriately sized dense model in the appropriate geographic location. PII-containing queries route to local endpoints before any LLM computation begins, making data residency violations structurally impossible. Simple queries reach small, fast models at a fraction of the cost. Our evaluation on 600 queries demonstrates 39% median latency reduction, 33-52% cost savings depending on query distribution, and generation throughput of 122-200 tokens/second versus 50-64 for the baseline. The encoder classifier achieves 99.2% accuracy with near-perfect PII recall at 7ms inference overhead, establishing pre-inference classification as a practical path to compliance-by-design LLM deployment.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在受监管行业（如金融、医疗、法律）部署大型语言模型（LLM）时，面临**合规性（PII数据不能跨境）**和**成本效率（简单查询不应消耗昂贵大模型）** 的双重约束。现有架构无法同时解决这两个问题：
  - 单一大模型：全量查询使用同一模型，无论复杂度，无地理路由能力，PII暴露风险高。
  - Mixture-of-Experts（MoE）：所有专家参数必须同时加载在内存中，路由发生在模型内部，无法在推理前进行数据敏感性判断或地理路由。
  - 已有路由方案（如FrugalGPT、RouteLLM、Hybrid LLM）仅优化成本，未整合合规检测和基于内容的地理路由。
- **研究含义**：提出一种**推理前分类器门控路由架构**，在查询进入任何LLM解码器之前，先对其复杂度和数据敏感性进行分类，从而从结构上消除PII数据驻留违规，同时显著降低成本和延迟，为受监管行业提供可部署的合规LLM推理方案。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 在LLM推理链路中插入一个**编码器分类器**，对每个查询进行**6分类**（复杂度 × 敏感性），根据分类结果路由到不同规模的**密集解码器模型**以及不同的**地理端点**。
- 简单查询 → 小模型（低成本、低延迟）；中等查询 → 中等模型；复杂查询 → 大模型（保留原能力）。
- 含PII的查询 → 强制路由到本地端点，禁止跨境。
- 使用**置信度回退**机制：当分类器置信度低于80%时，默认路由到最严格的“复杂+PII”标签，确保合规。

### 关键技术细节
- **分类器结构**：Bidirectional Transformer Encoder（类似BERT），12层，768隐藏维度，12注意力头，约1.34亿参数。
- **训练**：从头开始，先在2 billion tokens英文文本上预训练（500K步，掩码语言模型），再在50,000个领域特定样本上微调（12,000步）。输入长度截断为128 tokens。
- **6个标签**：由3个复杂度等级（简单simple、中等medium、复杂complex）和2个敏感性等级（非敏感non-PII、敏感PII）叉乘得到。
- **路由逻辑**：路由层（CPU）根据分类标签选择目标模型和区域，转发查询。整个决策在LLM推理开始前完成。
- **模型层级**：支持任意数目的密集模型层（文中使用3层），每层独立部署，无共享故障域。
- **部署**：容器化 + 基础设施即代码（IaC），可一键复制到任何司法管辖区。

1. **延迟显著降低**：门控路由在600 queries上中位延迟降低39%。简单查询2–4倍更快（490–840ms vs 1850ms），中等查询1.7–3.3倍更快（1140–2250ms vs 3750ms），复杂查询持平。

7. **结构优势**：每层独立故障域、独立扩展、组件级升级、地理路由、审计追踪，这些是MoE无法提供的。
