## 摘要
稀疏混合专家（MoE）架构通过top-$k$路由仅激活一小部分专家，从而在固定推理预算下扩展LLM参数。虽然这保持了因果性并适用于自回归语言模型，但离散的top-$k$算子不可微分，迫使每个输入激活固定数量的专家，导致计算利用效率低下。我们提出SoftMoE，它用截断的软top-$k$ LapSum松弛替代离散路由，允许基于梯度的专家路由优化。我们进一步参数化每层的平均激活专家数量，并施加全局预算约束，使模型能够学习如何跨层分配专家容量。SoftMoE与自回归建模完全兼容，在语言建模和下游任务上达到与稀疏MoE相当或更优的性能，同时激活的专家显著减少。值得注意的是，学习到的分配高度非均匀，较深层次的层激活更多专家。源代码已公开$^\dagger$。

## Abstract
Sparse Mixture-of-Experts (MoE) architectures enable scaling LLM parameters under a fixed inference budget by activating only a small subset of experts via top-$k$ routing. While this preserves causality and suits autoregressive language models, the discrete top-$k$ operator is not differentiable, forcing a fixed number of active experts per input and resulting in inefficient use of computation. We propose SoftMoE, which replaces discrete routing with a truncated soft top-$k$ LapSum relaxation, allowing gradient-based optimization of expert routing. We further parameterize the mean number of active experts per layer and impose a global budget constraint, enabling the model to learn how to allocate expert capacity across layers. SoftMoE remains fully compatible with autoregressive modeling and achieves performance comparable to or better than sparse MoE on language modeling and downstream tasks, while activating significantly fewer experts. Notably, the learned allocation is highly non-uniform, with later layers activating more experts. The source code is publicly available$^\dagger$.

# 论文总结：SoftMoE: Soft Differentiable Routing for Mixture-of-Experts in LLMs

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：稀疏混合专家（MoE）架构通过 top-𝑘 路由仅激活部分专家，在固定推理预算下扩展了 LLM 参数规模。但离散的 top-𝑘 算子不可微分，导致：
  - 每个输入必须激活固定数量的专家，计算效率低下；
  - 无法通过梯度优化路由决策，训练与推理之间存在不一致；
  - 专家容量无法跨层自适应分配，造成计算资源浪费。
- **背景**：已有工作尝试可微分路由（如 token 混合、专家选择、最优传输），但往往破坏自回归因果性，或引入额外跨输入依赖，不适用于因果语言模型。
- **整体含义**：SoftMoE 通过可微分的软路由替代硬选择，实现端到端梯度优化，并允许模型自动学习每层专家分配，从而在保持自回归兼容性的同时提升计算效率。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 用 **截断的软 top-𝑘 LapSum 松弛** 替代离散 top-𝑘，使路由权重连续可微，同时保持稀疏性。
- 通过 **参数化每层平均激活专家数** 并施加 **全局预算约束**，让模型自动学习跨层专家容量分配。

### 关键技术细节
1. **SoftTopK 算子**：
   - 基于 LapSum 函数（Laplace CDF 之和）求解可微阈值 $\tilde{x}$，满足 $\sum_i F_{\text{Lap}}(r_i - \tilde{x}) = k$，其中 $r_i$ 是门控分数。
   - 软选择权重 $\tilde{p}_i = F_{\text{Lap}}(r_i - \tilde{x})$，连续且满足 $\sum \tilde{p}_i = k$，且每个元素在 $[0,1]$ 内。
   - 计算复杂度 $O(n)$（$n$ 为专家数），无需排序或迭代。

2. **截断操作**：
   - 对低于阈值 $\tau$ 的权重置零（相当于 shifted ReLU），恢复稀疏激活。
   - 梯度仅在权重 $> \tau$ 时流动，低于 $\tau$ 时为零，保证可微性。
   - 激活专家数因输入而异，实现按 token 自适应计算。

3. **学习跨层专家分配**：
   - 参数化每层平均激活专家数 $k_l$，采用软最大化重参数化：$\pi = \text{softmax}(\eta)$, $k_l = \pi_l \cdot (K - L) + 1$。
   - 全局约束：$\sum_l k_l = K$，且 $k_l \ge 1$（$L$ 为层数，$K$ 为全局预算）。
   - 整个分配可端到端梯度优化。

4. **兼容性**：每个 token 独立处理，不涉及跨 token 混合，完全兼容自回归解码。

- **下游任务**：PIQA（物理常识推理）、HellaSwag（句子补全）、ARC-E（基础科学问答），均为零样本评估。

### Benchmark 与对比方法
- **基线**：标准稀疏 MoE（Switch Transformer）架构，激活专家数 $k \in \{1,2,3,4\}$。
- **SoftMoE 变体**：

  - SoftMoE：学习跨层专家分配。
- **配置**：初始平均 $k \in \{1, 1.5, 2\}$，扩张因子 $\alpha \in \{2,4\}$。

## 6. 主要结论与发现
1. **软路由提升效率**：SoftMoE* 在相同预算下，激活专家数平均减少 17%–24%（C4/OWT），同时语言建模损失更低或相当。
2. **跨层分配的适应性**：模型自动将预算集中到**后几层**（前 3 层吸收约 50% 预算），前层和中间层使用更少专家。该模式在训练早期（~25k 步）快速形成。
3. **下游性能**：在 HellaSwag 上始终优于稀疏 MoE；PIQA 上多数配置更好或相当；ARC-E 上在 OWT 下最优，C4 下保持竞争力。
4. **学习分配带来额外增益**：SoftMoE（有学习分配）通常进一步降低损失，但激活专家数略高于 SoftMoE*，反映模型倾向将额外容量分配给后层。
5. **计算开销可忽略**：LapSum 的 $O(n)$ 复杂度相对 $n \times d$ 门控矩阵计算可忽略，不抵消激活专家减少带来的收益。

## 7. 优点
- **创新性**：首次将 LapSum 可微排序算子用于 MoE 路由，实现端到端梯度优化，同时保持稀疏性和自回归因果性。
- **实用性**：
  - 允许模型按输入动态调整激活专家数，而非固定 $\text{top-}k$。
  - 可学习跨层计算预算分配，揭示深层需要更多容量的有趣现象。
  - 实现简单，计算开销低，易于集成到现有框架（Megatron-LM）。
- **实验严谨性**：对比了同架构下的多种设置，消融了关键超参数（阈值 $\tau$、jitter、学习分配），并提供了训练/推理效率数据。

## 8. 不足与局限
- **规模局限**：仅测试 1.63B 参数的模型，未在更大规模（如 100B+）上验证，结论可能随规模变化。
- **语言与任务覆盖**：仅使用英文语料（OWT、C4）和英文下游任务，未评估多语言或多模态。
- **潜在风险**：
  - 截断阈值 $\tau$ 调节了稀疏度，但取值敏感（需从初始批次统计），可能需要手动调优。
  - 预算学习依赖于全局约束 $\sum k_l = K$，若 $K$ 设置不当可能影响性能。
  - 实验未讨论在非常深的模型（如 50+ 层）中是否会出现分配不稳定或收敛困难。
- **偏差**：所有模型在相同随机种子下训练，但不同配置的收敛速度略有差异，少量运行可能不完全代表统计显著差异。
