## 摘要
测试时扩展提升了语言模型的推理能力，但现有方法往往面临一个困难的权衡：长思维链采样仍然是单线程的，而句子级或解决方案级搜索可能计算量大且难以端到端训练。我们引入了局部分支路由（LBR），这是一个令牌级别的测试时扩展框架，它扩展一个小型的局部前瞻树，将所有采样分支通过语言模型前向传播，并使用一个轻量级路由器来选择要承诺的深度-1子树。通过在候选局部未来的隐藏状态上进行路由，LBR允许每个令牌决策利用超出根节点下一个令牌分布的证据，同时避免完整的解决方案级搜索。由此产生的剪枝-移位-增长解码过程保留了离散分支的身份，并定义了一个可处理的树轨迹似然：新增长的节点在首次采样时被计数，路由器决策被赋予显式概率。这使得能够使用可验证奖励进行端到端强化学习，基于与离散令牌RLVR相同的似然比原则联合优化基础模型和路由器。在合成层次规划任务上，LBR表明候选隐藏状态提供了有用的路由证据。在数学推理基准上，LBR在Pass@1和Pass@32上都优于离散思维链、普通离散令牌RLVR以及RL兼容的软令牌分支基线。这些结果表明，轻量级局部分支为语言模型测试时扩展提供了一种高效、可训练且离散的形式。

## Abstract
Test-time scaling improves language-model reasoning, but existing approaches often face a difficult trade-off: long chain-of-thought sampling remains single-threaded, while sentence- or solution-level search can be computationally expensive and hard to train end-to-end. We introduce Local Branch Routing (LBR), a token-level test-time scaling framework that expands a small local lookahead tree, forwards all sampled branches through the language model, and uses a lightweight router to select the depth-1 subtree to commit. By routing over the hidden states of candidate local futures, LBR allows each token decision to use evidence beyond the root next-token distribution while avoiding full solution-level search. The resulting prune-shift-grow decoding process preserves discrete branch identities and defines a tractable tree-trajectory likelihood: newly grown nodes are counted when first sampled, and router decisions are assigned explicit probabilities. This enables end-to-end reinforcement learning with verifiable rewards, jointly optimizing the base model and router under the same likelihood-ratio principle as discrete-token RLVR. On synthetic hierarchical-planning tasks, LBR shows that post-candidate hidden states provide useful routing evidence. On mathematical reasoning benchmarks, LBR improves both Pass@1 and Pass@32 over discrete chain-of-thought, vanilla discrete-token RLVR, and RL-compatible soft-token branching baselines. These results suggest that lightweight local branching offers an efficient, trainable, and discrete form of language-model test-time scaling.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有语言模型测试时扩展方法面临效率与可训练性的权衡。长思维链（CoT）采样仍然是单线程的，无法并行探索；而搜索式方法（如树搜索）虽然引入了宽度，但在句子级或解决方案级进行搜索计算开销大，且难以作为单一可训练解码策略进行端到端优化。
- **整体含义**：本文提出**局部分支路由（Local Branch Routing, LBR）**，一种**令牌级（token-level）**的测试时扩展框架，通过在小范围局部前瞻树中分支并路由，使每个令牌决策能够利用候选未来隐藏状态，而非仅依赖根节点下一个令牌分布。LBR保留了离散分支身份，定义了可处理的树轨迹似然，支持端到端强化学习，从而在效率和训练性之间取得更好的平衡。

## 2. 论文提出的方法论

### 核心思想
- LBR在每个解码步骤维护一个**滚动局部前瞻树**（深度L，宽度K），通过四个阶段实现：生长（grow）、路由（route）、剪枝（prune）、移位-增长（shift-regrow）。
- 路由器基于已前向传播的隐藏状态选择深度1子树，选定的令牌被提交，其余分支被剪枝，选中的子树前移并生长新层恢复深度L。
- 整个过程的随机性仅来自生长新节点（从语言模型采样）和路由器决策，因此轨迹似然可分解为两部分，从而可以用似然比原则结合RLVR（基于可验证奖励的强化学习）进行端到端训练。

### 关键技术细节
- **局部树生长**：从当前前缀开始，每个活动节点从过滤的模型分布（温度、top-p等）中采样K个子节点，并全部前向传播得到隐藏状态。
- **路由器（Set-Attention Router）**：轻量级路由器，先独立编码每个深度1候选子树（L=1时为单个令牌的隐藏状态），然后通过跨子树集注意力（set attention）进行对比，最后通过共享MLP和softmax输出路由分布。该设计保证排列不变性，并让路由器在对比中评估候选。
- **树轨迹似然**：$\log p_{\theta,\phi}(\mathcal{F} | q) = \sum_{t} \sum_{v \in \mathcal{G}_t} \log \tilde{\pi}_\theta (\text{tok}(v) | \text{ctx}(v)) + \sum_t \log \rho_\phi(k_t^* | x_{<t}, \mathcal{A}_t; \theta)$。第一项是生长节点的语言模型对数概率（仅首次采样时计数），第二项是路由器决策的对数概率。
- **联合训练**：使用GRPO算法优化基础模型参数θ和路由器参数ϕ，损失函数为$L_{LBR}(\theta,\phi; \mathcal{F}) = -A(\mathcal{F}) \log p_{\theta,\phi}(\mathcal{F}|q)$，其中$A$是基于验证奖励计算的优势。
- **因果树注意（Tree-Causal Attention）**：在生长阶段利用前缀共享机制批量前向传播新节点，提高效率。

## 6. 论文的主要结论与发现

1. **LBR在数学推理上全面优于所有基线**：在1.5B和7B两个规模、6个基准上，LBR（L=1或L=2）的Pass@1和Pass@32均超过离散CoT、离散RLVR和软令牌RLVR。例如1.5B模型在MATH500上Pass@1达75.0%（基线最高67.5%），在AIME'24上Pass@1达15.1%（基线最高11.8%）。
2. **更深局部前瞻（L=2）通常带来进一步提升**，尤其是7B骨干下，性能增益显著（如AIME'24 Pass@1从23.9%升至28.0%）。
3. **路由器对比性关键**：跨子树集注意力（对比路由）显著优于独立路由器，验证了路由决策需要对比兄弟分支。
4. **LBR在合成任务中展示机制优势**：后候选隐藏状态比前分支状态包含更多关于最终目标的信息；软令牌合并导致概念模糊（概念身份探针准确率低于0.8），而LBR保留离散身份。
5. **训练动态揭示联合优化**：路由器熵下降，基础LM在路由位置熵也下降，表明模型和路由器协同适应，路由更果断而不损害采样多样性。

## 7. 优点

- **方法创新**：提出令牌级离散分支路由，兼顾效率（局部前瞻）和端到端可训练性（树轨迹似然+RLVR），与软令牌方法不同，保持离散性，避免概念模糊。
- **路由器设计**：基于隐藏状态的集注意力路由器，能够捕获后候选信息，且对比性强，比独立评分更有效。
- **算法效率**：通过树因果注意和前缀共享，前向传播开销可控；轻量路由器参数量小。
- **实验验证充分**：从合成任务到多个数学基准，消融和动态分析完整，解释性实验（探针）有力支撑动机。
- **通用性**：可适配不同模型大小，解码过程与标准解码兼容，可替换现有解码策略。

## 8. 不足与局限

- **局部性限制**：LBR仅利用局部前瞻窗口内的信息，无法实现全局规划或完整解决方案级搜索。对于需要长距离依赖或深层嵌套推理的任务，可能效果有限。
- **深度-成本权衡**：增大局部前瞻深度L可以增强路由信号，但带来的前向传播计算开销线性增长（需前向更多层节点），存在速度-效果权衡，未提供自适应深度的方案。
- **实验覆盖局限**：仅测试了数学推理和合成规划任务，未涉及其他类型推理（如常识推理、代码生成、多跳问答）或实际应用（如对话、翻译），通用性有待验证。
- **偏差风险**：合成任务基于特定图结构，可能偏向于LBR的机制；数学推理基准可能带有数据分布偏差。验证奖励器仅使用二元答案匹配，可能漏掉部分正确推理过程。
- **缺乏与其他树搜索方法的直接比较**：如Tree of Thoughts、MCTS-DPO等虽然粒度不同（句子级），但未在相同设置下对比性能与计算成本，难以全面评估LBR的竞争地位。
- **缺乏推理开销分析**：虽提到“高效”，但未具体量化相对于普通解码或软令牌方法增加的计算时间或内存。
