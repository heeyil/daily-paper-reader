## 摘要
基于大语言模型（LLM）的智能体越来越多地应用于交互式文本环境，包括网页导航、代码编辑、工具使用以及长期对话。然而，许多智能体仍然主要是反应式的，即将观察映射到动作，而缺乏对这些环境如何结构化及演变的显式模型。这激发了文本世界模型（TWMs）的研究：即基于文本状态的转移模型，给定一个状态和候选动作，预测生成的网页、终端输出、API响应或用户回复，从而支持规划、高效学习和原则性评估。我们系统地梳理了面向基于LLM的智能体的文本世界模型，围绕一个正式框架和智能体生命周期组织：（1）基础，定义文本世界模型并通过状态表示和基础领域进行表征；（2）构建，对“LLM作为世界模型”和“代码作为世界模型”范式进行分类，并综述构建方法；（3）应用，考察世界模型如何在训练时通过经验合成以及在推理时通过规划、验证和适应来支持智能体；（4）评估，涵盖对世界模型本身的评估以及将其用作智能体评估环境两个方面。我们旨在整合这一快速发展的领域，澄清其设计空间，并强调未来研究的开放挑战。

## Abstract
Large language model (LLM)-based agents are increasingly used in interactive textual environments, from web navigation and code editing to tool use and long-horizon dialogue. Yet many remain largely reactive, mapping observations to actions without an explicit model of how these environments are structured and evolve. This motivates text world models (TWMs): transition models over textual states that, given a state and a candidate action, predict the resulting webpage, terminal output, API response, or user reply, thereby supporting planning, efficient learning, and principled evaluation. We systematically review text world models for LLM-based agents, organized around a formal framework and the agent lifecycle: (1) Foundations, defining text world models and characterizing them by state representation and grounding domain; (2) Construction, taxonomizing LLM-as-WM and code-as-WM paradigms and reviewing methods for building them; (3) Application, examining how world models support agents at training time through experience synthesis and at inference time through planning, verification, and adaptation; and (4) Evaluation, covering both evaluation of the world model itself and its use as an evaluation environment for agents. We aim to consolidate this rapidly developing area, clarify its design space, and highlight open challenges for future research.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：基于大语言模型（LLM）的智能体在交互式文本环境（如网页导航、代码编辑、工具使用、长对话）中，多数仍为**反应式**：仅根据当前观察直接映射到动作，缺乏对环境的结构化、演化规律的显式建模。这导致智能体难以进行多步规划、高效学习，也无法在部署前进行可靠评估。
- **研究动机**：将“世界模型”（World Model）引入文本域——即**文本世界模型（TWM）**——通过预测给定状态和动作后的下一个文本状态（网页、终端输出、API响应、用户回复等），为智能体提供规划、训练和评估的基础设施。TWM 填补了智能体与环境之间的“鸿沟”。
- **整体含义**：该综述首次系统性地整理了 TWM 研究领域，提供了一个统一的框架（形式化定义+两轴分类+生命周期视角），并识别了开放挑战，旨在推动该方向的进一步发展。

## 2. 方法论

### 核心思想与框架

- **形式化定义**：文本世界模型 \( M: S \times A \to TS \)，给定智能体可见状态 \( s_t \) 和候选动作 \( a_t \)，输出后继状态的文本渲染 \( \hat{s}_{t+1} \)。
- **两轴分类**：
  - **状态表示**（自然语言/结构化表示/可执行代码）；
  - **基础领域**（物理/数字/社会/抽象世界）。
- **生命周期视角**：构建（§3）→ 训练时使用（§4）→ 推理时使用（§5）→ 评估（§6）。

### 关键技术细节（三大构建范式）

1. **LLM 作为世界模型（LLM-as-WM）**：
   - **学习型**：使用 SFT 或 RL（GRPO、DPO）在轨迹数据上微调 LLM。预测目标可以是完整状态或仅状态变化（delta）。数据来源从真实环境采集到完全合成。规模从数千到万亿 token（Code World Model 5T token）。
   - **提示型**：通过上下文学习（ICL）、检索增强（RAG）或自演化记忆，不更新参数。依赖 LLM 原有知识。
2. **代码作为世界模型（Code-as-WM）**：
   - 由 LLM 生成可执行代码（PDDL、Python、HTML 仿真器），由执行器运行，保证确定性、可验证性。
   - 大规模环境合成：通过自动验证和多样化生成，扩展环境数量（如 EnvScaler 191 个沙盒，AWM 1000+ 环境）。

### 训练时使用（§4）

- **内部化**：将世界模型知识融入智能体参数（如 SPA、RWML），或显式出现在推理链中（如 Dyna-Think、Dyna-Mind）。
- **作为训练环境**：离线合成轨迹（WebSynthesis）→ 在线环境（DreamGym）→ 共演化（DynaWeb）。
- **用户模拟**：训练 RL 智能体时模拟用户行为（UserRL、Echo-N1），并注重用户模型真实性（HumanLM）。

### 推理时使用（§5）

- **作为模拟器**：浅层 lookahead（一步或几步）或深层树搜索（MCTS、RAP、LATS）。
- **作为验证器**：对候选动作排序、筛选或重生成（SWE-World、CUWM、FOREAGENT、WAC）。

## 6. 主要结论与发现

1. **TWM 极大提升了智能体能力**：在规划、训练效率、安全性（验证）三方面均有显著正收益。
2. **预测粒度与领域特性相关**：Delta 预测在冗余观察（如 Web 页面）中更高效；完整状态预测适用于紧凑环境。
3. **学习范式趋势**：从 SFT 向基于奖励的训练（RL）演变，奖励从表面匹配转向行为一致性。
4. **代码作为世界模型**：提供确定性、可验证的转移，适合规则明确的环境，但受限于代码表达能力。
5. **用户模拟仍需突破**：当前提示型 LLM 模拟用户系统性地高估智能体能力，需要专门训练的用户模型（如 UserLM）。
6. **评估指标需要多维**：单步准确性不足以反映世界模型质量，任务成功与环境理解可分离（Task2Quiz 现象），行为保持指标更可靠。
7. **世界模型与策略的耦合**：紧耦合（共享参数）有利于一致性，解耦有利于复用，尚无统一最优方案。

## 7. 优点

- **第一个系统性综述**：为 TWM 领域提供了统一的定义、分类和生命周期框架，填补了现有世界模型综述主要关注视觉/控制域的空白。
- **结构清晰**：按构建→训练→推理→评估分节，每节又细分子方向，便于读者快速定位。
- **全面覆盖**：收录超过 100 篇代表性工作，时间跨度从 2023 到 2026，展示了快速扩张的现状。
- **跨范式比较**：明确对比了 LLM-as-WM 与 Code-as-WM 的优缺点、适用场景及发展趋势。
- **指出开放问题**：如世界模型-策略耦合、推理世界模型、持续适应等，为未来研究指明方向。

- **领域覆盖不完全**：尽管覆盖面广，但某些新兴领域（如具身机器人中的文本工作模型、多模态融合）讨论较少。
