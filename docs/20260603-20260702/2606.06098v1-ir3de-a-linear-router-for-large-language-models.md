## 摘要
基础大型语言模型（LLMs）在广泛的一般任务上展现出熟练度，并通过领域专家LLMs在各种专门任务上取得了显著成果。随着可用LLM列表的不断增长，推理路由器被提出，以针对每个提示选择最合适的LLM。然而，现有的路由方法要么优化从弱到强的通用型LLMs的成本，要么需要大量训练来支持领域专家路由。在本文中，我们提出IR3DE，一种基于岭回归的领域专家路由器，为每个提示提供廉价且快速的路由决策。我们在两种因果语言建模（CLM）设置中评估IR3DE，其中任务是对所有领域进行下一个token预测，以及一种推理设置，其中每个领域都有其独特的推理任务。尽管是线性路由器，IR3DE在两种CLM设置中实现了与其他基线相当的性能，并在推理设置中超越了它们，归一化性能达到98.4%。此外，IR3DE允许添加或移除新的领域专家，而无需从头重新训练路由器，从而可以服务于动态的LLM集合，对路由器本身的影响最小。我们的代码可在 github.com/gensyn-ai/IR3DE 获取。

## Abstract
Foundational Large Language Models (LLMs) demonstrate proficiency on a wide range of general tasks, and achieve remarkable results on various specialized tasks via domain-expert LLMs. With the ever-growing list of available LLMs, inference routers are being proposed to select the most appropriate LLM for each prompt. However, existing routing methods either optimize cost across weak-to-strong generalist LLMs or require substantial training to support domain-expertise routing. In this paper, we propose IR3DE, a Ridge Regression-based Router for Domain Experts that provides cheap and fast routing decisions for each prompt. We evaluate IR3DE in two Causal Language Modeling (CLM) settings where the tasks are next-token prediction for all domains, and one reasoning setting where each domain has its own distinct reasoning task. Despite being a linear router, IR3DE achieves performance comparable to the other baselines in both CLM settings, and surpassing them in the reasoning setting, with a normalized performance of 98.4%. Moreover, IR3DE enables the addition or removal of new domain experts without requiring the router to be retrained from scratch, allowing a dynamic set of LLMs to be served with minimal disruption to the router itself. Our code is available at: github.com/gensyn-ai/IR3DE.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将路由建模为**基于 token 嵌入的线性分类**问题。通过岭回归闭式解一次性学习线性权重，再根据 token 级预测的置信度（熵）筛选出高置信 tokens，最终通过多数投票确定目标专家。
- **技术细节**：
  1. **Token Router (TR)**：  
     - 输入文本 \( x \) 经 tokenizer 和预训练嵌入层 \( E \) 得到 token 嵌入矩阵 \( E(T(x)) \in \mathbb{R}^{T \times h} \)。  
     - 线性变换 \( W \in \mathbb{R}^{h \times C} \)（\( C \) 为专家数）后接 softmax，输出每个 token 的领域概率 \( R(x)_t = \text{softmax}(E(T(x))W) \)。  
     - 训练：使用所有领域样本的 token 嵌入矩阵 \( E(T(X)) \) 和 one-hot 标签 \( Y \)，求解岭回归问题：  
       \[
       W^* = (E(T(X))^\top E(T(X)) + \lambda I_h)^{-1} E(T(X))^\top Y.
       \]  
       该解为闭式，可通过批量计算实现异步、去中心化训练，支持动态增减专家。
  2. **Sample Route Selector (SRS)**：  
     - 对输入文本的每个 token，计算 softmax 概率向量的 Shannon 熵。  
     - 仅保留熵最小的前 \( k \) 个 token（高置信度），让这些 token 通过 argmax 投票，最终采用多数投票决定路由目标。  
     - **动机**：常见 tokens（如“the”）在不同领域出现次数相近，其预测熵高，应被排除以减少噪声。
  3. **变体**：  
     - **IR3DE-all**：使用全部 tokens（上限 1024 tokens）投票。  
     - **IR3DE-avg**：对 token 嵌入取平均，直接取 softmax 最大概率的领域。

### 3. 实验设计：数据集/场景、基准、对比方法

- **三种实验设置**：
  1. **CLM**：因果语言建模，任务为所有领域的下一个 token 预测。  
     - 基础模型：115M Llama3，在 M2D2 的 5 个领域（coding、math、physics、history、philosophy）上进行微调。  
     - 评估指标：perplexity（归一化为相对领域专家性能的百分比，PPL越低越好，归一化后越高越好）。
  2. **CLM_large**：类似 CLM，但使用 1B Llama3 基础模型，4 个领域（math、biology、legal、dialogue）。数据集分别来自 OpenWebMath、peS2o、Pile of Law、UltraChat 200k。
  3. **Reasoning**：每个领域有独立的下游推理任务。  
     - 专家模型：Llama3-3B 领域专家（来自 MergeBench），4 个领域（math、multilingual、coding、instruction following）。  
     - 数据集：GSM8k（math）、M_ARC（multilingual）、HumanEval（coding）、IFEval（instruction following）。  
     - 评估指标：各领域标准指标（accuracy、pass@1、task compliance rate），同样归一化为相对专家自身性能的百分比。
- **对比方法**：  
  - 领域专家自身（作为上限参考）、Experts average、Random routing。  
  - **MoDEM**：使用 DeBERTa v3 分类器（small 44M / large 304M），需集中训练。  
  - **kNN router**：基于 BERT 嵌入的 k 近邻投票（\( k \in \{1,5,10\} \)）。  
  - 省略了 BERT-router、MLP-router（因 MoDEM 已代表更强的 LM 分类器）。
- **结果报告**：所有结果已归一化百分比（越高越好），数值超过 100 表示因生成随机性导致的“超专家自身”情况。

### 6. 论文的主要结论与发现

- **性能**：在 CLM 设置中，IR3DE-all 实现了与 kNN router 相当的归一化平均性能（100.0%）。在 Reasoning 设置中，IR3DE（top-k 熵过滤）达到 **98.4%** 平均归一化性能，超过第二名 kNN router（97.6%）。  
- **动态性**：IR3DE 支持在不解体路由器的前提下**动态添加/移除专家**（通过异步更新统计量 \( A, B \) 后重新计算权重），这是其他方法不具备的。  
- **有效性**：熵过滤（选择 entropy 最小的 top-k tokens）显著提升了路由准确率，避免噪声 token 干扰。图 3 显示存在最优 \( k \) 值（CLM: 100, CLM_large: 10, Reasoning: 10）。  
- **极低开销**：路由计算仅需一次嵌入前向 + 小矩阵乘，不如 LM 基分类器繁重。

### 7. 优点：方法或实验设计上的亮点

- **方法亮点**：
  - **线性闭式解**：无需梯度迭代训练，一次计算即得到最优权重，适合快速部署。  
  - **去中心化和隐私友好**：各领域数据集可单独处理并异步聚合统计量，无需集中存储。  
  - **动态扩展**：可通过增量更新权重实现专家增减，不影响已有路由。  
  - **轻量**：仅需嵌入层（可与专家共享或使用小嵌入），计算量远低于额外语言模型。
- **实验设计亮点**：
  - 归一化指标消除了不同领域表现尺度差异，可公平比较路由方法。  
  - 包含三个不同难度和任务类型的设置，验证泛化性。  
  - 消融实验清晰说明了熵过滤（top-k）的作用。
