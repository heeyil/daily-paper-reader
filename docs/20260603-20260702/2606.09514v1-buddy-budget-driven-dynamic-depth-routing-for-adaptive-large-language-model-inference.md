## 摘要
大型语言模型（LLM）由于其深度和参数规模而推理成本高昂。深度剪枝可以通过跳过冗余的Transformer块来降低延迟，但现有方法（i）在用户指定的计算预算下控制能力有限，且（ii）通常固定路由路径，无法在解码过程中根据上下文增长进行自适应调整。我们提出Buddy，一种预算驱动的动态深度路由框架。Buddy使用轻量级决策模块，根据输入对中间层进行评分，并确定性地执行top-k层以满足给定预算。为支持解码时自适应，Buddy重用第一层KV缓存作为低开销的全局上下文来源，并在每次路由决策前将其与最新令牌表示进行池化。当未提供明确预算时，可选的预算预测器会估计输入相关的计算量，以平衡质量与效率。在Llama系列和Qwen模型上的实验表明，Buddy与强静态剪枝基线相比具有竞争力，且通常能改进精度-计算权衡，同时独特地支持严格的预算控制、解码时重路由以及单个训练模型内的多预算设置。

## Abstract
Large language models (LLMs) incur high inference cost due to their depth and parameter scale. Depth pruning can reduce latency by skipping redundant Transformer blocks, but existing methods (i) provide limited control under user-specific compute budgets and (ii) typically fix the routing path, failing to adapt as the context grows during decoding. We propose Buddy, a budget-driven dynamic depth routing framework. Buddy uses a lightweight Decision Module to score intermediate layers conditioned on the input and deterministically executes the top-k layers to satisfy a given budget. To support decode-time adaptation, Buddy reuses the first-layer KV cache as a low-overhead global context source and pools it together with the newest token representation before each routing decision. When no explicit budget is provided, an optional Budget Predictor estimates an input-dependent compute level to balance quality and efficiency. Experiments on Llama-family and Qwen models show that Buddy is competitive with strong static pruning baselines and often improves the accuracy-compute trade-off, while uniquely supporting strict budget control, decode-time rerouting, and multiple budgets within a single trained model.

# 论文详细中文总结

## 1. 核心问题与整体含义（研究动机和背景）

大型语言模型（LLM）因其深度和参数规模，推理延迟高、成本大。深度剪枝（跳过冗余Transformer块）是一种有效降低延迟的方法。然而现有深度剪枝方法存在两个关键缺陷：
- （i）对用户指定的计算预算缺乏严格且灵活的掌控，实际计算量不可预测；
- （ii）通常在预填充阶段确定固定的路由路径并在解码阶段复用，而实际上随生成上下文增长，各层的相对重要性会动态变化。

因此，本文旨在设计一种能同时适应输入难度和用户预算、在解码阶段能动态调整执行路径的推理机制，使得单一部署的LLM能够可靠地服务异构计算约束。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
Buddy提出预算驱动的动态深度路由框架。给定输入和可选的计算预算，Buddy选择要执行的层子集，使之既满足预算约束，又保持任务性能。

### 关键技术细节

1. **决策模块（Decision Module）**：
   - 轻量级双层MLP（Linear-SiLU-Linear），根据当前上下文对中间候选层（去掉首尾固定层）输出分数。
   - 融合离线先验（通过ΔPPL、Taylor/Fisher、余弦相似度等指标计算并归一化）得到最终分数。
   - 用户预算`b`（执行层比例）转换为整数`k = round(b·L) - 2`，选择得分最高的`k`个中间层执行，其余跳过，保持残差对齐。
   - 训练时采用直通估计器（STE）使离散选择可导。

2. **KV感知规划器（KV-aware Planner）**：
   - 在自回归解码阶段，重用第一层KV缓存提供轻量级全局摘要，与最新令牌表示连接后输入决策模块，使路由路径随解码过程动态更新。

3. **预算预测器（Budget Predictor）**：
   - 当用户未提供预算时，使用GRPO（组相对策略优化）训练一个小型分类器，从离散动作空间（执行中间层数）中选择最优计算量，平衡质量与效率。

### 训练方式
- 采用LoRA（r=8, 所有线性模块）与基础模型联合微调，使用Alpaca数据集。
- 决策模块与LoRA参数同时更新，随机采样不同稀疏度训练，包含全执行情况以保证梯度更新。

## 6. 主要结论与发现

1. **性能**：Buddy在大多数稀疏度下达到最佳或次优平均精度，尤其在25%-50%稀疏度下表现突出；退化曲线平滑。
2. **速度**：相比密集模型，50%稀疏度时预填充加速1.91倍（Alpaca），解码加速1.64倍；开销极小（决策模块仅占~0.003% FLOPs）。
3. **动态适应性**：路由路径在解码期间确实变化（尤其在中等稀疏度），且重新计算比复用预填充路径带来更好ROUGE分数。
4. **预算灵活性**：单个训练模型可服务多个不同预算，无需训练多个模型。
5. **消融表明**：Value状态作为输入最佳；Taylor先验优于其他；融合系数ω=0.1最优；第一层KV缓存作为上下文提取层效果稳健。

## 7. 优点

- **严格预算控制**：通过top-k选择确定执行层数，用户可精确设定计算量。
- **解码自适应**：利用首层KV缓存低成本获取全局上下文，实现解码阶段路径动态更新。
- **单模型多预算**：训练一次即可适应不同稀疏度，降低部署和训练成本。
- **轻量高效**：决策模块FLOPs占比极低（<0.005%），实际运行延迟增加约5.67%（表8）。
- **创新性组合**：将先验知识融合、STE训练、GRPO预算预测等有机结合，覆盖多场景。
- **实验全面**：多模型、多基准、多方法对比，并深入分析路由行为和自适应效果。

## 8. 不足与局限

- **VRAM不降低**：所有Transformer块常驻GPU内存（因为需要动态选择），内存开销未减少。
- **批处理低效**：不同序列可能选择不同执行路径，导致KV缓存缺失；当前用零填充占位，未来需更高效策略（如选择性缓存共享、分组）。
- **规模限制**：实验仅在7B-13B模型上进行，未验证更大规模（如70B）的可扩展性。
- **困难任务退化**：如GSM8K上精度下降显著（12.5%稀疏度时Llama2-7B从14.03降到7.81），说明复杂推理任务中深度剪枝仍有挑战。
- **先验依赖**：需要在小验证集上计算离线先验，可能增加额外校准成本。
- **训练复杂度**：需两阶段训练（SFT+GRPO），且GRPO对超参数敏感。
