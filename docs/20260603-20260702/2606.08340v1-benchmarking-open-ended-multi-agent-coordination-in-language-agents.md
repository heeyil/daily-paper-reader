## 摘要
随着语言模型越来越多地被部署为自主智能体，它们必须在开放式交互任务中与其它智能体进行长期协调。然而，现有评估很少同时测试这些需求，而是侧重于单智能体任务、短交互或高度结构化的多智能体环境。我们提出了$alem$，一个基于JAX的开放式多智能体协调基准，构建在类似Craftax的动力学基础上。Alem将程序化生成的协调任务、软专业化、通信和可控协调难度嵌入到一个具有探索、制作、交易和战斗的长期生存世界中。我们在同质团队中零样本评估了13个现代LLM，并以训练过的MARL智能体作为参考点。当前的LLM智能体远未解决alem任务，平均仅约6%的归一化回报，但它们的失败并非均匀分布。在最困难的协调设置下，零样本的Gemini-3.1-Pro-High接近训练了十亿步的MARL智能体，而GPT-5.4-High在基础任务奖励上表现强劲，但在协调奖励上却低得多。这种对比表明，个体任务能力并不等同于协调能力。消融实验显示，通信对协调贡献最大，而记忆和推理在用于维护多步计划时也有帮助。总体而言，我们的结果将协调识别为前沿LLM智能体的一个独特瓶颈，区别于单智能体能力。Alem使这一瓶颈可测量，并为开发能够通信、分配角色和执行共享计划的智能体提供了一个受控测试平台。代码见https://github.com/alem-world/alem-env。

## Abstract
As language models are increasingly deployed as autonomous agents, they must coordinate with others over long horizons in open-ended interactive tasks. Yet existing evaluations rarely test these demands together, instead emphasising single-agent tasks, short interactions, or highly structured multi-agent settings. We introduce $alem$, a JAX-based benchmark for open-ended multi-agent coordination built on Craftax-like dynamics. Alem embeds procedurally generated coordination tasks, soft specialisation, communication, and controllable coordination difficulty into a long-horizon survival world with exploration, crafting, trading, and combat. We evaluate $13$ modern LLMs zero-shot within homogeneous teams, with trained MARL agents as reference points. Current LLM agents remain far from solving alem, averaging only ~6% normalised return, but their failures are not uniform. On the hardest coordination setting, zero-shot Gemini-3.1-Pro-High approaches MARL agents trained for one billion steps, while GPT-5.4-High achieves strong base-task reward but much lower coordination reward. This contrast shows that individual task competence does not imply coordination competence. Ablations show that communication is the largest contributor to coordination, while memory and reasoning help when used to maintain multi-step plans. Overall, our results identify coordination as a distinct bottleneck for frontier LLM agents, separate from single-agent capabilities. Alem makes this bottleneck measurable and provides a controlled testbed for developing agents that communicate, allocate roles, and execute shared plans. Code is available at https://github.com/alem-world/alem-env.

- **核心思想**：在类似Craftax的开放世界中，通过**程序化生成**多种协调任务（同步操作、交接、长期依赖），并引入**软专业化**（任何智能体可尝试任何角色操作，但非专家有概率失败）、**显式通信通道**和**可调节的协调难度参数**α（0-1），使环境既能测试基础任务能力，又能专门测量协调能力。
- **关键技术细节**：
  - 环境建模为部分可观察随机博弈（POSG），支持像素、符号和文本三种观测接口。
  - 协调任务覆盖四种类型：**同步（Synchronous）**、**交接（Handover）**、**软同步（Soft）**和**硬同步（Hard）**，通过参数控制时间紧密度和所需智能体数量。
  - 合作事件包括资源请求/给予、队友复活、同步挖掘/建造/战斗等，并新增27个协调成就。
  - 采用JAX实现，支持单GPU上十亿步级MARL训练（约两天内完成）。
  - LLM智能体使用ReAct风格的文本提示，包含系统规则、历史观察、当前状态和可选推理/通信/记忆字段。

### 3. 实验设计：数据集/场景、基准、对比方法

- **场景**：ALEM包含9个程序化生成的关卡（Overworld、Gnomish Mines、Fire Realm等），每关包含多种协调任务。难度分为Easy（α=0.3）、Medium（α=0.6）、Hard（α=0.9）。
- **基准**：训练了4种MARL算法（IPPO、MAPPO、PQN-VDN、HyperMARL-IPPO）作为参考点，训练步数分别为100M和1B（以及在Hard设置下额外3B步的扩展实验）。
- **对比方法**：零样本评估了13个现代LLM，包括闭源模型（GPT-5.4-High、Gemini-3.1-Pro-High）和开源模型（Gemma-4系列、Llama-3.1/3.3系列、Qwen-3.5/3.6系列），在同质团队（3个相同模型智能体）中运行。此外还进行了消融实验和异质团队实验。

### 4. 资源与算力

- **MARL训练**：使用单个NVIDIA L40S 48GB GPU，配合JAX 0.4.38，一次完整的1B步训练约在两天内完成。扩展实验中3B步训练也仅需数天。
- **LLM评估**：使用3个NVIDIA A100 80GB GPU，每个开源模型20个种子×3个难度，闭源模型10个种子，耗时10-30小时/每个难度配置。
- **推理服务**：开源模型通过vLLM框架加载，闭源模型通过各自API调用。

- **总实验规模**：13个LLM × 3个难度 × 10~20个种子，加上4种MARL基线 × 2个训练步数 × 5个种子，以及消融实验（移除通信、记忆、推理）、异质团队实验、单智能体校准实验等，总计超过2000次评估。
- **充分性**：实验覆盖了同质、异质团队，多种协调难度和任务类型，并分解了基础奖励与协调奖励。消融实验定量分析了各组件贡献，定性分析了推理和通信内容。所有结果附有95%置信区间（使用rliable库的bootstrap方法），统计比较合理。
- **客观性**：使用了标准化的评估协议（零样本、固定提示模板），并报告了动作解析成功率（大多模型>99%），排除了接口理解因素。然而，闭源模型种子数较少（10个），可能引入一定偏差。

### 6. 论文的主要结论与发现

1. **当前LLM远未解决ALEM**：平均归一化总回报仅约6%；最先进的零样本模型Gemini-3.1-Pro-High在Hard设置下协调回报17.5%，接近1B步训练的MARL（17.6%），但总体仍很低。
2. **基础任务能力不能保证协调能力**：GPT-5.4-High基础奖励高（11.8%），但协调奖励仅4.2%，远低于Gemini-3.1-Pro-High的17.5%。说明两者是独立瓶颈。
3. **通信是协调的最大贡献者**：移除通信导致协调奖励大幅下降（Gemini从17.5%→5.3%，Gemma从8.8%→3.8%），基础奖励变化较小。
4. **记忆的效果取决于使用方式**：Gemini的scratchpad多用于前瞻性计划（71.5%多行、75.3%含未来词），移除后协调奖励下降；Gemma多用于状态摘要，记忆移除影响小。
5. **推理同时支持基础和协调能力**：降低推理水平使两种奖励均下降；但Gemini在无推理时倾向于用更长的消息和scratchpad进行补偿，但仍无法完全恢复性能。
6. **异质团队表现接近同质团队均值**：混合模型团队并未达到最强成员水平，说明协调能力无法通过简单组合强模型获得。
7. **同类型协调任务难度差异大**：交接任务（Handover）相对容易，而硬同步（Synchronous-Hard）和建造任务（Construction）对LLM和MARL都极具挑战。

### 7. 优点

- **全面的协调测试维度**：同时覆盖同步、交接、软/硬、长期依赖等多种协调类型，且通过单一参数α控制难度，保持世界布局不变，便于归因。
- **丰富的接口和可扩展性**：提供像素、符号、文本三种接口，既支持传统RL也支持LLM；JAX实现速度极快，支持大规模实验。
- **细致的消融实验**：系统分析通信、记忆、推理三个组件的影响，并结合定性内容分析（消息和scratchpad内容），揭示了模型内部行为差异。
- **分离基础与协调能力的度量**：通过分别报告Base%和Coord%，清晰展示协调瓶颈独立于基础任务能力。
- **信息论诊断**：验证了环境确实引入了多智能体依赖性，而非仅增加单智能体难度。

### 8. 不足与局限

- **模型覆盖不全**：未评估视觉-语言模型（VLM）和Claude系列模型，未来工作可补充。
- **闭源模型种子数较少**：GPT和Gemini仅10个种子，置信区间可能不够稳定；但报告了95% CI，结论整体稳健。
- **评估仅限零样本**：未测试微调或提示工程等适应方法，因此当前得分不代表模型潜力上限。
- **异质团队实验初步**：仅测试了两种团队组合（同家族和前三强混合），未系统研究更多组合和规模效应。
- **缺乏跨回合记忆机制**：LLM智能体当前没有跨回合持续学习能力，而ALEM允许未来研究终身学习。
- **分析工具较简单**：消息和scratchpad的定量分析基于手动定义的词表（如未来词、命令词），可能不适用于所有模型输出风格。
- **环境复杂度可能限制泛化**：虽然程序化生成减少记忆，但具体游戏机制（如Craftax-like）与真实世界仍有差距，协调发现可能无法直接迁移。
