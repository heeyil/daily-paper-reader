## 摘要
大型语言模型（LLMs）在性能与成本之间存在权衡，更强大的模型会带来更高的开销。LLM路由旨在通过将查询发送给最合适的模型来降低成本同时保持性能。然而，现有方法无法很好地适应不同用户的成本-性能偏好。为了弥补这一差距，我们引入了一种新颖的感知式LLM路由范式，实现个性化且以用户为中心的成本-性能优化，通过少量交互高效学习用户的隐式偏好。为了应对异构用户需求的挑战，我们将偏好特征建模为上下文赌博机中的一组不同任务，并提出了MetaRouter，一种专为偏好感知的LLM路由设计的元学习框架。实验结果表明，MetaRouter在分布内和分布外任务上均优于强基线。此外，它在学习用户偏好方面表现出高效性，对可路由LLM的变化具有鲁棒性，并且可扩展到多模型路由。

## Abstract
Large language models (LLMs) present a trade-off between performance and cost, where more powerful models incur greater expense. LLM routing aims to mitigate expenses while maintaining performance by sending queries to the most suitable model. However, existing methods cannot perform well for different user cost-performance preferences. To address this gap, we introduce a novel perceptive LLM routing paradigm for personalized and user-centric cost-performance optimization, which efficiently learns users' implicit preferences through little interaction. To handle the challenge of heterogeneous user needs, we formulate preference profiles as a set of distinct tasks in contextual bandit and propose MetaRouter, a meta-learning framework designed for preference-aware LLM routing. Experimental results show that MetaRouter outperforms strong baselines on both in-distribution and out-of-distribution tasks. Furthermore, it exhibits high efficiency in learning user preferences, robustness to changes in the routable LLMs, and scalability to multi-model routing.

## 1. 核心问题与整体含义（研究动机和背景）

大型语言模型（LLMs）在性能与成本之间存在固有权衡：强大模型（如 GPT-4）性能优异但 API 费用高昂，轻量模型（如 Phi-3）成本低但能力有限。LLM 路由旨在为每条查询自动选择最合适的模型，以在保持性能的同时降低成本。然而，现有路由方法存在两大局限：一是依赖用户手动配置参数（如成本-性能权重），不够直观且难以找到最优设置；二是当用户偏好改变时，需要重新训练整个模型，效率低下。因此，如何从少量用户交互中**隐式学习个性化偏好**并快速适应不同用户，成为关键挑战。本文提出**感知式 LLM 路由**（Perceptive LLM Routing）范式，并通过元学习框架 MetaRouter 实现该范式。

## 2. 方法论

### 核心思想
将不同用户的成本-性能偏好建模为上下文赌博机（Contextual Bandit）中的**不同任务**，利用元学习训练一个可在少量交互后快速适应新偏好的路由策略。

### 关键技术细节
- **问题形式化**：每个查询 $x$ 为状态，动作 $a \in \{S, L\}$（小/大模型），奖励 $R(x,a)$ 基于用户效用函数 $s(o) = w \cdot q(o) - (1-w) \cdot p(o)$，其中 $q(o)$ 为响应质量（如 LLM 裁判评分），$p(o)$ 为成本（基于 token 数和价格）。不同 $w$ 对应不同偏好任务。
- **上下文编码器 $E_\phi$**：在适应阶段，系统展示两模型对同一查询的响应，用户选择偏好其一（考虑性能与价格），形成成对比较上下文 $c = (x, S \succ L)$。编码器通过置换不变聚合 $E_\phi(z|c_{1:N}) = \frac{1}{N}\sum f_\phi(c_n)$ 生成隐式任务变量 $z$（10维向量）。
- **策略网络 $\pi_\theta$**：采用门控残差机制（Gated Residual Mechanism），将查询嵌入 $h_{obs}$ 与通过门控网络产生的调制向量 $g = \tanh(\text{MLP}_{\text{gate}}(z))$ 进行残差结合：$h_{mod} = h_{obs} + (h_{obs} \odot g)$，再输入 MLP 输出路由概率。奖励设计使用基线减法 $\tanh(\lambda (s(o_a) - \bar{s}(x)))$，并加入熵正则化鼓励探索。
- **元训练（Meta-training）**：在模拟的偏好任务分布上训练，每个任务生成 $c_{1:N}$ 和一批查询，对查询嵌入注入高斯噪声以增强鲁棒性。通过策略梯度同时更新编码器和策略网络。

## 6. 主要结论与发现

- MetaRouter 在 **ID 和 OOD 任务上均显著优于所有基线**，例如 RouteLLM 上 HV 提升 8.8%，IGD 降低 40.6%。
- **仅需约 6 个成对比较上下文**即可达到接近饱和的性能，用户负担极低。
- **对模型变更鲁棒**：当候选 LLM 换为全新模型对时不需重新训练，仍优于基线。
- **可扩展至多模型路由**（5 个模型）且保持优势。
- **对复杂效用函数（Cobb-Douglas）和不同质量指标（BART）均表现稳健**。

## 7. 优点

- **用户中心范式**：首次将元学习用于 LLM 路由，无需手动配置参数，从隐式偏好中学习。
- **高效适应**：仅需少量成对比较（~6 个）即可捕获用户偏好，实用性强。
- **模型无关性强**：训练后可直接适应新模型对，无需重新训练，降低部署成本。
- **设计创新**：门控残差机制有效融合查询与偏好表征，噪声注入增强泛化，奖励设计使用基线减法与 tanh 归一化稳定训练。
- **实验全面**：覆盖多种任务、分布内外、扩展场景，消融和敏感性分析充分验证设计。

## 8. 不足与局限

- **用户反馈模拟**：实验中使用合成偏好（基于固定权重 $w$）而非真实用户，可能低估真实交互中的噪声和不确定性。真实用户偏好更复杂且可能随时间漂移（论文假定适应期间偏好稳定）。
- **质量度量依赖**：主要使用 LLM 裁判作为质量信号，其自身存在偏差，替换为 BART 后仍有依赖自动评价的风险。
- **奖励函数形式局限**：仅考虑性能与成本的线性/拟线性组合，忽略了延迟、隐私等维度，计划在 future work 中扩展。

- **OOD 任务上仍有差距**：如 Omni-MATH 上 HV 低于 Matrix Factorization，说明在高度未见过分布下泛化能力仍有提升空间。
