Abstract 

Agentic language models must learn when to call tools, when to consume tool responses, and when to answer directly. This makes multi-teacher on-policy distillation a natural training strategy: one teacher can specialize in tool calls, another in direct responses, and the student can learn from both on its own generated distribution. We show that this strategy can induce a behavior shift that is invisible from aggregate losses alone. In a two-teacher tool-use setting, vanilla generalized knowledge distillation improves tool-call recall but also moves the model toward over-calling, where it calls tools on examples that should be answered directly. Aggregate explanations are insufficient: tool-call samples do not receive more token exposure, and full-sequence per-token divergence is not larger for the tool-call teacher. We instead analyze behavior leverage imbalance : local token-level signals at mode-entry and structural positions, such as <tool call> and function names, can have disproportionate control over the global generation mode. We propose Soft Clamp, a per-token divergence calibration method that dynamically compresses extreme token-level Jensen-Shannon divergence while preserving nonzero gradients. On APIGen-MT, Soft Clamp reduces over-calling from 13.7% to 9.0% relative to vanilla GKD while matching its decision accuracy. In a BFCL multi-turn diagnostic, it also lowers tool-call loops and repeated calls among GKD variants. These results suggest that multi-teacher OPD should monitor where teacher signals act, not only how large they are in aggregate. 

# 1 Introduction 

Agentic language models often act through interleaved dialogue and tool-use trajectories, such as 

System → User → ToolCall 1 → ToolResponse 1 → ToolCall 2 → ToolResponse 2 → Response 1 → User → · · · .

Along such trajectories, the model must decide not only how to use a tool, but also whether to use one and when to stop using tools. This decision is brittle in multi-turn systems. A model that overuses tools increases latency and cost, and it can enter repeated tool-call loops after receiving observations. A model that underuses tools fails tasks that require external information or actions. Multi-teacher on-policy distillation (MOPD) appears well matched to this problem [ 11 ]. A tool-call teacher can specialize in structured function calls; a response teacher can specialize in direct natural-language answers. The student then learns from teacher distributions on its own rollouts, as in generalized knowledge distillation (GKD) [1]. This setup offers specialization without manually merging heterogeneous behaviors. We find a failure mode that is not captured by loss curves or aggregate teacher contributions alone. In our APIGen-MT tool-use setting, vanilla GKD improves decision accuracy and tool-call recall, but it also raises over-calling. The model becomes better at calling tools when tools are needed, yet it becomes more likely to call tools when a direct answer is appropriate. 1arXiv:2607.07050v1 [cs.CL] 8 Jul 2026 Several aggregate explanations are not supported by the training logs. Tool-call samples do not dominate token exposure. Full-sequence per-token Jensen-Shannon divergence (JSD) and a gradient proxy are not larger for the tool-call teacher. In several runs, response tokens receive more exposure and larger aggregate divergence. The final behavior therefore cannot be explained by sample count, token count, or total divergence alone. We argue that the missing variable is behavior leverage imbalance . A token-level learning signal matters not only by its magnitude, but also by where it lands in the generation process. In a tool-use model, mode-entry tokens such as <tool call> , function names, and structural markers can switch the model into a tool-call trajectory. A relatively small local signal at such positions can affect the entire continuation. Natural-language response signals are often spread across many content tokens. They can improve style, factuality, and detail, yet they may exert less direct control over the discrete tool-call versus response boundary. This hypothesis predicts that decision-boundary metrics should explain the final behavior better than full-sequence averages do. This observation leads to a targeted mitigation. We propose Soft Clamp, a token-level calibration rule for GKD. For each token divergence di, Soft Clamp sets a dynamic threshold C = k mean( d) inside a batch. Tokens below the threshold are unchanged. Tokens above the threshold are scaled by a detached factor,

Our contributions are: • We identify behavior leverage imbalance as a concrete failure mode in multi-teacher OPD for agentic tool-use models. • We show that aggregate token exposure and full-sequence divergence do not explain the observed tool-call shift, whereas response-side decision pressure gives a more targeted diagnostic of the final over-calling behavior. • We introduce Soft Clamp, a lightweight per-token divergence calibration method that preserves gradients while compressing extreme token-level signals. • We evaluate on APIGen-MT, BFCL, When2Call, and a BFCL multi-turn loop diagnostic. Soft Clamp reduces over-calling among GKD variants while preserving APIGen decision accuracy. We do not claim to introduce multi-teacher distillation itself, nor do we claim that all teacher contributions should be equalized. Our claim is narrower: when different teachers supervise different generation modes, high-leverage local signals can shift global behavior, and token-level calibration can reduce that shift. 

# 2 Problem Setup 

2.1 Multi-teacher on-policy distillation 

Let x be a dialogue context and y = ( y1, . . . , y T ) a student-generated continuation. In on-policy distillation, the student samples from its current policy and then learns from teacher distributions evaluated on the sampled trajectory. In our setting, each training example has a teacher tag m ∈ { tool , response } that routes the example to a tool-call teacher or a response teacher. We use GKD-style token losses based on the divergence between teacher and student next-token distri-butions [ 1]. 2.2 Tool-call versus response supervision 

The tool-call teacher and response teacher start from the same base model. The tool-call teacher is supervised on examples whose final target is a structured tool call. The response teacher is supervised on examples whose final target is a direct natural-language answer. Training uses the final assistant turn as the supervised target; previous turns serve as context. For tool-call examples, our training implementation consumes the final assistant target as text. We therefore pre-render structured tool calls into the model chat template’s textual tool-call form; Appendix A.2 gives an example. This is an implementation-specific serialization detail, not a requirement of multi-teacher OPD itself: other training frameworks may represent tool calls with different structured fields or templates. The detail matters for our experiments because treating the final target as a generic assistant response without the expected rendering can make the model learn an invalid schema even when decision metrics appear plausible. 

2.3 Behavior leverage 

We define behavior leverage as the degree to which a token position controls the future generation mode. Mode-entry tokens have high behavior leverage because changing them redirects the continuation. In tool-use generation, <tool call> and function names are high-leverage positions. Once the model enters a tool-call block, later tokens must follow a structured schema, and the dialogue may continue with tool observations. This differs from ordinary token difficulty. A content token in a direct response can have high loss, but correcting it may only change a local phrase. A mode-entry token with comparable loss can change whether the whole output becomes a tool call. Multi-teacher OPD becomes vulnerable when one teacher’s signals concentrate on high-leverage positions and another teacher’s signals spread across lower-leverage content positions.

4.2 Comparison with hard clipping and global reweighting 

Hard clipping tests whether directly truncating extreme token losses is enough. It is simple, but it can remove much of the learning signal from difficult high-leverage tokens. Global reweighting changes token weights according to their batch-relative divergence. It is useful as a contrast, but it scales broad regions of the loss rather than isolating local extremes. Soft Clamp is deliberately narrower. It leaves ordinary tokens unchanged, preserves gradients for extreme tokens, and adapts the threshold to the current batch scale. It does not explicitly identify mode-entry tokens; instead, it limits the effect of any extreme local divergence that could dominate training. 4.3 Format anchoring 

Structured tool calls introduce a second risk: schema drift. In early pure OPD runs, some checkpoints produced invalid tool-call strings even when the training data used the correct XML-style target format. Our main runs therefore include a supervised loss with weight 0.3 as a format anchor. This anchor is not the proposed method; it is a controlled training component that keeps the tool-call schema parseable while we compare GKD variants. A pure OPD schema-drift study remains useful future work. 

# 5 Experiments 

5.1 Models and training 

All models use Qwen3.5-9B as the base. The tool-call teacher and response teacher are initialized from the same base and trained on their respective target types. They serve as online supervisors, not as deployable balanced baselines. The main comparison therefore reports the base model, a base SFT model trained on the mixed tool-use data, and four GKD students: vanilla GKD, Hard Clip, Global Reweight, and Soft Clamp. The GKD students share the same data, teachers, training length, and core OPD settings. The main GKD runs use sf t alpha = 0 .3 as a format anchor.

5.5 Multi-turn tool-call loops 

The multi-turn diagnostic connects the decision metric to deployment behavior. Vanilla GKD makes 1.494 tool calls per turn, triggers Loop@3 on 14.8% of turns, and repeats the same call on 16.7% of turns. Soft Clamp lowers these values to 1.268, 10.1%, and 11.1%, respectively. The final-answer rate rises from 89.6% to 94.1%. This result does not replace official BFCL scoring, but it captures the practical risk of an excessive tool-call prior under a controlled interaction harness. 

# 6 Related Work and Limitations 

On-policy and generalized distillation. On-policy distillation trains the student on its own generated distribution, reducing train-test mismatch compared with purely offline distillation. GKD instantiates this idea with divergence-based teacher supervision [ 1 ]. MiniLLM and related work study reverse-KL-style distillation and student-generated samples [ 10 , 17 ]. Our work does not modify the basic OPD objective. It studies how heterogeneous teacher signals act on behavior boundaries inside that objective. 

Multi-teacher OPD. Recent work explores multi-teacher OPD, teacher routing, specialization, and robust-ness [ 4 , 7 , 11 , 13 , 14 , 20 ]. These methods often focus on how to combine or select teachers. Balancing losses and gradients. Multi-objective training has long used loss weighting and gradient balancing to manage competing tasks [ 2 , 9]. Distillation-specific variants also adjust token weights or teacher contributions [ 8, 15 , 16 ]. Soft Clamp differs in scope. It is not a global task-weighting rule. It compresses extreme per-token divergence values while preserving gradients, which targets local domination at high-leverage positions. 

Tool-use behavior drift. Tool-use and agentic training can produce behavior drift, including tool overuse, tool-use collapse, and repeated calls [ 3, 5 , 6, 12 , 18 , 19 ]. These studies often examine RL or agentic search settings. We find a related behavior shift in supervised multi-teacher OPD, which suggests that overuse can arise before RL, through local distillation signals at mode-entry positions. 

Limitations. First, our experiments focus on one base model and a two-teacher tool-call versus response setting. The mechanism should be tested in other heterogeneous-teacher settings, such as refusal versus helpfulness, code versus natural language, and short answers versus long reasoning. Second, the reported numbers are point estimates from the current training runs; multi-seed studies would better separate systematic effects from run-to-run variation. Third, Soft Clamp is a mitigation, not a theory-optimal solution. Some extreme token-level signals may encode useful specialization; excessive compression could underfit them. Fourth, our step-level evidence uses teacher-forced process metrics. The observed alignment between response-side decision pressure and final over-calling should be read as diagnostic evidence, not as a complete causal proof. Checkpoint-wise free-generation probes would give a more direct trajectory of over-calling during training. # 7 Conclusion 

Multi-teacher OPD can shift model behavior even when aggregate teacher contributions look balanced. In tool-use distillation, local signals at mode-entry and structural positions have high behavior leverage: they can move the model into a tool-call trajectory and affect the entire continuation. This helps explain why vanilla GKD can improve tool-call recall while also increasing over-calling. Soft Clamp offers a simple calibration rule for this failure mode. By dynamically compressing extreme per-token divergence and preserving gradients, it reduces over-calling and multi-turn loop risk among GKD variants while maintaining APIGen decision accuracy. The broader lesson is diagnostic: multi-teacher OPD should track not only how much signal each teacher supplies, but also where that signal acts and whether it pushes behavior boundaries.

A.1 Training data and supervision 

The main GKD experiments use two training splits derived from APIGen-MT: 15,419 tool-call examples and 15,420 response examples. Each example is routed to one teacher by its teacher tag. The tool-call teacher supervises examples whose final target is a structured tool call, and the response teacher supervises examples whose final target is a natural-language answer. Only the final assistant turn is used as the supervised target. Earlier turns provide dialogue context. In our ms-swift/Megatron implementation, the training backend treats this final assistant target as text. For tool-call examples, we therefore pre-render the final target into the model chat template’s XML-style textual form, with tags such as <tool call> , <function=...> , and <parameter=...> . This preprocessing is backend-specific rather than an algorithmic requirement. It is important in our setup because leaving a tool call as an unrendered object can make the model learn an invalid tool-call schema. 

A.2 Tool-call target rendering 

For structured tool-call examples, the final supervised target is rendered as text before training in our backend. A representative target is: 

> <tool_call> <function=get_flight_cost> <parameter=travel_to> LAX </parameter> </function> </tool_call>

This appendix example is illustrative; the experiments use the rendered targets from the APIGen-MT-derived training split. Other frameworks may serialize the same structured call differently. 

A.3 Main training configuration 

Table 7 lists the shared training configuration for the four GKD variants. All variants use the same data, teachers, model initialization, training length, and batch settings. They differ only in the loss modifier: none for vanilla GKD, hard clipping for Hard Clip, batch-relative reweighting for Global Reweight, and dynamic token compression for Soft Clamp. The supervised anchor is not part of the proposed method. It is held fixed across all GKD variants to keep structured tool-call outputs parseable while comparing the distillation losses.

The BFCL multi-turn diagnostic evaluates whether a model with a stronger tool-call prior becomes less usable in an interactive tool environment. It uses 800 BFCL tasks and 3,136 user turns. When the model emits a tool call, the harness returns a simulated tool observation and continues the dialogue until the model gives a final answer or reaches the maximum step limit. We report five diagnostic metrics. Tool calls per turn counts the average number of tool calls emitted per user turn. Loop@3 and Loop@5 measure whether the model keeps calling tools for at least three or five consecutive steps. Max-step hit records turns that end by reaching the harness limit. Repeat same call 

records repeated calls to the same function with the same arguments. Final answer records whether the model eventually returns a non-tool final answer. This diagnostic is not a replacement for official BFCL scoring. It is a usability stress test for the specific failure mode studied in this paper: a model can improve should-call recall while becoming too willing to enter tool-call trajectories during multi-turn interaction. 

# E Schema Drift and Format Anchoring 

Early pure OPD runs showed that a model can preserve the high-level decision signal while drifting away from the required tool-call schema. This was most visible as invalid XML-style tool-call strings. The issue is separate from behavior leverage imbalance: one concerns whether the model chooses the tool-call mode, and the other concerns whether the generated tool-call string is parseable. The main experiments therefore use a small supervised loss with sft alpha =0.3 as a format anchor. Because this anchor is shared by all GKD variants, it does not explain the difference between vanilla GKD, Hard Clip, Global Reweight, and Soft Clamp. A pure OPD schema-drift experiment is useful as a separate robustness study, but it is not required for the main comparison in this paper. 17
