## 摘要
条件计算可以将语言模型质量与每令牌推理成本解耦，但领先的技术孤立地在单一维度上操作：混合专家（MoE）使FFN稀疏化，混合深度（MoD）跳过整个Transformer块，KV缓存量化压缩注意力内存。我们认为这三个决策（注意力分辨率、专家选择和缓存位宽）是强耦合的，应该联合做出：一个足够稀有以至于需要完整注意力的令牌，无论哪个专家处理它，也可能需要高精度缓存。我们引入了TriRoute，一个跨所有三个轴共享的轻量级控制器，对于每一层的每个令牌，发出协调策略：(i) 注意力模式（跳过/局部/全局），(ii) FFN专家的稀疏集（带有一个恢复MoD的空专家），以及(iii) KV缓存位宽。该控制器通过异质松弛（用于分类决策的Gumbel-Softmax配合直通估计，以及用于专家的负载均衡top-k门控）在拉格朗日预算约束下进行端到端训练，该约束将平均计算和内存成本变为可控旋钮。我们识别了朴素联合训练中的跨轴路由崩溃级联，其中一个轴上的崩溃会传播到其他轴，并通过逐轴归一化和耦合感知平衡损失来解决它。在从160M到1.3B参数的仅解码器模型上，在计算最优令牌计数下，TriRoute在匹配推理FLOPs和内存的情况下帕累托支配了最佳的独立MoD+MoE+KV量化组合，同时更好地保留了在稀有实体、代码和算术上的尾例鲁棒性，这些是纯困惑度优化所侵蚀的。事后分析揭示了可解释的结构：控制器将完整注意力和高精度缓存分配给句首位置、稀有子词和命名实体，而廉价地路由功能词。

## Abstract
Conditional computation can decouple language model quality from per-token inference cost, yet leading techniques act on a single axis in isolation: Mixture-of-Experts (MoE) sparsifies the FFN, Mixture-of-Depths (MoD) skips whole transformer blocks, and KV-cache quantization compresses attention memory. We argue these three decisions (attention resolution, expert selection, and cache bit-width) are strongly coupled and should be made jointly: a token rare enough to warrant full attention may also need high-precision caching regardless of which expert processes it. We introduce TriRoute, a single lightweight controller shared across all three axes that, for every token at every layer, emits a coordinated policy: (i) an attention mode (skip/local/full), (ii) a sparse set of FFN experts (with a null expert recovering MoD), and (iii) a KV-cache bit-width. The controller trains end-to-end via a heterogeneous relaxation (Gumbel-Softmax with straight-through estimation for categorical decisions and load-balanced top-k gating for experts) under a Lagrangian budget constraint that turns the average compute and memory cost into a controllable knob. We identify a cross-axis routing-collapse cascade in naive joint training, where collapse on one axis propagates to the others, and address it with per-axis normalization and a coupling-aware balancing loss. On decoder-only models from 160M to 1.3B parameters at compute-optimal token counts, TriRoute Pareto-dominates the best independent MoD+MoE+KV-quantization combination at matched inference FLOPs and memory, while better preserving tail-case robustness on rare entities, code, and arithmetic that pure perplexity optimization erodes. Post-hoc analysis reveals interpretable structure: the controller allocates full attention and high-precision cache to sentence-initial positions, rare subwords, and named entities, while cheaply routing function words.

## 1. 核心问题与整体含义（研究动机和背景）

当前大语言模型（LLM）在推理时，每个 token 消耗的计算和内存高度均匀，导致资源浪费。为打破这种均匀性，三个主流技术分别作用在不同维度：
- **MoE（混合专家）**：使前馈网络（FFN）稀疏化，每个 token 仅激活少量专家。
- **MoD（混合深度）**：允许 token 跳过整个 Transformer 块。
- **KV缓存量化**：压缩注意力缓存的内存占用。

然而，这些技术各自独立优化，忽略了三个决策之间的强耦合关系。例如，一个稀有实体 token 可能不需要复杂的 FFN 变换（适合 MoD 跳过），但却需要完整的注意力分辨率和高精度的 KV 缓存，以便后续 token 正确检索。而一个功能词则恰好相反。因此，最优的资源分配是一个**跨轴的向量决策**，而非单轴标量。论文提出 **TriRoute**，通过单个联合控制器，为每个 token 同时决策注意力模式、专家选择和缓存位宽，实现资源按需分配。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
TriRoute 在每个 Transformer 层前插入一个轻量级共享控制器，为每个 token 输出一个协调策略 \( \pi_t^{(\ell)} = (a_t, e_t, b_t) \)，分别控制三个轴：
- **注意力模式 \( a_t \)**：`skip`（跳过注意力）、`local`（局部窗口）、`full`（完整因果注意力），控制 token 的读操作。
- **专家选择 \( e_t \)**：从 E 个 FFN 专家（含一个空专家 \( f_0 \equiv 0 \)）中选出 top-k，控制 token 的 FFN 变换。空专家等价于 MoD 的跳过。
- **KV缓存位宽 \( b_t \)**：2/4/8/16 比特，控制 token 自己的 K/V 被存储的精度，决定未来读的保真度。

### 关键技术细节

1. **共享控制器结构**  
   控制器由一个两层 MLP 主干和三个线性头组成，输入为 token 的归一化残差状态和廉价侧特征（位置、边界、熵等）。输出 logits 分别对应三个轴的选项。

2. **异构梯度估计**  
   - 专家选择：采用标准的 softmax top-k 门控，梯度通过选中专家的门控权重传播。
   - 注意力模式和位宽：采用 **Straight-Through Gumbel-Softmax**（ST-Gumbel），前向传递决策的 one-hot，反向传播通过 soft 样本的梯度。温度从 2.0 退火到 0.5。
   - **跨轴梯度平衡**：每个轴的梯度被其运行时的梯度范数 EMA 归一化，防止注意力轴主导共享主干。

3. **可微成本模型**  
   为 FLOPs 和内存建立可微成本函数：
   \[
   c_{\text{flops}}^t = \sum_m \tilde{y}_{a_t,m} \kappa_a^m(t) + \sum_{j\in S_t} \bar{p}_{e_t,j} \kappa_e,\quad c_{\text{mem}}^t = \sum_b \tilde{y}_{b_t,b} (2 d_{kv} b)
   \]
   各项成本归一化为密集模型的百分比。

4. **耦合感知平衡——防止跨轴路由崩塌**  
   朴素联合训练中，一旦一个轴崩塌（如注意力过早全跳过），会降低共享主干的方差，诱发其他轴相继崩塌。解决方法：
   - **逐轴白化**：在每个头前对共享表示进行去相关。
   - **边际熵下限**：当某个轴的使用熵低于比例 \( \zeta \) 时施加惩罚，避免完全崩塌。

5. **单一预算旋钮的在线拉格朗日控制器**  
   通过拉格朗日乘子 \( \lambda_r \)（对 FLOPs 和内存分别）将平均成本约束在目标 \( C_r^* \) 以下，乘子由对偶上升自适应更新。用户只需设定一个总目标成本，无需调三个稀疏超参数。

6. **训练与推理**  
   训练时端到端优化式 (13)，包含语言模型损失、平衡损失和预算约束损失。推理时取硬决策，仅执行选中的计算，保持因果、单遍、兼容批量和 KV 分页。

## 6. 主要结论与发现

1. **联合路由支配独立帕累托前沿**：  
   TriRoute 在相等计算和内存下，困惑度更低、下游准确率更高，且随着预算紧张，优势更明显。1.3B 模型在 55% FLOPs / 40% 内存下接近密集模型质量。

2. **尾域鲁棒性显著提升**：  
   独立组合在稀有实体、代码、数学和长上下文上性能大幅下降（稀有实体 +2.1 ppl），而 TriRoute 仅退化 0.4 ppl，GSM8K 只降 0.3 点。表明联合控制器能将资源集中在关键 token 上。

3. **控制器学习到可解释的结构**：
   - 读操作（注意力）和写操作（缓存精度）强耦合，集中在句子起始、稀有子词、命名实体上；功能词被廉价处理。
   - FFN 计算（专家激活）与注意力解耦：稀有实体常跳过 FFN 但保留高注意力和高精度缓存——这是块级 MoD 无法表达的。
   - 早期层广泛使用注意力，后期层变得选择性；专家使用随深度增加。
   - 聚类分析揭示了“功能词/廉价”、“实体/锚点”、“计算”、“边界/沉没”等可解释集群。

## 7. 优点

1. **统一视角**：首次将三个独立的条件计算轴（注意力、专家、缓存）整合于单一学习型控制器下，解决耦合问题。
2. **方法创新**：
   - 异构松弛与梯度平衡，使分类和实数值决策可共同训练。
   - 识别并解决了跨轴路由崩塌，提出白化+熵下限的耦合感知平衡。
   - 在线拉格朗日预算控制器，单一旋钮即可扫描整个帕累托前沿。
3. **实验设计严谨**：匹配预算、尾域分析、实际吞吐测量，避免仅依赖于理论 FLOPs。
4. **可解释性**：通过事后分析揭示路由策略的语言结构和语言直觉一致，增加了方法的可信度。
5. **持续开源**：提供参考 PyTorch 实现，便于复现和采用。

## 8. 不足与局限

1. **FLOP 到延迟的差距**：实际加速比（1.7x）低于理论 FLOP 比率（1.8x），原因包括：

   - 混合精度缓存需要额外分组开销。
   - 控制器本身引入同步点。
2. **训练成本与稳定性**：联合训练比单轴路由更敏感，引入多个额外超参数（α, β, γ, ζ, ρλ）。虽然论文称同组超参数跨规模迁移，但未提供极端稀疏下的验证。
