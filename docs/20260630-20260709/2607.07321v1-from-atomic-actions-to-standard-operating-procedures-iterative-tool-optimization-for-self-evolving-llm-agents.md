## 摘要
工具利用使大型语言模型（LLM）代理能够与现实世界交互并解决复杂任务。然而，现有的代理框架主要依赖于由细粒度原子动作（例如，基本的文件I/O或单次搜索）组成的静态工具集，这迫使代理为每个重复的工作流程重新发明低级逻辑，导致推理开销增加和失败率上升。在本研究中，我们提出代理可以通过将这些原子动作合成为可重用的标准操作程序（SOP）来实现自我进化，这些SOP作为可调用的高阶工具封装了多步逻辑。我们进一步引入了EvoSOP框架，该框架使代理能够从执行轨迹中提取SOP，并通过构建、合并、评估和剪枝的系统生命周期迭代优化工具集。大量实验表明，与基线相比，EvoSOP显著提高了任务成功率，同时大幅减少了交互轮数。我们的分析还揭示，迭代工具优化培养了可靠且高效的工具使用模式，为自进化代理的发展提供了可扩展的路径。

## Abstract
Tool utilization enables Large Language Model (LLM) agents to interact with the real world and resolve complex tasks. However, existing agent frameworks predominantly rely on static toolsets composed of granular atomic actions (e.g., basic file I/O or single-turn search), which forces agents to reinvent low-level logic for every recurring workflow, leading to increased reasoning overhead and failure rates. In this study, we propose that agents can achieve self-evolution by synthesizing these atomic actions into reusable Standard Operating Procedures (SOPs), which function as callable higher-order tools that encapsulate multi-step logic. We further introduce EvoSOP, a framework that empowers agents to extract SOPs from execution trajectories and iteratively optimize the toolset through a systematic lifecycle of construction, merging, evaluation, and pruning. Extensive experiments demonstrate that EvoSOP significantly boosts task success rates while substantially reducing the number of interaction rounds compared to baselines. Our analysis also reveals that iterative tool optimization fosters reliable and efficient tool-use patterns, providing a scalable pathway for the development of self-evolving agents.

# 论文总结：从原子动作到标准操作程序——面向自进化LLM代理的迭代工具优化

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：现有 LLM 代理框架严重依赖静态、细粒度的**原子动作**工具集（如文件 I/O、单次搜索等），迫使代理为每个重复性工作流重新编排低级逻辑，导致**推理开销大**、**失败率高**，缺乏人类问题解决中的**层次化效率**（即使用标准操作程序 SOP 封装多步逻辑）。
- **背景**：尽管已有研究通过动态代码生成扩展工具集（如 Voyager、CRAFT、ASI），但这些方法将工具创建视为**一次性事件**，缺乏长期管理机制，导致工具集**臃肿**（冗余、低效工具堆积），反而干扰决策、降低性能。
- **整体含义**：提出代理可通过**将原子动作合成为可重用的高阶 SOP** 实现**自我进化**，并设计**迭代优化生命周期**（构建、合并、评估、剪枝）来系统性地管理工具集，避免退化。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 将执行轨迹中**频繁共现、逻辑耦合**的原子动作序列提取并封装为**可调用的标准操作程序（SOP）**，通过**非参数化结构优化**（不更新 LLM 参数）实现代理自我进化。
- 将迭代优化类比机器学习流水线：执行轨迹 = 数据获取，任务执行 = 前向传播，SOP 提取/合并/剪枝 = 后向传播（符号化梯度），合并剪枝 = 正则化，工具集稳定 = 学习率衰减。

### 关键技术模块（四个协作模块）
1. **构造函数（Constructor）**：从历史执行轨迹中识别**逻辑耦合的动作段**（连续调用、结果依赖、高频组合等），通过 `f_extract` 和 `f_rewrite` 将其转换为**包含条件判断和错误处理的 SOP 源代码**（含语义 schema）。
2. **合并器（Merger）**：对同一批次内构建的 SOP 进行**功能重叠分析**，合并冗余 SOP 为**更通用、更高阶**的 SOP。采用**非破坏性合并**策略（保留原始 SOP 直至验证阶段），避免过早丢失高质量变体。
3. **评估器（Evaluator）**：将候选 SOP 加入工具集 `F'`，让代理**重新执行训练任务**，生成新轨迹 `ˆξ`，收集每个 SOP 的**实际效用数据**（成功/失败/异常等）。
4. **审查器（Reviewer）**：基于评估轨迹，对每个 SOP 调用状态分类（最优、部分效用、中性、负面干扰、实现缺陷），对**低质量、高错误率、冗余或罕用**的 SOP 执行**剪枝**，保留高质量 SOP。

### 训练流程（算法 1）
- **小批量（Mini-batching）**：将初始轨迹分为小批量，每轮仅从当前批次轨迹中提取新 SOP。
- **检查点（Checkpointing）**：每轮结束存档工具集和评估日志，选择**训练成功率最高**的轮次作为最终模型，避免激进剪枝。
- **迭代轮次**：前 5 轮从轨迹构造 SOP，后 5 轮仅执行合并/评估/剪枝（不再生成新 SOP），共 10 轮。

## 3. 实验设计
### 数据集 / 场景

- **Tau2Bench**（Telecom 子集，Solo 模式）：涉及手机网络连接问题解决，工具空间更大、状态转换更复杂。

### 基准方法（对比）
- **基础代理方法**：ReAct、DFSDT（带 DFS 回溯的 agent）。
- **工具增强方法**：

## 7. 优点
- **方法论创新**：首次将代理工具优化类比为机器学习训练流程（前向/后向传播、正则化、学习率衰减），提供理论层次。
- **系统性生命周期管理**：超越一次性工具创建，通过构建-合并-评估-剪枝闭环，避免工具集膨胀和性能退化。
- **模型无关**：无需更新 LLM 参数，可集成任意黑盒模型，实用性强。
- **实验设计扎实**：跨多个模型、两个不同难度的基准，多次运行报告标准误差，提供可视化工具演化分析。
- **案例可解释性**：跟踪单个 SOP 的生命周期，直观展示框架如何区分高质量、冗余和不良工具。
