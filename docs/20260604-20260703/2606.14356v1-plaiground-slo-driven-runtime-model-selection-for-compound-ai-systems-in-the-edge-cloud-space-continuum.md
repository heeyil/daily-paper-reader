## 摘要
3D计算连续体统一了边缘、云和空间，其上的应用需要将多个AI任务（如目标检测、时间序列分析和自然语言处理）组合成复合AI系统。这些系统必须满足关于准确性、延迟和成本的严格服务水平目标（SLO）。维持复合AI系统SLO合规性的一个关键机制是运行时模型选择，即针对每个工作流任务动态切换AI模型。然而，现有的分布式和复合AI框架本身并不支持运行时模型选择。我们提出了PLAIground，一个使复合AI系统能够进行运行时模型选择的框架。PLAIground引入了可复合AI模型（CAIM）抽象，通过任务和数据契约将任务语义与AI模型实现解耦，从而无需更改工作流即可切换模型。此外，PLAIground引入了Pixie，一种SLO驱动的运行时模型选择算法，该算法在执行过程中动态选择每个任务最合适的模型。我们在两个真实的复合AI工作流上的评估表明，Pixie在保持SLO合规性的同时实现了高达91.3%的准确性，而固定模型策略要么违反成本和延迟预算高达21倍，要么准确性目标偏差4%。

## Abstract
Applications in the 3D Computing Continuum, which unifies edge, cloud, and space, require combining multiple AI tasks such as object detection, time-series analytics, and natural language processing into Compound AI systems. These systems must satisfy stringent Service Level Objectives (SLOs) on accuracy, latency, and cost. A key mechanism for maintaining SLO compliance of Compound AI systems is runtime model selection, where AI models are dynamically switched for each workflow task. However, existing distributed and compound AI frameworks do not natively support runtime model selection. We present PLAIground, a framework that enables runtime model selection for Compound AI systems. PLAIground introduces Compoundable AI Model (CAIM) abstraction, which decouples task semantics from AI model implementations via Task and Data Contracts, enabling model switching without workflow changes. Additionally, PLAIground introduces Pixie, an SLO-driven runtime model selection algorithm, which dynamically selects the most suitable model for each task during execution. Our evaluation on two realistic Compound AI workflows demonstrates that Pixie achieves up to 91.3% accuracy while maintaining SLO compliance where fixed-model strategies either violate cost and latency budgets up to 21x or miss accuracy targets by 4%.

- **核心问题**：现有分布式和复合AI框架（如LangChain、DSPy）将任务定义与特定AI模型实现紧密耦合，不支持**运行时模型选择**。开发者必须硬编码模型调用、数据解析逻辑，无法在运行时动态切换模型以适应环境变化。因此，在资源受限且动态的3D连续体中，缺乏一种机制能**在不修改工作流代码的前提下**，针对每个任务持续选择最合适的AI模型以满足SLO。
- **整体含义**：解决该问题对于实现自适应、高能效、高可靠的复合AI系统至关重要，可直接应用于**早期野火检测、智能问答**等需要多模型协同且实时性强的场景。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
PLAIground提出**复合AI模型（CAIM）** 抽象，将任务语义与AI模型实现彻底解耦；并设计**Pixie**算法，在运行时通过监控实际资源消耗，动态调整每个CAIM的模型选择以维持SLO合规。

### 关键技术细节

#### (1) CAIM抽象
- **Task Contract**：声明任务的功能需求（如目标检测、文本生成）和非功能需求（SLO：如准确性下限、延迟上限、成本预算）。
- **Data Contract**：定义严格的输入/输出数据模式（如图像、边界框、JSON），作为数据流标准化层，保证模型切换时下游任务仍接收一致格式的数据。
- **System Contract**：由平台提供，包含候选模型列表、性能画像（准确性、延迟、能耗等）及部署配置。开发者只需提供Task和Data Contract，运行时自动绑定可用的模型。

#### (2) Pixie算法（Algorithm 1）
- **输入**：有序候选模型集合（System Contract）、系统SLO阈值、观察窗口大小k、阈值τ_low和τ_high。
- **初始化**：选择满足SLO的限制内准确性最高的模型（贪婪选择）。
- **运行时循环**：
  1. 每次请求后更新**滑动平均窗口**（大小为k）记录观测到的资源指标（如延迟、能耗）。
  2. 计算每个系统SLO的**归一化间隙**：`g = min_i (L_i - avg(W,R_i)) / L_i`，找出最受约束的SLO。
  3. 判断当前状态：
     - 若 `g < τ_low`：SLO压力过大 → **降级**到资源消耗更低的模型。
     - 若 `g > τ_high`：SLO余量充足 → **升级**到准确性更高的模型。
     - 否则：保持当前模型。
  4. 每次切换后重置观察窗口，进入冷却期（连续k次观测不评估），防止噪声导致震荡。
- **SLO分解**：对于工作流级别的累积SLO（如总成本、端到端延迟），按各候选模型平均资源消耗的比例分解到每个CAIM。

> **算法流程文字说明**：  
> 初始化模型（最高准确率满足SLO） → 循环工作流处理每批请求 → 若已积累k次观测 → 计算最紧SLO的归一化间隙 → 若间隙 < τ_low，执行降级；若间隙 > τ_high，执行升级；否则不动 → 重置窗口 → 执行模型后观测指标并更新窗口。

1. **Pixie是唯一同时满足所有SLO的策略**：在Wildfire Detection中，Greedy-Quality消耗能量1246J（超标21倍），有效准确率仅33.8%；Pixie消耗438J（低于预算450J），有效准确率91.3%（比Greedy-Cost提升2.8%）。

## 8. 不足与局限
1. **实验覆盖有限**：仅测试两个场景，未涵盖更复杂的复合工作流（如串联10+任务）、非单调性系统（假设每个步骤提升accuracy会提升整体，但实际中有噪声或互斥）等。需要更多领域和更大规模的验证。
2. **依赖单调性假设**：论文明确基于“改善单个步骤准确性不会损害端到端准确性”的单调性。若系统存在步骤间的冲突（如分类器与下游模型协同问题），Pixie的独立选择可能不优。
3. **SLO分解过于简单**：工作级SLO分解仅按平均资源消耗比例分配，未考虑模型间的变异性（如高峰期延迟波动）。任务级SLO（如最小准确性）的分解被标记为未来工作，实际系统中难以保证。

5. **参数敏感性未讨论**：τ_low和τ_high、窗口大小k对性能有影响，论文未进行消融或灵敏度分析，可能在不同工作负载下需手工调整。
6. **冷启动与模型切换频率**：尽管单次切换快，但频繁切换可能导致模型缓存未命中或系统抖动，论文未评估切换频率对整体性能的负面影响。
7. **噪声处理**：滑动平均可平滑噪声，但未讨论异常值处理（如瞬间延迟尖峰）或过拟合到短时波动的问题。
